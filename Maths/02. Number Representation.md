# 2. Number Representation

Every number a computer works with has to be stored as bits — this lesson covers the different **bases** used to represent numbers, how to convert between them, how characters relate to integers, and what happens when a number doesn't fit in its allocated space (overflow/underflow), plus the Java types built to handle numbers of any size.

## 1. Decimal

### Theory

Decimal (base 10) is the number system humans use every day — it has 10 digits (`0`–`9`), and each position represents a power of 10.

```
5347 = 5×10³ + 3×10² + 4×10¹ + 7×10⁰
```

In Java, integer literals written normally (`5347`) are assumed to be decimal. This is the default base for all arithmetic and printing (`System.out.println`) unless you explicitly ask for another base.

### Code

```java
public class Decimal {
    public static void main(String[] args) {
        int decimalValue = 5347;
        System.out.println("Decimal value: " + decimalValue);

        // Manually expanding by place value
        int expanded = 5 * 1000 + 3 * 100 + 4 * 10 + 7 * 1;
        System.out.println("Expanded (5*10^3 + 3*10^2 + 4*10^1 + 7*10^0): " + expanded);
    }
}
```

### Example

```
Decimal value: 5347
Expanded (5*10^3 + 3*10^2 + 4*10^1 + 7*10^0): 5347
```

## 2. Binary

### Theory

Binary (base 2) uses only two digits — `0` and `1` — and is the native language of computer hardware (on/off, high/low voltage). Each position represents a power of 2.

```
1011 (binary) = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11 (decimal)
```

Java lets you write binary literals directly using the `0b` prefix (Java 7+), and provides `Integer.toBinaryString()` / `Integer.parseInt(str, 2)` to convert to/from binary text.

### Code

```java
public class Binary {
    public static void main(String[] args) {
        int binaryLiteral = 0b1011; // written directly as binary
        System.out.println("0b1011 as decimal: " + binaryLiteral);

        // Decimal -> Binary string
        int number = 11;
        String binaryString = Integer.toBinaryString(number);
        System.out.println("11 in binary: " + binaryString);

        // Binary string -> Decimal
        int parsedBack = Integer.parseInt("1011", 2);
        System.out.println("\"1011\" (base 2) as decimal: " + parsedBack);
    }
}
```

### Example

```
0b1011 as decimal: 11
11 in binary: 1011
"1011" (base 2) as decimal: 11
```

## 3. Octal

### Theory

Octal (base 8) uses digits `0`–`7`. It was historically popular because 3 binary bits map exactly to 1 octal digit (groups of 3). Java treats any integer literal **starting with a leading `0`** as octal — a classic gotcha (`010` is `8`, not ten!).

```
17 (octal) = 1×8¹ + 7×8⁰ = 8 + 7 = 15 (decimal)
```

### Code

```java
public class Octal {
    public static void main(String[] args) {
        int octalLiteral = 017; // leading 0 => interpreted as octal
        System.out.println("017 as decimal: " + octalLiteral);

        // Decimal -> Octal string
        int number = 15;
        String octalString = Integer.toOctalString(number);
        System.out.println("15 in octal: " + octalString);

        // Octal string -> Decimal
        int parsedBack = Integer.parseInt("17", 8);
        System.out.println("\"17\" (base 8) as decimal: " + parsedBack);
    }
}
```

### Example

```
017 as decimal: 15
15 in octal: 17
"17" (base 8) as decimal: 15
```

> **Gotcha:** `int x = 010;` does NOT equal `10`. It equals `8`, because the leading zero makes Java parse it as octal.

## 4. Hexadecimal

### Theory

Hexadecimal (base 16) uses digits `0`–`9` plus letters `A`–`F` (representing 10–15). It's widely used for memory addresses, color codes, and bitwise/byte-level work because 4 binary bits map exactly to 1 hex digit (a full byte = exactly 2 hex digits). Java literals use the `0x` prefix.

```
2F (hex) = 2×16¹ + 15×16⁰ = 32 + 15 = 47 (decimal)
```

### Code

```java
public class Hexadecimal {
    public static void main(String[] args) {
        int hexLiteral = 0x2F; // written directly as hex
        System.out.println("0x2F as decimal: " + hexLiteral);

        // Decimal -> Hex string
        int number = 47;
        String hexString = Integer.toHexString(number);
        System.out.println("47 in hex: " + hexString);

        // Hex string -> Decimal
        int parsedBack = Integer.parseInt("2F", 16);
        System.out.println("\"2F\" (base 16) as decimal: " + parsedBack);
    }
}
```

### Example

```
0x2F as decimal: 47
47 in hex: 2f
"2F" (base 16) as decimal: 47
```

## 5. Base Conversion

### Theory

Java's `Integer` (and `Long`) class provides general-purpose base conversion methods that work for **any radix from 2 to 36**, not just 2, 8, 10, 16:

- `Integer.toString(number, radix)` — decimal → any base (as a `String`)
- `Integer.parseInt(str, radix)` — any base string → decimal (`int`)

This is the toolkit you reach for whenever a problem asks you to convert between arbitrary bases (a very common DSA/interview question).

### Code

```java
public class BaseConversion {
    public static void main(String[] args) {
        int number = 100;

        // Decimal to any base
        System.out.println("100 in base 2:  " + Integer.toString(number, 2));
        System.out.println("100 in base 8:  " + Integer.toString(number, 8));
        System.out.println("100 in base 16: " + Integer.toString(number, 16));
        System.out.println("100 in base 36: " + Integer.toString(number, 36));

        // Any base back to decimal
        System.out.println("\"1100100\" (base 2) -> decimal:  " + Integer.parseInt("1100100", 2));
        System.out.println("\"144\" (base 8) -> decimal:      " + Integer.parseInt("144", 8));
        System.out.println("\"64\" (base 16) -> decimal:      " + Integer.parseInt("64", 16));
        System.out.println("\"2S\" (base 36) -> decimal:      " + Integer.parseInt("2S", 36));
    }
}
```

### Example

```
100 in base 2:  1100100
100 in base 8:  144
100 in base 16: 64
100 in base 36: 2s

"1100100" (base 2) -> decimal:  100
"144" (base 8) -> decimal:      100
"64" (base 16) -> decimal:      100
"2S" (base 36) -> decimal:      100
```

## 6. Digit Representation

### Theory

To manually work with the individual digits of a number (rather than using a built-in library method), the classic pattern is repeated `% 10` (extract last digit) and `/ 10` (drop last digit). This works for any base — swap `10` for the target radix.

This pattern underlies a huge number of DSA problems: reversing a number, checking palindromes, digit sum, counting digits, etc.

### Code

```java
public class DigitRepresentation {
    public static void main(String[] args) {
        int number = 4021;

        System.out.println("Digits of " + number + " (from last to first):");
        int n = number;
        while (n != 0) {
            int lastDigit = n % 10;
            System.out.println(lastDigit);
            n = n / 10;
        }

        // Building the reversed number back up
        int reversed = 0;
        n = number;
        while (n != 0) {
            reversed = reversed * 10 + (n % 10);
            n /= 10;
        }
        System.out.println("Reversed: " + reversed);

        // Sum of digits
        int sum = 0;
        n = number;
        while (n != 0) {
            sum += n % 10;
            n /= 10;
        }
        System.out.println("Digit sum: " + sum);
    }
}
```

### Example

```
Digits of 4021 (from last to first):
1
2
0
4
Reversed: 1204
Digit sum: 7
```

## 7. Character ↔ Integer conversion

### Theory

In Java, `char` is internally a 16-bit unsigned integer holding a Unicode code point. This means:

- **char → int**: just assign it (implicit widening), or subtract `'0'` to convert a **digit character** to its numeric value.
- **int → char**: cast explicitly, or add `'0'` to a digit value (0–9) to get its character form.
- `Character.getNumericValue(char)` is a safer, more general alternative that also handles letters (`'A'` → 10, etc.) and out-of-range input.

```
'7' - '0' = 7   (because '0'..'9' are contiguous in Unicode: 48..57)
```

### Code

```java
public class CharIntConversion {
    public static void main(String[] args) {
        char digitChar = '7';

        // char -> int (numeric value of the digit)
        int digitValue = digitChar - '0';
        System.out.println("'7' - '0' = " + digitValue);

        // int -> char
        int value = 9;
        char asChar = (char) ('0' + value);
        System.out.println("'0' + 9 (as char) = " + asChar);

        // Implicit char -> int (uses Unicode code point, not digit value)
        int codePoint = digitChar;
        System.out.println("Unicode code point of '7': " + codePoint);

        // Safer built-in alternative
        System.out.println("Character.getNumericValue('7'): " + Character.getNumericValue('7'));
        System.out.println("Character.getNumericValue('A'): " + Character.getNumericValue('A'));
    }
}
```

### Example

```
'7' - '0' = 7
'0' + 9 (as char) = 9
Unicode code point of '7': 55
Character.getNumericValue('7'): 7
Character.getNumericValue('A'): 10
```

## 8. Overflow

### Theory

Overflow happens when a computed value is **larger** than the maximum a type can hold. Java does **not** throw an exception for primitive overflow — it silently wraps around (two's complement wraparound), which can introduce hard-to-find bugs.

```
Integer.MAX_VALUE + 1 == Integer.MIN_VALUE   (wraps around)
```

### Code

```java
public class Overflow {
    public static void main(String[] args) {
        int max = Integer.MAX_VALUE;
        System.out.println("Integer.MAX_VALUE:     " + max);
        System.out.println("MAX_VALUE + 1:         " + (max + 1)); // silently wraps

        // Detecting overflow safely with Math.addExact (throws instead of wrapping)
        try {
            int result = Math.addExact(max, 1);
            System.out.println("This won't print: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Math.addExact caught overflow: " + e.getMessage());
        }
    }
}
```

### Example

```
Integer.MAX_VALUE:     2147483647
MAX_VALUE + 1:         -2147483648
Math.addExact caught overflow: integer overflow
```

## 9. Underflow

### Theory

Underflow is the mirror image of overflow: a computed value is **smaller** (more negative) than the minimum a type can hold, and it wraps around to the top of the range. For floating-point types, "underflow" can also refer to a very small non-zero result rounding to `0.0` — Java handles this gracefully by returning `0.0` rather than crashing.

### Code

```java
public class Underflow {
    public static void main(String[] args) {
        int min = Integer.MIN_VALUE;
        System.out.println("Integer.MIN_VALUE:     " + min);
        System.out.println("MIN_VALUE - 1:         " + (min - 1)); // wraps to MAX_VALUE

        try {
            int result = Math.subtractExact(min, 1);
            System.out.println("This won't print: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Math.subtractExact caught underflow: " + e.getMessage());
        }

        // Floating-point underflow: extremely small result rounds to 0.0, no crash
        double tiny = Double.MIN_VALUE / 2;
        System.out.println("Double.MIN_VALUE / 2:  " + tiny);
    }
}
```

### Example

```
Integer.MIN_VALUE:     -2147483648
MIN_VALUE - 1:         2147483647
Math.subtractExact caught underflow: integer overflow
Double.MIN_VALUE / 2:  0.0
```

## 10. Integer limits (Java)

### Theory

Java's numeric types each have a fixed size and a fixed range. Picking the right type — or reaching for an arbitrary-precision type — is a core skill for avoiding overflow bugs.

| Type | Size | Range |
|---|---|---|
| `int` | 32-bit | `-2,147,483,648` to `2,147,483,647` |
| `long` | 64-bit | `-9,223,372,036,854,775,808` to `9,223,372,036,854,775,807` |
| `BigInteger` | Arbitrary precision | Unbounded (limited only by memory) — for exact integer math beyond `long` |
| `BigDecimal` | Arbitrary precision | Unbounded — for exact **decimal** math (money, precise fractions) where `double` rounding errors are unacceptable |

> **When to use which:** `int` for everyday counters/indices, `long` when values might exceed ~2.1 billion (factorials, big sums, timestamps in milliseconds), `BigInteger` when even `long` isn't enough (e.g. `100!`), and `BigDecimal` whenever exact decimal precision matters (never use `double` for money).

### Code

```java
import java.math.BigInteger;
import java.math.BigDecimal;

public class IntegerLimits {
    public static void main(String[] args) {
        // --- Integer ---
        System.out.println("Integer.MAX_VALUE: " + Integer.MAX_VALUE);
        System.out.println("Integer.MIN_VALUE: " + Integer.MIN_VALUE);

        // --- Long ---
        System.out.println("Long.MAX_VALUE:    " + Long.MAX_VALUE);
        System.out.println("Long.MIN_VALUE:    " + Long.MIN_VALUE);

        long bigProduct = 100_000L * 100_000L; // 'L' suffix / long variable avoids int overflow
        System.out.println("100000L * 100000: " + bigProduct);

        // --- BigInteger: exact arbitrary-precision integers ---
        BigInteger factorial20 = BigInteger.ONE;
        for (int i = 1; i <= 20; i++) {
            factorial20 = factorial20.multiply(BigInteger.valueOf(i));
        }
        System.out.println("20! (BigInteger):  " + factorial20);
        // Note: 20! = 2,432,902,008,176,640,000 -- fits in a long,
        // but 21! onward would overflow long and needs BigInteger.

        // --- BigDecimal: exact decimal arithmetic ---
        BigDecimal a = new BigDecimal("0.1");
        BigDecimal b = new BigDecimal("0.2");
        System.out.println("0.1 + 0.2 (BigDecimal): " + a.add(b));

        // Why not double for money — the classic floating-point rounding artifact:
        double doubleSum = 0.1 + 0.2;
        System.out.println("0.1 + 0.2 (double):     " + doubleSum);
    }
}
```

### Example

```
Integer.MAX_VALUE: 2147483647
Integer.MIN_VALUE: -2147483648
Long.MAX_VALUE:    9223372036854775807
Long.MIN_VALUE:    -9223372036854775808
100000L * 100000: 10000000000
20! (BigInteger):  2432902008176640000
0.1 + 0.2 (BigDecimal): 0.3
0.1 + 0.2 (double):     0.30000000000000004
```

## Quick Reference Cheat Sheet

| Base | Radix | Java prefix / method | Digits used |
|---|---|---|---|
| Decimal | 10 | (default) | `0`–`9` |
| Binary | 2 | `0b...` / `parseInt(s, 2)` | `0`–`1` |
| Octal | 8 | `0...` (leading zero!) / `parseInt(s, 8)` | `0`–`7` |
| Hexadecimal | 16 | `0x...` / `parseInt(s, 16)` | `0`–`9`, `A`–`F` |

| Type | Size | Overflow-safe alternative |
|---|---|---|
| `int` | 32-bit | `long`, or `Math.addExact`/`multiplyExact` to detect overflow |
| `long` | 64-bit | `BigInteger` |
| `double` | 64-bit float (imprecise) | `BigDecimal` (exact decimal) |
