# 24. Recurrence Relations

This lesson widens the lens: recurrences that come from DP problems, ones with a derivable closed form, and the general case of any **linear** recurrence, solved fast via [matrix exponentiation](<./17. ## 1. DP Recurrence

### Theory

Many dynamic-programming problems reduce to recognizing a recurrence between subproblem answers. **Classic example — Climbing Stairs:** with `n` steps and the ability to climb `1` or `2` steps at a time, the number of distinct ways to reach step `n` is the number of ways to reach step `n-1` (then take one final 1-step) **plus** the number of ways to reach step `n-2` (then take one final 2-step):

```
ways(n) = ways(n-1) + ways(n-2)
ways(1) = 1, ways(2) = 2
```

This is exactly the Fibonacci recurrence, just shifted — a common pattern where a real-world counting problem turns out to secretly be a well-known sequence.

### Code

```java
public class DpRecurrence {
    public static void main(String[] args) {
        for (int n = 1; n <= 5; n++) {
            System.out.println("Ways to climb " + n + " stairs: " + climbStairs(n));
        }
    }

    static long climbStairs(int n) {
        if (n <= 2) return n;
        long[] dp = new long[n + 1];
        dp[1] = 1;
        dp[2] = 2;
        for (int i = 3; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }
        return dp[n];
    }
}
```

### Example

```
Ways to climb 1 stairs: 1
Ways to climb 2 stairs: 2
Ways to climb 3 stairs: 3
Ways to climb 4 stairs: 5
Ways to climb 5 stairs: 8
```

### Problem (Basic): House Robber

**Theory:** **Classic example — House Robber:** a robber can't rob two **adjacent** houses; maximize total loot from a row of houses with given values. This is a different *shape* of recurrence from Climbing Stairs — instead of **summing** the ways from the two previous states, it takes the **max** of two choices at each house: either skip it (`dp[i-1]`) or rob it and add its value to the best result from two houses back (`dp[i-2] + nums[i]`), since the adjacent house can't also be robbed.

```java
public class HouseRobber {
    public static void main(String[] args) {
        System.out.println("rob([1,2,3,1]): " + rob(new int[] { 1, 2, 3, 1 }));
        System.out.println("rob([2,7,9,3,1]): " + rob(new int[] { 2, 7, 9, 3, 1 }));
    }

    static int rob(int[] nums) {
        int prev2 = 0, prev1 = 0; // best loot using zero houses, then one house
        for (int value : nums) {
            int current = Math.max(prev1, prev2 + value); // skip this house, or rob it
            prev2 = prev1;
            prev1 = current;
        }
        return prev1;
    }
}
```

```
rob([1,2,3,1]): 4
rob([2,7,9,3,1]): 12
```

`[1,2,3,1]`: robbing houses at values `1` and `3` (indices `0` and `2`, non-adjacent) gives `4`, beating any other combination. `[2,7,9,3,1]`: robbing houses at values `2`, `9`, and `1` (indices `0`, `2`, `4` — every other house) gives `12`, better than robbing the single largest house (`9`) alone or any adjacent-respecting pair.

## 2. Mathematical Recurrence

### Theory

Some recurrences have a **closed-form solution** derivable through algebra rather than needing DP at all. **Classic example — Tower of Hanoi:** moving `n` disks takes `T(n) = 2×T(n-1) + 1` moves (move `n-1` disks out of the way, move the largest disk, then move the `n-1` disks back on top). Expanding this recurrence reveals the closed form:

```
T(n) = 2×T(n-1) + 1,  T(0) = 0
     = 2ⁿ - 1
```

### Code

```java
public class MathematicalRecurrence {
    public static void main(String[] args) {
        for (int n = 0; n <= 5; n++) {
            System.out.println("hanoi(" + n + ") recurrence=" + hanoiRecurrence(n)
                    + ", closed form=" + hanoiClosedForm(n));
        }
    }

    static long hanoiRecurrence(int n) {
        if (n == 0) return 0;
        return 2 * hanoiRecurrence(n - 1) + 1;
    }

    static long hanoiClosedForm(int n) {
        return (1L << n) - 1; // 2^n - 1
    }
}
```

### Example

```
hanoi(0) recurrence=0, closed form=0
hanoi(1) recurrence=1, closed form=1
hanoi(2) recurrence=3, closed form=3
hanoi(3) recurrence=7, closed form=7
hanoi(4) recurrence=15, closed form=15
hanoi(5) recurrence=31, closed form=31
```

Both approaches agree at every `n` — the closed form just skips straight to the answer in O(1) (or O(log n) for the bit shift) instead of O(n) recursive calls.

## 3. Linear Recurrence

### Theory

A **linear recurrence** expresses each term as a fixed linear combination of a constant number of previous terms — Fibonacci, Tribonacci, and the Tower of Hanoi recurrence are all special cases. Any such recurrence can be written as **matrix multiplication**, which means it inherits [matrix exponentiation](<./17. Matrix Mathematics.md#2-matrix-exponentiation>)'s **O(log n)** speed — essential once `n` is too large for a loop.

**Example recurrence:** `a(n) = 2×a(n-1) + 3×a(n-2)`, with `a(0)=0, a(1)=1`. In matrix form:

```
[a(n+1)]   [2 3]   [a(n)  ]
[a(n)  ] = [1 0] × [a(n-1)]
```

### Code

```java
public class LinearRecurrence {
    public static void main(String[] args) {
        int n = 10;
        System.out.println("a(10) [iterative]: " + linearRecurrenceIterative(n));
        System.out.println("a(10) [matrix exponentiation]: " + linearRecurrenceMatrix(n));
    }

    // O(n)
    static long linearRecurrenceIterative(int n) {
        if (n == 0) return 0;
        if (n == 1) return 1;
        long prev2 = 0, prev1 = 1;
        for (int i = 2; i <= n; i++) {
            long current = 2 * prev1 + 3 * prev2;
            prev2 = prev1;
            prev1 = current;
        }
        return prev1;
    }

    // O(log n), scales to huge n
    static long linearRecurrenceMatrix(int n) {
        if (n == 0) return 0;
        long[][] base = { { 2, 3 }, { 1, 0 } };
        long[][] result = matrixPower(base, n - 1);
        // [a(n), a(n-1)]^T = base^(n-1) * [a(1), a(0)]^T = base^(n-1) * [1, 0]^T
        return result[0][0] * 1 + result[0][1] * 0;
    }

    static long[][] matrixPower(long[][] base, int exponent) {
        int size = base.length;
        long[][] result = identity(size);
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = multiply(result, base);
            base = multiply(base, base);
            exponent >>= 1;
        }
        return result;
    }

    static long[][] multiply(long[][] a, long[][] b) {
        int n = a.length, k = b.length, m = b[0].length;
        long[][] result = new long[n][m];
        for (int i = 0; i < n; i++)
            for (int j = 0; j < m; j++)
                for (int x = 0; x < k; x++)
                    result[i][j] += a[i][x] * b[x][j];
        return result;
    }

    static long[][] identity(int size) {
        long[][] id = new long[size][size];
        for (int i = 0; i < size; i++) id[i][i] = 1;
        return id;
    }
}
```

### Example

```
a(10) [iterative]: 14762
a(10) [matrix exponentiation]: 14762
```

Both methods agree — the matrix version is the one that stays fast even if `n` were `10^18`.

## Quick Reference Cheat Sheet

| Type | Example | Best approach |
|---|---|---|
| DP Recurrence | Climbing Stairs, House Robber | Bottom-up array, O(n) |
| Mathematical Recurrence | Tower of Hanoi | Derive the closed form algebraically, O(1)/O(log n) |
| Linear Recurrence (general) | `a(n) = c1·a(n-1) + c2·a(n-2) + ...` | Matrix exponentiation, O(log n) |
