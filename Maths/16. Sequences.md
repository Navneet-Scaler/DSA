This lesson covers the classic named sequences, the trick for computing huge terms fast ([Matrix Exponentiation](<./17. ## 1. Fibonacci

### Theory

The Fibonacci sequence is defined by `F(n) = F(n-1) + F(n-2)`, with `F(0) = 0, F(1) = 1`. Three ways to compute it, in increasing order of speed:

1. **Naive recursion** — directly implements the recurrence, but recomputes the same subproblems repeatedly: **O(2ⁿ)**.
2. **Iterative** — build up from the base cases with a loop, keeping only the last two values: **O(n)**.
3. **Matrix exponentiation** — Fibonacci has a matrix form that lets you jump straight to the `n`-th term using [binary exponentiation](<./07. Powers and Exponentiation.md#3-binary-exponentiation>): **O(log n)**, essential when `n` is huge (e.g. `10^18`) and the answer is wanted `mod p`.

```
[F(n+1)]   [1 1]ⁿ   [F(1)]
[F(n)  ] = [1 0]  ×  [F(0)]
```

### Code

```java
public class Fibonacci {
    public static void main(String[] args) {
        System.out.println("fib(10) [iterative]: " + fibIterative(10));

        long MOD = 1_000_000_007L;
        System.out.println("fib(50) mod 1e9+7 [matrix exponentiation]: " + fibMatrix(50, MOD));
    }

    // O(n)
    static long fibIterative(int n) {
        long a = 0, b = 1;
        for (int i = 0; i < n; i++) {
            long next = a + b;
            a = b;
            b = next;
        }
        return a;
    }

    // O(log n), safe for huge n via matrix exponentiation
    static long fibMatrix(long n, long mod) {
        if (n == 0) return 0;
        long[][] base = { { 1, 1 }, { 1, 0 } };
        long[][] result = matPow(base, n, mod);
        return result[0][1];
    }

    static long[][] matPow(long[][] m, long p, long mod) {
        int size = m.length;
        long[][] result = identity(size);
        while (p > 0) {
            if ((p & 1) == 1) result = matMultiply(result, m, mod);
            m = matMultiply(m, m, mod);
            p >>= 1;
        }
        return result;
    }

    static long[][] matMultiply(long[][] a, long[][] b, long mod) {
        int n = a.length;
        long[][] result = new long[n][n];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                long sum = 0;
                for (int k = 0; k < n; k++) {
                    sum = (sum + a[i][k] * b[k][j]) % mod;
                }
                result[i][j] = sum;
            }
        }
        return result;
    }

    static long[][] identity(int size) {
        long[][] id = new long[size][size];
        for (int i = 0; i < size; i++) id[i][i] = 1;
        return id;
    }
}
```

### Example

```
fib(10) [iterative]: 55
fib(50) mod 1e9+7 [matrix exponentiation]: 586268941
```

`fib(50) = 12586269025`, and `12586269025 mod (10^9+7) = 586268941` — the matrix method gives the exact same answer without ever materializing the full (much larger) value.

### Problem (Basic): Climbing Stairs

**Theory:** **Classic example — Climbing Stairs:** count the number of distinct ways to climb `n` stairs, taking either `1` or `2` steps at a time. The last step taken is either a `1`-step (from stair `n-1`) or a `2`-step (from stair `n-2`), so the total ways to reach stair `n` is the **sum** of the ways to reach each of those — exactly the Fibonacci recurrence, just shifted and with different base cases (`ways(1)=1, ways(2)=2` instead of `F(0)=0, F(1)=1`).

```java
public class ClimbingStairs {
    public static void main(String[] args) {
        System.out.println("climbStairs(2): " + climbStairs(2));
        System.out.println("climbStairs(3): " + climbStairs(3));
        System.out.println("climbStairs(5): " + climbStairs(5));
    }

    static int climbStairs(int n) {
        if (n <= 2) return n;
        int a = 1, b = 2;
        for (int i = 3; i <= n; i++) {
            int next = a + b;
            a = b;
            b = next;
        }
        return b;
    }
}
```

```
climbStairs(2): 2
climbStairs(3): 3
climbStairs(5): 8
```

`climbStairs(5) = 8` matches `fib(6)` in the standard 0-indexed Fibonacci numbering — the same underlying recurrence, wearing a different problem's name.

## 2. Arithmetic Progression

### Theory

An **Arithmetic Progression (AP)** is a sequence with a constant **difference** `d` between consecutive terms: `a₁, a₁+d, a₁+2d, ...`. Both the `n`-th term and the sum of the first `n` terms have direct **O(1)** formulas — no loop needed:

```
nth term:        aₙ = a₁ + (n-1) × d
sum of n terms:  Sₙ = n × (2a₁ + (n-1)d) / 2
```

### Code

```java
public class ArithmeticProgression {
    public static void main(String[] args) {
        long a1 = 3, d = 5;
        int n = 10;

        System.out.println("10th term (a1=3, d=5): " + nthTerm(a1, d, n));
        System.out.println("Sum of first 10 terms: " + sumOfN(a1, d, n));
    }

    static long nthTerm(long a1, long d, int n) {
        return a1 + (long) (n - 1) * d;
    }

    static long sumOfN(long a1, long d, int n) {
        return (long) n * (2 * a1 + (long) (n - 1) * d) / 2;
    }
}
```

### Example

```
10th term (a1=3, d=5): 48
Sum of first 10 terms: 255
```

## 3. Geometric Progression

### Theory

A **Geometric Progression (GP)** has a constant **ratio** `r` between consecutive terms: `a₁, a₁r, a₁r², ...`. Like AP, both the `n`-th term and the partial sum have closed forms — but the sum formula needs a **special case when `r = 1`** (the general formula would divide by zero):

```
nth term:                  aₙ = a₁ × r^(n-1)
sum of n terms (r ≠ 1):    Sₙ = a₁ × (rⁿ - 1) / (r - 1)
sum of n terms (r = 1):    Sₙ = a₁ × n
```

### Code

```java
public class GeometricProgression {
    public static void main(String[] args) {
        long a1 = 2, r = 3;
        int n = 6;

        System.out.println("6th term (a1=2, r=3): " + nthTerm(a1, r, n));
        System.out.println("Sum of first 6 terms: " + sumOfN(a1, r, n));
    }

    static long nthTerm(long a1, long r, int n) {
        return a1 * (long) Math.pow(r, n - 1);
    }

    static long sumOfN(long a1, long r, int n) {
        if (r == 1) return a1 * n; // special case: avoid dividing by (r - 1) = 0
        return a1 * ((long) Math.pow(r, n) - 1) / (r - 1);
    }
}
```

### Example

```
6th term (a1=2, r=3): 486
Sum of first 6 terms: 728
```

## 4. Tribonacci

### Theory

The **Tribonacci** sequence extends Fibonacci's idea to three preceding terms instead of two: `T(n) = T(n-1) + T(n-2) + T(n-3)`, with base cases `T(0)=0, T(1)=0, T(2)=1`. Same iterative approach as Fibonacci, just tracking three running values instead of two.

### Code

```java
public class Tribonacci {
    public static void main(String[] args) {
        for (int n = 0; n < 10; n++) {
            System.out.println("T(" + n + ") = " + tribonacci(n));
        }
    }

    static long tribonacci(int n) {
        if (n == 0 || n == 1) return 0;
        if (n == 2) return 1;

        long a = 0, b = 0, c = 1;
        for (int i = 3; i <= n; i++) {
            long next = a + b + c;
            a = b;
            b = c;
            c = next;
        }
        return c;
    }
}
```

### Example

```
T(0) = 0
T(1) = 0
T(2) = 1
T(3) = 1
T(4) = 2
T(5) = 4
T(6) = 7
T(7) = 13
T(8) = 24
T(9) = 44
```

## 5. Catalan Numbers

### Theory

**Catalan numbers** `Cₙ` count an enormous variety of combinatorial structures: the number of valid balanced-parenthesis sequences of length `2n`, the number of distinct binary search trees with `n` nodes, the number of ways to triangulate a convex polygon, and more. They satisfy the recurrence:

```
C₀ = 1
Cₙ = Σ (Cᵢ × C(n-1-i))   for i = 0 to n-1
```

There's also a direct **O(1)** formula (given `nCr`, see [Lesson 14](<./14. Combinatorics.md#3-ncr>)):

```
Cₙ = C(2n, n) / (n + 1)
```

### Code

```java
public class CatalanNumbers {
    public static void main(String[] args) {
        long[] table = buildCatalanTable(7);
        for (int n = 0; n <= 7; n++) {
            System.out.println("C(" + n + ") = " + table[n]);
        }
    }

    // Direct recurrence, O(n^2)
    static long[] buildCatalanTable(int n) {
        long[] catalan = new long[n + 1];
        catalan[0] = 1;
        for (int i = 1; i <= n; i++) {
            long sum = 0;
            for (int j = 0; j < i; j++) {
                sum += catalan[j] * catalan[i - 1 - j];
            }
            catalan[i] = sum;
        }
        return catalan;
    }
}
```

### Example

```
C(0) = 1
C(1) = 1
C(2) = 2
C(3) = 5
C(4) = 14
C(5) = 42
C(6) = 132
C(7) = 429
```

### Problem (Basic): Unique Binary Search Trees

**Theory:** **Classic example — Unique Binary Search Trees:** count how many structurally distinct BSTs can be built using exactly `n` distinct values `1..n`. Picking any value `i` as the root splits the remaining values into a **left subtree** (the `i-1` smaller values) and a **right subtree** (the `n-i` larger values) — since BST structure only depends on subtree *size*, not the actual values, the count is `Σ (ways to arrange i-1 values) × (ways to arrange n-i values)` for every choice of root `i` — precisely the Catalan number recurrence.

```java
public class UniqueBinarySearchTrees {
    public static void main(String[] args) {
        System.out.println("numTrees(0): " + numTrees(0));
        System.out.println("numTrees(3): " + numTrees(3));
        System.out.println("numTrees(4): " + numTrees(4));
    }

    static long numTrees(int n) {
        long[] catalan = new long[n + 1];
        catalan[0] = 1;
        for (int i = 1; i <= n; i++) {
            long sum = 0;
            for (int j = 0; j < i; j++) {
                sum += catalan[j] * catalan[i - 1 - j];
            }
            catalan[i] = sum;
        }
        return catalan[n];
    }
}
```

```
numTrees(0): 1
numTrees(3): 5
numTrees(4): 14
```

`numTrees(3) = 5` matches `C₃` from the Catalan table above directly — every BST-counting instance of this shape reduces to the same underlying sequence.

## Quick Reference Cheat Sheet

| Sequence | Recurrence | Fast computation |
|---|---|---|
| Fibonacci | `F(n) = F(n-1) + F(n-2)` | Matrix exponentiation, O(log n) (→ Climbing Stairs) |
| Arithmetic Progression | `aₙ = aₙ₋₁ + d` | Closed form, O(1) |
| Geometric Progression | `aₙ = aₙ₋₁ × r` | Closed form, O(1) (special-case `r = 1`) |
| Tribonacci | `T(n) = T(n-1)+T(n-2)+T(n-3)` | Iterative, O(n) |
| Catalan Numbers | `Cₙ = Σ Cᵢ·C(n-1-i)` | `C(2n,n)/(n+1)`, O(1) given `nCr` (→ Unique Binary Search Trees) |
