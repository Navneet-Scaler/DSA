# 9. Factorials

Factorials are the backbone of combinatorics (permutations, combinations, `nCr`) and show up constantly in probability and counting problems. They also grow *ferociously* fast — this lesson covers both computation styles, where overflow hits, how to count trailing zeros without computing the whole factorial, and how to handle factorials too large for any primitive type.

## 1. Factorial

### Theory

The factorial of a non-negative integer `n`, written `n!`, is the product of all positive integers up to `n`:

```
n! = n × (n-1) × (n-2) × ... × 2 × 1
0! = 1     (defined by convention, not computed — the "empty product")
```

Factorials grow **extremely fast** — `13!` already overflows a 32-bit `int`, and `21!` overflows even a 64-bit `long`. Choosing the right data type (or switching to `BigInteger`) is the central challenge of working with factorials in Java.

---

## 2. Recursive Factorial

### Theory

Factorial has a natural recursive definition — `n! = n × (n-1)!`, with base case `0! = 1`. This maps directly onto a recursive function. The downside: each call adds a stack frame, so very large `n` risks a `StackOverflowError` — the iterative version (next section) avoids this entirely.

### Code

```java
public class RecursiveFactorial {
    public static void main(String[] args) {
        System.out.println("5! (recursive): " + factorial(5));
        System.out.println("0! (recursive): " + factorial(0));
    }

    static long factorial(int n) {
        if (n == 0) return 1;
        return n * factorial(n - 1);
    }
}
```

### Example

```
5! (recursive): 120
0! (recursive): 1
```

## 3. Iterative Factorial

### Theory

The loop-based version avoids recursion overhead and stack-depth limits entirely — it's the preferred approach for larger `n`. But it doesn't avoid **overflow**: watch the type carefully.

| Value | `int` (32-bit) safe? | `long` (64-bit) safe? |
|---|---|---|
| `12!` = 479,001,600 | Yes | Yes |
| `13!` = 6,227,020,800 | **No — overflows** | Yes |
| `20!` = 2,432,902,008,176,640,000 | No | Yes |
| `21!` = 51,090,942,171,709,440,000 | No | **No — overflows** |

### Code

```java
public class IterativeFactorial {
    public static void main(String[] args) {
        System.out.println("5! (iterative):  " + factorial(5));

        // Overflow demonstration
        int factorial13AsInt = factorialInt(13);
        System.out.println("13! as int (overflowed): " + factorial13AsInt);

        long factorial13AsLong = factorial(13);
        System.out.println("13! as long (correct):   " + factorial13AsLong);
    }

    static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }

    static int factorialInt(int n) {
        int result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }
}
```

### Example

```
5! (iterative):  120
13! as int (overflowed): 1932053504
13! as long (correct):   6227020800
```

## 4. Trailing Zeros

### Theory

Directly computing `n!` just to count its trailing zeros is wasteful (and impossible once it overflows). Instead, use **Legendre's formula**: a trailing zero comes from a factor of `10 = 2 × 5`, and factors of `5` are always scarcer than factors of `2` in `n!`, so the number of trailing zeros equals the number of times `5` divides into `n!`:

```
trailingZeros(n!) = ⌊n/5⌋ + ⌊n/25⌋ + ⌊n/125⌋ + ...   (until the term becomes 0)
```

This runs in **O(log₅ n)** — no factorial computation needed at all.

### Code

```java
public class TrailingZerosInFactorial {
    public static void main(String[] args) {
        System.out.println("Trailing zeros in 10!:  " + trailingZeros(10));
        System.out.println("Trailing zeros in 25!:  " + trailingZeros(25));
        System.out.println("Trailing zeros in 100!: " + trailingZeros(100));
    }

    static int trailingZeros(int n) {
        int count = 0;
        for (long p = 5; p <= n; p *= 5) {
            count += n / p;
        }
        return count;
    }
}
```

### Example

```
Trailing zeros in 10!:  2
Trailing zeros in 25!:  6
Trailing zeros in 100!: 24
```

## 5. Large Factorials

### Theory

Once `n` exceeds `20`, even `long` isn't enough — the only exact option is `java.math.BigInteger`, which supports arbitrary-precision integer arithmetic (at the cost of being slower than primitive math).

### Code

```java
import java.math.BigInteger;

public class LargeFactorials {
    public static void main(String[] args) {
        System.out.println("25! (BigInteger): " + factorial(25));
    }

    static BigInteger factorial(int n) {
        BigInteger result = BigInteger.ONE;
        for (int i = 2; i <= n; i++) {
            result = result.multiply(BigInteger.valueOf(i));
        }
        return result;
    }
}
```

### Example

```
25! (BigInteger): 15511210043330985984000000
```

## 6. Factorial Modulo

### Theory

When a problem asks for `n!` **modulo** some large prime (commonly `10^9 + 7`), you don't need `BigInteger` at all — just apply the modulus after every multiplication, keeping every intermediate value bounded within a `long`. This is the same trick used in [Modular Exponentiation](<./7. Powers and Exponentiation.md#4-modular-exponentiation>):

```
(x * y) % m = ((x % m) * (y % m)) % m
```

> **For repeated queries** (e.g. many `nCr mod p` calls), precompute a `factorial[]` array once up to the largest `n` needed, so each query becomes an **O(1)** array lookup instead of recomputing the product from scratch.

### Code

```java
public class FactorialModulo {
    public static void main(String[] args) {
        long MOD = 1_000_000_007L;

        System.out.println("20! mod 1e9+7: " + factorialMod(20, MOD));
        System.out.println("50! mod 1e9+7: " + factorialMod(50, MOD));

        // Precomputed array for repeated queries
        long[] factMod = precomputeFactorials(20, MOD);
        System.out.println("Precomputed 20! mod 1e9+7: " + factMod[20]);
        System.out.println("Precomputed 10! mod 1e9+7: " + factMod[10]);
    }

    static long factorialMod(int n, long mod) {
        long result = 1;
        for (int i = 2; i <= n; i++) {
            result = (result * i) % mod;
        }
        return result;
    }

    static long[] precomputeFactorials(int n, long mod) {
        long[] fact = new long[n + 1];
        fact[0] = 1;
        for (int i = 1; i <= n; i++) {
            fact[i] = (fact[i - 1] * i) % mod;
        }
        return fact;
    }
}
```

### Example

```
20! mod 1e9+7: 146326063
50! mod 1e9+7: 318608048
Precomputed 20! mod 1e9+7: 146326063
Precomputed 10! mod 1e9+7: 3628800
```

## Quick Reference Cheat Sheet

| Task | Approach |
|---|---|
| Small `n` (≤ 12) | `int` is safe |
| Medium `n` (13–20) | Use `long` |
| Large `n` (≥ 21) | Use `BigInteger` for the exact value |
| Trailing zeros in `n!` | Legendre's formula — count factors of 5, don't compute `n!` |
| `n!` modulo `p` (single query) | Loop multiplying `% p` each step |
| `n!` modulo `p` (many queries) | Precompute a `factorial[]` array once, O(1) lookups after |
