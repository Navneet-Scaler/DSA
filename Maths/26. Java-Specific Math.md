## 1. Math Class

### Theory

`java.lang.Math` is a **final class of static methods** — it can't be instantiated (its constructor is private) and every method is called directly on the class, like `Math.sqrt(x)`. It also exposes two constants: `Math.PI` and `Math.E`, both `double`s carrying full precision. Every method covered in this lesson lives here.

---

## 2. Math.pow

### Theory

`Math.pow(base, exponent)` computes a power using **floating point**, returning a `double` even for integer inputs. For exact integer powers or modular exponentiation, prefer manual [binary exponentiation](<./07. Powers and Exponentiation.md#3-binary-exponentiation>) instead — full theory and the negative/fractional-exponent gotchas are in [Lesson 7](<./07. Powers and Exponentiation.md>).

### Code

```java
public class MathPow {
    public static void main(String[] args) {
        System.out.println("Math.pow(2, 10): " + Math.pow(2, 10));
    }
}
```

### Example

```
Math.pow(2, 10): 1024.0
```

## 3. Math.sqrt

### Theory

`Math.sqrt(x)` returns the square root as a `double`. For an **exact integer** square root (no floating-point rounding risk), use the [binary-search approach](<./21. Binary Search on Answer.md#1-square-root>) from Lesson 21 instead.

### Code

```java
public class MathSqrt {
    public static void main(String[] args) {
        System.out.println("Math.sqrt(64): " + Math.sqrt(64));
    }
}
```

### Example

```
Math.sqrt(64): 8.0
```

## 5. Math.max

### Theory

`Math.max(a, b)` returns the larger of two values, overloaded for all primitive numeric types. For finding the max across an **array**, loop and track the running maximum (or use `Arrays.stream(arr).max()` for an `int[]`).

### Code

```java
public class MathMax {
    public static void main(String[] args) {
        System.out.println("Math.max(3, 7): " + Math.max(3, 7));
    }
}
```

### Example

```
Math.max(3, 7): 7
```

## 9. Math.round

### Theory

`Math.round(x)` rounds to the nearest integer using `floor(x + 0.5)` — which means halfway values round toward **positive infinity**, not away from zero (`Math.round(-4.5)` is `-4`, not `-5`). Full explanation, plus `BigDecimal`'s more controllable `RoundingMode` alternatives, is in [Lesson 11](<./11. Decimal Mathematics.md#3-rounding>).

### Code

```java
public class MathRound {
    public static void main(String[] args) {
        System.out.println("Math.round(4.5): " + Math.round(4.5));
        System.out.println("Math.round(-4.5): " + Math.round(-4.5)); // -4, not -5
    }
}
```

### Example

```
Math.round(4.5): 5
Math.round(-4.5): -4
```

## 10. Random

### Theory

`java.util.Random` generates pseudo-random values, deterministically reproducible from a given seed — the same seed always produces the same sequence, which is exactly what makes it useful for reproducible tests. Full coverage — shuffling, reservoir sampling, and uniform distributions — is in [Lesson 19](<./19. Probability.md>).

### Code

```java
import java.util.Random;

public class RandomExample {
    public static void main(String[] args) {
        Random random = new Random(99);
        System.out.println("Three calls to nextInt(100), seed 99:");
        System.out.println("  " + random.nextInt(100));
        System.out.println("  " + random.nextInt(100));
        System.out.println("  " + random.nextInt(100));
    }
}
```

### Example

```
Three calls to nextInt(100), seed 99:
  87
  58
  29
```

## 11. BigInteger

### Theory

`java.math.BigInteger` represents integers of **arbitrary size**, limited only by available memory — essential once a value exceeds `long`'s range (`~9.2 × 10^18`). Full coverage (including its use for large factorials) is in [Lesson 2](<./02. Number Representation.md#10-integer-limits-java>) and [Lesson 9](<./09. Factorials.md#5-large-factorials>).

### Code

```java
import java.math.BigInteger;

public class BigIntegerExample {
    public static void main(String[] args) {
        BigInteger result = BigInteger.ONE;
        for (int i = 2; i <= 30; i++) {
            result = result.multiply(BigInteger.valueOf(i));
        }
        System.out.println("30!: " + result);
    }
}
```

### Example

```
30!: 265252859812191058636308480000000
```

## 12. BigDecimal

### Theory

`java.math.BigDecimal` represents **decimal** numbers exactly, with explicit control over rounding — essential for money or any calculation where `double`'s binary floating-point rounding is unacceptable. Full coverage (precision, rounding modes, significant digits) is in [Lesson 11](<./11. Decimal Mathematics.md>).

### Code

```java
import java.math.BigDecimal;
import java.math.RoundingMode;

public class BigDecimalExample {
    public static void main(String[] args) {
        BigDecimal result = BigDecimal.valueOf(22)
                .divide(BigDecimal.valueOf(7), 6, RoundingMode.HALF_UP);
        System.out.println("22/7 to 6 decimal places: " + result);
    }
}
```

### Example

```
22/7 to 6 decimal places: 3.142857
```
