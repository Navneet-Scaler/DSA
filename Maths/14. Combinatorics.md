## 1. Permutations

### Theory

A **permutation** is an arrangement of items where **order matters**. Arranging all `n` distinct items has `n!` possible orders. Arranging only `r` of the `n` items (an "r-permutation") is written `P(n, r)` or `nPr`:

```
nPr = n! / (n - r)!
```

Intuition: there are `n` choices for the first position, `n-1` for the second, ..., down to `n-r+1` for the `r`-th position — multiplying those `r` shrinking terms is equivalent to `n! / (n-r)!`.

### Problem 1 (Advanced): Permutation Sequence

**Theory:** Find the `k`-th permutation (in lexicographic order) of `1, 2, ..., n` **directly**, without generating and sorting all `n!` permutations. The key insight: fixing the **first** digit divides all permutations into `n` equal-sized blocks of `(n-1)!` permutations each — so `k / (n-1)!` (using 0-indexed `k`) tells you exactly which first digit to pick. Removing that digit from the pool and repeating with the remainder `k % (n-1)!` peels off one digit at a time, in `O(n²)` (dominated by the `O(n)` removal from a list, done `n` times).

```java
import java.util.ArrayList;
import java.util.List;

public class PermutationSequence {
    public static void main(String[] args) {
        System.out.println("getPermutation(3, 3): " + getPermutation(3, 3));
        System.out.println("getPermutation(4, 9): " + getPermutation(4, 9));
        System.out.println("getPermutation(3, 1): " + getPermutation(3, 1));
    }

    static String getPermutation(int n, int k) {
        List<Integer> numbers = new ArrayList<>();
        int[] fact = new int[n + 1];
        fact[0] = 1;
        for (int i = 1; i <= n; i++) {
            numbers.add(i);
            fact[i] = fact[i - 1] * i;
        }

        k--; // convert to 0-indexed
        StringBuilder result = new StringBuilder();
        for (int i = n; i >= 1; i--) {
            int blockSize = fact[i - 1];
            int index = k / blockSize;
            result.append(numbers.remove(index));
            k %= blockSize;
        }
        return result.toString();
    }
}
```

```
getPermutation(3, 3): 213
getPermutation(4, 9): 2314
getPermutation(3, 1): 123
```

For `n=3`: the 6 permutations in order are `123, 132, 213, 231, 312, 321` — the 3rd one is indeed `"213"`, found directly by picking each digit's block rather than generating all 6.

### Problem 2 (Medium): Count Numbers with Unique Digits

**Theory:** Count how many numbers in `[0, 10^n)` have **all distinct digits** (no repeated digit). Rather than checking every number individually, count directly by digit-length: numbers of exactly `length` digits (no leading zero) have `9` choices for the first digit, then `9, 8, 7, ...` shrinking choices for each subsequent digit (an `nPr`-style count, since order/position matters and repeats aren't allowed) — sum this across every length from `1` to `n`, plus `1` for the number `0` itself.

```java
public class CountNumbersWithUniqueDigits {
    public static void main(String[] args) {
        for (int n = 0; n <= 3; n++) {
            System.out.println("countNumbersWithUniqueDigits(" + n + "): " + countNumbersWithUniqueDigits(n));
        }
    }

    static int countNumbersWithUniqueDigits(int n) {
        if (n == 0) return 1;

        int total = 10; // all single-digit numbers 0-9
        int uniqueDigitsCount = 9; // count of length-`length` numbers with all-unique digits
        int availableDigits = 9;

        for (int length = 2; length <= n; length++) {
            uniqueDigitsCount *= availableDigits;
            total += uniqueDigitsCount;
            availableDigits--;
        }
        return total;
    }
}
```

```
countNumbersWithUniqueDigits(0): 1
countNumbersWithUniqueDigits(1): 10
countNumbersWithUniqueDigits(2): 91
countNumbersWithUniqueDigits(3): 739
```

For `n=2` (numbers `0` to `99`): `10` single-digit numbers, plus `9 × 9 = 81` two-digit numbers with distinct digits (first digit `1-9`, second digit anything but the first) — `10 + 81 = 91`.

## 2. Combinations

### Theory

A **combination** is a selection of items where **order does not matter** — choosing `{1, 2, 3}` is the same as choosing `{3, 2, 1}`. Since each combination of `r` items can be arranged in `r!` different orders, combinations are permutations **divided by the overcounting factor** `r!`:

```
nCr = nPr / r! = n! / (r! × (n - r)!)
```

## 3. nCr

### Theory

`nCr` ("n choose r") counts combinations, as above. There are three practical ways to compute it, each suited to a different situation:

1. **Direct factorial formula** — simple, but factorials overflow fast (see [Lesson 9](<./09. Factorials.md>)), so only safe for small `n`.
2. **Pascal's Triangle recurrence** — avoids factorials and overflow entirely, good for building a full table of values.
3. **Modular arithmetic with precomputed factorials + modular inverse** — the standard approach for large `n` where the answer is wanted `mod p` (see [Lesson 13](<./13. Modular Arithmetic.md#4-mod-division>)).

### Code

```java
public class NCr {
    public static void main(String[] args) {
        System.out.println("C(5, 3) [direct factorial]: " + nCrDirect(5, 3));

        long MOD = 1_000_000_007L;
        long[] fact = precomputeFactorials(1000, MOD);
        System.out.println("C(1000, 500) mod 1e9+7 [precomputed + modular inverse]: "
                + nCrMod(1000, 500, fact, MOD));
    }

    static long nCrDirect(int n, int r) {
        return factorial(n) / (factorial(r) * factorial(n - r));
    }

    static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) result *= i;
        return result;
    }

    static long[] precomputeFactorials(int n, long mod) {
        long[] fact = new long[n + 1];
        fact[0] = 1;
        for (int i = 1; i <= n; i++) fact[i] = (fact[i - 1] * i) % mod;
        return fact;
    }

    static long nCrMod(int n, int r, long[] fact, long mod) {
        if (r < 0 || r > n) return 0;
        long denominator = (fact[r] * fact[n - r]) % mod;
        return (fact[n] * modInverse(denominator, mod)) % mod;
    }

    static long modInverse(long a, long mod) {
        return modPow(a, mod - 2, mod); // valid since mod (1e9+7) is prime -- Fermat's Little Theorem
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = (result * base) % mod;
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
C(5, 3) [direct factorial]: 10
C(1000, 500) mod 1e9+7 [precomputed + modular inverse]: 159835829
```

## 4. nPr

### Theory

`nPr` counts permutations directly, as introduced above:

```
nPr = n! / (n - r)!
```

### Code

```java
public class NPr {
    public static void main(String[] args) {
        System.out.println("P(5, 3): " + nPr(5, 3));
    }

    static long nPr(int n, int r) {
        return factorial(n) / factorial(n - r);
    }

    static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) result *= i;
        return result;
    }
}
```

### Example

```
P(5, 3): 60
```

## 5. Pascal's Triangle

### Theory

Pascal's Triangle is a triangular array where each entry is `nCr` for its row `n` and position `r`, built purely from **addition** (no factorials, no overflow risk for the values themselves) using the recurrence:

```
C(n, r) = C(n-1, r-1) + C(n-1, r)
C(n, 0) = C(n, n) = 1
```

Intuition: to choose `r` items from `n`, either the `n`-th item is **excluded** (choose `r` from the remaining `n-1`) or **included** (choose `r-1` from the remaining `n-1`) — these two cases are disjoint and cover everything.

### Code

```java
public class PascalsTriangle {
    public static void main(String[] args) {
        int[][] triangle = buildPascalsTriangle(6);
        for (int[] row : triangle) {
            StringBuilder line = new StringBuilder();
            for (int value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }

    static int[][] buildPascalsTriangle(int rows) {
        int[][] triangle = new int[rows][];
        for (int n = 0; n < rows; n++) {
            triangle[n] = new int[n + 1];
            triangle[n][0] = triangle[n][n] = 1;
            for (int r = 1; r < n; r++) {
                triangle[n][r] = triangle[n - 1][r - 1] + triangle[n - 1][r];
            }
        }
        return triangle;
    }
}
```

### Example

```
1
1 1
1 2 1
1 3 3 1
1 4 6 4 1
1 5 10 10 5 1
```

Row `5` reads `1 5 10 10 5 1` — matching `C(5,0)=1, C(5,1)=5, C(5,2)=10, C(5,3)=10, C(5,4)=5, C(5,5)=1`.

### Problem (Basic): Pascal's Triangle II — Single Row

**Theory:** Return **only** row `rowIndex` of Pascal's Triangle, without building every row above it. Rather than the additive recurrence (which needs the previous full row), build the row directly left to right using the multiplicative relationship between consecutive entries in the same row: `C(n, i) = C(n, i-1) × (n - i + 1) / i` — each entry is derived from its immediate left neighbor with one multiplication and one division, needing only `O(rowIndex)` space instead of `O(rowIndex²)` for the full triangle.

```java
import java.util.ArrayList;
import java.util.List;

public class PascalsTriangleII {
    public static void main(String[] args) {
        System.out.println("getRow(3): " + getRow(3));
        System.out.println("getRow(0): " + getRow(0));
        System.out.println("getRow(5): " + getRow(5));
    }

    static List<Integer> getRow(int rowIndex) {
        List<Integer> row = new ArrayList<>();
        row.add(1);
        for (int i = 1; i <= rowIndex; i++) {
            long next = (long) row.get(i - 1) * (rowIndex - i + 1) / i;
            row.add((int) next);
        }
        return row;
    }
}
```

```
getRow(3): [1, 3, 3, 1]
getRow(0): [1]
getRow(5): [1, 5, 10, 10, 5, 1]
```

Same values as the full-triangle build, but row `5` is produced directly in one pass rather than as a byproduct of building rows `0` through `4` first.

## 6. Inclusion-Exclusion

### Theory

The **Inclusion-Exclusion Principle** counts the size of a **union** of sets without double-counting overlaps. For two sets:

```
|A ∪ B| = |A| + |B| - |A ∩ B|
```

For three sets, you add back the triple overlap that got subtracted too many times:

```
|A ∪ B ∪ C| = |A| + |B| + |C| - |A∩B| - |A∩C| - |B∩C| + |A∩B∩C|
```

**Classic application:** counting numbers up to `N` divisible by `3` or `5` (the "FizzBuzz sum" problem) — you can't just add "multiples of 3" and "multiples of 5" because multiples of `15` would be counted twice.

### Code

```java
public class InclusionExclusion {
    public static void main(String[] args) {
        int n = 30;
        long divisibleBy3Or5 = countDivisibleByAOrB(n, 3, 5);
        System.out.println("Numbers from 1 to " + n + " divisible by 3 or 5: " + divisibleBy3Or5);
    }

    static long countDivisibleByAOrB(int n, int a, int b) {
        long countA = n / a;
        long countB = n / b;
        long lcmAB = lcm(a, b);
        long countBoth = n / lcmAB;
        return countA + countB - countBoth; // inclusion-exclusion for two sets
    }

    static long lcm(int a, int b) {
        return (long) a / gcd(a, b) * b;
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
Numbers from 1 to 30 divisible by 3 or 5: 14
```

### Problem (Medium): Count Numbers Divisible by At Least One of Three Divisors

**Theory:** Extend the two-set case to **three** divisors, directly implementing the 3-set formula from the Theory section above: `|A∪B∪C| = |A|+|B|+|C| - |A∩B|-|A∩C|-|B∩C| + |A∩B∩C|`. Each pairwise or triple "intersection count" (numbers divisible by **both**/all of a group of divisors) is just "numbers divisible by their [LCM](<./06. GCD and LCM.md#4-lcm>)" — reusing LCM the same way the two-set version did.

```java
public class CountDivisibleByAtLeastOneOfThree {
    public static void main(String[] args) {
        System.out.println("countDivisibleByAtLeastOneOfThree(30,2,3,5): "
                + countDivisibleByAtLeastOneOfThree(30, 2, 3, 5));
        System.out.println("countDivisibleByAtLeastOneOfThree(100,2,3,5): "
                + countDivisibleByAtLeastOneOfThree(100, 2, 3, 5));
    }

    static long countDivisibleByAtLeastOneOfThree(int n, int a, int b, int c) {
        long countA = n / a, countB = n / b, countC = n / c;
        long countAB = n / lcm(a, b);
        long countAC = n / lcm(a, c);
        long countBC = n / lcm(b, c);
        long countABC = n / lcm(lcm(a, b), c);
        return countA + countB + countC - countAB - countAC - countBC + countABC;
    }

    static long lcm(int a, int b) {
        return (long) a / gcd(a, b) * b;
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

```
countDivisibleByAtLeastOneOfThree(30,2,3,5): 22
countDivisibleByAtLeastOneOfThree(100,2,3,5): 74
```

Of the numbers `1` to `30`, `22` are divisible by `2`, `3`, or `5` — the triple-overlap term `countABC` (multiples of `30`) corrects for `30` itself being subtracted three times (once per pair) and needing to be added back once.

## 7. Stars and Bars

### Theory

**Stars and Bars** counts the number of ways to distribute `n` identical items into `k` distinct groups (each group can receive zero or more items). Picture `n` stars (items) and `k-1` bars (dividers splitting them into `k` groups) arranged in a row — the count is the number of ways to place those bars among the stars:

```
ways = C(n + k - 1, k - 1)
```

**Classic application:** "how many ways can you give `n` identical candies to `k` children?" (children may receive zero candies).

### Code

```java
public class StarsAndBars {
    public static void main(String[] args) {
        int candies = 5, children = 3;
        System.out.println("Ways to distribute " + candies + " candies among " + children
                + " children: " + waysToDistribute(candies, children));
    }

    static long waysToDistribute(int items, int groups) {
        return nCr(items + groups - 1, groups - 1);
    }

    static long nCr(int n, int r) {
        return factorial(n) / (factorial(r) * factorial(n - r));
    }

    static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) result *= i;
        return result;
    }
}
```

### Example

```
Ways to distribute 5 candies among 3 children: 21
```

## Quick Reference Cheat Sheet

| Concept | Formula |
|---|---|
| Permutations (all `n`) | `n!` (→ Permutation Sequence, Count Numbers with Unique Digits) |
| Permutations (`r` of `n`) — nPr | `n! / (n-r)!` |
| Combinations — nCr | `n! / (r! × (n-r)!)` |
| nCr (modular, large n) | precompute `factorial[]` mod `p`, use modular inverse for division |
| Pascal's Triangle | `C(n,r) = C(n-1,r-1) + C(n-1,r)` (→ Pascal's Triangle II, single-row) |
| Inclusion-Exclusion (2 sets) | `|A∪B| = |A| + |B| - |A∩B|` (→ 3-set version: Count Divisible by At Least One of Three) |
| Stars and Bars | `C(n + k - 1, k - 1)` — distribute `n` items into `k` groups |
