# 3. Digits

Almost every "number manipulation" interview question — palindrome check, Armstrong numbers, digit DP, checksum validation — reduces to pulling a number apart digit by digit. This lesson builds the standard toolkit for working with individual digits in Java, using the repeated **`% 10` (extract) / `/ 10` (drop)** pattern introduced in the previous lesson.

## 1. Extract Digit

### Theory

To pull out the digit at a specific position `k` (counting from the right, starting at `0`), divide by `10^k` to shift that digit into the units place, then take `% 10` to isolate it:

```
digit at position k = (n / 10^k) % 10
```

For `n = 4021`: position `0` → `1`, position `1` → `2`, position `2` → `0`, position `3` → `4`.

### Code

```java
public class ExtractDigit {
    public static void main(String[] args) {
        int n = 4021;

        for (int k = 0; k < 4; k++) {
            int digit = (n / (int) Math.pow(10, k)) % 10;
            System.out.println("Digit at position " + k + " (from right): " + digit);
        }
    }
}
```

### Example

```
Digit at position 0 (from right): 1
Digit at position 1 (from right): 2
Digit at position 2 (from right): 0
Digit at position 3 (from right): 4
```

## 2. Count Digits

### Theory

Two common approaches:

1. **Loop-based** — repeatedly divide by 10, counting iterations until the number becomes `0`. Reliable and works for any size.
2. **Log-based** — `(int) Math.log10(n) + 1`. Faster (no loop) but only valid for `n > 0`, and can be off by one due to floating-point imprecision on values that sit exactly on a power of 10 — so the loop method is the safer default.

### Code

```java
public class CountDigits {
    public static void main(String[] args) {
        int n = 4021;

        // Loop-based (safe for all cases, including n == 0)
        int count = 0;
        int temp = n;
        if (temp == 0) {
            count = 1; // zero itself has exactly one digit
        }
        while (temp != 0) {
            count++;
            temp /= 10;
        }
        System.out.println("Digit count (loop): " + count);

        // Log-based (fast, but only for n > 0)
        int logCount = (int) Math.log10(n) + 1;
        System.out.println("Digit count (log10): " + logCount);
    }
}
```

### Example

```
Digit count (loop): 4
Digit count (log10): 4
```

## 3. Reverse Digits

### Theory

Build the reversed number by repeatedly taking the last digit off `n` and appending it to a `reversed` accumulator (which shifts left — multiplies by 10 — before adding the new digit).

```
reversed = reversed * 10 + (n % 10)
n = n / 10
```

> Reversing a number that ends in zero(s) naturally **drops** those zeros — `1200` reversed is `21`, not `0021`, since leading zeros aren't stored in an `int`.

### Code

```java
public class ReverseDigits {
    public static void main(String[] args) {
        int n = 4021;
        int reversed = 0;
        int temp = n;

        while (temp != 0) {
            int lastDigit = temp % 10;
            reversed = reversed * 10 + lastDigit;
            temp /= 10;
        }
        System.out.println("Reverse of " + n + ": " + reversed);

        // The trailing-zero gotcha
        int withZeros = 1200;
        int reversedZeros = 0;
        temp = withZeros;
        while (temp != 0) {
            reversedZeros = reversedZeros * 10 + (temp % 10);
            temp /= 10;
        }
        System.out.println("Reverse of " + withZeros + ": " + reversedZeros);
    }
}
```

### Example

```
Reverse of 4021: 1204
Reverse of 1200: 21
```

## 4. Sum of Digits

### Theory

Extract each digit with `% 10` and accumulate a running total, dropping the digit with `/ 10` each time — the same skeleton loop used throughout this lesson.

> **Common use:** repeatedly summing digits until a single digit remains gives the **digital root** — useful in divisibility and checksum problems.

### Code

```java
public class SumOfDigits {
    public static void main(String[] args) {
        int n = 4021;
        int sum = 0;
        int temp = n;

        while (temp != 0) {
            sum += temp % 10;
            temp /= 10;
        }
        System.out.println("Sum of digits of " + n + ": " + sum);
    }
}
```

### Example

```
Sum of digits of 4021: 7
```

## 5. Product of Digits

### Theory

Same loop skeleton, but the accumulator starts at `1` and **multiplies** instead of adds.

> **Gotcha:** if any digit is `0`, the entire product becomes `0` — this is expected mathematical behavior, not a bug, but it's easy to forget when a problem expects the product of *non-zero* digits only (in which case you must explicitly skip zero digits).

### Code

```java
public class ProductOfDigits {
    public static void main(String[] args) {
        int n = 4021;
        long product = 1;
        int temp = n;

        while (temp != 0) {
            product *= (temp % 10);
            temp /= 10;
        }
        System.out.println("Product of digits of " + n + ": " + product); // contains a 0 digit!

        // Product ignoring zero digits
        long productSkipZero = 1;
        temp = n;
        while (temp != 0) {
            int digit = temp % 10;
            if (digit != 0) {
                productSkipZero *= digit;
            }
            temp /= 10;
        }
        System.out.println("Product ignoring zero digits: " + productSkipZero);
    }
}
```

### Example

```
Product of digits of 4021: 0
Product ignoring zero digits: 8
```

## 6. Leading Zeros

### Theory

In decimal, an `int` never actually *stores* leading zeros — `0042` and `42` are the same value, so "leading zeros" of a plain number is `0` by definition. Leading zeros become meaningful in two real situations:

1. **Fixed-width formatting** — e.g. padding `42` to `"0042"` for display (invoice numbers, timestamps).
2. **Binary/bit-level leading zeros** — `Integer.numberOfLeadingZeros(n)` counts how many of the 32 bits before the first `1` bit are zero. This is a genuinely useful DSA tool: it tells you the position of the most-significant set bit, which is the basis for "next power of 2", bit-length calculations, and more.

### Code

```java
public class LeadingZeros {
    public static void main(String[] args) {
        // 1. Fixed-width string padding
        int value = 42;
        String padded = String.format("%05d", value); // pad to width 5 with leading zeros
        System.out.println("42 padded to width 5: " + padded);

        // 2. Bit-level leading zeros
        int n = 5; // binary: ...00000101
        System.out.println("Integer.numberOfLeadingZeros(5): " + Integer.numberOfLeadingZeros(n));
        System.out.println("Integer.numberOfLeadingZeros(1): " + Integer.numberOfLeadingZeros(1));
    }
}
```

### Example

```
42 padded to width 5: 00042
Integer.numberOfLeadingZeros(5): 29
Integer.numberOfLeadingZeros(1): 31
```

## 7. Trailing Zeros

### Theory

Two flavors again:

1. **Decimal trailing zeros of a number** — how many times the number is divisible by 10, found with a loop dividing by 10 while the remainder is 0.
2. **Bit-level trailing zeros** — `Integer.numberOfTrailingZeros(n)` counts zero bits after the last `1` bit, i.e. the position of the lowest set bit. This underlies classic bit-manipulation tricks like isolating the lowest set bit (`n & -n`).

> **Related classic problem:** counting trailing zeros in `N!` (factorial) — that's really "how many times does 10 divide N!", which reduces to counting factors of 5 (since factors of 2 are always more plentiful), via Legendre's formula: `⌊N/5⌋ + ⌊N/25⌋ + ⌊N/125⌋ + ...`. That's a distinct, more advanced problem — flagged here since it's often confused with plain digit trailing zeros.

### Code

```java
public class TrailingZeros {
    public static void main(String[] args) {
        // 1. Decimal trailing zeros
        int n = 1200;
        int count = 0;
        int temp = n;
        while (temp != 0 && temp % 10 == 0) {
            count++;
            temp /= 10;
        }
        System.out.println("Trailing zeros in " + n + ": " + count);

        // 2. Bit-level trailing zeros
        int bits = 8; // binary: 1000
        System.out.println("Integer.numberOfTrailingZeros(8): " + Integer.numberOfTrailingZeros(bits));

        // Isolating the lowest set bit using n & -n
        int x = 12; // binary: 1100
        System.out.println("Lowest set bit of 12 (12 & -12): " + (x & -x));
    }
}
```

### Example

```
Trailing zeros in 1200: 2
Integer.numberOfTrailingZeros(8): 3
Lowest set bit of 12 (12 & -12): 4
```

## 8. Digit Frequency

### Theory

To count how many times each digit (`0`–`9`) appears in a number, use a small array of size 10 as a frequency table: extract each digit and increment its corresponding bucket.

### Code

```java
public class DigitFrequency {
    public static void main(String[] args) {
        int n = 122333;
        int[] frequency = new int[10];
        int temp = n;

        while (temp != 0) {
            int digit = temp % 10;
            frequency[digit]++;
            temp /= 10;
        }

        System.out.println("Digit frequencies in " + n + ":");
        for (int digit = 0; digit <= 9; digit++) {
            if (frequency[digit] > 0) {
                System.out.println(digit + " -> " + frequency[digit] + " time(s)");
            }
        }
    }
}
```

### Example

```
Digit frequencies in 122333:
1 -> 1 time(s)
2 -> 2 time(s)
3 -> 3 time(s)
```

## 9. Last Digit

### Theory

The simplest digit extraction of all — the last digit of `n` is just `n % 10`. This single fact underlies checking even/odd-last-digit patterns, computing units-digit cycles (e.g. units digit of powers), and every loop in this lesson.

### Code

```java
public class LastDigit {
    public static void main(String[] args) {
        int n = 4021;
        System.out.println("Last digit of " + n + ": " + (n % 10));

        // Works for negative numbers too (sign follows dividend, see Modulo lesson)
        int negative = -4021;
        System.out.println("Last digit of " + negative + ": " + (negative % 10));
    }
}
```

### Example

```
Last digit of 4021: 1
Last digit of -4021: -1
```

## 10. First Digit

### Theory

Two common approaches:

1. **Loop-based** — keep dividing by 10 until the number is a single digit (`< 10`).
2. **Log-based** — compute the number of digits `d` via `Math.log10`, then divide by `10^(d-1)`.

### Code

```java
public class FirstDigit {
    public static void main(String[] args) {
        int n = 4021;

        // 1. Loop-based
        int temp = n;
        while (temp >= 10) {
            temp /= 10;
        }
        System.out.println("First digit (loop): " + temp);

        // 2. Log-based
        int digitCount = (int) Math.log10(n) + 1;
        int firstDigit = n / (int) Math.pow(10, digitCount - 1);
        System.out.println("First digit (log10): " + firstDigit);
    }
}
```

### Example

```
First digit (loop): 4
First digit (log10): 4
```

## Quick Reference Cheat Sheet

| Task | Core expression |
|---|---|
| Extract digit at position `k` | `(n / 10^k) % 10` |
| Count digits | loop `n /= 10` counting iterations, or `(int) Math.log10(n) + 1` |
| Reverse digits | `reversed = reversed * 10 + n % 10` each iteration |
| Sum of digits | accumulate `n % 10` each iteration |
| Product of digits | multiply `n % 10` each iteration (watch for `0` digits) |
| Leading zeros (bits) | `Integer.numberOfLeadingZeros(n)` |
| Trailing zeros (decimal) | loop while `n % 10 == 0` |
| Trailing zeros (bits) | `Integer.numberOfTrailingZeros(n)` |
| Digit frequency | `int[10]` bucket array, increment on each extracted digit |
| Last digit | `n % 10` |
| First digit | loop until `n < 10`, or divide by `10^(digitCount - 1)` |
