# 15. Counting Techniques

"How many X are there?" is one of the most common question shapes in DSA — count digits, count qualifying numbers, count arrangements. This lesson gathers the core counting patterns, including **Digit DP**, the technique for counting numbers with a digit-level property across a huge range without enumerating every number.

## 1. Counting Digits

### Theory

Two common digit-counting tasks: the digit count of **one** number (see [Lesson 3](<./03. Digits.md#2-count-digits>) and [Lesson 8](<./08. Logarithms.md#4-digit-count-using-logs>)), and the **total digit count across a range** — e.g. "how many digits total appear when writing 1 to N?" This is a common building block for problems about digit strings.

### Code

```java
public class CountingDigits {
    public static void main(String[] args) {
        System.out.println("Total digits from 1 to 100: " + totalDigitsInRange(1, 100));
    }

    static long totalDigitsInRange(int from, int to) {
        long total = 0;
        for (int n = from; n <= to; n++) {
            total += String.valueOf(n).length();
        }
        return total;
    }
}
```

### Example

```
Total digits from 1 to 100: 192
```

### Problem (Advanced): Number of Digit One

**Theory:** Count the **total number of times** the digit `1` appears across every number from `1` to `n` (not how many numbers contain a `1` — how many `1`s total, since a number like `11` contributes two). Rather than a loop over every number and every digit (too slow for `n` up to `10^9`), process **one digit position at a time** (units, tens, hundreds, ...): for each position, split `n` into the digits to its left (`high`), the digit at that position (`cur`), and the digits to its right (`low`), then count how many times `1` lands at that position using a case split on whether `cur` is `0`, `1`, or `≥2` — each case has a direct closed-form count based on `high` and `low`.

```java
public class NumberOfDigitOne {
    public static void main(String[] args) {
        System.out.println("countDigitOne(13): " + countDigitOne(13));
        System.out.println("countDigitOne(100): " + countDigitOne(100));
        System.out.println("countDigitOne(234): " + countDigitOne(234));
    }

    static long countDigitOne(int n) {
        long count = 0;
        for (long i = 1; i <= n; i *= 10) {
            long divider = i * 10;
            long high = n / divider;
            long cur = (n / i) % 10;
            long low = n % i;

            if (cur == 0) {
                count += high * i;
            } else if (cur == 1) {
                count += high * i + low + 1;
            } else {
                count += (high + 1) * i;
            }
        }
        return count;
    }
}
```

```
countDigitOne(13): 6
countDigitOne(100): 21
countDigitOne(234): 154
```

`1..13` contains six `1`s total: `1, 10, 11 (two 1s), 12, 13` — matching the closed-form position-by-position count exactly, without ever looping through all 13 numbers digit-by-digit.

## 2. Counting Numbers

### Theory

Counting how many numbers in `[1, n]` satisfy a simple property (like divisibility) often has a direct **O(1)** formula instead of a loop — e.g. numbers divisible by `k` up to `n` is just `⌊n / k⌋` (see [Lesson 4: Divisibility](<./04. Divisibility.md>) and [Inclusion-Exclusion](<./14. Combinatorics.md#6-inclusion-exclusion>) for combining multiple conditions).

### Code

```java
public class CountingNumbers {
    public static void main(String[] args) {
        System.out.println("Numbers from 1 to 100 divisible by 7: " + countDivisible(100, 7));
    }

    static long countDivisible(int n, int k) {
        return n / k;
    }
}
```

### Example

```
Numbers from 1 to 100 divisible by 7: 14
```

## 3. Frequency Counting

### Theory

Frequency counting tallies how often each distinct value appears — the same array/hash-map bucket pattern used for [Digit Frequency](<./03. Digits.md#8-digit-frequency>), generalized to any values (characters, array elements, etc.). It's the backbone of anagram checks, mode-finding, and histogram-style problems.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class FrequencyCounting {
    public static void main(String[] args) {
        String s = "mississippi";
        Map<Character, Integer> freq = countFrequency(s);
        System.out.println("Frequency of characters in \"" + s + "\": " + freq);
    }

    static Map<Character, Integer> countFrequency(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) {
            freq.merge(c, 1, Integer::sum);
        }
        return freq;
    }
}
```

### Example

```
Frequency of characters in "mississippi": {m=1, s=4, p=2, i=4}
```

## 4. Digit DP Basics

### Theory

**Digit DP** counts numbers in a range `[0, N]` satisfying a digit-level property (e.g. "no digit is 7") *without* checking every number one by one — essential when `N` is up to `10^18`. It processes `N`'s digits one at a time (most significant first), tracking:

- **`pos`** — which digit position we're filling.
- **`tight`** — whether the digits chosen so far exactly match `N`'s prefix (constraining what digits are still allowed) or have already gone strictly below it (meaning any digit `0-9` is now free).
- Any problem-specific state (e.g. "have we placed a non-zero digit yet", "count of a digit seen so far").

At each position, try every valid digit (`0` to `9`, or up to `N`'s digit if `tight`), recurse, and memoize on `(pos, tight, ...state)` to avoid recomputation.

### Code

```java
public class DigitDpBasics {
    static int[] digits;
    static Long[][][] memo;

    public static void main(String[] args) {
        System.out.println("Count of numbers in [0, 50] with no digit 7: " + countNoSeven(50));
    }

    static long countNoSeven(int n) {
        digits = String.valueOf(n).chars().map(c -> c - '0').toArray();
        memo = new Long[digits.length + 1][2][2];
        long countIncludingZero = solve(0, 1, 0); // 1 = tight, 0 = "started" flag unused as bound here
        return countIncludingZero - 1; // subtract 1 to exclude 0 itself if problem wants 1..n; 0 has no digit 7 so it was counted
    }

    static long solve(int pos, int tight, int startedFlag) {
        if (pos == digits.length) return 1;
        if (memo[pos][tight][startedFlag] != null) return memo[pos][tight][startedFlag];

        int limit = (tight == 1) ? digits[pos] : 9;
        long total = 0;
        for (int d = 0; d <= limit; d++) {
            if (d == 7) continue; // skip the forbidden digit
            int nextTight = (tight == 1 && d == limit) ? 1 : 0;
            total += solve(pos + 1, nextTight, 1);
        }
        return memo[pos][tight][startedFlag] = total;
    }
}
```

### Example

```
Count of numbers in [0, 50] with no digit 7: 45
```

### Problem (Advanced): Numbers At Most N Given Digit Set

**Theory:** Given a set of allowed **digits** (e.g. `{1, 3, 5, 7}`), count how many positive integers `≤ N` can be built using only those digits (repetition allowed). Split into two cases: numbers with **fewer digits** than `N` are unconstrained — any digit from the allowed set at any position, giving `K^length` choices for each shorter length. Numbers with **exactly as many digits as `N`** need a digit-DP-style walk: at each position, any allowed digit **strictly less** than `N`'s digit there contributes `K^(remaining positions)` freely-fillable numbers; an allowed digit **equal** to `N`'s digit continues the constrained ("tight") path; if no allowed digit ever equals `N`'s digit at some position, the tight path dies there (no number matches `N`'s prefix beyond that point).

```java
public class NumbersAtMostNGivenDigitSet {
    public static void main(String[] args) {
        System.out.println("atMostNGivenDigitSet([1,3,5,7], 100): "
                + atMostNGivenDigitSet(new String[] { "1", "3", "5", "7" }, 100));
        System.out.println("atMostNGivenDigitSet([1,4,9], 1000000000): "
                + atMostNGivenDigitSet(new String[] { "1", "4", "9" }, 1000000000));
        System.out.println("atMostNGivenDigitSet([7], 8): "
                + atMostNGivenDigitSet(new String[] { "7" }, 8));
    }

    static long atMostNGivenDigitSet(String[] digits, int n) {
        String s = String.valueOf(n);
        int len = s.length();
        int k = digits.length;
        long total = 0;

        // Numbers with fewer digits than n: fully unconstrained
        for (int length = 1; length < len; length++) {
            total += (long) Math.pow(k, length);
        }

        // Numbers with exactly as many digits as n
        boolean matchedAllPositions = true;
        for (int i = 0; i < len; i++) {
            char target = s.charAt(i);
            boolean foundEqual = false;
            for (String d : digits) {
                if (d.charAt(0) < target) {
                    total += (long) Math.pow(k, len - i - 1);
                } else if (d.charAt(0) == target) {
                    foundEqual = true;
                }
            }
            if (!foundEqual) {
                matchedAllPositions = false;
                break;
            }
        }
        if (matchedAllPositions) total += 1; // n itself is achievable digit-by-digit

        return total;
    }
}
```

```
atMostNGivenDigitSet([1,3,5,7], 100): 20
atMostNGivenDigitSet([1,4,9], 1000000000): 29523
atMostNGivenDigitSet([7], 8): 1
```

With digits `{1,3,5,7}` and `n=100`: every 1-digit (`4`) and 2-digit (`4²=16`) combination qualifies (`20` total), and no 3-digit number `≤ 100` can be built (the smallest 3-digit number using these digits is `111 > 100`).

## 5. Counting Arrangements

### Theory

Counting **distinct arrangements** of a collection with repeated elements needs the **multiset permutation formula**: divide the total arrangements (`n!`) by the factorial of each repeated element's count, since swapping identical elements produces the same arrangement.

```
distinct arrangements = n! / (c₁! × c₂! × ... × cₖ!)
```

where `c₁, c₂, ..., cₖ` are the counts of each distinct value.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class CountingArrangements {
    public static void main(String[] args) {
        System.out.println("Distinct arrangements of \"AABB\": " + countArrangements("AABB"));
    }

    static long countArrangements(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);

        long denominator = 1;
        for (int count : freq.values()) {
            denominator *= factorial(count);
        }
        return factorial(s.length()) / denominator;
    }

    static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) result *= i;
        return result;
    }
}
```

### Example

```
Distinct arrangements of "AABB": 6
```

`AABB`'s 6 distinct arrangements: `AABB, ABAB, ABBA, BAAB, BABA, BBAA`.

## Quick Reference Cheat Sheet

| Technique | Approach |
|---|---|
| Counting Digits (a range) | sum of `String.valueOf(n).length()` per number, or a closed-form digit-block formula (→ Number of Digit One) |
| Counting Numbers (a property) | direct formula when possible (e.g. `n / k` for divisibility) |
| Frequency Counting | hash map / fixed-size array bucket, one pass |
| Digit DP | recursion over digit positions with a `tight` flag + memoization (→ Numbers At Most N Given Digit Set) |
| Counting Arrangements | `n! / (c₁! × c₂! × ... × cₖ!)` for a multiset |
