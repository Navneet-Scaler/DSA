## 1. Bit Operations

### Theory

Four operations cover almost every bit-manipulation need — setting, clearing, toggling, and checking a specific bit position `i` (0-indexed from the right):

```
set bit i:    n | (1 << i)
clear bit i:  n & ~(1 << i)
toggle bit i: n ^ (1 << i)
check bit i:  (n & (1 << i)) != 0
```

### Code

```java
public class BitOperations {
    public static void main(String[] args) {
        int n = 0b1010; // 10

        System.out.println("n = " + Integer.toBinaryString(n));
        System.out.println("Set bit 0: " + Integer.toBinaryString(setBit(n, 0)));
        System.out.println("Clear bit 1: " + Integer.toBinaryString(clearBit(n, 1)));
        System.out.println("Toggle bit 3: " + Integer.toBinaryString(toggleBit(n, 3)));
        System.out.println("Check bit 1: " + checkBit(n, 1));
        System.out.println("Check bit 0: " + checkBit(n, 0));
    }

    static int setBit(int n, int i) { return n | (1 << i); }
    static int clearBit(int n, int i) { return n & ~(1 << i); }
    static int toggleBit(int n, int i) { return n ^ (1 << i); }
    static boolean checkBit(int n, int i) { return (n & (1 << i)) != 0; }
}
```

### Example

```
n = 1010
Set bit 0: 1011
Clear bit 1: 1000
Toggle bit 3: 10
Check bit 1: true
Check bit 0: false
```

## 2. XOR Math

### Theory

XOR (`^`) has algebraic properties that make it uniquely useful:

```
a ^ a = 0        (anything XORed with itself cancels out)
a ^ 0 = a         (XOR with 0 is the identity)
XOR is commutative and associative — order doesn't matter
```

**Classic application — find the single number that appears once while everything else appears in pairs:** XOR the entire array together; every paired value cancels to `0`, leaving only the unpaired value.

### Code

```java
public class XorMath {
    public static void main(String[] args) {
        int[] arr = { 4, 1, 2, 1, 2 };
        System.out.println("Single (unpaired) number: " + findSingleNumber(arr));
    }

    static int findSingleNumber(int[] arr) {
        int result = 0;
        for (int value : arr) {
            result ^= value; // paired values cancel to 0, leaving the unpaired one
        }
        return result;
    }
}
```

### Example

```
Single (unpaired) number: 4
```

`1^1=0`, `2^2=0`, leaving `4^0^0 = 4`.

### Problem (Basic): Missing Number

**Theory:** Given an array containing `n` distinct numbers from `0` to `n` with exactly **one** missing, find it. XOR every array value together **with every index from `0` to `n`** (note: `n+1` numbers in the full range, but only `n` array slots, so index `n` itself is included as an extra XOR term) — every number that's actually present cancels against its matching index, leaving only the missing value unpaired.

```java
public class MissingNumber {
    public static void main(String[] args) {
        System.out.println("missingNumber([3,0,1]): " + missingNumber(new int[] { 3, 0, 1 }));
        System.out.println("missingNumber([0,1]): " + missingNumber(new int[] { 0, 1 }));
    }

    static int missingNumber(int[] nums) {
        int result = nums.length; // account for index n, which has no array slot to pair with
        for (int i = 0; i < nums.length; i++) {
            result ^= i ^ nums[i];
        }
        return result;
    }
}
```

```
missingNumber([3,0,1]): 2
missingNumber([0,1]): 2
```

`[3,0,1]` should contain `{0,1,2,3}` minus one — XORing every index `0..3` against every value `{3,0,1}` cancels `0`, `1`, and `3`, leaving `2`.

## 3. Powers of Two

### Theory

A power of two has exactly **one bit set** in binary (`8 = 1000`, `16 = 10000`). Subtracting `1` from a power of two flips that single bit off and every bit below it on (`8-1=7=0111`), so a power of two ANDed with itself minus one is always `0` — this is the fast exact test (also mentioned in [Lesson 8: Logarithms](<./08. Logarithms.md#3-log2>)):

```
isPowerOfTwo(n) = n > 0 AND (n & (n - 1)) == 0
```

### Code

```java
public class PowersOfTwo {
    public static void main(String[] args) {
        System.out.println("16 is a power of 2: " + isPowerOfTwo(16));
        System.out.println("18 is a power of 2: " + isPowerOfTwo(18));
    }

    static boolean isPowerOfTwo(int n) {
        return n > 0 && (n & (n - 1)) == 0;
    }
}
```

### Example

```
16 is a power of 2: true
18 is a power of 2: false
```

### Problem (Medium): Power of Four

**Theory:** A power of four is a power of two where the single set bit sits **only at an even position** (`1=2⁰, 4=2², 16=2⁴, ...` — never at an odd position like `2=2¹` or `8=2³`). So: first apply the same power-of-two check, then additionally verify the set bit doesn't fall on an odd position, using a mask with `1`s at every odd bit position (`0xAAAAAAAA` in hex — binary `...10101010`): if `n & 0xAAAAAAAA` is nonzero, the set bit is at an odd position, disqualifying it.

```java
public class PowerOfFour {
    public static void main(String[] args) {
        for (int n : new int[] { 16, 5, 1, 8 }) {
            System.out.println("isPowerOfFour(" + n + "): " + isPowerOfFour(n));
        }
    }

    static boolean isPowerOfFour(int n) {
        return n > 0
                && (n & (n - 1)) == 0       // power of two
                && (n & 0xAAAAAAAA) == 0;   // set bit is at an even position
    }
}
```

```
isPowerOfFour(16): true
isPowerOfFour(5): false
isPowerOfFour(1): true
isPowerOfFour(8): false
```

`8 = 2³` is a power of two, but its set bit sits at odd position `3` — the mask check correctly rejects it, while `16 = 2⁴` (even position `4`) passes.

## 4. Gray Code

### Theory

**Gray code** is a binary numbering system where **consecutive values differ by exactly one bit** — useful for minimizing bit-flips in hardware counters, error correction, and combinatorial generation problems. The `n`-th Gray code value converts directly from ordinary binary:

```
gray(n) = n XOR (n >> 1)
```

### Code

```java
public class GrayCode {
    public static void main(String[] args) {
        System.out.println("3-bit Gray code sequence:");
        for (int i = 0; i < 8; i++) {
            int g = gray(i);
            System.out.println("  " + i + " -> " + String.format("%3s", Integer.toBinaryString(g)).replace(' ', '0'));
        }
    }

    static int gray(int n) {
        return n ^ (n >> 1);
    }
}
```

### Example

```
3-bit Gray code sequence:
  0 -> 000
  1 -> 001
  2 -> 011
  3 -> 010
  4 -> 110
  5 -> 111
  6 -> 101
  7 -> 100
```

Check any adjacent pair — `011` (3) and `010` (2) differ in exactly one bit, and so does every other consecutive pair, including the wraparound from `100` (7's Gray code) back to `000` (0's).

## 5. Population Count

### Theory

The **population count** (popcount) of a number is how many bits are set to `1`. **Brian Kernighan's trick** counts them efficiently: `n & (n-1)` always clears the **lowest set bit**, so repeating this until `n` becomes `0` counts exactly as many times as there were set bits — **O(number of set bits)**, faster than checking all 32 bit positions individually. Java also has a direct built-in: `Integer.bitCount(n)`.

### Code

```java
public class PopulationCount {
    public static void main(String[] args) {
        int n = 29; // binary: 11101

        System.out.println("popcount(29) [Brian Kernighan]: " + popcountKernighan(n));
        System.out.println("popcount(29) [Integer.bitCount]: " + Integer.bitCount(n));
    }

    static int popcountKernighan(int n) {
        int count = 0;
        while (n != 0) {
            n &= (n - 1); // clears the lowest set bit
            count++;
        }
        return count;
    }
}
```

### Example

```
popcount(29) [Brian Kernighan]: 4
popcount(29) [Integer.bitCount]: 4
```

`29 = 11101` in binary — 4 bits set, confirmed by both methods.

### Problem (Advanced): Counting Bits

**Theory:** **Classic example — Counting Bits:** compute the popcount of **every** number from `0` to `n`, not just one. Running Brian Kernighan's trick independently for each number costs `O(n log(max))` total. A DP relation does better: `bits[i] = bits[i >> 1] + (i & 1)` — shifting `i` right by one drops its last bit, so the popcount of `i` is the popcount of `i >> 1` (already computed, since `i >> 1 < i`) plus `1` if that dropped bit was a `1`. This computes the entire table in **O(n)** total.

```java
public class CountingBits {
    public static void main(String[] args) {
        System.out.println("countBits(5): " + java.util.Arrays.toString(countBits(5)));
        System.out.println("countBits(2): " + java.util.Arrays.toString(countBits(2)));
    }

    static int[] countBits(int n) {
        int[] result = new int[n + 1];
        for (int i = 1; i <= n; i++) {
            result[i] = result[i >> 1] + (i & 1);
        }
        return result;
    }
}
```

```
countBits(5): [0, 1, 1, 2, 1, 2]
countBits(2): [0, 1, 1]
```

`countBits(5)`: `5 = 101` has `2` set bits, computed as `bits[2] (=1, since 2=10) + (5 & 1 = 1) = 2` — reusing the already-computed answer for `2` instead of recounting `5`'s bits from scratch.

## Quick Reference Cheat Sheet

| Operation | Expression |
|---|---|
| Set bit `i` | `n \| (1 << i)` |
| Clear bit `i` | `n & ~(1 << i)` |
| Toggle bit `i` | `n ^ (1 << i)` |
| Check bit `i` | `(n & (1 << i)) != 0` |
| XOR self-cancel | `a ^ a = 0`, `a ^ 0 = a` (→ Single Number, Missing Number) |
| Power-of-two check | `n > 0 && (n & (n-1)) == 0` (→ Power of Four) |
| Gray code | `n ^ (n >> 1)` |
| Population count | Brian Kernighan's `n &= (n-1)` loop, or `Integer.bitCount(n)` (→ Counting Bits, O(n) DP) |
