# 25. String Mathematics

When a number is too large for even a `long` — or arrives as a string in the first place — arithmetic has to be done **digit by digit**, exactly like the addition and multiplication you learned by hand in school. This lesson covers that grade-school-algorithm approach in Java, plus the binary-string and decimal-string variants of the same idea. (For when `BigInteger`/`BigDecimal` is an acceptable shortcut instead, see [Lesson 26: Java-Specific Math](<./26. Java-Specific Math.md>).)

## 1. Big Number Addition

### Theory

To add two numbers given as strings (too large for any primitive type), simulate grade-school column addition: walk both strings from the **rightmost** digit, add corresponding digits plus any carry, record the result digit, and carry the rest forward. Continue until both strings *and* the carry are exhausted.

### Code

```java
public class BigNumberAddition {
    public static void main(String[] args) {
        System.out.println("99999999999999999999 + 1 = " + addStrings("99999999999999999999", "1"));
    }

    static String addStrings(String num1, String num2) {
        StringBuilder result = new StringBuilder();
        int i = num1.length() - 1, j = num2.length() - 1;
        int carry = 0;

        while (i >= 0 || j >= 0 || carry != 0) {
            int d1 = (i >= 0) ? num1.charAt(i) - '0' : 0;
            int d2 = (j >= 0) ? num2.charAt(j) - '0' : 0;
            int total = d1 + d2 + carry;

            result.append(total % 10);
            carry = total / 10;
            i--;
            j--;
        }
        return result.reverse().toString();
    }
}
```

### Example

```
99999999999999999999 + 1 = 100000000000000000000
```

## 2. Multiply Strings

### Theory

Multiplying two `n`-digit and `m`-digit numbers produces at most `n+m` digits. The grade-school algorithm multiplies every digit pair and accumulates into the correct result position: multiplying `num1[i]` by `num2[j]` contributes to result positions `i+j` (higher digit) and `i+j+1` (lower digit) when indices are counted from the left.

### Code

```java
public class MultiplyStrings {
    public static void main(String[] args) {
        System.out.println("123 * 456 = " + multiply("123", "456"));
    }

    static String multiply(String num1, String num2) {
        int n = num1.length(), m = num2.length();
        int[] result = new int[n + m];

        for (int i = n - 1; i >= 0; i--) {
            for (int j = m - 1; j >= 0; j--) {
                int mul = (num1.charAt(i) - '0') * (num2.charAt(j) - '0');
                int p1 = i + j, p2 = i + j + 1;
                int total = mul + result[p2];

                result[p2] = total % 10;
                result[p1] += total / 10;
            }
        }

        StringBuilder sb = new StringBuilder();
        for (int digit : result) sb.append(digit);

        // Strip leading zeros (but keep at least one digit, in case the product is 0)
        int start = 0;
        while (start < sb.length() - 1 && sb.charAt(start) == '0') start++;
        return sb.substring(start);
    }
}
```

### Example

```
123 * 456 = 56088
```

## 3. Binary Strings

### Theory

The same digit-by-digit addition pattern from Section 1 applies to **binary** strings — just carry in base `2` instead of base `10`. This is the classic "Add Binary" problem: each column sums to at most `1+1+1=3`, so the result digit is `total % 2` and the carry is `total / 2`.

### Code

```java
public class BinaryStrings {
    public static void main(String[] args) {
        System.out.println("1010 + 1011 (binary) = " + addBinary("1010", "1011"));
    }

    static String addBinary(String a, String b) {
        StringBuilder result = new StringBuilder();
        int i = a.length() - 1, j = b.length() - 1;
        int carry = 0;

        while (i >= 0 || j >= 0 || carry != 0) {
            int d1 = (i >= 0) ? a.charAt(i) - '0' : 0;
            int d2 = (j >= 0) ? b.charAt(j) - '0' : 0;
            int total = d1 + d2 + carry;

            result.append(total % 2);
            carry = total / 2;
            i--;
            j--;
        }
        return result.reverse().toString();
    }
}
```

### Example

```
1010 + 1011 (binary) = 10101
```

`1010₂ = 10`, `1011₂ = 11`, and `10 + 11 = 21 = 10101₂` — matches.

## 4. Decimal Strings

### Theory

Once numbers are represented as decimal strings (too large to parse into a primitive), even **comparing** two of them needs care — you can't just use `<` on the strings directly, since `"9" > "10"` lexicographically even though `9 < 10` numerically. The fix: compare **lengths** first (a longer digit string is always the bigger number, assuming no leading zeros); only fall back to lexicographic comparison when the lengths are equal.

### Code

```java
public class DecimalStrings {
    public static void main(String[] args) {
        String a = "12345678901234567890";
        String b = "998877665544332211";

        System.out.println("Compare(" + a + ", " + b + "): " + compare(a, b));
    }

    // Returns positive if a > b, negative if a < b, 0 if equal (assumes no leading zeros, non-negative)
    static int compare(String a, String b) {
        if (a.length() != b.length()) {
            return a.length() - b.length();
        }
        return a.compareTo(b); // same length -> lexicographic comparison matches numeric comparison
    }
}
```

### Example

```
Compare(12345678901234567890, 998877665544332211): 2
```

A positive result means `a > b` — `a` has `20` digits versus `b`'s `18`, so it's larger regardless of the actual digit values.

## Quick Reference Cheat Sheet

| Task | Approach |
|---|---|
| Big Number Addition | column-wise addition from the right, carrying base-10 overflow |
| Multiply Strings | grade-school multiplication, accumulate into a `(n+m)`-length digit array |
| Binary Strings (Add Binary) | same column addition, base-2 carry instead of base-10 |
| Decimal String Comparison | compare lengths first, then lexicographically only if lengths match |
