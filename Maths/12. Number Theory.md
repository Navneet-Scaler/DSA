## 1. Euler Totient

### Theory

Euler's Totient function, `φ(n)`, counts how many integers in `[1, n]` are **coprime** to `n` (see [Lesson 6: Coprime Numbers](<./6. GCD and LCM.md#6-coprime-numbers>)). It has a closed form based on `n`'s distinct prime factors `p₁, p₂, ..., pₖ`:

```
φ(n) = n × (1 - 1/p₁) × (1 - 1/p₂) × ... × (1 - 1/pₖ)
```

Computing it for a single `n` just needs `n`'s prime factorization (O(√n), same technique as [Lesson 5: Prime Factorization](<./5. Prime Numbers.md#6-prime-factorization>)). For computing `φ` for **every** number up to `N`, a sieve-style approach (similar to the Linear Sieve) is far faster than factorizing each number individually.

### Code

```java
public class EulerTotient {
    public static void main(String[] args) {
        System.out.println("phi(9):  " + phi(9));
        System.out.println("phi(10): " + phi(10));
        System.out.println("phi(36): " + phi(36));

        int[] phiTable = sieveOfTotients(10);
        System.out.print("phi(1..10) via sieve: ");
        for (int i = 1; i <= 10; i++) System.out.print(phiTable[i] + " ");
        System.out.println();
    }

    // Single-number totient via prime factorization, O(sqrt(n))
    static int phi(int n) {
        int result = n;
        int temp = n;
        for (int p = 2; (long) p * p <= temp; p++) {
            if (temp % p == 0) {
                while (temp % p == 0) temp /= p;
                result -= result / p;
            }
        }
        if (temp > 1) {
            result -= result / temp;
        }
        return result;
    }

    // Totient for every number 1..n at once, O(n log log n)
    static int[] sieveOfTotients(int n) {
        int[] phi = new int[n + 1];
        for (int i = 0; i <= n; i++) phi[i] = i;

        for (int i = 2; i <= n; i++) {
            if (phi[i] == i) { // i is prime (untouched so far)
                for (int j = i; j <= n; j += i) {
                    phi[j] -= phi[j] / i;
                }
            }
        }
        return phi;
    }
}
```

### Example

```
phi(9):  6
phi(10): 4
phi(36): 12
phi(1..10) via sieve: 1 1 2 2 4 2 6 4 6 4
```

## 2. Modular Arithmetic

### Theory

Modular arithmetic treats numbers as "wrapping around" after reaching a modulus `m` — like a clock face. The key identities let you take the modulus **at every step** of a computation instead of at the end, which is what keeps intermediate values from overflowing:

```
(a + b) % m = ((a % m) + (b % m)) % m
(a - b) % m = ((a % m) - (b % m) + m) % m     <- add m to stay non-negative (see Lesson 1: Modulo)
(a * b) % m = ((a % m) * (b % m)) % m
```

### Code

```java
public class ModularArithmetic {
    public static void main(String[] args) {
        long m = 5;
        System.out.println("(-7) mod 5, kept non-negative: " + (((-7 % m) + m) % m));

        long a = 123456789L, b = 987654321L, mod = 1_000_000_007L;
        System.out.println("(a * b) mod 1e9+7: " + ((a % mod) * (b % mod)) % mod);
    }
}
```

### Example

```
(-7) mod 5, kept non-negative: 3
(a * b) mod 1e9+7: 259106859
```

## 3. Modular Inverse

### Theory

The modular inverse of `a` modulo `m`, written `a⁻¹`, is the number satisfying `a × a⁻¹ ≡ 1 (mod m)`. It exists **only when `gcd(a, m) = 1`** (i.e. `a` and `m` are coprime), and there are two standard ways to compute it:

1. **Extended Euclid** ([Lesson 6](<./6. GCD and LCM.md#2-extended-euclid>)) — works for **any** modulus, prime or not, as long as `gcd(a, m) = 1`.
2. **Fermat's Little Theorem** (below) — only works when `m` is **prime**: `a⁻¹ ≡ a^(m-2) (mod m)`, computed via [Modular Exponentiation](<./7. Powers and Exponentiation.md#4-modular-exponentiation>).

### Code

```java
public class ModularInverse {
    public static void main(String[] args) {
        System.out.println("Inverse of 3 mod 11 (Extended Euclid): " + modInverseExtEuclid(3, 11));
        System.out.println("Inverse of 3 mod 11 (Fermat, since 11 is prime): " + modInverseFermat(3, 11));
    }

    // Works for any m with gcd(a, m) == 1
    static long modInverseExtEuclid(long a, long m) {
        long[] result = extGcd(a, m); // {gcd, x, y}
        return ((result[1] % m) + m) % m;
    }

    static long[] extGcd(long a, long b) {
        if (b == 0) return new long[] { a, 1, 0 };
        long[] sub = extGcd(b, a % b);
        long x = sub[2];
        long y = sub[1] - (a / b) * sub[2];
        return new long[] { sub[0], x, y };
    }

    // Only valid when m is prime
    static long modInverseFermat(long a, long primeMod) {
        return modPow(a, primeMod - 2, primeMod);
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = (result * base) % mod;
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
Inverse of 3 mod 11 (Extended Euclid): 4
Inverse of 3 mod 11 (Fermat, since 11 is prime): 4
```

Both agree: `3 × 4 = 12 ≡ 1 (mod 11)`. ✓

## 4. Chinese Remainder Theorem

### Theory

The **Chinese Remainder Theorem (CRT)** solves a system of simultaneous congruences with pairwise **coprime** moduli:

```
x ≡ a₁ (mod m₁)
x ≡ a₂ (mod m₂)
```

It guarantees a unique solution modulo `m₁ × m₂`. The two-congruence case can be combined directly using the extended Euclidean algorithm on the moduli, then extended to more congruences by repeatedly merging one more equation in at a time.

### Code

```java
public class ChineseRemainderTheorem {
    public static void main(String[] args) {
        long x = crt(2, 3, 3, 5);
        System.out.println("x = " + x + " satisfies x % 3 = " + (x % 3) + ", x % 5 = " + (x % 5));
    }

    // Combines x ≡ a1 (mod m1) and x ≡ a2 (mod m2), where gcd(m1, m2) == 1
    static long crt(long a1, long m1, long a2, long m2) {
        long[] eg = extGcd(m1, m2); // eg = {gcd, p, q} such that m1*p + m2*q = gcd
        long gcd = eg[0], p = eg[1];
        long lcm = m1 * m2;
        long diff = a2 - a1;

        long t = ((diff / gcd) % (m2 / gcd) * p) % (m2 / gcd);
        long x = (a1 + m1 * t) % lcm;
        return ((x % lcm) + lcm) % lcm;
    }

    static long[] extGcd(long a, long b) {
        if (b == 0) return new long[] { a, 1, 0 };
        long[] sub = extGcd(b, a % b);
        long x = sub[2];
        long y = sub[1] - (a / b) * sub[2];
        return new long[] { sub[0], x, y };
    }
}
```

### Example

```
x = 8 satisfies x % 3 = 2, x % 5 = 3
```

## 5. Fermat's Little Theorem

### Theory

**Fermat's Little Theorem** states: if `p` is prime and `a` is not divisible by `p`, then:

```
a^(p-1) ≡ 1 (mod p)
```

Two big practical uses:

1. **Modular inverse** when the modulus is prime: `a⁻¹ ≡ a^(p-2) (mod p)` (shown in the Modular Inverse section above).
2. **Fermat primality test** — a probabilistic primality check: if `a^(n-1) mod n ≠ 1` for some `a`, then `n` is **definitely composite**. If it equals `1`, `n` is *probably* prime (a small number of "Carmichael number" exceptions exist, which is why more robust probabilistic tests like Miller-Rabin are preferred in practice).

### Code

```java
public class FermatLittleTheorem {
    public static void main(String[] args) {
        System.out.println("2^6 mod 7 (p=7 prime): " + modPow(2, 6, 7));   // expect 1
        System.out.println("3^10 mod 11 (p=11 prime): " + modPow(3, 10, 11)); // expect 1
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = (result * base) % mod;
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
2^6 mod 7 (p=7 prime): 1
3^10 mod 11 (p=11 prime): 1
```

## 6. Wilson's Theorem

### Theory

**Wilson's Theorem** states a number `p > 1` is prime **if and only if**:

```
(p-1)! ≡ -1 (mod p)
```

It's a beautiful theoretical characterization of primality, but **not practical** as an actual primality test — computing `(p-1)!` is far more expensive than the [Optimized √N Check](<./5. Prime Numbers.md#3-optimized-n-check>) from Lesson 5. It's included here for its role as a foundational number-theory result (and it does occasionally show up directly in competitive-programming problems).

### Code

```java
public class WilsonTheorem {
    public static void main(String[] args) {
        int[] candidates = { 5, 7, 6, 8 };
        for (int p : candidates) {
            long factorialMod = factorialMod(p - 1, p);
            long expected = p - 1; // -1 mod p, expressed as a non-negative value
            System.out.println("(" + (p - 1) + ")! mod " + p + " = " + factorialMod
                    + "  ->  Wilson says " + p + " is prime: " + (factorialMod == expected));
        }
    }

    static long factorialMod(int n, long mod) {
        long result = 1;
        for (int i = 1; i <= n; i++) {
            result = (result * i) % mod;
        }
        return result;
    }
}
```

### Example

```
(4)! mod 5 = 4  ->  Wilson says 5 is prime: true
(6)! mod 7 = 6  ->  Wilson says 7 is prime: true
(5)! mod 6 = 0  ->  Wilson says 6 is prime: false
(7)! mod 8 = 0  ->  Wilson says 8 is prime: false
```

## 7. Bézout Identity

### Theory

**Bézout's Identity** guarantees that for any integers `a` and `b`, there exist integers `x` and `y` such that:

```
a*x + b*y = gcd(a, b)
```

This was introduced practically in [Lesson 6: Extended Euclid](<./6. GCD and LCM.md#2-extended-euclid>) — the Extended Euclidean Algorithm is precisely the constructive proof of this theorem, and it's the reason the Modular Inverse (section 3 above) works: when `gcd(a, m) = 1`, Bézout's identity gives `a*x + m*y = 1`, meaning `a*x ≡ 1 (mod m)` — so `x` **is** `a`'s modular inverse.

### Code

```java
public class BezoutIdentity {
    public static void main(String[] args) {
        long[] result = extGcd(3, 11); // {gcd, x, y}
        System.out.println("gcd(3, 11) = " + result[0] + ", x = " + result[1] + ", y = " + result[2]);
        System.out.println("Check: 3*x + 11*y = " + (3 * result[1] + 11 * result[2]));
        System.out.println("Since gcd is 1, x mod 11 is 3's modular inverse: " + (((result[1] % 11) + 11) % 11));
    }

    static long[] extGcd(long a, long b) {
        if (b == 0) return new long[] { a, 1, 0 };
        long[] sub = extGcd(b, a % b);
        long x = sub[2];
        long y = sub[1] - (a / b) * sub[2];
        return new long[] { sub[0], x, y };
    }
}
```

### Example

```
gcd(3, 11) = 1, x = 4, y = -1
Check: 3*x + 11*y = 1
Since gcd is 1, x mod 11 is 3's modular inverse: 4
```

## Quick Reference Cheat Sheet

| Concept | Key formula | Requires |
|---|---|---|
| Euler Totient | `φ(n) = n × Π(1 - 1/p)` over distinct primes `p \| n` | — |
| Modular Arithmetic | apply `% m` after every `+`, `-`, `*` step | — |
| Modular Inverse | `a⁻¹` such that `a × a⁻¹ ≡ 1 (mod m)` | `gcd(a, m) = 1` |
| — via Extended Euclid | from `a*x + m*y = gcd(a,m) = 1`, `x mod m` is the inverse | any coprime `m` |
| — via Fermat | `a⁻¹ ≡ a^(m-2) (mod m)` | `m` must be **prime** |
| Chinese Remainder Theorem | combine `x ≡ aᵢ (mod mᵢ)` into one congruence mod `Πmᵢ` | moduli pairwise coprime |
| Fermat's Little Theorem | `a^(p-1) ≡ 1 (mod p)` | `p` prime, `p ∤ a` |
| Wilson's Theorem | `(p-1)! ≡ -1 (mod p)` iff `p` is prime | theoretical — not efficient in practice |
| Bézout Identity | `a*x + b*y = gcd(a, b)` has integer solutions | always true, for any `a, b` |
