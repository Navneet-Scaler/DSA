# 17. Matrix Mathematics

Matrices show up whenever a problem has a **linear transformation** applied repeatedly — computing huge Fibonacci-like sequences ([Lesson 16](<./16. Sequences.md#1-fibonacci>)), image/grid rotations, and graph adjacency computations. This lesson covers the operations you need: multiplying, exponentiating, and the two structural operations (transpose, rotation) that reshape a matrix in place.

## 1. Matrix Multiplication

### Theory

Multiplying an `n×k` matrix `A` by a `k×m` matrix `B` produces an `n×m` result, where each entry is a **dot product** of a row of `A` and a column of `B`:

```
result[i][j] = Σ (A[i][x] × B[x][j])   for x = 0 to k-1
```

The inner dimensions must match (`A`'s column count = `B`'s row count). This naive approach is **O(n × m × k)** — cubic for square matrices.

### Code

```java
public class MatrixMultiplication {
    public static void main(String[] args) {
        long[][] a = { { 1, 2 }, { 3, 4 } };
        long[][] b = { { 5, 6 }, { 7, 8 } };

        long[][] product = multiply(a, b);
        printMatrix(product);
    }

    static long[][] multiply(long[][] a, long[][] b) {
        int n = a.length, k = b.length, m = b[0].length;
        long[][] result = new long[n][m];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                long sum = 0;
                for (int x = 0; x < k; x++) {
                    sum += a[i][x] * b[x][j];
                }
                result[i][j] = sum;
            }
        }
        return result;
    }

    static void printMatrix(long[][] m) {
        for (long[] row : m) {
            StringBuilder line = new StringBuilder();
            for (long value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }
}
```

### Example

```
19 22
43 50
```

`[[1,2],[3,4]] × [[5,6],[7,8]] = [[19,22],[43,50]]` — e.g. `result[0][0] = 1×5 + 2×7 = 19`.

### Problem (Advanced): Sparse Matrix Multiplication

**Theory:** When both matrices are **mostly zeros** (a "sparse" matrix, the same idea behind the [Sparse Vector Dot Product](<../Arrays/23. Advanced Array Algorithms.md#5-sparse-array-idea>) from Arrays Lesson 23), the naive triple loop wastes almost all its work multiplying by zero. Instead, skip any multiplication where either factor is already known to be zero: only iterate over `mat1`'s **non-zero** entries, and for each one, only combine it with `mat2`'s non-zero entries in the matching row — turning the cost from `O(n×m×k)` down to roughly `O(non-zero entries)`.

```java
public class SparseMatrixMultiplication {
    public static void main(String[] args) {
        int[][] mat1 = { { 1, 0, 0 }, { -1, 0, 3 } };
        int[][] mat2 = { { 7, 0, 0 }, { 0, 0, 0 }, { 0, 0, 1 } };

        int[][] result = multiply(mat1, mat2);
        for (int[] row : result) System.out.println(java.util.Arrays.toString(row));
    }

    static int[][] multiply(int[][] mat1, int[][] mat2) {
        int n = mat1.length, k = mat1[0].length, m = mat2[0].length;
        int[][] result = new int[n][m];

        for (int i = 0; i < n; i++) {
            for (int x = 0; x < k; x++) {
                if (mat1[i][x] == 0) continue; // skip zero entries entirely
                for (int j = 0; j < m; j++) {
                    if (mat2[x][j] != 0) {
                        result[i][j] += mat1[i][x] * mat2[x][j];
                    }
                }
            }
        }
        return result;
    }
}
```

```
[7, 0, 0]
[-7, 0, 3]
```

Most of the `2×3` and `3×3` matrices are zero — the skip-on-zero check avoids ever computing a `0 × anything` term, which dominates the naive approach when sparsity is high.

## 2. Matrix Exponentiation

### Theory

Just like [binary exponentiation](<./07. Powers and Exponentiation.md#3-binary-exponentiation>) computes `a^n` in O(log n) multiplications, the same squaring trick works for **square matrices**, since matrix multiplication is associative:

```
Aⁿ = (A^(n/2))²              if n is even
Aⁿ = (A^(n/2))² × A          if n is odd
A⁰ = Identity matrix
```

This is what makes computing the `n`-th Fibonacci number in O(log n) possible (see [Lesson 16](<./16. Sequences.md#1-fibonacci>)) — and more generally, any linear recurrence can be expressed as repeated multiplication by a fixed matrix, then fast-forwarded with exponentiation.

### Code

```java
public class MatrixExponentiation {
    public static void main(String[] args) {
        long[][] a = { { 1, 2 }, { 3, 4 } };

        long[][] result = matrixPower(a, 5);
        printMatrix(result);
    }

    static long[][] matrixPower(long[][] base, int exponent) {
        int size = base.length;
        long[][] result = identity(size);

        while (exponent > 0) {
            if ((exponent & 1) == 1) {
                result = multiply(result, base);
            }
            base = multiply(base, base);
            exponent >>= 1;
        }
        return result;
    }

    static long[][] multiply(long[][] a, long[][] b) {
        int n = a.length, k = b.length, m = b[0].length;
        long[][] result = new long[n][m];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                long sum = 0;
                for (int x = 0; x < k; x++) sum += a[i][x] * b[x][j];
                result[i][j] = sum;
            }
        }
        return result;
    }

    static long[][] identity(int size) {
        long[][] id = new long[size][size];
        for (int i = 0; i < size; i++) id[i][i] = 1;
        return id;
    }

    static void printMatrix(long[][] m) {
        for (long[] row : m) {
            StringBuilder line = new StringBuilder();
            for (long value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }
}
```

### Example

```
1069 1558
2337 3406
```

`[[1,2],[3,4]]^5 = [[1069,1558],[2337,3406]]`, computed in just 3 squarings instead of 4 sequential multiplications.

### Problem (Medium): Tiling a 2×N Board with Dominoes

**Theory:** Count the number of ways to completely tile a `2×n` board using `1×2` dominoes (placed either horizontally or vertically). The last column is filled either by one **vertical** domino (leaving a `2×(n-1)` board to tile) or two **horizontal** dominoes stacked (leaving a `2×(n-2)` board) — giving the same recurrence shape as Fibonacci: `ways(n) = ways(n-1) + ways(n-2)`, with `ways(1)=1, ways(2)=2`. For huge `n` (needed `mod p`), this becomes another direct application of matrix exponentiation rather than a linear scan.

```java
public class TilingTwoByN {
    public static void main(String[] args) {
        long MOD = 1_000_000_007L;
        for (int n : new int[] { 1, 2, 3, 4, 30 }) {
            System.out.println("numTilings(" + n + "): " + numTilings(n, MOD));
        }
    }

    static long numTilings(int n, long mod) {
        if (n <= 2) return n;
        long[][] base = { { 1, 1 }, { 1, 0 } };
        long[][] power = matrixPower(base, n - 2, mod);
        // [ways(n), ways(n-1)] = base^(n-2) * [ways(2), ways(1)] = base^(n-2) * [2, 1]
        return (power[0][0] * 2 + power[0][1] * 1) % mod;
    }

    static long[][] matrixPower(long[][] base, int exponent, long mod) {
        int size = base.length;
        long[][] result = identity(size);
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = multiply(result, base, mod);
            base = multiply(base, base, mod);
            exponent >>= 1;
        }
        return result;
    }

    static long[][] multiply(long[][] a, long[][] b, long mod) {
        int n = a.length, k = b.length, m = b[0].length;
        long[][] result = new long[n][m];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                long sum = 0;
                for (int x = 0; x < k; x++) sum = (sum + a[i][x] * b[x][j]) % mod;
                result[i][j] = sum;
            }
        }
        return result;
    }

    static long[][] identity(int size) {
        long[][] id = new long[size][size];
        for (int i = 0; i < size; i++) id[i][i] = 1;
        return id;
    }
}
```

```
numTilings(1): 1
numTilings(2): 2
numTilings(3): 3
numTilings(4): 5
numTilings(30): 1346269
```

`numTilings(30) = 1346269` matches the direct O(n) recurrence exactly, but computed via `O(log n)` matrix squarings — the same jump in efficiency [Lesson 16's Fibonacci](<./16. Sequences.md#1-fibonacci>) gets from matrix exponentiation, applied to a differently-named but identically-shaped recurrence.

## 3. Identity Matrix

### Theory

The **identity matrix** `I` is the matrix equivalent of the number `1` — multiplying any matrix by it leaves the matrix unchanged (`A × I = I × A = A`). It has `1`s along the main diagonal and `0`s everywhere else. It's the required **starting value** for matrix exponentiation (the same role `1` plays as the starting accumulator in [binary exponentiation](<./07. Powers and Exponentiation.md#3-binary-exponentiation>)).

### Code

```java
public class IdentityMatrix {
    public static void main(String[] args) {
        long[][] id = identity(3);
        for (long[] row : id) {
            StringBuilder line = new StringBuilder();
            for (long value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }

    static long[][] identity(int size) {
        long[][] id = new long[size][size];
        for (int i = 0; i < size; i++) {
            id[i][i] = 1;
        }
        return id;
    }
}
```

### Example

```
1 0 0
0 1 0
0 0 1
```

## 4. Rotation

### Theory

Rotating an `n×n` matrix **90° clockwise** moves each element `M[i][j]` to position `[j][n-1-i]` in the result — equivalently, "the first column becomes the last row read top-to-bottom" (or, done in place: transpose the matrix, then reverse each row).

### Code

```java
public class Rotation {
    public static void main(String[] args) {
        int[][] m = { { 1, 2, 3 }, { 4, 5, 6 }, { 7, 8, 9 } };
        int[][] rotated = rotate90Clockwise(m);
        printMatrix(rotated);
    }

    static int[][] rotate90Clockwise(int[][] m) {
        int n = m.length;
        int[][] result = new int[n][n];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                result[j][n - 1 - i] = m[i][j];
            }
        }
        return result;
    }

    static void printMatrix(int[][] m) {
        for (int[] row : m) {
            StringBuilder line = new StringBuilder();
            for (int value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }
}
```

### Example

```
7 4 1
8 5 2
9 6 3
```

The first column `[1, 4, 7]` (top to bottom) has become the first row `[7, 4, 1]` — that's the signature of a 90° clockwise rotation.

## 5. Transpose

### Theory

The **transpose** of a matrix flips it across its main diagonal: row `i`, column `j` becomes row `j`, column `i` (`Mᵀ[j][i] = M[i][j]`). Unlike rotation, transpose works on **non-square** matrices too — an `n×m` matrix transposes into an `m×n` matrix.

### Code

```java
public class Transpose {
    public static void main(String[] args) {
        int[][] m = { { 1, 2, 3 }, { 4, 5, 6 } };
        int[][] transposed = transpose(m);
        printMatrix(transposed);
    }

    static int[][] transpose(int[][] m) {
        int rows = m.length, cols = m[0].length;
        int[][] result = new int[cols][rows];
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                result[j][i] = m[i][j];
            }
        }
        return result;
    }

    static void printMatrix(int[][] m) {
        for (int[] row : m) {
            StringBuilder line = new StringBuilder();
            for (int value : row) line.append(value).append(" ");
            System.out.println(line.toString().trim());
        }
    }
}
```

### Example

```
1 4
2 5
3 6
```

A `2×3` matrix transposes into a `3×2` matrix — row `[1,2,3]` becomes column `[1,2,3]` read top to bottom.

## Quick Reference Cheat Sheet

| Operation | Complexity | Notes |
|---|---|---|
| Matrix Multiplication | O(n×m×k) | Inner dimensions must match (→ Sparse Matrix Multiplication) |
| Matrix Exponentiation | O(size³ × log n) | Same squaring trick as binary exponentiation (→ Tiling a 2×N Board) |
| Identity Matrix | O(n) to build | Neutral element for multiplication |
| Rotation (90° CW) | O(n²) | `result[j][n-1-i] = M[i][j]` |
| Transpose | O(rows × cols) | Works on non-square matrices too |
