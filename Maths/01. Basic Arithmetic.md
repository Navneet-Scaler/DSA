Every algorithm eventually boils down to arithmetic on numbers — how they're added, split, rounded, or compared. This lesson covers the fundamental operations in **Java**, along with the quirks (integer overflow, negative modulo, truncation vs. flooring) that trip people up in interviews and competitive programming.

## 1. Addition

### Theory

Addition (`+`) combines two numbers into their sum. In Java, addition works on all numeric primitive types (`int`, `long`, `double`, `float`) and also acts as **string concatenation** when either operand is a `String`.

> **Watch out — Integer Overflow:** `int` in Java is 32-bit, ranging from `-2,147,483,648` to `2,147,483,647`. Adding two large `int`s can silently wrap around instead of throwing an error. Use `long` when sums might exceed this range.

### Code

```java
public class Addition {
    public static void main(String[] args) {
        int a = 15;
        int b = 27;
        int sum = a + b;
        System.out.println("Sum: " + sum);

        // Overflow demonstration
        int max = Integer.MAX_VALUE;
        System.out.println("MAX_VALUE:        " + max);
        System.out.println("MAX_VALUE + 1:    " + (max + 1)); // wraps to MIN_VALUE

        // Safe addition using long
        long safeSum = (long) max + 1;
        System.out.println("Safe sum (long):  " + safeSum);
    }
}
```

### Example

| Expression | Result |
|---|---|
| `15 + 27` | `42` |
| `Integer.MAX_VALUE + 1` | `-2147483648` (overflow!) |
| `(long) Integer.MAX_VALUE + 1` | `2147483648` (correct) |

```
Sum: 42
MAX_VALUE:        2147483647
MAX_VALUE + 1:    -2147483648
Safe sum (long):  2147483648
```

## 2. Subtraction

### Theory

Subtraction (`-`) finds the difference between two numbers: `a - b` means "how far is `b` from `a`". Like addition, it can also overflow at the boundaries of the data type (e.g., `Integer.MIN_VALUE - 1` wraps to `Integer.MAX_VALUE`).

### Code

```java
public class Subtraction {
    public static void main(String[] args) {
        int a = 50;
        int b = 18;
        System.out.println("Difference: " + (a - b));

        // Order matters!
        System.out.println("a - b: " + (a - b));
        System.out.println("b - a: " + (b - a));

        // Underflow demonstration
        int min = Integer.MIN_VALUE;
        System.out.println("MIN_VALUE - 1: " + (min - 1)); // wraps to MAX_VALUE
    }
}
```

### Example

```
Difference: 32
a - b: 32
b - a: -32
MIN_VALUE - 1: 2147483647
```

> Note how `a - b` and `b - a` are exact negatives of each other — subtraction is **not commutative**.

## 3. Multiplication

### Theory

Multiplication (`*`) repeatedly adds a number to itself `n` times: `a * b` is `a` added to itself `b` times. It grows results much faster than addition/subtraction, so overflow happens far more easily — this is the operation most likely to silently break in competitive programming.

> **Rule of thumb:** if two `int`s could multiply to something bigger than ~2.1 billion, cast at least one operand to `long` first.

### Code

```java
public class Multiplication {
    public static void main(String[] args) {
        int a = 12;
        int b = 8;
        System.out.println("Product: " + (a * b));

        // Overflow demonstration
        int big1 = 100_000;
        int big2 = 100_000;
        System.out.println("int * int:        " + (big1 * big2)); // overflows

        // Fix: cast to long BEFORE multiplying
        long safe = (long) big1 * big2;
        System.out.println("(long) cast fix:  " + safe);
    }
}
```

### Example

```
Product: 96
int * int:        1410065408   <-- WRONG (overflowed)
(long) cast fix:  10000000000  <-- correct
```

## 4. Division

### Theory

Division (`/`) splits a number into equal parts. In Java, the behavior depends entirely on the **types** of the operands:

- `int / int` → **truncates** toward zero, discarding the remainder (this is "integer division", covered next).
- If **either** operand is `double`/`float`, the result is a full decimal (floating-point) division.
- Dividing by zero:
  - `int / 0` → throws `ArithmeticException: / by zero`
  - `double / 0.0` → returns `Infinity`, `-Infinity`, or `NaN` (no exception)

### Code

```java
public class Division {
    public static void main(String[] args) {
        int a = 7;
        int b = 2;

        System.out.println("int / int:       " + (a / b));       // truncates
        System.out.println("double / int:     " + ((double) a / b));
        System.out.println("(double)7/(double)2: " + (7.0 / 2.0));

        // Divide by zero behavior
        System.out.println("5.0 / 0.0:        " + (5.0 / 0.0));   // Infinity
        System.out.println("-5.0 / 0.0:       " + (-5.0 / 0.0));  // -Infinity
        System.out.println("0.0 / 0.0:        " + (0.0 / 0.0));   // NaN

        try {
            System.out.println(a / 0); // throws ArithmeticException
        } catch (ArithmeticException e) {
            System.out.println("int / 0 threw:   " + e.getMessage());
        }
    }
}
```

### Example

```
int / int:       3
double / int:     3.5
(double)7/(double)2: 3.5
5.0 / 0.0:        Infinity
-5.0 / 0.0:       -Infinity
0.0 / 0.0:        NaN
int / 0 threw:   / by zero
```

## 5. Integer Division

### Theory

Integer division is what happens when **both operands are integer types** (`int`, `long`, etc.): the result is truncated — the decimal part is **cut off, not rounded**, and truncation happens **toward zero**.

| Expression | Mathematical result | Java `int` result |
|---|---|---|
| `7 / 2` | 3.5 | `3` |
| `-7 / 2` | -3.5 | `-3` (not `-4`) |
| `7 / -2` | -3.5 | `-3` (not `-4`) |

This "toward zero" behavior is different from **floor division** (used in Python), which always rounds *down*. This distinction matters a lot when negative numbers are involved — see [Floor](#9-floor) below.

### Code

```java
public class IntegerDivision {
    public static void main(String[] args) {
        System.out.println("7 / 2:   " + (7 / 2));    // 3
        System.out.println("-7 / 2:  " + (-7 / 2));   // -3 (truncation, not floor)
        System.out.println("7 / -2:  " + (7 / -2));   // -3
        System.out.println("-7 / -2: " + (-7 / -2));  // 3
    }
}
```

### Example

```
7 / 2:   3
-7 / 2:  -3
7 / -2:  -3
-7 / -2: 3
```

## 6. Modulo (%)

### Theory

The modulo operator (`%`) returns the **remainder** after integer division. In Java, the result satisfies:

```
a = (a / b) * b + (a % b)
```

Because Java's integer division truncates toward zero, **the sign of the result always matches the sign of the dividend (`a`)** — not the divisor. This is different from languages like Python, where the result takes the sign of the divisor.

| Expression | Result | Why |
|---|---|---|
| `7 % 3` | `1` | 7 = 2×3 + 1 |
| `-7 % 3` | `-1` | sign follows dividend (`-7`) |
| `7 % -3` | `1` | sign follows dividend (`7`) |
| `-7 % -3` | `-1` | sign follows dividend (`-7`) |

> **Common use case:** `n % 2 == 0` checks even/odd. But for negative `n`, prefer `Math.floorMod(n, 2)` if you specifically need a non-negative result.

### Code

```java
public class Modulo {
    public static void main(String[] args) {
        System.out.println("7 % 3:   " + (7 % 3));
        System.out.println("-7 % 3:  " + (-7 % 3));
        System.out.println("7 % -3:  " + (7 % -3));
        System.out.println("-7 % -3: " + (-7 % -3));

        // Math.floorMod always returns a result with the sign of the divisor
        System.out.println("Math.floorMod(-7, 3): " + Math.floorMod(-7, 3));
    }
}
```

### Example

```
7 % 3:   1
-7 % 3:  -1
7 % -3:  1
-7 % -3: -1
Math.floorMod(-7, 3): 2
```

## 7. Negative Numbers

### Theory

Java stores signed integers using **two's complement** representation. For an `int` (32 bits):

- The most significant bit represents the sign.
- Range: `-2,147,483,648` to `2,147,483,647` — notice there is **one more negative value** than positive, because zero occupies a "positive" slot.
- This asymmetry means `Math.abs(Integer.MIN_VALUE)` cannot be represented as a positive `int` — it overflows back to `Integer.MIN_VALUE` itself! (See [Absolute Value](#8-absolute-value) below.)

Negating a number uses the unary minus operator (`-x`), which is equivalent to `0 - x`.

### Code

```java
public class NegativeNumbers {
    public static void main(String[] args) {
        int x = 42;
        System.out.println("Negation of 42: " + (-x));
        System.out.println("Negation of -42: " + (-(-x)));

        System.out.println("Integer.MIN_VALUE:  " + Integer.MIN_VALUE);
        System.out.println("Integer.MAX_VALUE:  " + Integer.MAX_VALUE);

        // The classic gotcha: negating MIN_VALUE overflows
        System.out.println("-Integer.MIN_VALUE: " + (-Integer.MIN_VALUE));
    }
}
```

### Example

```
Negation of 42: -42
Negation of -42: 42
Integer.MIN_VALUE:  -2147483648
Integer.MAX_VALUE:  2147483647
-Integer.MIN_VALUE: -2147483648   <-- overflow gotcha!
```

## 8. Absolute Value

### Theory

The absolute value of a number is its distance from zero, always non-negative: `|x|`. Java provides `Math.abs()`, overloaded for `int`, `long`, `float`, and `double`.

> **The one edge case to memorize:** `Math.abs(Integer.MIN_VALUE)` returns `Integer.MIN_VALUE` (still negative!) because `+2147483648` has no representation in a 32-bit signed `int`. The safe fix is to widen to `long` first.

### Code

```java
public class AbsoluteValue {
    public static void main(String[] args) {
        System.out.println("Math.abs(-15): " + Math.abs(-15));
        System.out.println("Math.abs(15):  " + Math.abs(15));
        System.out.println("Math.abs(-3.14): " + Math.abs(-3.14));

        // The gotcha
        System.out.println("Math.abs(Integer.MIN_VALUE): " + Math.abs(Integer.MIN_VALUE));

        // Safe fix using long
        long safeAbs = Math.abs((long) Integer.MIN_VALUE);
        System.out.println("Safe abs (long):              " + safeAbs);
    }
}
```

### Example

```
Math.abs(-15): 15
Math.abs(15):  15
Math.abs(-3.14): 3.14
Math.abs(Integer.MIN_VALUE): -2147483648   <-- still negative!
Safe abs (long):              2147483648
```

## 9. Floor

### Theory

Flooring rounds a number **down** to the nearest integer — toward negative infinity. `Math.floor()` operates on `double`s and returns a `double`.

| Value | Floor |
|---|---|
| `3.7` | `3.0` |
| `3.2` | `3.0` |
| `-3.2` | `-4.0` |
| `-3.7` | `-4.0` |

> This is exactly the behavior you get "for free" with `int / int` division **only when both numbers are positive**. For negatives, Java's `/` truncates toward zero, while floor rounds toward negative infinity — they disagree. Use `Math.floorDiv()` when you need true floor division on integers.

### Code

```java
public class Floor {
    public static void main(String[] args) {
        System.out.println("Math.floor(3.7):  " + Math.floor(3.7));
        System.out.println("Math.floor(-3.2): " + Math.floor(-3.2));

        // Integer floor division vs truncating division
        System.out.println("-7 / 2 (truncates):        " + (-7 / 2));       // -3
        System.out.println("Math.floorDiv(-7, 2):      " + Math.floorDiv(-7, 2)); // -4
    }
}
```

### Example

```
Math.floor(3.7):  3.0
Math.floor(-3.2): -4.0
-7 / 2 (truncates):        -3
Math.floorDiv(-7, 2):      -4
```

## 10. Ceiling

### Theory

Ceiling rounds a number **up** to the nearest integer — toward positive infinity. `Math.ceil()` operates on `double`s and returns a `double`.

| Value | Ceiling |
|---|---|
| `3.2` | `4.0` |
| `3.7` | `4.0` |
| `-3.7` | `-3.0` |
| `-3.2` | `-3.0` |

> **Common interview trick:** to compute `ceil(a / b)` for positive integers `a, b` without using floating point, use the formula:
> ```
> ceilDiv = (a + b - 1) / b
> ```
> Java 9+ also provides `Math.ceilDiv(int, int)` directly (Java 18+ for the two-argument version used below).

### Code

```java
public class Ceiling {
    public static void main(String[] args) {
        System.out.println("Math.ceil(3.2):  " + Math.ceil(3.2));
        System.out.println("Math.ceil(-3.7): " + Math.ceil(-3.7));

        // Integer ceiling division trick (no floating point)
        int a = 7, b = 2;
        int ceilDiv = (a + b - 1) / b;
        System.out.println("ceil(7 / 2) via formula: " + ceilDiv);

        // Built-in (Java 18+)
        System.out.println("Math.ceilDiv(7, 2):      " + Math.ceilDiv(a, b));
    }
}
```

### Example

```
Math.ceil(3.2):  4.0
Math.ceil(-3.7): -3.0
ceil(7 / 2) via formula: 4
Math.ceilDiv(7, 2):      4
```

## Quick Reference Cheat Sheet

| Operation | Operator / Method | Overflow risk | Sign behavior |
|---|---|---|---|
| Addition | `a + b` | Yes | — |
| Subtraction | `a - b` | Yes | Not commutative |
| Multiplication | `a * b` | Yes (grows fastest) | — |
| Division | `a / b` | Only `MIN_VALUE / -1` | Truncates toward zero |
| Integer Division | `a / b` (both int) | Same as above | Toward zero, not floor |
| Modulo | `a % b` | No | Follows sign of dividend `a` |
| Absolute Value | `Math.abs(x)` | Yes, for `Integer.MIN_VALUE` | Always non-negative (except that edge case) |
| Floor | `Math.floor(x)` / `Math.floorDiv(a,b)` | No | Rounds toward −∞ |
| Ceiling | `Math.ceil(x)` / `Math.ceilDiv(a,b)` | No | Rounds toward +∞ |
