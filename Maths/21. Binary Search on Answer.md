# 21. Binary Search on Answer

Binary search doesn't need a sorted array — it works on any **monotonic condition**: if "does `x` satisfy the requirement?" flips from false to true (or true to false) exactly once as `x` increases, you can binary search directly on `x`, the *answer itself*, instead of an index. This lesson applies that pattern to root-finding and general monotonic decision problems.

## 1. Square Root

### Theory

Finding `⌊√n⌋` (the integer square root, floor-rounded) is a textbook case: the condition "`x² ≤ n`" is **monotonic** — true for small `x`, false for large `x`, flipping exactly once. Binary search for the largest `x` where it's still true, without ever calling `Math.sqrt`.

### Code

```java
public class SquareRoot {
    public static void main(String[] args) {
        System.out.println("Integer sqrt of 50: " + integerSqrt(50));
        System.out.println("Integer sqrt of 49: " + integerSqrt(49));
    }

    static long integerSqrt(long n) {
        long lo = 0, hi = n, answer = 0;
        while (lo <= hi) {
            long mid = lo + (hi - lo) / 2;
            if (mid * mid <= n) {
                answer = mid; // mid works, but a larger value might too
                lo = mid + 1;
            } else {
                hi = mid - 1; // mid overshoots, search smaller
            }
        }
        return answer;
    }
}
```

### Example

```
Integer sqrt of 50: 7
Integer sqrt of 49: 7
```

`7² = 49 ≤ 50` but `8² = 64 > 50`, so `⌊√50⌋ = 7`. And `49` is a perfect square, landing exactly on `7`.

## 2. nth Root

### Theory

The same idea generalizes directly: to find `⌊ⁿ√n⌋` (the integer `k`-th root), binary search on the condition "`x^k ≤ n`" instead of "`x² ≤ n`" — everything else about the search is identical.

### Code

```java
public class NthRoot {
    public static void main(String[] args) {
        System.out.println("Integer cube root of 27: " + integerNthRoot(27, 3));
        System.out.println("Integer cube root of 30: " + integerNthRoot(30, 3));
    }

    static long integerNthRoot(long n, int k) {
        long lo = 0, hi = n, answer = 0;
        while (lo <= hi) {
            long mid = lo + (hi - lo) / 2;
            if (power(mid, k, n) <= n) {
                answer = mid;
                lo = mid + 1;
            } else {
                hi = mid - 1;
            }
        }
        return answer;
    }

    // Computes mid^k, but bails out early (returns a value > n) once it would overflow past n
    static long power(long base, int exponent, long cap) {
        long result = 1;
        for (int i = 0; i < exponent; i++) {
            result *= base;
            if (result > cap) return result; // overshot already, no need to keep multiplying
        }
        return result;
    }
}
```

### Example

```
Integer cube root of 27: 3
Integer cube root of 30: 3
```

`3³ = 27 ≤ 30` but `4³ = 64 > 30`, so `⌊³√30⌋ = 3`.

## 3. Search Over a Mathematical Function

### Theory

The general pattern: whenever you can write **"can `x` achieve the goal?"** as a monotonic yes/no check, binary search directly on `x` — even when `x` represents something abstract, like a *speed* or a *capacity*, not just a number being squared. **Classic example — "Koko Eating Bananas"** (a well-known binary-search-on-answer problem): Koko must eat all banana piles within `h` hours; eating at speed `k` bananas/hour, pile `p` takes `⌈p/k⌉` hours. As `k` increases, total hours needed **decreases monotonically** — so binary search for the smallest `k` where the total time is still `≤ h`.

### Code

```java
public class SearchOverFunction {
    public static void main(String[] args) {
        int[] piles = { 3, 6, 7, 11 };
        int hours = 8;

        System.out.println("Minimum eating speed: " + minEatingSpeed(piles, hours));
    }

    static int minEatingSpeed(int[] piles, int h) {
        int lo = 1, hi = max(piles);
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (hoursNeeded(piles, mid) <= h) {
                hi = mid; // mid works; try to do at least as well with something smaller
            } else {
                lo = mid + 1; // mid is too slow; need a faster speed
            }
        }
        return lo;
    }

    static long hoursNeeded(int[] piles, int speed) {
        long hours = 0;
        for (int pile : piles) {
            hours += (pile + speed - 1) / speed; // ceiling division, see Lesson 1
        }
        return hours;
    }

    static int max(int[] arr) {
        int m = arr[0];
        for (int value : arr) m = Math.max(m, value);
        return m;
    }
}
```

### Example

```
Minimum eating speed: 4
```

At speed `4`: `⌈3/4⌉+⌈6/4⌉+⌈7/4⌉+⌈11/4⌉ = 1+2+2+3 = 8` hours — exactly the limit. Speed `3` would need `1+2+3+4=10` hours, too slow.

## Quick Reference Cheat Sheet

| Variant | Monotonic condition | Search space |
|---|---|---|
| Square Root | `x² ≤ n` | `x` in `[0, n]` |
| nth Root | `x^k ≤ n` | `x` in `[0, n]` |
| Search Over a Function | any "can `x` achieve the goal?" check that flips once | `x` in whatever range the answer could plausibly be |

**General template:**

```
lo, hi = smallest possible answer, largest possible answer
while lo < hi (or lo <= hi, depending on the exact variant):
    mid = lo + (hi - lo) / 2
    if condition(mid) holds:
        shrink toward mid (this could be the answer, or a better one might be smaller)
    else:
        move past mid (mid doesn't work, need to search further)
```
