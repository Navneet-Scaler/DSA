## 1. Mod Addition

### Theory

```
(a + b) mod m = ((a mod m) + (b mod m)) mod m
```

Reducing each operand `mod m` **before** adding keeps the sum small and avoids overflow when `a` and `b` are themselves huge (e.g. accumulated sums in a loop). Since two values already `< m` can sum to at most `2m - 2`, a single final `% m` (or even just one conditional subtraction) is always enough to bring it back in range.

### Code

```java
public class ModAddition {
    public static void main(String[] args) {
        long m = 13;
        System.out.println("(7 + 3) mod 13: " + modAdd(7, 3, m));
    }

    static long modAdd(long a, long b, long m) {
        return ((a % m) + (b % m)) % m;
    }
}
```

### Example

```
(7 + 3) mod 13: 10
```

## 2. Mod Subtraction

### Theory

```
(a - b) mod m = ((a mod m) - (b mod m) + m) mod m
```

Java's `%` can return a **negative** result when `a < b` (see [Lesson 1: Modulo](<./1. Basic Arithmetic.md#6-modulo->) — the sign follows the dividend). Adding `m` before the final `% m` guarantees a non-negative result in `[0, m-1]`, which is what "modular" values are expected to look like.

### Code

```java
public class ModSubtraction {
    public static void main(String[] args) {
        long m = 13;
        System.out.println("(7 - 3) mod 13: " + modSub(7, 3, m));
        System.out.println("(3 - 7) mod 13: " + modSub(3, 7, m)); // would be negative without the fix
    }

    static long modSub(long a, long b, long m) {
        return (((a % m) - (b % m)) % m + m) % m;
    }
}
```

### Example

```
(7 - 3) mod 13: 4
(3 - 7) mod 13: 9
```

## 3. Mod Multiplication

### Theory

```
(a * b) mod m = ((a mod m) * (b mod m)) mod m
```

Reducing operands first helps, but there's a deeper overflow trap: if `m` itself is large (say, close to `10^18`, as some problems use), then even two values `< m` can multiply to something that **overflows a 64-bit `long`** (`long` maxes out around `9.2 × 10^18`). The general-purpose fix is **`mulmod` via binary multiplication** ("Russian peasant multiplication") — the same doubling trick as binary exponentiation, but for multiplication: it never lets an intermediate value exceed roughly `2m`.

```
mulmod(a, b, m):
    result = 0
    a = a mod m
    while b > 0:
        if b's lowest bit is 1: result = (result + a) mod m
        a = (a * 2) mod m
        b >>= 1
    return result
```

### Code

```java
public class ModMultiplication {
    public static void main(String[] args) {
        long mod = 1_000_000_007L;
        long x = 999999999999999989L;
        long y = 999999999999999999L;

        // Direct approach - fine as long as m is small enough that (m-1)*(m-1) fits in a long
        System.out.println("Direct (x % mod) * (y % mod) % mod: " + ((x % mod) * (y % mod)) % mod);

        // Safe approach for ANY modulus, even ones where a*b could overflow long
        long bigMod = 1_000_000_000_000_000_009L; // ~10^18
        long a = 987654321987654321L;
        long b = 123456789123456789L;
        System.out.println("mulmod with huge modulus: " + mulmod(a, b, bigMod));
    }

    static long mulmod(long a, long b, long m) {
        long result = 0;
        a %= m;
        while (b > 0) {
            if ((b & 1) == 1) {
                result = (result + a) % m;
            }
            a = (a * 2) % m;
            b >>= 1;
        }
        return result;
    }
}
```

### Example

```
Direct (x % mod) * (y % mod) % mod: 1824
mulmod with huge modulus: 249809486904130499
```

## 4. Mod Division

### Theory

Division is the odd one out: there is **no** identity like `(a / b) mod m = ((a mod m) / (b mod m)) mod m` — integer/modular division doesn't distribute that way. Instead, "dividing by `b`" under a modulus means **multiplying by `b`'s modular inverse**:

```
(a / b) mod m = (a * b⁻¹) mod m
```

This only works when `b⁻¹` exists, i.e. `gcd(b, m) = 1` (see [Modular Inverse](#6-inverse-mod) below and [Lesson 12](<./12. Number Theory.md#3-modular-inverse>)).

### Code

```java
public class ModDivision {
    public static void main(String[] args) {
        long m = 13;
        long a = 7, b = 3;

        long bInverse = modInverse(b, m);
        long division = (a * bInverse) % m;

        System.out.println("Inverse of 3 mod 13: " + bInverse);
        System.out.println("7 / 3 mod 13: " + division);
        System.out.println("Check: (result * 3) mod 13 should be 7 -> " + (division * 3) % m);
    }

    static long modInverse(long a, long m) {
        long[] eg = extGcd(a, m);
        return ((eg[1] % m) + m) % m;
    }

    static long[] extGcd(long a, long b) {
        if (b == 0) return new long[] { a, 1, 0 };
        long[] sub = extGcd(b, a % b);
        long x = sub[2];
        long y = sub[1] - (a / b) * sub[2];
        return new long[] { sub[0], x, y };
    }
}
```

### Example

```
Inverse of 3 mod 13: 9
7 / 3 mod 13: 11
Check: (result * 3) mod 13 should be 7 -> 7
```

## 5. Fast Mod Power

### Theory

Computing `base^exponent mod m` efficiently is the same **binary exponentiation** technique from [Lesson 7](<./7. Powers and Exponentiation.md#4-modular-exponentiation>), squaring the base and halving the exponent, taking `% m` at every step so values never explode:

```
result = 1
while exponent > 0:
    if exponent's lowest bit is 1: result = (result * base) mod m
    base = (base * base) mod m
    exponent >>= 1
```

Runs in **O(log(exponent))** — essential whenever the exponent is huge.

### Code

```java
public class FastModPower {
    public static void main(String[] args) {
        System.out.println("3^100 mod 13: " + modPow(3, 100, 13));
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) {
                result = (result * base) % mod;
            }
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
3^100 mod 13: 3
```

## 6. Inverse Mod

### Theory

Recapping [Lesson 12](<./12. Number Theory.md#3-modular-inverse>): the modular inverse `a⁻¹` satisfies `a × a⁻¹ ≡ 1 (mod m)`, and exists iff `gcd(a, m) = 1`. Two ways to compute it:

1. **Extended Euclid** — works for any coprime `m` (prime or not).
2. **Fermat's Little Theorem** — `a⁻¹ ≡ a^(m-2) (mod m)`, but **only when `m` is prime**.

This is the piece that makes [Mod Division](#4-mod-division) possible — without it, "dividing" under a modulus wouldn't be well-defined.

### Code

```java
public class InverseMod {
    public static void main(String[] args) {
        System.out.println("Inverse of 3 mod 13 (Extended Euclid): " + modInverseExtEuclid(3, 13));
        System.out.println("Inverse of 3 mod 13 (Fermat, 13 is prime): " + modInverseFermat(3, 13));
    }

    static long modInverseExtEuclid(long a, long m) {
        long[] eg = extGcd(a, m);
        return ((eg[1] % m) + m) % m;
    }

    static long[] extGcd(long a, long b) {
        if (b == 0) return new long[] { a, 1, 0 };
        long[] sub = extGcd(b, a % b);
        long x = sub[2];
        long y = sub[1] - (a / b) * sub[2];
        return new long[] { sub[0], x, y };
    }

    static long modInverseFermat(long a, long primeMod) {
        return modPow(a, primeMod - 2, primeMod);
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) result = (result * base) % mod;
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
Inverse of 3 mod 13 (Extended Euclid): 9
Inverse of 3 mod 13 (Fermat, 13 is prime): 9
```

## Quick Reference Cheat Sheet

| Operation | Formula |
|---|---|
| Mod Addition | `((a % m) + (b % m)) % m` |
| Mod Subtraction | `(((a % m) - (b % m)) % m + m) % m` |
| Mod Multiplication | `((a % m) * (b % m)) % m`, or `mulmod` (doubling) if `m` is huge |
| Mod Division | `(a * modInverse(b, m)) % m` — never divide directly |
| Fast Mod Power | binary exponentiation, taking `% m` at every squaring step |
| Inverse Mod | Extended Euclid (any coprime `m`), or `a^(m-2) mod m` (prime `m` only) |
