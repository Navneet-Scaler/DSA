# 6. GCD / LCM

GCD (Greatest Common Divisor) and LCM (Least Common Multiple) show up everywhere — simplifying fractions, cycle-length problems, modular arithmetic, and cryptography (RSA leans directly on Extended Euclid). This lesson builds from the basic Euclidean algorithm up to its extended form, an alternative bit-based approach, and the properties that make GCD/LCM so useful.

## 1. Euclidean Algorithm

### Theory

The Euclidean algorithm computes `gcd(a, b)` using the identity:

```
gcd(a, b) = gcd(b, a % b)
gcd(a, 0) = a
```

Intuition: any number that divides both `a` and `b` must also divide `a % b` (the remainder), so the pair `(a, b)` can be replaced with the smaller pair `(b, a % b)` without changing the GCD. Repeating this shrinks the numbers rapidly — the algorithm runs in **O(log(min(a, b)))**, extremely fast even for huge numbers.

### Code

```java
public class EuclideanAlgorithm {
    public static void main(String[] args) {
        System.out.println("gcd(48, 18) [iterative]: " + gcdIterative(48, 18));
        System.out.println("gcd(48, 18) [recursive]: " + gcdRecursive(48, 18));
        System.out.println("gcd(0, 5): " + gcdIterative(0, 5));
    }

    static int gcdIterative(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }

    static int gcdRecursive(int a, int b) {
        if (b == 0) return a;
        return gcdRecursive(b, a % b);
    }
}
```

### Example

```
gcd(48, 18) [iterative]: 6
gcd(48, 18) [recursive]: 6
gcd(0, 5): 5
```

> Java also ships a built-in: `java.math.BigInteger` has a `.gcd()` method if you're already working with `BigInteger` values.

## 2. Extended Euclid

### Theory

The **Extended Euclidean Algorithm** doesn't just find `gcd(a, b)` — it also finds integers `x` and `y` satisfying **Bézout's identity**:

```
a*x + b*y = gcd(a, b)
```

This is the foundation for computing **modular multiplicative inverses** (needed constantly in modular arithmetic and RSA-style cryptography): if `gcd(a, m) = 1`, then the `x` from `a*x + m*y = 1` is exactly `a`'s inverse mod `m`.

It works by unwinding the recursive Euclidean algorithm backward, expressing the GCD at each step in terms of `x` and `y` from the step before.

### Code

```java
public class ExtendedEuclid {
    public static void main(String[] args) {
        int[] result = extGcd(30, 20);
        System.out.println("extGcd(30, 20) -> gcd=" + result[0] + ", x=" + result[1] + ", y=" + result[2]);
        System.out.println("Check: 30*x + 20*y = " + (30 * result[1] + 20 * result[2]));

        int[] result2 = extGcd(35, 15);
        System.out.println("extGcd(35, 15) -> gcd=" + result2[0] + ", x=" + result2[1] + ", y=" + result2[2]);
    }

    // Returns {gcd, x, y} such that a*x + b*y = gcd
    static int[] extGcd(int a, int b) {
        if (b == 0) {
            return new int[] { a, 1, 0 };
        }
        int[] sub = extGcd(b, a % b);
        int gcd = sub[0];
        int x1 = sub[1];
        int y1 = sub[2];

        int x = y1;
        int y = x1 - (a / b) * y1;
        return new int[] { gcd, x, y };
    }
}
```

### Example

```
extGcd(30, 20) -> gcd=10, x=1, y=-1
Check: 30*x + 20*y = 10
extGcd(35, 15) -> gcd=5, x=1, y=-2
```

## 3. Binary GCD

### Theory

Also known as **Stein's algorithm**, this computes GCD using only **subtraction and bit shifts** — no division or modulo — which can be faster on hardware where division is expensive. It relies on these identities:

- If both `a` and `b` are even: `gcd(a, b) = 2 * gcd(a/2, b/2)`
- If `a` is even and `b` is odd: `gcd(a, b) = gcd(a/2, b)`
- If both are odd: `gcd(a, b) = gcd(|a - b|, min(a, b))` (since `a - b` is even when both are odd)

### Code

```java
public class BinaryGcd {
    public static void main(String[] args) {
        System.out.println("binaryGcd(48, 18): " + binaryGcd(48, 18));
    }

    static int binaryGcd(int a, int b) {
        if (a == 0) return b;
        if (b == 0) return a;

        int shift = 0;
        while (((a | b) & 1) == 0) { // both even -> factor out common 2s
            a >>= 1;
            b >>= 1;
            shift++;
        }
        while ((a & 1) == 0) a >>= 1; // remove remaining factors of 2 from a

        while (b != 0) {
            while ((b & 1) == 0) b >>= 1; // remove remaining factors of 2 from b
            if (a > b) {
                int temp = a;
                a = b;
                b = temp;
            }
            b = b - a;
        }
        return a << shift;
    }
}
```

### Example

```
binaryGcd(48, 18): 6
```

## 4. LCM

### Theory

The **Least Common Multiple** of `a` and `b` is the smallest positive number divisible by both. It relates to GCD through:

```
lcm(a, b) = (a * b) / gcd(a, b)
```

> **Overflow gotcha:** compute `a / gcd(a, b)` **first**, then multiply by `b` — dividing before multiplying keeps intermediate values smaller and avoids overflowing `int`/`long` when `a * b` alone would be too large.

### Code

```java
public class Lcm {
    public static void main(String[] args) {
        System.out.println("lcm(4, 6): " + lcm(4, 6));
        System.out.println("lcm(12, 18): " + lcm(12, 18));
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }

    static int lcm(int a, int b) {
        return (a / gcd(a, b)) * b; // divide first, then multiply, to reduce overflow risk
    }
}
```

### Example

```
lcm(4, 6): 12
lcm(12, 18): 36
```

## 5. GCD Properties

### Theory

A handful of identities make GCD easy to reason about and simplify:

| Property | Statement |
|---|---|
| Base case | `gcd(a, 0) = a` |
| Recursive step | `gcd(a, b) = gcd(b, a % b)` |
| Commutative | `gcd(a, b) = gcd(b, a)` |
| Scaling | `gcd(k*a, k*b) = k * gcd(a, b)` |
| GCD–LCM relation | `gcd(a, b) * lcm(a, b) = a * b` |
| Coprimality after dividing | `gcd(a/gcd(a,b), b/gcd(a,b)) = 1` |

### Code

```java
public class GcdProperties {
    public static void main(String[] args) {
        int a = 12, b = 18, k = 2;

        System.out.println("gcd(" + a + ", " + b + "): " + gcd(a, b));

        // Scaling property
        System.out.println("gcd(" + (k * a) + ", " + (k * b) + "): " + gcd(k * a, k * b)
                + "  (expected " + (k * gcd(a, b)) + ")");

        // GCD-LCM relation: gcd * lcm == a * b
        int g = gcd(a, b);
        int l = (a / g) * b;
        System.out.println("gcd*lcm = " + (g * l) + ",  a*b = " + (a * b));

        // After dividing by GCD, the results are coprime
        System.out.println("gcd(a/g, b/g) = " + gcd(a / g, b / g));
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
gcd(12, 18): 6
gcd(24, 36): 12  (expected 12)
gcd*lcm = 216,  a*b = 216
gcd(a/g, b/g) = 1
```

## 6. Coprime Numbers

### Theory

Two numbers are **coprime** (or **relatively prime**) if their GCD is `1` — they share no common factor other than `1`. Note that coprime numbers don't need to be prime themselves (`8` and `15` are coprime, but neither is prime). Coprimality is the condition required for a modular inverse to exist, and it's central to Euler's Totient function (counting numbers `≤ n` coprime to `n`).

### Code

```java
public class CoprimeNumbers {
    public static void main(String[] args) {
        System.out.println("8 and 15 coprime: " + isCoprime(8, 15));
        System.out.println("8 and 12 coprime: " + isCoprime(8, 12));
    }

    static boolean isCoprime(int a, int b) {
        return gcd(a, b) == 1;
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
8 and 15 coprime: true
8 and 12 coprime: false
```

| Concept | Formula / approach |
|---|---|
