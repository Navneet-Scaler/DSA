# 6. GCD / LCM

GCD (Greatest Common Divisor) and LCM (Least Common Multiple) show up everywhere — simplifying fractions, cycle-length problems, modular arithmetic, and cryptography (RSA leans directly on Extended Euclid). This lesson builds from the basic Euclidean algorithm up to its extended form, an alternative bit-based approach, and the properties that make GCD/LCM so useful.

## 1. Euclidean Algorithm

### Theory

The Euclidean algorithm computes `gcd(a, b)` using the identity:

```
gcd(a, b) = gcd(b, a % b)
gcd(a, 0) = a
```

Intuition: any number that divides both `a` and `b` must also divide `a % b` (the remainder), so the pair `(a, b)` can be replaced with the smaller pair `(b, a % b)` without changing the GCD. Repeating this shrinks the numbers rapidly — the algorithm runs in **O(log(min(a, b)))**, extremely fast even for huge numbers.

### Code

```java
public class EuclideanAlgorithm {
    public static void main(String[] args) {
        System.out.println("gcd(48, 18) [iterative]: " + gcdIterative(48, 18));
        System.out.println("gcd(48, 18) [recursive]: " + gcdRecursive(48, 18));
        System.out.println("gcd(0, 5): " + gcdIterative(0, 5));
    }

    static int gcdIterative(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }

    static int gcdRecursive(int a, int b) {
        if (b == 0) return a;
        return gcdRecursive(b, a % b);
    }
}
```

### Example

```
gcd(48, 18) [iterative]: 6
gcd(48, 18) [recursive]: 6
gcd(0, 5): 5
```

> Java also ships a built-in: `java.math.BigInteger` has a `.gcd()` method if you're already working with `BigInteger` values.

### Problem 1 (Basic): GCD of an Array

**Theory:** Extend pairwise GCD to a whole array — the GCD of `n` numbers is computed by folding the two-number GCD across the array: `gcd(a, b, c) = gcd(gcd(a, b), c)`, since any divisor common to all three numbers must in particular be a common divisor of the first two.

```java
public class GcdOfArray {
    public static void main(String[] args) {
        System.out.println("gcdOfArray([24,36,60]): " + gcdOfArray(new int[] { 24, 36, 60 }));
    }

    static int gcdOfArray(int[] nums) {
        int result = nums[0];
        for (int i = 1; i < nums.length; i++) {
            result = gcd(result, nums[i]);
        }
        return result;
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

```
gcdOfArray([24,36,60]): 12
```

### Problem 2 (Advanced): Find GCD of Strings

**Theory:** A delightfully different application of the same idea — the "GCD" of two strings is the largest string `t` such that both `str1` and `str2` are formed by repeating `t` some whole number of times. Two strings have such a common "divisor" string **only if** `str1 + str2 == str2 + str1` (concatenating them in either order gives the same result — this holds exactly when both are built from repeats of a shared base string). If so, the answer is the prefix of length `gcd(len(str1), len(str2))` — the string-length analogue of numeric GCD.

```java
public class GcdOfStrings {
    public static void main(String[] args) {
        System.out.println("gcdOfStrings('ABCABC','ABC'): " + gcdOfStrings("ABCABC", "ABC"));
        System.out.println("gcdOfStrings('ABABAB','ABAB'): " + gcdOfStrings("ABABAB", "ABAB"));
        System.out.println("gcdOfStrings('LEET','CODE'): '" + gcdOfStrings("LEET", "CODE") + "'");
    }

    static String gcdOfStrings(String str1, String str2) {
        if (!(str1 + str2).equals(str2 + str1)) return "";
        int gcdLength = gcd(str1.length(), str2.length());
        return str1.substring(0, gcdLength);
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

```
gcdOfStrings('ABCABC','ABC'): ABC
gcdOfStrings('ABABAB','ABAB'): AB
gcdOfStrings('LEET','CODE'): ''
```

`"ABABAB"` (6 chars) and `"ABAB"` (4 chars): `gcd(6,4) = 2`, and the first `2` characters `"AB"` indeed repeats to build both strings. `"LEET"`/`"CODE"` fail the concatenation-equality check entirely, so there's no common base string at all.

## 2. Extended Euclid

### Theory

The **Extended Euclidean Algorithm** doesn't just find `gcd(a, b)` — it also finds integers `x` and `y` satisfying **Bézout's identity**:

```
a*x + b*y = gcd(a, b)
```

This is the foundation for computing **modular multiplicative inverses** (needed constantly in modular arithmetic and RSA-style cryptography): if `gcd(a, m) = 1`, then the `x` from `a*x + m*y = 1` is exactly `a`'s inverse mod `m`.

It works by unwinding the recursive Euclidean algorithm backward, expressing the GCD at each step in terms of `x` and `y` from the step before.

### Code

```java
public class ExtendedEuclid {
    public static void main(String[] args) {
        int[] result = extGcd(30, 20);
        System.out.println("extGcd(30, 20) -> gcd=" + result[0] + ", x=" + result[1] + ", y=" + result[2]);
        System.out.println("Check: 30*x + 20*y = " + (30 * result[1] + 20 * result[2]));

        int[] result2 = extGcd(35, 15);
        System.out.println("extGcd(35, 15) -> gcd=" + result2[0] + ", x=" + result2[1] + ", y=" + result2[2]);
    }

    // Returns {gcd, x, y} such that a*x + b*y = gcd
    static int[] extGcd(int a, int b) {
        if (b == 0) {
            return new int[] { a, 1, 0 };
        }
        int[] sub = extGcd(b, a % b);
        int gcd = sub[0];
        int x1 = sub[1];
        int y1 = sub[2];

        int x = y1;
        int y = x1 - (a / b) * y1;
        return new int[] { gcd, x, y };
    }
}
```

### Example

```
extGcd(30, 20) -> gcd=10, x=1, y=-1
Check: 30*x + 20*y = 10
extGcd(35, 15) -> gcd=5, x=1, y=-2
```

### Problem (Advanced): Modular Multiplicative Inverse

**Theory:** Find `x` such that `(a * x) % m == 1` — the "reciprocal" of `a` under modular arithmetic, which only exists when `gcd(a, m) = 1` (i.e., `a` and `m` are coprime). Run Extended Euclid on `(a, m)`: the returned `x` from Bézout's identity `a*x + m*y = gcd(a, m) = 1` already **is** the modular inverse — it just needs to be normalized into the range `[0, m)` with `((x % m) + m) % m`, since Extended Euclid can return a negative `x`.

```java
public class ModularInverse {
    public static void main(String[] args) {
        System.out.println("modInverse(3, 11): " + modInverse(3, 11));
        System.out.println("modInverse(10, 17): " + modInverse(10, 17));
    }

    static int modInverse(int a, int m) {
        int[] result = extGcd(a, m);
        int gcd = result[0], x = result[1];
        if (gcd != 1) throw new IllegalArgumentException("Inverse doesn't exist (a and m aren't coprime)");
        return ((x % m) + m) % m;
    }

    static int[] extGcd(int a, int b) {
        if (b == 0) return new int[] { a, 1, 0 };
        int[] sub = extGcd(b, a % b);
        int x = sub[2];
        int y = sub[1] - (a / b) * sub[2];
        return new int[] { sub[0], x, y };
    }
}
```

```
modInverse(3, 11): 4
modInverse(10, 17): 12
```

`3 × 4 = 12 ≡ 1 (mod 11)` — confirming `4` is indeed `3`'s modular inverse mod `11`. This is the standard technique for computing "division" under a modulus, used constantly in [Modular Arithmetic (Lesson 13)](<./13. Modular Arithmetic.md>).

## 3. Binary GCD

### Theory

Also known as **Stein's algorithm**, this computes GCD using only **subtraction and bit shifts** — no division or modulo — which can be faster on hardware where division is expensive. It relies on these identities:

- If both `a` and `b` are even: `gcd(a, b) = 2 * gcd(a/2, b/2)`
- If `a` is even and `b` is odd: `gcd(a, b) = gcd(a/2, b)`
- If both are odd: `gcd(a, b) = gcd(|a - b|, min(a, b))` (since `a - b` is even when both are odd)

### Code

```java
public class BinaryGcd {
    public static void main(String[] args) {
        System.out.println("binaryGcd(48, 18): " + binaryGcd(48, 18));
    }

    static int binaryGcd(int a, int b) {
        if (a == 0) return b;
        if (b == 0) return a;

        int shift = 0;
        while (((a | b) & 1) == 0) { // both even -> factor out common 2s
            a >>= 1;
            b >>= 1;
            shift++;
        }
        while ((a & 1) == 0) a >>= 1; // remove remaining factors of 2 from a

        while (b != 0) {
            while ((b & 1) == 0) b >>= 1; // remove remaining factors of 2 from b
            if (a > b) {
                int temp = a;
                a = b;
                b = temp;
            }
            b = b - a;
        }
        return a << shift;
    }
}
```

### Example

```
binaryGcd(48, 18): 6
```

## 4. LCM

### Theory

The **Least Common Multiple** of `a` and `b` is the smallest positive number divisible by both. It relates to GCD through:

```
lcm(a, b) = (a * b) / gcd(a, b)
```

> **Overflow gotcha:** compute `a / gcd(a, b)` **first**, then multiply by `b` — dividing before multiplying keeps intermediate values smaller and avoids overflowing `int`/`long` when `a * b` alone would be too large.

### Code

```java
public class Lcm {
    public static void main(String[] args) {
        System.out.println("lcm(4, 6): " + lcm(4, 6));
        System.out.println("lcm(12, 18): " + lcm(12, 18));
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }

    static int lcm(int a, int b) {
        return (a / gcd(a, b)) * b; // divide first, then multiply, to reduce overflow risk
    }
}
```

### Example

```
lcm(4, 6): 12
lcm(12, 18): 36
```

### Problem (Basic): LCM of an Array

**Theory:** Extend pairwise LCM to a whole array using the same fold-left approach as [GCD of an Array](#problem-1-basic-gcd-of-an-array): `lcm(a, b, c) = lcm(lcm(a, b), c)`, since the smallest number divisible by all three must in particular be divisible by the LCM of the first two.

```java
public class LcmOfArray {
    public static void main(String[] args) {
        System.out.println("lcmOfArray([4,6,8]): " + lcmOfArray(new int[] { 4, 6, 8 }));
    }

    static long lcmOfArray(int[] nums) {
        long result = nums[0];
        for (int i = 1; i < nums.length; i++) {
            result = lcm(result, nums[i]);
        }
        return result;
    }

    static long lcm(long a, long b) {
        return (a / gcd(a, b)) * b;
    }

    static long gcd(long a, long b) {
        while (b != 0) {
            long temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

```
lcmOfArray([4,6,8]): 24
```

`24` is the smallest number divisible by `4`, `6`, and `8` — verified by folding: `lcm(4,6)=12`, then `lcm(12,8)=24`.

## 5. GCD Properties

### Theory

A handful of identities make GCD easy to reason about and simplify:

| Property | Statement |
|---|---|
| Base case | `gcd(a, 0) = a` |
| Recursive step | `gcd(a, b) = gcd(b, a % b)` |
| Commutative | `gcd(a, b) = gcd(b, a)` |
| Scaling | `gcd(k*a, k*b) = k * gcd(a, b)` |
| GCD–LCM relation | `gcd(a, b) * lcm(a, b) = a * b` |
| Coprimality after dividing | `gcd(a/gcd(a,b), b/gcd(a,b)) = 1` |

### Code

```java
public class GcdProperties {
    public static void main(String[] args) {
        int a = 12, b = 18, k = 2;

        System.out.println("gcd(" + a + ", " + b + "): " + gcd(a, b));

        // Scaling property
        System.out.println("gcd(" + (k * a) + ", " + (k * b) + "): " + gcd(k * a, k * b)
                + "  (expected " + (k * gcd(a, b)) + ")");

        // GCD-LCM relation: gcd * lcm == a * b
        int g = gcd(a, b);
        int l = (a / g) * b;
        System.out.println("gcd*lcm = " + (g * l) + ",  a*b = " + (a * b));

        // After dividing by GCD, the results are coprime
        System.out.println("gcd(a/g, b/g) = " + gcd(a / g, b / g));
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
gcd(12, 18): 6
gcd(24, 36): 12  (expected 12)
gcd*lcm = 216,  a*b = 216
gcd(a/g, b/g) = 1
```

## 6. Coprime Numbers

### Theory

Two numbers are **coprime** (or **relatively prime**) if their GCD is `1` — they share no common factor other than `1`. Note that coprime numbers don't need to be prime themselves (`8` and `15` are coprime, but neither is prime). Coprimality is the condition required for a modular inverse to exist, and it's central to Euler's Totient function (counting numbers `≤ n` coprime to `n`).

### Code

```java
public class CoprimeNumbers {
    public static void main(String[] args) {
        System.out.println("8 and 15 coprime: " + isCoprime(8, 15));
        System.out.println("8 and 12 coprime: " + isCoprime(8, 12));
    }

    static boolean isCoprime(int a, int b) {
        return gcd(a, b) == 1;
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
8 and 15 coprime: true
8 and 12 coprime: false
```

## Quick Reference Cheat Sheet

| Concept | Formula / approach |
|---|---|
| Euclidean GCD | `gcd(a, b) = gcd(b, a % b)`, base case `gcd(a, 0) = a` (→ GCD of an Array, GCD of Strings) |
| Extended Euclid | finds `x, y` with `a*x + b*y = gcd(a, b)` (→ Modular Multiplicative Inverse) |
| Binary GCD | GCD via bit shifts + subtraction, no division |
| LCM | `(a / gcd(a, b)) * b` — divide before multiplying (→ LCM of an Array) |
| GCD–LCM relation | `gcd(a, b) * lcm(a, b) = a * b` |
| Scaling property | `gcd(k*a, k*b) = k * gcd(a, b)` |
| Coprime | `gcd(a, b) == 1` |
