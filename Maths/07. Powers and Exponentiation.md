# 7. Powers & Exponentiation

Raising a number to a power seems trivial until the exponent is huge — computing `a^n` the naive way takes `n` multiplications, which is far too slow when `n` is in the millions (or when the result needs to stay inside a modulus for cryptography-style problems). This lesson goes from the naive loop to **binary exponentiation** — the O(log n) technique that underlies fast power, modular exponentiation, matrix exponentiation, and more.

## 1. Power

### Theory

`a^n` means multiplying `a` by itself `n` times. The naive implementation is a simple loop — **O(n)** multiplications. Java's built-in `Math.pow(double, double)` also computes powers, but it works in **floating point**, so results for large integer powers can lose precision — use a manual `long`-based loop (or the faster methods below) when you need exact integer results.

### Code

```java
public class Power {
    public static void main(String[] args) {
        System.out.println("2^10 (manual loop): " + power(2, 10));
        System.out.println("2^10 (Math.pow):    " + Math.pow(2, 10));
    }

    static long power(long base, int exponent) {
        long result = 1;
        for (int i = 0; i < exponent; i++) {
            result *= base;
        }
        return result;
    }
}
```

### Example

```
2^10 (manual loop): 1024
2^10 (Math.pow):    1024.0
```

## 2. Fast Power

### Theory

Instead of multiplying `n` times, notice that `a^n` can be built from `a^(n/2)` **squared** — halving the exponent at each step instead of decrementing it by 1. This recursive divide-and-conquer approach runs in **O(log n)** multiplications instead of O(n):

```
a^n = (a^(n/2))^2              if n is even
a^n = (a^(n/2))^2 * a          if n is odd
a^0 = 1
```

### Code

```java
public class FastPower {
    public static void main(String[] args) {
        System.out.println("3^13 (fast power, recursive): " + fastPower(3, 13));
    }

    static long fastPower(long base, int exponent) {
        if (exponent == 0) return 1;

        long half = fastPower(base, exponent / 2);
        long halfSquared = half * half;

        if (exponent % 2 == 0) {
            return halfSquared;
        } else {
            return halfSquared * base;
        }
    }
}
```

### Example

```
3^13 (fast power, recursive): 1594323
```

## 3. Binary Exponentiation

### Theory

Binary exponentiation is the **iterative** cousin of fast power, and it's the version you'll use most in practice (no recursion overhead, no stack depth concerns). It works directly off the exponent's **binary representation**: `n` in binary is a sum of powers of 2, so `a^n` is a product of `a^(2^k)` terms — one for each set bit of `n`.

```
result = 1
while n > 0:
    if n's lowest bit is 1:
        result *= base
    base *= base      // base becomes base^2, base^4, base^8, ...
    n >>= 1            // move to the next bit
```

### Code

```java
public class BinaryExponentiation {
    public static void main(String[] args) {
        System.out.println("3^13 (binary exponentiation): " + power(3, 13));
    }

    static long power(long base, int exponent) {
        long result = 1;
        while (exponent > 0) {
            if ((exponent & 1) == 1) {
                result *= base;
            }
            base *= base;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
3^13 (binary exponentiation): 1594323
```

`13` in binary is `1101` — so `3^13 = 3^8 * 3^4 * 3^1`, computed in just 4 iterations instead of 13 multiplications.

## 4. Modular Exponentiation

### Theory

In competitive programming and cryptography, results are usually taken **modulo** some large number (commonly `10^9 + 7`) because the true value of `a^n` would otherwise be astronomically large and overflow even a `long`. Modular exponentiation applies the modulus **at every squaring step**, keeping intermediate values bounded, while still running in **O(log n)**:

```
(x * y) % m = ((x % m) * (y % m)) % m
```

This identity means you can take the modulus after every multiplication without changing the final answer.

### Code

```java
public class ModularExponentiation {
    public static void main(String[] args) {
        long MOD = 1_000_000_007L;

        System.out.println("2^13 mod 1e9+7:      " + modPow(2, 13, MOD));
        System.out.println("7^100000 mod 1e9+7:  " + modPow(7, 100000, MOD));
    }

    static long modPow(long base, long exponent, long mod) {
        long result = 1;
        base %= mod;
        while (exponent > 0) {
            if ((exponent & 1) == 1) {
                result = (result * base) % mod;
            }
            base = (base * base) % mod;
            exponent >>= 1;
        }
        return result;
    }
}
```

### Example

```
2^13 mod 1e9+7:      8192
7^100000 mod 1e9+7:  183133701
```

`7^100000` has tens of thousands of digits — impossible to compute directly — yet `modPow` returns the answer instantly by never letting intermediate values exceed the modulus.

## 5. Negative Powers

### Theory

Mathematically, `a^(-n) = 1 / a^n`. There's an important type distinction in Java:

- With **integer types** (`int`/`long`), `1 / a^n` is integer division — it truncates to `0` for any `|a| > 1` (since `1` divided by a bigger integer is less than 1). Negative integer powers are only meaningful with **floating-point** types.
- With **`double`**, `Math.pow(a, -n)` behaves correctly and returns the expected fractional result.

### Code

```java
public class NegativePowers {
    public static void main(String[] args) {
        System.out.println("Math.pow(2, -3):        " + Math.pow(2, -3));

        // Integer division gotcha
        long positivePow = 2 * 2 * 2; // 2^3
        long wrongNegative = 1 / positivePow; // integer division truncates to 0
        System.out.println("1 / 2^3 (int division): " + wrongNegative);

        // Correct way: cast to double first
        double correctNegative = 1.0 / positivePow;
        System.out.println("1.0 / 2^3 (double):     " + correctNegative);
    }
}
```

### Example

```
Math.pow(2, -3):        0.125
1 / 2^3 (int division): 0
1.0 / 2^3 (double):     0.125
```

## 6. Floating Powers

### Theory

Fractional exponents represent roots: `a^(1/2)` is the square root of `a`, `a^(1/3)` is the cube root, and so on. Java's `Math.pow(double, double)` handles these directly, but two things to keep in mind:

- Results are subject to **floating-point precision limits** — don't expect exact equality when comparing computed roots.
- `Math.pow` returns `NaN` for a **negative base with a non-integer exponent** (e.g. the cube root of `-8` is a well-defined real number, `-2`, but `Math.pow(-8, 1.0/3.0)` still returns `NaN` because Java can't tell fractional exponents like `1/3` apart from ones that would require a complex result).

### Code

```java
public class FloatingPowers {
    public static void main(String[] args) {
        System.out.println("2^0.5 (sqrt of 2):   " + Math.pow(2, 0.5));
        System.out.println("Math.sqrt(2):        " + Math.sqrt(2));

        System.out.println("8^(1/3) (cube root): " + Math.pow(8, 1.0 / 3));

        // The negative-base gotcha
        System.out.println("(-8)^(1/3) via Math.pow: " + Math.pow(-8, 1.0 / 3)); // NaN, not -2
        System.out.println("Math.cbrt(-8):            " + Math.cbrt(-8));         // correct: -2.0
    }
}
```

### Example

```
2^0.5 (sqrt of 2):   1.4142135623730951
Math.sqrt(2):        1.4142135623730951
8^(1/3) (cube root): 2.0
(-8)^(1/3) via Math.pow: NaN
Math.cbrt(-8):            -2.0
```

> **Takeaway:** use `Math.sqrt()` for square roots and `Math.cbrt()` for cube roots — they handle negative inputs correctly, unlike a general `Math.pow(x, 1.0/3)` call.

## Quick Reference Cheat Sheet

| Technique | Complexity | Use case |
|---|---|---|
| Naive Power | O(n) | Teaching only — avoid for large `n` |
| Fast Power (recursive) | O(log n) | Exact integer powers |
| Binary Exponentiation (iterative) | O(log n) | Preferred — no recursion overhead |
| Modular Exponentiation | O(log n) | Powers where the answer is taken `% mod` |
| Negative Powers | — | Requires floating-point (`double`) to be meaningful |
| Floating Powers | — | `Math.pow`, but prefer `Math.sqrt`/`Math.cbrt` for roots |
