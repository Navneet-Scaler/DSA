## 1. Numerator/Denominator

### Theory

A fraction `n/d` has a **numerator** (`n`, the top — how many parts you have) and a **denominator** (`d`, the bottom — how many parts make a whole). Java has no built-in fraction type, so fractions are usually represented as a pair of integers (or a small class wrapping them). One convention worth adopting early: keep the **sign only on the numerator** and the denominator always positive — it avoids ambiguity like `-1/-2` vs `1/2` vs `-1/2` all meaning different things depending on where the minus sign sits.

### Code

```java
public class NumeratorDenominator {
    public static void main(String[] args) {
        int numerator = -3;
        int denominator = -4;

        // Normalize: push sign onto numerator, keep denominator positive
        if (denominator < 0) {
            numerator = -numerator;
            denominator = -denominator;
        }

        System.out.println("Normalized fraction: " + numerator + "/" + denominator);
    }
}
```

### Example

```
Normalized fraction: 3/4
```

## 2. Simplification

### Theory

A fraction is in **lowest terms** when its numerator and denominator share no common factor other than `1` — i.e. `gcd(n, d) == 1`. Simplifying just means dividing both by their GCD (see [Lesson 6: GCD and LCM](<./06. GCD and LCM.md>)).

```
simplified = (n / gcd(n, d)) / (d / gcd(n, d))
```

### Code

```java
public class Simplification {
    public static void main(String[] args) {
        int[] simplified = simplify(8, 12);
        System.out.println("8/12 simplified: " + simplified[0] + "/" + simplified[1]);
    }

    static int[] simplify(int n, int d) {
        int g = gcd(Math.abs(n), Math.abs(d));
        return new int[] { n / g, d / g };
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
8/12 simplified: 2/3
```

## 3. Mixed Fractions

### Theory

An **improper fraction** (numerator ≥ denominator) can be rewritten as a **mixed fraction**: a whole number plus a proper fraction remainder. This is just integer division and modulo:

```
whole = n / d       (integer division)
remainder = n % d
mixed = whole  remainder/d
```

### Code

```java
public class MixedFractions {
    public static void main(String[] args) {
        int n = 17, d = 5;
        int whole = n / d;
        int remainder = n % d;

        System.out.println(n + "/" + d + " as a mixed fraction: " + whole + " " + remainder + "/" + d);
    }
}
```

### Example

```
17/5 as a mixed fraction: 3 2/5
```

## 4. Fraction Addition

### Theory

To add two fractions, put them over a **common denominator** — the simplest (though not always smallest) common denominator is just the product of the two denominators:

```
n1/d1 + n2/d2 = (n1*d2 + n2*d1) / (d1*d2)
```

Always **simplify the result** afterward — cross-multiplying like this doesn't guarantee lowest terms.

### Code

```java
public class FractionAddition {
    public static void main(String[] args) {
        int[] sum = add(1, 4, 1, 6);
        System.out.println("1/4 + 1/6 = " + sum[0] + "/" + sum[1]);
    }

    static int[] add(int n1, int d1, int n2, int d2) {
        int n = n1 * d2 + n2 * d1;
        int d = d1 * d2;
        return simplify(n, d);
    }

    static int[] simplify(int n, int d) {
        int g = gcd(Math.abs(n), Math.abs(d));
        return new int[] { n / g, d / g };
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
1/4 + 1/6 = 5/12
```

## 5. Fraction Multiplication

### Theory

Multiplying fractions is simpler than adding — multiply numerators together, and denominators together, then simplify:

```
n1/d1 * n2/d2 = (n1*n2) / (d1*d2)
```

### Code

```java
public class FractionMultiplication {
    public static void main(String[] args) {
        int[] product = multiply(2, 3, 3, 4);
        System.out.println("2/3 * 3/4 = " + product[0] + "/" + product[1]);
    }

    static int[] multiply(int n1, int d1, int n2, int d2) {
        int n = n1 * n2;
        int d = d1 * d2;
        return simplify(n, d);
    }

    static int[] simplify(int n, int d) {
        int g = gcd(Math.abs(n), Math.abs(d));
        return new int[] { n / g, d / g };
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
}
```

### Example

```
2/3 * 3/4 = 1/2
```

## 6. Decimal Conversion

### Theory

Converting a fraction to a decimal is just division: `(double) n / d`. Some fractions convert **exactly** (`1/8 = 0.125`, because `8` is a power of `2`, which divides evenly into the binary-based `double` representation). Others, like `1/3`, **cannot** be represented exactly in binary floating point at all — the `double` result is only an approximation, no matter how many digits you print.

### Code

```java
public class DecimalConversion {
    public static void main(String[] args) {
        System.out.println("1/8 as decimal: " + ((double) 1 / 8)); // exact
        System.out.println("1/3 as decimal: " + ((double) 1 / 3)); // approximation only
    }
}
```

### Example

```
1/8 as decimal: 0.125
1/3 as decimal: 0.3333333333333333
```

## 7. Recurring Decimals

### Theory

Some fractions produce a decimal that **repeats forever** (`1/3 = 0.333...`), while others terminate (`1/2 = 0.5`) or repeat only after a non-repeating prefix (`1/6 = 0.1666...`). Rather than relying on an imprecise `double`, you can detect the exact repeating pattern by simulating **long division** and tracking every remainder you've seen:

- If the remainder ever becomes `0`, the decimal **terminates**.
- If a remainder **repeats**, the digits generated since it last appeared form the **repeating cycle** — wrap them in parentheses, the standard recurring-decimal notation.

This is a classic interview problem (LeetCode's "Fraction to Recurring Decimal").

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class RecurringDecimals {
    public static void main(String[] args) {
        System.out.println("1/2 -> " + fractionToDecimal(1, 2));
        System.out.println("1/3 -> " + fractionToDecimal(1, 3));
        System.out.println("1/6 -> " + fractionToDecimal(1, 6));
        System.out.println("22/7 -> " + fractionToDecimal(22, 7));
    }

    static String fractionToDecimal(long numerator, long denominator) {
        if (numerator % denominator == 0) {
            return String.valueOf(numerator / denominator);
        }

        StringBuilder result = new StringBuilder();
        if ((numerator < 0) ^ (denominator < 0)) {
            result.append("-");
        }
        numerator = Math.abs(numerator);
        denominator = Math.abs(denominator);

        result.append(numerator / denominator);
        long remainder = numerator % denominator;
        result.append(".");

        Map<Long, Integer> seenAtPosition = new HashMap<>();
        StringBuilder fractionalPart = new StringBuilder();

        while (remainder != 0) {
            if (seenAtPosition.containsKey(remainder)) {
                int start = seenAtPosition.get(remainder);
                fractionalPart.insert(start, "(");
                fractionalPart.append(")");
                break;
            }
            seenAtPosition.put(remainder, fractionalPart.length());
            remainder *= 10;
            fractionalPart.append(remainder / denominator);
            remainder %= denominator;
        }

        return result.append(fractionalPart).toString();
    }
}
```

### Example

```
1/2 -> 0.5
1/3 -> 0.(3)
1/6 -> 0.1(6)
22/7 -> 3.(142857)
```

## Quick Reference Cheat Sheet

| Task | Formula / approach |
|---|---|
| Normalize sign | keep denominator positive, sign on numerator |
| Simplify | divide both by `gcd(n, d)` |
| Mixed fraction | `whole = n / d`, `remainder = n % d` |
| Add | `(n1*d2 + n2*d1) / (d1*d2)`, then simplify |
| Multiply | `(n1*n2) / (d1*d2)`, then simplify |
| Decimal (approx) | `(double) n / d` |
| Recurring decimal (exact) | simulate long division, track remainders seen, wrap repeat in `()` |
