# 11. Hashing

Hashing trades a small amount of memory for O(1) average-time lookups — turning "have I seen this?" or "where is this value?" from an O(n) scan into a constant-time check. This lesson covers Java's two core hash-based collections and the array-problem patterns built on top of them.

## 1. HashMap

### Theory

A `HashMap<K, V>` stores key-value pairs with **O(1) average-case** `put`, `get`, and `containsKey` — the key is hashed to find its bucket directly, instead of scanning. This is the workhorse structure for almost every "remember something about a value" array pattern in this lesson.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapExample {
    public static void main(String[] args) {
        Map<String, Integer> ages = new HashMap<>();
        ages.put("Alice", 30);
        ages.put("Bob", 25);

        System.out.println("Alice's age: " + ages.get("Alice"));
        System.out.println("Contains 'Carol': " + ages.containsKey("Carol"));
    }
}
```

### Example

```
Alice's age: 30
Contains 'Carol': false
```

## 2. HashSet

### Theory

A `HashSet<T>` stores **unique** values only, with O(1) average `add` and `contains` — it's a `HashMap` with no meaningful value, just keys. Used whenever the question is purely "is this present?" rather than "what's associated with this?"

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class HashSetExample {
    public static void main(String[] args) {
        Set<Integer> seen = new HashSet<>();
        seen.add(5);
        seen.add(10);

        System.out.println("Contains 5: " + seen.contains(5));
        System.out.println("Contains 7: " + seen.contains(7));
    }
}
```

### Example

```
Contains 5: true
Contains 7: false
```

## 4. Seen Elements

### Theory

The simplest hashing pattern: walk the array once, and use a `HashSet` to remember every value encountered so far. **Classic example — Contains Duplicate:** the moment a value is seen a second time, report `true`.

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class SeenElements {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 1 };
        System.out.println("Contains duplicate: " + containsDuplicate(arr));
    }

    static boolean containsDuplicate(int[] arr) {
        Set<Integer> seen = new HashSet<>();
        for (int value : arr) {
            if (!seen.add(value)) return true; // add() returns false if already present
        }
        return false;
    }
}
```

### Example

```
Contains duplicate: true
```

## 5. Index Mapping

### Theory

Beyond just "have I seen this value," a `HashMap<value, index>` remembers **where** a value occurred — enabling O(n) solutions to problems that would otherwise need an O(n²) nested search. **Classic example — Two Sum (unsorted array):** for each element, check whether its *complement* (`target - value`) has already been seen; if so, its stored index plus the current index form the answer.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class IndexMapping {
    public static void main(String[] args) {
        int[] arr = { 2, 7, 11, 15 };
        int[] result = twoSum(arr, 9);
        System.out.println("Indices: [" + result[0] + ", " + result[1] + "]");
    }

    static int[] twoSum(int[] arr, int target) {
        Map<Integer, Integer> valueToIndex = new HashMap<>();
        for (int i = 0; i < arr.length; i++) {
            int complement = target - arr[i];
            if (valueToIndex.containsKey(complement)) {
                return new int[] { valueToIndex.get(complement), i };
            }
            valueToIndex.put(arr[i], i);
        }
        return new int[] { -1, -1 };
    }
}
```

### Example

```
Indices: [0, 1]
```

`arr[0] + arr[1] = 2 + 7 = 9` — found in a single O(n) pass, no sorting or nested loop required (compare with the sorted-array [Pair Sum](<./08. Two Pointers.md#5-pair-sum>) approach, which needs the array sorted first).

## 6. Prefix Hashing

### Theory

Combining a `HashMap` with a running [prefix sum](<./05. Prefix Techniques.md#1-prefix-sum>) solves subarray-aggregate problems in **O(n)**. **Classic example — Subarray Sum Equals K:** as you accumulate the running prefix sum, a subarray ending at the current position sums to `k` exactly when `(currentPrefixSum - k)` was a **previously seen** prefix sum — track how many times each prefix sum has occurred in a map, and add that count whenever the target difference has been seen before.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class PrefixHashing {
    public static void main(String[] args) {
        int[] arr = { 1, 1, 1 };
        System.out.println("Subarrays summing to 2: " + subarraySumEqualsK(arr, 2));
    }

    static int subarraySumEqualsK(int[] arr, int k) {
        Map<Integer, Integer> prefixSumCounts = new HashMap<>();
        prefixSumCounts.put(0, 1); // an empty prefix (sum 0) has occurred once, before any elements

        int count = 0, prefixSum = 0;
        for (int value : arr) {
            prefixSum += value;
            count += prefixSumCounts.getOrDefault(prefixSum - k, 0);
            prefixSumCounts.merge(prefixSum, 1, Integer::sum);
        }
        return count;
    }
}
```

### Example

```
Subarrays summing to 2: 2
```

`[1,1,1]` has two subarrays summing to `2`: indices `[0,1]` and `[1,2]`.

## 7. Longest Consecutive Sequence

### Theory

Find the length of the longest run of **consecutive integers** in an unsorted array, in O(n) — sorting first would cost O(n log n), which is one step too slow for the intended solution. Put every value in a `HashSet`, then for each value, only start counting a sequence if `value - 1` is **not** in the set (meaning this value is a sequence's true starting point) — this guarantees every sequence gets counted from its start exactly once, so the total work across all starting points is still O(n).

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class LongestConsecutiveSequence {
    public static void main(String[] args) {
        int[] nums = { 100, 4, 200, 1, 3, 2 };
        System.out.println("Longest consecutive sequence: " + longestConsecutive(nums));
    }

    static int longestConsecutive(int[] nums) {
        Set<Integer> numSet = new HashSet<>();
        for (int n : nums) numSet.add(n);

        int best = 0;
        for (int n : numSet) {
            if (!numSet.contains(n - 1)) { // only start counting from a true sequence start
                int length = 1;
                while (numSet.contains(n + length)) length++;
                best = Math.max(best, length);
            }
        }
        return best;
    }
}
```

### Example

```
Longest consecutive sequence: 4
```

`[1, 2, 3, 4]` is the longest run — `100` and `200` are isolated, each contributing a sequence of length `1`.

## 8. Isomorphic Strings

### Theory

Two strings are isomorphic if there's a **consistent one-to-one character mapping** from one to the other (every occurrence of a character in `s` always maps to the same character in `t`, and vice versa). Two hash maps — one for each direction — enforce this: as you walk both strings together, reject immediately if either map ever demands a conflicting mapping.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class IsomorphicStrings {
    public static void main(String[] args) {
        System.out.println("'egg' / 'add': " + isIsomorphic("egg", "add"));
        System.out.println("'foo' / 'bar': " + isIsomorphic("foo", "bar"));
    }

    static boolean isIsomorphic(String s, String t) {
        if (s.length() != t.length()) return false;

        Map<Character, Character> mapST = new HashMap<>();
        Map<Character, Character> mapTS = new HashMap<>();

        for (int i = 0; i < s.length(); i++) {
            char a = s.charAt(i), b = t.charAt(i);
            if (mapST.containsKey(a) && mapST.get(a) != b) return false;
            if (mapTS.containsKey(b) && mapTS.get(b) != a) return false;
            mapST.put(a, b);
            mapTS.put(b, a);
        }
        return true;
    }
}
```

### Example

```
'egg' / 'add': true
'foo' / 'bar': false
```

`e→a, g→d` is consistent both ways for `"egg"`/`"add"`. For `"foo"`/`"bar"`, `f→b` is fine, but `o` would need to map to **both** `a` and `r` — inconsistent, so `false`.

## 9. Intersection of Two Arrays (Java Set Operations)

### Theory

Rather than hand-rolling "for each element in A, check if it's in B" with an explicit loop, Java's `Set` interface exposes the actual **set-theory operations** directly as methods: `retainAll` computes an **intersection** in place, `addAll` a **union**, and `removeAll` a **difference** (set subtraction). Backing the sets with `HashSet` keeps every one of these O(n) average case, and using sets naturally de-duplicates the result too.

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class IntersectionOfTwoArrays {
    public static void main(String[] args) {
        int[] nums1 = { 1, 2, 2, 1 };
        int[] nums2 = { 2, 2 };
        System.out.println("Intersection: " + intersection(nums1, nums2));
    }

    static Set<Integer> intersection(int[] nums1, int[] nums2) {
        Set<Integer> set1 = new HashSet<>();
        for (int n : nums1) set1.add(n);

        Set<Integer> set2 = new HashSet<>();
        for (int n : nums2) set2.add(n);

        set1.retainAll(set2); // keep only elements present in both sets -- true set intersection
        return set1;
    }
}
```

### Example

```
Intersection: [2]
```

`1` only appears in `nums1`, and `2` appears in both — `retainAll` leaves only the shared element, with duplicates in either input naturally collapsed by the `Set`.

## Quick Reference Cheat Sheet

| Pattern | Structure | Example use case |
|---|---|---|
| HashMap | key → value | any "remember something about a value" need |
| HashSet | unique values only | "has this been seen?" checks |
| Frequency Map | value → count | see [Lesson 4](<./04. Frequency Problems.md>) |
| Seen Elements | `HashSet`, check-then-add | Contains Duplicate |
| Index Mapping | value → index | Two Sum (unsorted array) |
| Prefix Hashing | prefix sum → count of occurrences | Subarray Sum Equals K |
| Longest Consecutive Sequence | `HashSet`, only count from true sequence starts | O(n) longest run of consecutive integers |
| Isomorphic Strings | two `HashMap`s, one per mapping direction | consistent character-to-character mapping |
| Intersection of Two Arrays | `Set.retainAll` / `addAll` / `removeAll` | intersection / union / difference of two arrays |
