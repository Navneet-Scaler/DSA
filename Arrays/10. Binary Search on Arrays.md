# 10. Binary Search on Arrays

Binary search cuts the search space in half at every step, turning an O(n) scan into **O(log n)** — but only on sorted data (or on the "monotonic answer space" pattern from [Maths Lesson 21](<../Maths/21. Binary Search on Answer.md>)). This lesson covers exact search plus the boundary-finding variants (lower/upper bound) that solve a much wider range of problems than exact match alone.

## 1. Basic Binary Search

### Theory

Given a sorted array, compare the target against the middle element: if equal, done; if the target is smaller, discard the right half; if larger, discard the left half. Repeat on the remaining half.

### Code

```java
public class BasicBinarySearch {
    public static void main(String[] args) {
        int[] arr = { 1, 3, 5, 7, 9, 11 };
        System.out.println("Index of 7: " + search(arr, 7));
    }

    static int search(int[] arr, int target) {
        int lo = 0, hi = arr.length - 1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] == target) return mid;
            else if (arr[mid] < target) lo = mid + 1;
            else hi = mid - 1;
        }
        return -1;
    }
}
```

### Example

```
Index of 7: 3
```

## 2. Lower Bound

### Theory

The **lower bound** of a target is the index of the **first** element that is `≥ target` — even if the exact value isn't present, this tells you where it would fit. Unlike exact search, this variant needs a **half-open** search space (`hi = length`, not `length - 1`) so it can express "the answer is past every element."

### Code

```java
public class LowerBound {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 2, 2, 3, 4 };
        System.out.println("Lower bound of 2: " + lowerBound(arr, 2));
    }

    static int lowerBound(int[] arr, int target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

### Example

```
Lower bound of 2: 1
```

Index `1` is the **first** occurrence of `2` in `[1, 2, 2, 2, 3, 4]`.

## 3. Upper Bound

### Theory

The **upper bound** is the index of the **first** element that is **strictly greater** than the target — one past the last occurrence of the target itself. Same template as [Lower Bound](#2-lower-bound), just with `<=` instead of `<` in the comparison.

### Code

```java
public class UpperBound {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 2, 2, 3, 4 };
        System.out.println("Upper bound of 2: " + upperBound(arr, 2));
    }

    static int upperBound(int[] arr, int target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] <= target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

### Example

```
Upper bound of 2: 4
```

Index `4` is where `3` sits — one past the last `2`. `upperBound - lowerBound` gives the **count** of the target in the array (here, `4 - 1 = 3` occurrences of `2`).

## 4. Search Insert Position

### Theory

"Where would this target be inserted to keep the array sorted?" is **exactly** the [Lower Bound](#2-lower-bound) query — if the target exists, its insert position is its first occurrence; if it doesn't exist, it's the index of the first larger element (functionally identical code).

### Code

```java
public class SearchInsertPosition {
    public static void main(String[] args) {
        int[] arr = { 1, 3, 5, 6 };
        System.out.println("Insert position for 5: " + searchInsert(arr, 5));
        System.out.println("Insert position for 2: " + searchInsert(arr, 2));
        System.out.println("Insert position for 7: " + searchInsert(arr, 7));
    }

    static int searchInsert(int[] arr, int target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

### Example

```
Insert position for 5: 2
Insert position for 2: 1
Insert position for 7: 4
```

## 5. Binary Search on Answer

### Theory

Full theory and derivation is in [Maths Lesson 21: Binary Search on Answer](<../Maths/21. Binary Search on Answer.md>) — the same "monotonic condition" idea applies directly to array problems where the search space isn't the array's *indices*, but some other value entirely (a capacity, a speed, a maximum). The array itself is just an input to the monotonic **check function**, not the thing being binary-searched.

**Array-specific example — minimum array-sum-splitting capacity:** given an array and a number of groups `k`, find the smallest "maximum subarray sum" achievable by splitting the array into `k` contiguous parts. As the candidate capacity increases, the number of parts needed to stay under it **decreases monotonically** — a textbook binary-search-on-answer setup.

## 6. Rotated Array Search

### Theory

A **rotated sorted array** (e.g. `[4,5,6,7,0,1,2]`, originally `[0,1,2,4,5,6,7]` rotated) is no longer globally sorted, but at every step, **at least one half around the midpoint is still sorted**. Binary search still works: check which half is sorted, then decide whether the target could be in that sorted half's range — if not, it must be in the other half.

### Code

```java
public class RotatedArraySearch {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 6, 7, 0, 1, 2 };
        System.out.println("Index of 0: " + search(arr, 0));
    }

    static int search(int[] arr, int target) {
        int lo = 0, hi = arr.length - 1;
        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] == target) return mid;

            if (arr[lo] <= arr[mid]) { // left half is sorted
                if (arr[lo] <= target && target < arr[mid]) {
                    hi = mid - 1;
                } else {
                    lo = mid + 1;
                }
            } else { // right half is sorted
                if (arr[mid] < target && target <= arr[hi]) {
                    lo = mid + 1;
                } else {
                    hi = mid - 1;
                }
            }
        }
        return -1;
    }
}
```

### Example

```
Index of 0: 4
```

## Quick Reference Cheat Sheet

| Variant | Finds | Search space |
|---|---|---|
| Basic Binary Search | exact index of target | `[0, length-1]` |
| Lower Bound | first index `≥ target` | `[0, length]` (half-open) |
| Upper Bound | first index `> target` | `[0, length]` (half-open) |
| Search Insert Position | same as Lower Bound | `[0, length]` |
| Binary Search on Answer | smallest/largest value satisfying a monotonic check | whatever range the answer could be in |
| Rotated Array Search | target in a rotated sorted array | `[0, length-1]`, checking which half is sorted |
