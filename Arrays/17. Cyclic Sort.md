# 17. Cyclic Sort

When an array's values are known to fall within a specific range tied to its own indices (like `1..n` or `0..n-1`), each value has one "correct" home position. **Cyclic Sort** repeatedly swaps each element into its correct position — in **O(n) time and O(1) extra space** — and once done, a single pass reveals any index that still doesn't hold its correct value, which is exactly where missing/duplicate-number problems hide their answer.

## 1. Missing Number

### Theory

Given an array of `n` numbers drawn from `0..n` (so `n+1` possible values, one missing, array size `n`), cyclic sort places each value `v` at index `v` — except `v = n` itself has no valid index in a size-`n` array, so it's simply skipped during placement. Whichever index ends up **not** holding its own value afterward is the missing number. (This is an alternate approach to the sum/XOR formulas already covered in [Lesson 4](<./04. Frequency Problems.md#8-missing-number>) — useful when you want the in-place, no-extra-arithmetic version.)

### Code

```java
public class MissingNumber {
    public static void main(String[] args) {
        int[] arr = { 3, 0, 1 };
        System.out.println("Missing number: " + findMissingNumber(arr));
    }

    static int findMissingNumber(int[] nums) {
        int n = nums.length;
        int i = 0;
        while (i < n) {
            int correct = nums[i];
            if (correct < n && nums[i] != nums[correct]) {
                swap(nums, i, correct);
            } else {
                i++;
            }
        }
        for (i = 0; i < n; i++) {
            if (nums[i] != i) return i;
        }
        return n;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
Missing number: 2
```

## 2. Duplicate Number

### Theory

Given `n+1` numbers all in `1..n` (guaranteeing at least one duplicate, by pigeonhole), a clever alternative to cyclic sort treats the array as an **implicit linked list**: `nums[i]` points to the "next" index `nums[i]`. A repeated value means two different indices point to the same place — creating a **cycle** — which Floyd's Tortoise and Hare algorithm (the same cycle-detection technique used for linked lists) finds in O(n) time, O(1) space, without modifying the array.

### Code

```java
public class DuplicateNumber {
    public static void main(String[] args) {
        int[] arr = { 1, 3, 4, 2, 2 };
        System.out.println("Duplicate number: " + findDuplicate(arr));
    }

    static int findDuplicate(int[] nums) {
        int slow = nums[0], fast = nums[0];
        do {
            slow = nums[slow];
            fast = nums[nums[fast]];
        } while (slow != fast);

        slow = nums[0];
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;
    }
}
```

### Example

```
Duplicate number: 2
```

## 3. First Missing Positive

### Theory

Given an unsorted array (possibly containing negatives, zeros, and values larger than `n`), find the **smallest missing positive integer**. Cyclic sort handles this directly: place each value `v` at index `v-1`, but only when `v` is actually in the valid range `1..n` (out-of-range values — negatives, zero, too-large — are simply left wherever they land, since they can never be the answer). The first index that doesn't hold `i+1` reveals the answer.

### Code

```java
public class FirstMissingPositive {
    public static void main(String[] args) {
        System.out.println("First missing positive [3,4,-1,1]: " + firstMissingPositive(new int[] { 3, 4, -1, 1 }));
        System.out.println("First missing positive [1,2,0]: " + firstMissingPositive(new int[] { 1, 2, 0 }));
    }

    static int firstMissingPositive(int[] nums) {
        int n = nums.length;
        int i = 0;
        while (i < n) {
            int correct = nums[i] - 1;
            if (correct >= 0 && correct < n && nums[i] != nums[correct]) {
                swap(nums, i, correct);
            } else {
                i++;
            }
        }
        for (i = 0; i < n; i++) {
            if (nums[i] != i + 1) return i + 1;
        }
        return n + 1;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
First missing positive [3,4,-1,1]: 2
First missing positive [1,2,0]: 3
```

## 4. Corrupt Pair

### Theory

Given an array meant to contain every number `1..n` exactly once, but where **one number got duplicated and another went missing** (so the array still has `n` elements, just with a repeat instead of the true missing value), cyclic sort finds **both** in one pass: place every value at its correct index, then whichever index breaks the pattern reveals the duplicate (the value sitting there) and the missing number (the index itself, 1-based).

### Code

```java
public class CorruptPair {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 2, 4 };
        int[] result = findCorruptPair(arr);
        System.out.println("Duplicate: " + result[0] + ", Missing: " + result[1]);
    }

    static int[] findCorruptPair(int[] nums) {
        int n = nums.length;
        int i = 0;
        while (i < n) {
            int correct = nums[i] - 1;
            if (nums[i] != nums[correct]) {
                swap(nums, i, correct);
            } else {
                i++;
            }
        }
        for (i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                return new int[] { nums[i], i + 1 }; // duplicate, missing
            }
        }
        return new int[] { -1, -1 };
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
Duplicate: 2, Missing: 3
```

## 5. Set Mismatch

### Theory

"Set Mismatch" is the same problem as [Corrupt Pair](#4-corrupt-pair) under a different name (used on some platforms) — one number is duplicated, one is missing, from an array meant to be `1..n`. As a contrast to the cyclic-sort approach, it can also be solved **algebraically**: if `actualSum` and `actualSumOfSquares` are computed from the array, and `expectedSum`/`expectedSumOfSquares` come from the formulas for `1..n` (see [Maths Lesson 22: Formula Derivation](<../Maths/22. Optimization.md#4-formula-derivation>)), then two equations in two unknowns (`duplicate - missing` and `duplicate² - missing²`) solve for both values directly — no array mutation needed, at the cost of relying on `long` arithmetic to avoid overflow in the squared terms.

### Code

```java
public class SetMismatch {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 2, 4 };
        int[] result = findErrorNums(arr);
        System.out.println("Duplicate: " + result[0] + ", Missing: " + result[1]);
    }

    static int[] findErrorNums(int[] nums) {
        int n = nums.length;
        long actualSum = 0, actualSumSquares = 0;
        long expectedSum = (long) n * (n + 1) / 2;
        long expectedSumSquares = (long) n * (n + 1) * (2L * n + 1) / 6;

        for (int value : nums) {
            actualSum += value;
            actualSumSquares += (long) value * value;
        }

        long sumDiff = expectedSum - actualSum;               // missing - duplicate
        long squareDiff = expectedSumSquares - actualSumSquares; // missing^2 - duplicate^2
        long sumTotal = squareDiff / sumDiff;                  // missing + duplicate

        long missing = (sumDiff + sumTotal) / 2;
        long duplicate = missing - sumDiff;

        return new int[] { (int) duplicate, (int) missing };
    }
}
```

### Example

```
Duplicate: 2, Missing: 3
```

Same answer as the cyclic-sort version — two different tools, same result.

## 6. Find All Duplicates

### Theory

A generalization of [Corrupt Pair](#4-corrupt-pair): given an array of `n` numbers where each value in `1..n` appears **once or twice**, find **every** value that appears twice (not just one). Same cyclic-sort placement pass; the final scan collects **every** mismatched index's value, instead of stopping at the first one.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class FindAllDuplicates {
    public static void main(String[] args) {
        int[] arr = { 4, 3, 2, 7, 8, 2, 3, 1 };
        System.out.println("All duplicates: " + findDuplicates(arr));
    }

    static List<Integer> findDuplicates(int[] nums) {
        int n = nums.length;
        int i = 0;
        while (i < n) {
            int correct = nums[i] - 1;
            if (nums[i] != nums[correct]) {
                swap(nums, i, correct);
            } else {
                i++;
            }
        }

        List<Integer> result = new ArrayList<>();
        for (i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                result.add(nums[i]);
            }
        }
        return result;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
All duplicates: [3, 2]
```

Both `2` and `3` appear twice in the input (order in the output may vary depending on swap history, but the *set* found is always correct).

## 7. Find All Missing Numbers

### Theory

The mirror of [Find All Duplicates](#6-find-all-duplicates): given an array of `n` numbers from `1..n` with some values **missing** (because others are duplicated to fill their slots), find **every** missing value. Identical cyclic-sort placement pass; the final scan collects every index whose expected value (`index + 1`) never got placed there.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class FindAllMissingNumbers {
    public static void main(String[] args) {
        int[] arr = { 4, 3, 2, 7, 8, 2, 3, 1 };
        System.out.println("All missing: " + findDisappearedNumbers(arr));
    }

    static List<Integer> findDisappearedNumbers(int[] nums) {
        int n = nums.length;
        int i = 0;
        while (i < n) {
            int correct = nums[i] - 1;
            if (nums[i] != nums[correct]) {
                swap(nums, i, correct);
            } else {
                i++;
            }
        }

        List<Integer> result = new ArrayList<>();
        for (i = 0; i < n; i++) {
            if (nums[i] != i + 1) {
                result.add(i + 1);
            }
        }
        return result;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
All missing: [5, 6]
```

`5` and `6` never appear in the input — both `2` and `3` are duplicated to fill their slots instead.

## Quick Reference Cheat Sheet

| Problem | Technique | Complexity |
|---|---|---|
| Missing Number | cyclic sort placement into `0..n-1` | O(n) time, O(1) space |
| Duplicate Number | Floyd's cycle detection on implicit linked list | O(n) time, O(1) space, no mutation |
| First Missing Positive | cyclic sort placement into `1..n`, skip out-of-range | O(n) time, O(1) space |
| Find All Duplicates | same as Corrupt Pair, collect every mismatch's value | O(n) time, O(1) space (excl. output) |
| Find All Missing Numbers | same placement pass, collect every mismatch's index | O(n) time, O(1) space (excl. output) |
| Corrupt Pair | cyclic sort, first mismatch reveals both answers | O(n) time, O(1) space |
| Set Mismatch | algebraic: sum + sum-of-squares equations | O(n) time, O(1) space, no mutation |
