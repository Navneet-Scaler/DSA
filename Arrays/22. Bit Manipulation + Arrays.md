## 1. Bitmask Arrays

### Theory

An `n`-element array's **subsets** can each be represented as an `n`-bit integer — bit `i` set means "index `i` is included" (the same encoding from [Problem-Solving: Power Set](<../Maths/14. Combinatorics.md>) generation). This becomes genuinely powerful as a **DP state**: `dp[mask]` can represent "the best achievable outcome using exactly the subset of elements marked in `mask`" — letting a DP explore all `2ⁿ` subsets systematically.

### Problem 1 (Basic): Sum of All Subset XOR Totals

**Theory:** For every subset of the array (including the empty subset, XOR total `0`), compute its XOR, and sum all of those totals across every subset. Direct bitmask enumeration: for each of the `2ⁿ` masks, XOR together the elements it selects.

```java
public class SumOfAllSubsetXorTotals {
    public static void main(String[] args) {
        System.out.println("Sum of all subset XOR totals [5,1,6]: " + subsetXORSum(new int[] { 5, 1, 6 }));
    }

    static int subsetXORSum(int[] nums) {
        int n = nums.length;
        int total = 0;

        for (int mask = 0; mask < (1 << n); mask++) {
            int xorValue = 0;
            for (int i = 0; i < n; i++) {
                if ((mask & (1 << i)) != 0) xorValue ^= nums[i];
            }
            total += xorValue;
        }
        return total;
    }
}
```

```
Sum of all subset XOR totals [5,1,6]: 28
```

### Problem 2 (Advanced): Partition to K Equal Sum Subsets

**Theory:** Can the array be split into `k` groups with equal sums? `dp[mask]` tracks the current group's running sum (mod target) achievable using exactly the elements in `mask`; transitioning adds one more unused element to the current group, wrapping to a fresh group whenever the target sum is hit.

### Code

```java
public class BitmaskArrays {
    public static void main(String[] args) {
        int[] nums = { 4, 3, 2, 3, 5, 2, 1 };
        System.out.println("Can partition into 4 equal-sum subsets: " + canPartitionKSubsets(nums, 4));
    }

    static boolean canPartitionKSubsets(int[] nums, int k) {
        int total = 0;
        for (int n : nums) total += n;
        if (total % k != 0) return false;
        int target = total / k;

        int n = nums.length;
        for (int num : nums) if (num > target) return false;

        int fullMask = 1 << n;
        int[] dp = new int[fullMask];
        java.util.Arrays.fill(dp, -1);
        dp[0] = 0; // empty subset -> current group sum is 0

        for (int mask = 0; mask < fullMask; mask++) {
            if (dp[mask] == -1) continue; // this subset combination isn't reachable
            int currentGroupSum = dp[mask];

            for (int i = 0; i < n; i++) {
                if ((mask & (1 << i)) == 0 && currentGroupSum + nums[i] <= target) {
                    int newMask = mask | (1 << i);
                    if (dp[newMask] == -1) {
                        dp[newMask] = (currentGroupSum + nums[i]) % target;
                    }
                }
            }
        }
        return dp[fullMask - 1] == 0;
    }
}
```

### Example

```
Can partition into 4 equal-sum subsets: true
```

`[4,3,2,3,5,2,1]` sums to `20`, split into 4 groups of `5` each (e.g. `{4,1}, {3,2}, {3,2}, {5}`) — the bitmask DP confirms this is achievable without searching every grouping explicitly by brute force.

## 2. XOR Patterns

### Theory

Beyond the counting-based XOR problems in [Lesson 21](<./21. XOR Arrays.md>), XOR supports a genuinely different, **search**-flavored pattern. **Classic example — Maximum XOR of Two Numbers in an Array:** find the pair whose XOR is largest. The greedy bit-by-bit approach builds the answer from the **most significant bit down**: at each bit position, tentatively assume that bit can be set in the answer, and check (using the set of number-prefixes seen so far) whether any pair of numbers actually achieves that — keeping the bit only if some pair confirms it.

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class XorPatterns {
    public static void main(String[] args) {
        int[] nums = { 3, 10, 5, 25, 2, 8 };
        System.out.println("Maximum XOR of two numbers: " + findMaximumXOR(nums));
    }

    static int findMaximumXOR(int[] nums) {
        int maxXor = 0;
        int mask = 0;

        for (int bit = 31; bit >= 0; bit--) {
            mask |= (1 << bit); // grow the prefix mask one more bit
            Set<Integer> prefixes = new HashSet<>();
            for (int num : nums) prefixes.add(num & mask);

            int candidate = maxXor | (1 << bit); // tentatively assume this bit can be set
            for (int prefix : prefixes) {
                if (prefixes.contains(candidate ^ prefix)) {
                    maxXor = candidate; // confirmed: some pair achieves this bit
                    break;
                }
            }
        }
        return maxXor;
    }
}
```

### Example

```
Maximum XOR of two numbers: 28
```

`5 ^ 25 = 28` (`00101 ^ 11001 = 11100`) — the largest achievable XOR from any pair in the array.

### Problem 2 (Medium): Total Hamming Distance

**Theory:** The Hamming distance between two numbers is how many bit positions differ. Sum this over **every pair** in the array — checking all O(n²) pairs directly is too slow for large arrays. Instead, handle **one bit position at a time**: if `ones` numbers have that bit set (and `zeros = n - ones` don't), every one-zero pair contributes exactly one differing bit at that position — so the contribution is `ones × zeros`, summed across all 32 bit positions.

```java
public class TotalHammingDistance {
    public static void main(String[] args) {
        System.out.println("Total Hamming distance [4,14,2]: " + totalHammingDistance(new int[] { 4, 14, 2 }));
    }

    static int totalHammingDistance(int[] nums) {
        int total = 0;
        for (int bit = 0; bit < 32; bit++) {
            int ones = 0;
            for (int num : nums) {
                ones += (num >> bit) & 1;
            }
            int zeros = nums.length - ones;
            total += ones * zeros;
        }
        return total;
    }
}
```

```
Total Hamming distance [4,14,2]: 6
```

## 3. OR/AND Problems

### Theory

**Problem 1 (Medium): Bitwise AND of Numbers Range** — given `[m, n]`, find the AND of every integer in that range without iterating through it directly (the range could be huge). Key insight: AND-ing a **long run** of consecutive integers together zeroes out every bit that ever differs between any two numbers in the range — only the **common binary prefix** shared by `m` and `n` survives. Find that common prefix by right-shifting both numbers until they're equal, then shift back.

### Code

```java
public class OrAndProblems {
    public static void main(String[] args) {
        System.out.println("Bitwise AND of range [5, 7]: " + rangeBitwiseAnd(5, 7));
    }

    static int rangeBitwiseAnd(int m, int n) {
        int shift = 0;
        while (m < n) {
            m >>= 1;
            n >>= 1;
            shift++;
        }
        return m << shift;
    }
}
```

### Example

```
Bitwise AND of range [5, 7]: 4
```

`5 & 6 & 7` = `101 & 110 & 111` = `100` = `4` — only the leading `1` bit survives, since bit positions below it flip somewhere across the range.

### Problem 2 (Advanced): Bitwise ORs of Subarrays

**Theory:** Count how many **distinct** results are possible from OR-ing together some contiguous subarray. Brute force checks all O(n²) subarrays; the key optimization is that the set of distinct OR-values ending at index `i` is small (at most 32, since each new element can only ever **add** bits to a running OR, never remove them, so the set can grow at most 32 times before saturating) — maintain that small set incrementally: OR the new element into every value from the previous index's set, plus the new element alone.

```java
import java.util.HashSet;
import java.util.Set;

public class BitwiseOrsOfSubarrays {
    public static void main(String[] args) {
        System.out.println("Distinct subarray ORs [1,1,2]: " + subarrayBitwiseORs(new int[] { 1, 1, 2 }));
    }

    static int subarrayBitwiseORs(int[] arr) {
        Set<Integer> result = new HashSet<>();
        Set<Integer> current = new HashSet<>();

        for (int num : arr) {
            Set<Integer> next = new HashSet<>();
            next.add(num);
            for (int prev : current) {
                next.add(prev | num);
            }
            current = next;
            result.addAll(current);
        }
        return result.size();
    }
}
```

```
Distinct subarray ORs [1,1,2]: 3
```

The distinct achievable OR values are `{1, 2, 3}` — `1` (from any single `1`), `2` (the single `2`), and `3` (`1|2`).

## Quick Reference Cheat Sheet

| Problem | Technique | Complexity |
|---|---|---|
| Sum of All Subset XOR Totals | brute-force bitmask enumeration | O(2ⁿ × n) |
| Partition to K Equal Sum Subsets | bitmask DP: `dp[mask]` = current group's running sum | O(2ⁿ × n) |
| Maximum XOR of Two Numbers | greedy bit-by-bit, prefix set per bit position | O(32n) |
| Total Hamming Distance | per-bit `ones × zeros` contribution | O(32n) |
| Bitwise AND of Numbers Range | shift both bounds until equal, capturing the common prefix | O(log(max value)) |
| Bitwise ORs of Subarrays | incremental small OR-set per position | O(32n) |
