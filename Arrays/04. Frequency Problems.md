# 4. Frequency Problems

Counting how often each value appears is one of the most common array sub-tasks — it powers anagram checks, majority-element problems, duplicate detection, and missing-number puzzles. This lesson covers the two core counting strategies and the classic problems built on top of them.

## 1. Counting Frequency

### Theory

Counting frequency means tallying how many times each distinct value occurs, in a single **O(n)** pass. There are two ways to store the tallies:

1. A **HashMap** — works for any value type, any range.
2. A **fixed-size array** ("bucket array") — faster and lighter, but only works when the possible values are small and bounded (like lowercase letters, or digits `0-9`).

---

## 2. HashMap Frequency

### Theory

When values could be anything (arbitrary integers, strings, unbounded range), a `HashMap<T, Integer>` maps each distinct value to its count — general-purpose, O(1) average lookup/update per element.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapFrequency {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 4, 6, 5, 4 };
        Map<Integer, Integer> freq = countFrequency(arr);
        System.out.println("Frequency map: " + freq);
    }

    static Map<Integer, Integer> countFrequency(int[] arr) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int value : arr) {
            freq.merge(value, 1, Integer::sum);
        }
        return freq;
    }
}
```

### Example

```
Frequency map: {4=3, 5=2, 6=1}
```

## 3. Array Frequency

### Theory

When the value range is small and known in advance (e.g. `0-9`, or `'a'-'z'`), a plain array as a **bucket** is faster than a HashMap — no hashing, no boxing, just direct indexing.

### Code

```java
import java.util.Arrays;

public class ArrayFrequency {
    public static void main(String[] args) {
        int[] arr = { 3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5 };
        int[] freq = countFrequency(arr);
        System.out.println("Frequency of digits 0-9: " + Arrays.toString(freq));
    }

    static int[] countFrequency(int[] arr) {
        int[] freq = new int[10]; // bucket for digits 0-9
        for (int value : arr) {
            freq[value]++;
        }
        return freq;
    }
}
```

### Example

```
Frequency of digits 0-9: [0, 2, 1, 2, 1, 3, 1, 0, 0, 1]
```

## 4. Character Frequency

### Theory

Counting character frequency in a string is the array-frequency pattern applied to letters: map `'a'-'z'` onto bucket indices `0-25` via `c - 'a'`.

### Code

```java
public class CharacterFrequency {
    public static void main(String[] args) {
        String s = "engineering";
        int[] freq = countCharFrequency(s);

        System.out.println("'e' count: " + freq['e' - 'a']);
        System.out.println("'n' count: " + freq['n' - 'a']);
        System.out.println("'g' count: " + freq['g' - 'a']);
    }

    static int[] countCharFrequency(String s) {
        int[] freq = new int[26];
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }
        return freq;
    }
}
```

### Example

```
'e' count: 3
'n' count: 3
'g' count: 2
```

## 5. Integer Frequency

### Theory

For arbitrary (possibly negative, possibly huge-range) integers, the array-bucket trick doesn't fit — fall back to the [HashMap approach](#2-hashmap-frequency). This is the same core idea as [Character Frequency](#4-character-frequency), just without the bounded-range shortcut.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class IntegerFrequency {
    public static void main(String[] args) {
        int[] arr = { -5, 100, -5, 42, 100, 100 };
        Map<Integer, Integer> freq = new HashMap<>();
        for (int value : arr) freq.merge(value, 1, Integer::sum);
        System.out.println("Integer frequency: " + freq);
    }
}
```

### Example

```
Integer frequency: {-5=2, 100=3, 42=1}
```

## 6. Majority Element

### Theory

The **majority element** appears **more than `n/2`** times. The elegant **Boyer-Moore Voting Algorithm** finds it in **O(n) time and O(1) space** — no frequency map needed: maintain a `candidate` and a `count`; when `count` hits `0`, switch candidates; increment on a match, decrement otherwise. Since the majority element outnumbers everything else combined, it always survives to the end.

> This assumes a majority element **is guaranteed to exist**; if not, a separate verification pass is needed.

### Code

```java
public class MajorityElement {
    public static void main(String[] args) {
        int[] arr = { 2, 2, 1, 1, 1, 2, 2 };
        System.out.println("Majority element: " + findMajority(arr));
    }

    static int findMajority(int[] arr) {
        int count = 0, candidate = 0;
        for (int value : arr) {
            if (count == 0) {
                candidate = value;
            }
            count += (value == candidate) ? 1 : -1;
        }
        return candidate;
    }
}
```

### Example

```
Majority element: 2
```

## 7. Duplicate Detection

### Theory

To find the **first repeated** value, walk the array once, tracking values already seen in a `HashSet` — the moment a value is seen again, it's a duplicate. **O(n) time, O(n) space.**

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class DuplicateDetection {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 2, 5 };
        System.out.println("First duplicate: " + findDuplicate(arr));
    }

    static int findDuplicate(int[] arr) {
        Set<Integer> seen = new HashSet<>();
        for (int value : arr) {
            if (!seen.add(value)) { // add() returns false if value was already present
                return value;
            }
        }
        return -1; // no duplicate found
    }
}
```

### Example

```
First duplicate: 2
```

## 8. Missing Number

### Theory

Given an array containing `n` distinct numbers from `0` to `n` with exactly **one** missing, two O(n)-time, O(1)-space tricks find it without any hashing:

1. **Sum formula** — the expected sum of `0..n` is `n(n+1)/2`; subtract the array's actual sum, and the difference is the missing number.
2. **XOR** — XOR every index `0..n` together with every array value; every present number cancels with its index pairing, leaving only the missing one (same cancellation idea as [Lesson 23: XOR Math](<./../Maths/23. Bit Mathematics.md#2-xor-math>)).

### Code

```java
public class MissingNumber {
    public static void main(String[] args) {
        int[] arr = { 3, 0, 1 }; // numbers 0..3, one missing (n=3)

        System.out.println("Missing number [sum formula]: " + findMissingSum(arr));
        System.out.println("Missing number [XOR]: " + findMissingXor(arr));
    }

    static int findMissingSum(int[] arr) {
        int n = arr.length;
        int expectedSum = n * (n + 1) / 2;
        int actualSum = 0;
        for (int value : arr) actualSum += value;
        return expectedSum - actualSum;
    }

    static int findMissingXor(int[] arr) {
        int n = arr.length;
        int result = 0;
        for (int i = 0; i <= n; i++) result ^= i;
        for (int value : arr) result ^= value;
        return result;
    }
}
```

### Example

```
Missing number [sum formula]: 2
Missing number [XOR]: 2
```

`arr = [3, 0, 1]` is missing `2` from the range `0..3`.

## Quick Reference Cheat Sheet

| Technique | Approach | When to use |
|---|---|---|
| HashMap Frequency | `HashMap<T, Integer>` | any value type, unbounded range |
| Array Frequency | fixed-size bucket array | small, known value range |
| Character Frequency | `int[26]` bucket, index via `c - 'a'` | lowercase letter counting |
| Majority Element | Boyer-Moore voting | element appearing `> n/2` times |
| Duplicate Detection | `HashSet`, check on insert | first repeated value |
| Missing Number | sum formula or XOR cancellation | exactly one missing number in `0..n` |
