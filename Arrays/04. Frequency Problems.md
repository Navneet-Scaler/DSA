# 4. Frequency Problems

Counting how often each value appears is one of the most common array sub-tasks — it powers anagram checks, majority-element problems, duplicate detection, and missing-number puzzles. This lesson covers the two core counting strategies and the classic problems built on top of them.

## 1. Counting Frequency

### Theory

Counting frequency means tallying how many times each distinct value occurs, in a single **O(n)** pass. There are two ways to store the tallies:

1. A **HashMap** — works for any value type, any range.
2. A **fixed-size array** ("bucket array") — faster and lighter, but only works when the possible values are small and bounded (like lowercase letters, or digits `0-9`).

---

## 2. HashMap Frequency

### Theory

When values could be anything (arbitrary integers, strings, unbounded range), a `HashMap<T, Integer>` maps each distinct value to its count — general-purpose, O(1) average lookup/update per element.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapFrequency {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 4, 6, 5, 4 };
        Map<Integer, Integer> freq = countFrequency(arr);
        System.out.println("Frequency map: " + freq);
    }

    static Map<Integer, Integer> countFrequency(int[] arr) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int value : arr) {
            freq.merge(value, 1, Integer::sum);
        }
        return freq;
    }
}
```

### Example

```
Frequency map: {4=3, 5=2, 6=1}
```

### The Modern Java Alternative: Streams + Collectors.groupingBy

The loop above is the version to *understand*, but modern Java code often reaches for the `Stream` API instead — `Collectors.groupingBy` groups elements by a classifier function, and pairing it with `Collectors.counting()` turns that grouping directly into a frequency map, with no explicit loop at all:

```java
import java.util.Arrays;
import java.util.Map;
import java.util.stream.Collectors;

public class StreamFrequency {
    public static void main(String[] args) {
        Integer[] arr = { 4, 5, 4, 6, 5, 4 };
        Map<Integer, Long> freq = Arrays.stream(arr)
                .collect(Collectors.groupingBy(value -> value, Collectors.counting()));
        System.out.println("Frequency map (streams): " + freq);
    }
}
```

```
Frequency map (streams): {4=3, 5=2, 6=1}
```

Same result, but notice the value type is `Long`, not `Integer` — `Collectors.counting()` always produces a `long` count. Also note `Arrays.stream()` needs a **boxed** `Integer[]`, not a primitive `int[]` — streams over primitives (`IntStream`) don't have a direct `groupingBy` collector, since grouping requires boxing the keys anyway.

## 3. Array Frequency

### Theory

When the value range is small and known in advance (e.g. `0-9`, or `'a'-'z'`), a plain array as a **bucket** is faster than a HashMap — no hashing, no boxing, just direct indexing.

### Code

```java
import java.util.Arrays;

public class ArrayFrequency {
    public static void main(String[] args) {
        int[] arr = { 3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5 };
        int[] freq = countFrequency(arr);
        System.out.println("Frequency of digits 0-9: " + Arrays.toString(freq));
    }

    static int[] countFrequency(int[] arr) {
        int[] freq = new int[10]; // bucket for digits 0-9
        for (int value : arr) {
            freq[value]++;
        }
        return freq;
    }
}
```

### Example

```
Frequency of digits 0-9: [0, 2, 1, 2, 1, 3, 1, 0, 0, 1]
```

## 4. Character Frequency

### Theory

Counting character frequency in a string is the array-frequency pattern applied to letters: map `'a'-'z'` onto bucket indices `0-25` via `c - 'a'`.

### Code

```java
public class CharacterFrequency {
    public static void main(String[] args) {
        String s = "engineering";
        int[] freq = countCharFrequency(s);

        System.out.println("'e' count: " + freq['e' - 'a']);
        System.out.println("'n' count: " + freq['n' - 'a']);
        System.out.println("'g' count: " + freq['g' - 'a']);
    }

    static int[] countCharFrequency(String s) {
        int[] freq = new int[26];
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }
        return freq;
    }
}
```

### Example

```
'e' count: 3
'n' count: 3
'g' count: 2
```

## 5. Integer Frequency

### Theory

For arbitrary (possibly negative, possibly huge-range) integers, the array-bucket trick doesn't fit — fall back to the [HashMap approach](#2-hashmap-frequency). This is the same core idea as [Character Frequency](#4-character-frequency), just without the bounded-range shortcut.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class IntegerFrequency {
    public static void main(String[] args) {
        int[] arr = { -5, 100, -5, 42, 100, 100 };
        Map<Integer, Integer> freq = new HashMap<>();
        for (int value : arr) freq.merge(value, 1, Integer::sum);
        System.out.println("Integer frequency: " + freq);
    }
}
```

### Example

```
Integer frequency: {-5=2, 100=3, 42=1}
```

## 6. Majority Element

### Theory

The **majority element** appears **more than `n/2`** times. The elegant **Boyer-Moore Voting Algorithm** finds it in **O(n) time and O(1) space** — no frequency map needed: maintain a `candidate` and a `count`; when `count` hits `0`, switch candidates; increment on a match, decrement otherwise. Since the majority element outnumbers everything else combined, it always survives to the end.

> This assumes a majority element **is guaranteed to exist**; if not, a separate verification pass is needed.

### Code

```java
public class MajorityElement {
    public static void main(String[] args) {
        int[] arr = { 2, 2, 1, 1, 1, 2, 2 };
        System.out.println("Majority element: " + findMajority(arr));
    }

    static int findMajority(int[] arr) {
        int count = 0, candidate = 0;
        for (int value : arr) {
            if (count == 0) {
                candidate = value;
            }
            count += (value == candidate) ? 1 : -1;
        }
        return candidate;
    }
}
```

### Example

```
Majority element: 2
```

### Problem 2 (Advanced): Majority Element II (appearing more than n/3 times)

**Theory:** Generalizes Boyer-Moore to find every element appearing **more than `n/3`** times. Since more than 2 such elements can never coexist (three elements each exceeding `n/3` would need over `n` total elements), track **two** candidates and two counters instead of one, decrementing both whenever an element matches neither. A final verification pass confirms each surviving candidate truly exceeds the threshold (unlike the exactly-`n/2` case, a qualifying element here isn't guaranteed to exist).

```java
import java.util.ArrayList;
import java.util.List;

public class MajorityElementII {
    public static void main(String[] args) {
        System.out.println("Majority (>n/3) in [3,2,3]: " + majorityElement(new int[] { 3, 2, 3 }));
        System.out.println("Majority (>n/3) in [1,1,1,3,3,2,2,2]: "
                + majorityElement(new int[] { 1, 1, 1, 3, 3, 2, 2, 2 }));
    }

    static List<Integer> majorityElement(int[] nums) {
        Integer candidate1 = null, candidate2 = null;
        int count1 = 0, count2 = 0;

        for (int n : nums) {
            if (candidate1 != null && candidate1 == n) {
                count1++;
            } else if (candidate2 != null && candidate2 == n) {
                count2++;
            } else if (count1 == 0) {
                candidate1 = n;
                count1 = 1;
            } else if (count2 == 0) {
                candidate2 = n;
                count2 = 1;
            } else {
                count1--;
                count2--;
            }
        }

        List<Integer> result = new ArrayList<>();
        count1 = count2 = 0;
        for (int n : nums) { // verification pass -- candidates aren't guaranteed to qualify
            if (candidate1 != null && n == candidate1) count1++;
            if (candidate2 != null && n == candidate2) count2++;
        }
        if (count1 > nums.length / 3) result.add(candidate1);
        if (count2 > nums.length / 3) result.add(candidate2);
        return result;
    }
}
```

```
Majority (>n/3) in [3,2,3]: [3]
Majority (>n/3) in [1,1,1,3,3,2,2,2]: [1, 2]
```

## 7. Duplicate Detection

### Theory

To find the **first repeated** value, walk the array once, tracking values already seen in a `HashSet` — the moment a value is seen again, it's a duplicate. **O(n) time, O(n) space.**

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class DuplicateDetection {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 2, 5 };
        System.out.println("First duplicate: " + findDuplicate(arr));
    }

    static int findDuplicate(int[] arr) {
        Set<Integer> seen = new HashSet<>();
        for (int value : arr) {
            if (!seen.add(value)) { // add() returns false if value was already present
                return value;
            }
        }
        return -1; // no duplicate found
    }
}
```

### Example

```
First duplicate: 2
```

### Problem 2 (Medium): Contains Duplicate II (nearby duplicates)

**Theory:** A stricter variant — report whether a duplicate exists **within index distance `k`** of each other, not just anywhere in the array. Track the **most recent index** each value was seen at; a duplicate only counts if the gap since that index is `≤ k`. Because only the *most recent* occurrence ever matters going forward, updating the map with the latest index (even when the check fails) keeps every future comparison correct.

```java
import java.util.HashMap;
import java.util.Map;

public class ContainsDuplicateNearby {
    public static void main(String[] args) {
        System.out.println("[1,2,3,1] k=3: " + containsNearbyDuplicate(new int[] { 1, 2, 3, 1 }, 3));
        System.out.println("[1,2,3,1,2,3] k=2: " + containsNearbyDuplicate(new int[] { 1, 2, 3, 1, 2, 3 }, 2));
    }

    static boolean containsNearbyDuplicate(int[] nums, int k) {
        Map<Integer, Integer> lastIndex = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            if (lastIndex.containsKey(nums[i]) && i - lastIndex.get(nums[i]) <= k) {
                return true;
            }
            lastIndex.put(nums[i], i);
        }
        return false;
    }
}
```

```
[1,2,3,1] k=3: true
[1,2,3,1,2,3] k=2: false
```

The two `1`s in the first array are `3` apart (`≤ k=3`), so it qualifies; in the second array every repeated value is `3` apart, exceeding `k=2`.

## 8. Missing Number

### Theory

Given an array containing `n` distinct numbers from `0` to `n` with exactly **one** missing, two O(n)-time, O(1)-space tricks find it without any hashing:

1. **Sum formula** — the expected sum of `0..n` is `n(n+1)/2`; subtract the array's actual sum, and the difference is the missing number.
2. **XOR** — XOR every index `0..n` together with every array value; every present number cancels with its index pairing, leaving only the missing one (same cancellation idea as [Lesson 23: XOR Math](<./../Maths/23. Bit Mathematics.md#2-xor-math>)).

### Code

```java
public class MissingNumber {
    public static void main(String[] args) {
        int[] arr = { 3, 0, 1 }; // numbers 0..3, one missing (n=3)

        System.out.println("Missing number [sum formula]: " + findMissingSum(arr));
        System.out.println("Missing number [XOR]: " + findMissingXor(arr));
    }

    static int findMissingSum(int[] arr) {
        int n = arr.length;
        int expectedSum = n * (n + 1) / 2;
        int actualSum = 0;
        for (int value : arr) actualSum += value;
        return expectedSum - actualSum;
    }

    static int findMissingXor(int[] arr) {
        int n = arr.length;
        int result = 0;
        for (int i = 0; i <= n; i++) result ^= i;
        for (int value : arr) result ^= value;
        return result;
    }
}
```

### Example

```
Missing number [sum formula]: 2
Missing number [XOR]: 2
```

`arr = [3, 0, 1]` is missing `2` from the range `0..3`.

## 9. Valid Anagram

### Theory

Two strings are anagrams if they contain **exactly the same characters with the same frequencies** — just count characters in both strings ([Character Frequency](#4-character-frequency)) and compare the resulting frequency arrays.

### Code

```java
public class ValidAnagram {
    public static void main(String[] args) {
        System.out.println("'anagram' / 'nagaram': " + isAnagram("anagram", "nagaram"));
    }

    static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;

        int[] count = new int[26];
        for (char c : s.toCharArray()) count[c - 'a']++;
        for (char c : t.toCharArray()) count[c - 'a']--;

        for (int c : count) {
            if (c != 0) return false;
        }
        return true;
    }
}
```

### Example

```
'anagram' / 'nagaram': true
```

## 10. Group Anagrams

### Theory

Given a list of strings, group every set of anagrams together. Instead of comparing every pair (O(n²) string comparisons), give every string a **canonical key** — its characters sorted alphabetically — so all anagrams of each other map to the identical key. A `HashMap<String, List<String>>` from key to group then does the grouping in a single O(n log L) pass (L = average string length, from the sort).

### Code

```java
import java.util.*;

public class GroupAnagrams {
    public static void main(String[] args) {
        String[] words = { "eat", "tea", "tan", "ate", "nat", "bat" };
        System.out.println("Groups: " + groupAnagrams(words));
    }

    static List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> groups = new HashMap<>();

        for (String s : strs) {
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars); // canonical form -- same for every anagram of s

            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(groups.values());
    }
}
```

### Example

```
Groups: [[eat, tea, ate], [tan, nat], [bat]]
```

## Quick Reference Cheat Sheet

| Technique | Approach | When to use |
|---|---|---|
| HashMap Frequency | `HashMap<T, Integer>` | any value type, unbounded range |
| Array Frequency | fixed-size bucket array | small, known value range |
| Character Frequency | `int[26]` bucket, index via `c - 'a'` | lowercase letter counting |
| Majority Element | Boyer-Moore voting (1 or 2 candidates) | element(s) appearing `> n/2` or `> n/3` times |
| Duplicate Detection | `HashSet`/`HashMap`, check on insert or index gap | first repeated value, or nearby duplicates within `k` |
| Missing Number | sum formula or XOR cancellation | exactly one missing number in `0..n` |
| Valid Anagram | compare character frequency arrays | two strings, same length |
| Group Anagrams | `HashMap` keyed by sorted-character canonical form | grouping many strings at once |
