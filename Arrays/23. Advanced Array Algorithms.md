## 1. Merge Intervals

### Theory

Given a list of `[start, end]ranges`, merge every pair that **overlaps** into one combined range. Sort by start time first — once sorted, any overlap can only happen with the **most recently merged** interval, so a single linear scan suffices: extend the last merged interval if the next one overlaps it, otherwise start a new merged interval.

### Code

```java
import java.util.*;

public class MergeIntervals {
    public static void main(String[] args) {
        int[][] intervals = { { 1, 3 }, { 2, 6 }, { 8, 10 }, { 15, 18 } };
        System.out.println("Merged: " + Arrays.deepToString(merge(intervals)));
    }

    static int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));

        List<int[]> merged = new ArrayList<>();
        for (int[] interval : intervals) {
            if (merged.isEmpty() || merged.get(merged.size() - 1)[1] < interval[0]) {
                merged.add(interval);
            } else {
                merged.get(merged.size() - 1)[1] = Math.max(merged.get(merged.size() - 1)[1], interval[1]);
            }
        }
        return merged.toArray(new int[0][]);
    }
}
```

### Example

```
Merged: [[1, 6], [8, 10], [15, 18]]
```

`[1,3]` and `[2,6]` overlap (`2 ≤ 3`) and merge into `[1,6]`; the rest don't overlap anything and pass through unchanged.

### Problem 2 (Medium): Insert Interval

**Theory:** Given an **already** non-overlapping, sorted list of intervals, insert a new interval and merge as needed. Rather than appending and re-running full [Merge Intervals](#1-merge-intervals) (which would re-sort unnecessarily), walk through in three phases: copy every interval ending entirely **before** the new one, merge every interval that **overlaps** it, then copy every interval starting entirely **after** it.

```java
import java.util.ArrayList;
import java.util.List;

public class InsertInterval {
    public static void main(String[] args) {
        int[][] intervals = { { 1, 3 }, { 6, 9 } };
        int[] newInterval = { 2, 5 };
        System.out.println("After insert: " + java.util.Arrays.deepToString(insert(intervals, newInterval)));
    }

    static int[][] insert(int[][] intervals, int[] newInterval) {
        List<int[]> result = new ArrayList<>();
        int i = 0, n = intervals.length;

        while (i < n && intervals[i][1] < newInterval[0]) {
            result.add(intervals[i++]); // entirely before the new interval
        }

        while (i < n && intervals[i][0] <= newInterval[1]) {
            newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
            newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
            i++;
        }
        result.add(newInterval);

        while (i < n) {
            result.add(intervals[i++]); // entirely after the new interval
        }
        return result.toArray(new int[0][]);
    }
}
```

```
After insert: [[1, 5], [6, 9]]
```

`[2,5]` overlaps `[1,3]` (merging into `[1,5]`) but not `[6,9]`, which passes through unchanged.

## 2. Sweep Line / Line Sweep

### Theory

"Sweep line" and "line sweep" are two names for the same technique: process a set of **events** (typically interval starts/ends) in **sorted order**, maintaining a running state that updates at each event — turning problems that look like they need checking every pair of intervals into a single linear-ish pass. **Classic example — Car Pooling:** a car picks up and drops off passengers along a route defined by `[numPassengers, start, end]` trips; determine whether the car's capacity is ever exceeded. Treat each trip as **two events** — passengers **join** at `start`, passengers **leave** at `end` — accumulate these into a [difference array](<./05. Prefix Techniques.md#5-difference-array>) indexed by location, then sweep through checking if the running passenger count ever exceeds capacity.

### Code

```java
public class SweepLine {
    public static void main(String[] args) {
        int[][] trips = { { 2, 1, 5 }, { 3, 3, 7 } };
        System.out.println("Capacity 4 works: " + carPooling(trips, 4));
        System.out.println("Capacity 5 works: " + carPooling(trips, 5));
    }

    static boolean carPooling(int[][] trips, int capacity) {
        int maxLocation = 0;
        for (int[] trip : trips) maxLocation = Math.max(maxLocation, trip[2]);

        int[] passengerChange = new int[maxLocation + 2]; // difference array over locations
        for (int[] trip : trips) {
            int numPassengers = trip[0], start = trip[1], end = trip[2];
            passengerChange[start] += numPassengers; // event: passengers join
            passengerChange[end] -= numPassengers;    // event: passengers leave
        }

        int currentPassengers = 0;
        for (int change : passengerChange) {
            currentPassengers += change;
            if (currentPassengers > capacity) return false;
        }
        return true;
    }
}
```

### Example

```
Capacity 4 works: false
Capacity 5 works: true
```

Between location `3` and `5`, both trips overlap: `2 + 3 = 5` passengers are aboard simultaneously — fits in a car with capacity `5`, but not `4`.

## 3. Coordinate Compression

### Theory

When values span a **huge range** but only a **small number of distinct values** actually appear (e.g. coordinates up to `10^9`, but only `1000` of them show up in the input), map each distinct value to its **rank** among the sorted unique values — shrinking the effective range down to the count of distinct values, which then fits comfortably into an array-indexed technique (like a difference array or a small hash-free lookup) that would otherwise need infeasible memory.

### Code

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;
import java.util.TreeSet;

public class CoordinateCompression {
    public static void main(String[] args) {
        int[] arr = { 100, 4, 200, 1, 3, 2 };
        System.out.println("Compressed ranks: " + Arrays.toString(compress(arr)));
    }

    static int[] compress(int[] arr) {
        TreeSet<Integer> sortedUnique = new TreeSet<>();
        for (int value : arr) sortedUnique.add(value);

        Map<Integer, Integer> rank = new HashMap<>();
        int index = 0;
        for (int value : sortedUnique) {
            rank.put(value, index++);
        }

        int[] result = new int[arr.length];
        for (int i = 0; i < arr.length; i++) {
            result[i] = rank.get(arr[i]);
        }
        return result;
    }
}
```

### Example

```
Compressed ranks: [4, 3, 5, 0, 2, 1]
```

The 6 huge, spread-out values `{1,2,3,4,100,200}` compress down to dense ranks `{0,1,2,3,4,5}` — `100` becomes rank `4`, `200` becomes rank `5`, and so on, preserving relative order.

### Problem (Basic): Rank Transform of an Array

**Theory:** A direct, named application of coordinate compression: replace every element with its **rank** among the distinct sorted values (starting at `1` instead of `0`). This is exactly the compression technique from above, with the output being the compressed values themselves rather than an intermediate step toward something else.

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;
import java.util.TreeSet;

public class RankTransform {
    public static void main(String[] args) {
        int[] arr = { 40, 10, 20, 30 };
        System.out.println("Ranks: " + Arrays.toString(arrayRankTransform(arr)));
    }

    static int[] arrayRankTransform(int[] arr) {
        TreeSet<Integer> sortedUnique = new TreeSet<>();
        for (int value : arr) sortedUnique.add(value);

        Map<Integer, Integer> rank = new HashMap<>();
        int r = 1;
        for (int value : sortedUnique) rank.put(value, r++);

        int[] result = new int[arr.length];
        for (int i = 0; i < arr.length; i++) result[i] = rank.get(arr[i]);
        return result;
    }
}
```

```
Ranks: [4, 1, 2, 3]
```

`10` is the smallest (rank `1`), `40` is the largest (rank `4`) — each element replaced by its position in sorted order.

## 5. Sparse Array Idea

### Theory

When an array is **mostly zeros/defaults** with only a few meaningful entries, storing it as a full-length array wastes memory proportional to its size rather than its actual content. A **sparse representation** — a `HashMap<Integer, T>` mapping only the **non-default indices** to their values — uses memory proportional to the number of *non-zero* entries instead. **Classic example — Sparse Vector Dot Product:** computing the dot product of two mostly-zero vectors only needs to multiply positions where **both** vectors have a non-zero entry; storing each vector as an index-to-value map skips every position where at least one side is zero automatically.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class SparseArrayIdea {
    public static void main(String[] args) {
        int[] nums1 = { 1, 0, 0, 2, 3 };
        int[] nums2 = { 0, 3, 0, 4, 0 };

        SparseVector v1 = new SparseVector(nums1);
        SparseVector v2 = new SparseVector(nums2);
        System.out.println("Dot product: " + v1.dotProduct(v2));
    }

    static class SparseVector {
        private final Map<Integer, Integer> nonZeroEntries = new HashMap<>();

        SparseVector(int[] nums) {
            for (int i = 0; i < nums.length; i++) {
                if (nums[i] != 0) {
                    nonZeroEntries.put(i, nums[i]);
                }
            }
        }

        int dotProduct(SparseVector other) {
            int result = 0;
            // Iterate the smaller map for efficiency -- only shared non-zero indices matter
            Map<Integer, Integer> smaller = (this.nonZeroEntries.size() <= other.nonZeroEntries.size())
                    ? this.nonZeroEntries : other.nonZeroEntries;
            Map<Integer, Integer> larger = (smaller == this.nonZeroEntries) ? other.nonZeroEntries : this.nonZeroEntries;

            for (Map.Entry<Integer, Integer> entry : smaller.entrySet()) {
                if (larger.containsKey(entry.getKey())) {
                    result += entry.getValue() * larger.get(entry.getKey());
                }
            }
            return result;
        }
    }
}
```

### Example

```
Dot product: 8
```

Only index `3` has a non-zero value in **both** vectors (`2 × 4 = 8`) — every other position is skipped without ever touching the full-length arrays.

## Quick Reference Cheat Sheet

| Technique | Core idea | Complexity |
|---|---|---|
| Merge Intervals | sort by start, extend or start a new merged range (→ Insert Interval: three-phase walk) | O(n log n) |
| Sweep Line / Line Sweep | process sorted events, maintain running state | O(n log n) (dominated by the sort) |
| Coordinate Compression | map huge sparse values to dense ranks (→ Rank Transform of an Array) | O(n log n) |
| Difference Array | O(1) range updates, reconstruct via prefix sum | see [Lesson 5](<./05. Prefix Techniques.md#5-difference-array>) |
| Sparse Array Idea | `HashMap` of non-default entries instead of a full array | memory ∝ non-zero count, not array length |
