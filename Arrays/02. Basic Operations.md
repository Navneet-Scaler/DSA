# 2. Basic Operations

Every array algorithm is built from a small set of primitive operations — walk through it, change a value, make room for a new one, close a gap, find something, or rearrange what's there. This lesson covers each one directly, since [Lesson 1](<./01. Array Fundamentals.md>) already covered *why* they have the time costs they do.

## 1. Traversal (Forward & Reverse)

### Theory

Traversal visits every element exactly once, in order. Forward traversal walks index `0` to `length-1`; reverse traversal walks `length-1` down to `0` — same cost (**O(n)**), just opposite direction, useful whenever a problem's logic depends on processing from the end (e.g. finding the last occurrence, or building a result backward).

### Code

```java
public class Traversal {
    public static void main(String[] args) {
        int[] arr = { 10, 20, 30, 40, 50 };

        System.out.print("Forward: ");
        for (int i = 0; i < arr.length; i++) System.out.print(arr[i] + " ");
        System.out.println();

        System.out.print("Reverse: ");
        for (int i = arr.length - 1; i >= 0; i--) System.out.print(arr[i] + " ");
        System.out.println();
    }
}
```

### Example

```
Forward: 10 20 30 40 50 
Reverse: 50 40 30 20 10 
```

## 2. Updating

### Theory

Updating an element at a known index is a direct write — **O(1)**, using the same address arithmetic that makes reads O(1) (see [Lesson 1](<./01. Array Fundamentals.md#3-memory-layout--contiguous-memory>)).

### Code

```java
import java.util.Arrays;

public class Updating {
    public static void main(String[] args) {
        int[] arr = { 10, 20, 30 };
        arr[1] = 99;
        System.out.println("After arr[1] = 99: " + Arrays.toString(arr));
    }
}
```

### Example

```
After arr[1] = 99: [10, 99, 30]
```

## 3. Insertion

### Theory

Since Java arrays can't grow, inserting a new element means creating a **new, larger array**, copying everything before the insertion point, placing the new value, then copying everything after it, shifted one slot right. This is **O(n)** — dominated by the shifting/copying.

### Code

```java
import java.util.Arrays;

public class Insertion {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        int[] result = insertAt(arr, 2, 99);
        System.out.println("Insert 99 at index 2: " + Arrays.toString(result));
    }

    static int[] insertAt(int[] arr, int index, int value) {
        int[] result = new int[arr.length + 1];
        for (int i = 0; i < index; i++) result[i] = arr[i];
        result[index] = value;
        for (int i = index; i < arr.length; i++) result[i + 1] = arr[i];
        return result;
    }
}
```

### Example

```
Insert 99 at index 2: [1, 2, 99, 3, 4, 5]
```

## 4. Deletion

### Theory

Deleting an element means shifting every element **after** it one slot left, closing the gap, then shrinking to a new array one slot smaller. Also **O(n)**.

### Code

```java
import java.util.Arrays;

public class Deletion {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        int[] result = deleteAt(arr, 2);
        System.out.println("Delete index 2: " + Arrays.toString(result));
    }

    static int[] deleteAt(int[] arr, int index) {
        int[] result = new int[arr.length - 1];
        for (int i = 0; i < index; i++) result[i] = arr[i];
        for (int i = index; i < result.length; i++) result[i] = arr[i + 1];
        return result;
    }
}
```

### Example

```
Delete index 2: [1, 2, 4, 5]
```

## 5. Searching

### Theory

**Linear search** checks every element until a match is found — **O(n)**, works on any array regardless of order. If the array is sorted, [binary search](<./10. Binary Search on Arrays.md>) does better at O(log n).

### Code

```java
public class Searching {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("Index of 4: " + linearSearch(arr, 4));
        System.out.println("Index of 9 (not present): " + linearSearch(arr, 9));
    }

    static int linearSearch(int[] arr, int target) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == target) return i;
        }
        return -1;
    }
}
```

### Example

```
Index of 4: 3
Index of 9 (not present): -1
```

## 6. Copying

### Theory

A copy duplicates an array's contents into a new backing array — necessary because assigning `arr2 = arr1` only copies the **reference**, leaving both variables pointing at the same underlying array (mutating one mutates the other). A manual element-by-element copy is **O(n)**; Java's built-in `Arrays.copyOf` (covered in [Lesson 3](<./03. Java Array APIs.md>)) does the same thing more concisely.

### Code

```java
import java.util.Arrays;

public class Copying {
    public static void main(String[] args) {
        int[] original = { 1, 2, 3 };
        int[] copy = manualCopy(original);
        copy[0] = 99; // shouldn't affect the original

        System.out.println("Original: " + Arrays.toString(original));
        System.out.println("Copy (after modifying copy[0]): " + Arrays.toString(copy));
    }

    static int[] manualCopy(int[] arr) {
        int[] copy = new int[arr.length];
        for (int i = 0; i < arr.length; i++) copy[i] = arr[i];
        return copy;
    }
}
```

### Example

```
Original: [1, 2, 3]
Copy (after modifying copy[0]): [99, 2, 3]
```

## 7. Swapping

### Theory

Swapping two elements exchanges their values using a **temporary variable** to avoid losing either one mid-swap. This single operation is the building block of sorting, reversal, and the Dutch National Flag pattern (see [Lesson 18](<./18. In-place Manipulation.md>)).

### Code

```java
import java.util.Arrays;

public class Swapping {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        swap(arr, 0, 2);
        System.out.println("After swapping index 0 and 2: " + Arrays.toString(arr));
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
After swapping index 0 and 2: [3, 2, 1]
```

## 8. Rotation

### Theory

Rotating an array shifts every element by `k` positions, wrapping the overflow around to the other end. The cleanest way to build the rotated array directly: for a **right** rotation by `k`, the last `k` elements move to the front, followed by everything else in order.

### Code

```java
import java.util.Arrays;

public class Rotation {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("Rotate right by 2: " + Arrays.toString(rotateRight(arr, 2)));
        System.out.println("Rotate left by 2: " + Arrays.toString(rotateLeft(arr, 2)));
    }

    static int[] rotateRight(int[] arr, int k) {
        int n = arr.length;
        k %= n;
        int[] result = new int[n];
        for (int i = 0; i < n; i++) {
            result[(i + k) % n] = arr[i];
        }
        return result;
    }

    static int[] rotateLeft(int[] arr, int k) {
        int n = arr.length;
        k %= n;
        int[] result = new int[n];
        for (int i = 0; i < n; i++) {
            result[i] = arr[(i + k) % n];
        }
        return result;
    }
}
```

### Example

```
Rotate right by 2: [4, 5, 1, 2, 3]
Rotate left by 2: [3, 4, 5, 1, 2]
```

## 9. Reversal

### Theory

Reversing in place uses **two pointers** starting at opposite ends, swapping and moving inward until they meet — **O(n)** time, **O(1)** extra space (no new array needed).

### Code

```java
import java.util.Arrays;

public class Reversal {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        reverse(arr);
        System.out.println("Reversed in place: " + Arrays.toString(arr));
    }

    static void reverse(int[] arr) {
        int lo = 0, hi = arr.length - 1;
        while (lo < hi) {
            int temp = arr[lo];
            arr[lo] = arr[hi];
            arr[hi] = temp;
            lo++;
            hi--;
        }
    }
}
```

### Example

```
Reversed in place: [5, 4, 3, 2, 1]
```

## 10. Merge

### Theory

Merging two **already-sorted** arrays into one sorted array is the core step of merge sort: compare the current front elements of each array, take the smaller one, and advance that array's pointer — repeat until one array is exhausted, then append whatever remains of the other. **O(n + m)**.

### Code

```java
import java.util.Arrays;

public class Merge {
    public static void main(String[] args) {
        int[] a = { 1, 3, 5 };
        int[] b = { 2, 4, 6 };
        System.out.println("Merged: " + Arrays.toString(mergeSorted(a, b)));
    }

    static int[] mergeSorted(int[] a, int[] b) {
        int[] result = new int[a.length + b.length];
        int i = 0, j = 0, k = 0;

        while (i < a.length && j < b.length) {
            result[k++] = (a[i] <= b[j]) ? a[i++] : b[j++];
        }
        while (i < a.length) result[k++] = a[i++];
        while (j < b.length) result[k++] = b[j++];

        return result;
    }
}
```

### Example

```
Merged: [1, 2, 3, 4, 5, 6]
```

## 11. Split

### Theory

Splitting divides one array into two (or more) parts — most simply, into halves. This is the counterpart to [merging](#10-merge), and the divide step of merge sort itself.

### Code

```java
import java.util.Arrays;

public class Split {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        int[][] halves = splitInHalf(arr);
        System.out.println("First half: " + Arrays.toString(halves[0]));
        System.out.println("Second half: " + Arrays.toString(halves[1]));
    }

    static int[][] splitInHalf(int[] arr) {
        int mid = arr.length / 2;
        int[] first = Arrays.copyOfRange(arr, 0, mid);
        int[] second = Arrays.copyOfRange(arr, mid, arr.length);
        return new int[][] { first, second };
    }
}
```

### Example

```
First half: [1, 2]
Second half: [3, 4, 5]
```

## 12. Resize

### Theory

Resizing manually replicates what `ArrayList` does internally (see [Lesson 1: Static vs Dynamic Arrays](<./01. Array Fundamentals.md#2-static-vs-dynamic-arrays>)): allocate a new array of the desired size, copy over as many elements as fit, and return the new array. Growing preserves all elements and fills the rest with defaults; shrinking truncates.

### Code

```java
import java.util.Arrays;

public class Resize {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        System.out.println("Grown to size 5: " + Arrays.toString(resize(arr, 5)));
        System.out.println("Shrunk to size 2: " + Arrays.toString(resize(arr, 2)));
    }

    static int[] resize(int[] arr, int newSize) {
        int[] result = new int[newSize];
        int limit = Math.min(arr.length, newSize);
        for (int i = 0; i < limit; i++) result[i] = arr[i];
        return result;
    }
}
```

### Example

```
Grown to size 5: [1, 2, 3, 0, 0]
Shrunk to size 2: [1, 2]
```

## Quick Reference Cheat Sheet

| Operation | Complexity | Notes |
|---|---|---|
| Traversal | O(n) | forward or reverse, same cost |
| Updating | O(1) | direct index write |
| Insertion | O(n) | requires a new, larger array |
| Deletion | O(n) | requires shifting + a new, smaller array |
| Searching (linear) | O(n) | works on unsorted data |
| Copying | O(n) | needed to avoid aliasing the same backing array |
| Swapping | O(1) | uses a temporary variable |
| Rotation | O(n) | build directly via modular index arithmetic |
| Reversal | O(n) time, O(1) space | two-pointer swap from both ends |
| Merge | O(n + m) | requires both inputs already sorted |
| Split | O(n) | inverse of merge |
| Resize | O(min(oldSize, newSize)) | copy what fits, default-fill or truncate the rest |
