# 8. Two Pointers

The two-pointer pattern tracks two positions in an array simultaneously instead of one, cutting many O(n²) nested-loop problems down to **O(n)**. This lesson covers the two directional flavors, the slow/fast variant, partitioning, and the classic pair/triplet/four-sum family of problems built on top.

## 1. Opposite Direction

### Theory

One pointer starts at index `0`, the other at `length-1`, and they move **toward each other**. This works beautifully on **sorted** arrays: at each step, comparing the two pointed-to values tells you which pointer to move, since moving `left` only increases the sum and moving `right` only decreases it. [Pair Sum](#5-pair-sum) below is the canonical example.

---

## 2. Same Direction

### Theory

Both pointers start at the beginning and move **forward**, but at different rates or with different roles — typically a `slow` pointer marking "where the next valid element should go" and a `fast` pointer scanning ahead. **Classic example — Remove Duplicates from Sorted Array:** `fast` scans every element; `slow` only advances (and copies `arr[fast]` into place) when a genuinely new value is found.

### Code

```java
import java.util.Arrays;

public class SameDirection {
    public static void main(String[] args) {
        int[] arr = { 1, 1, 2, 2, 3 };
        int newLength = removeDuplicates(arr);
        System.out.println("New length: " + newLength);
        System.out.println("Array prefix: " + Arrays.toString(Arrays.copyOf(arr, newLength)));
    }

    static int removeDuplicates(int[] arr) {
        if (arr.length == 0) return 0;
        int slow = 0;
        for (int fast = 1; fast < arr.length; fast++) {
            if (arr[fast] != arr[slow]) {
                slow++;
                arr[slow] = arr[fast];
            }
        }
        return slow + 1;
    }
}
```

### Example

```
New length: 3
Array prefix: [1, 2, 3]
```

## 3. Slow/Fast Pointer

### Theory

A special case of same-direction pointers where `fast` moves **twice as quickly** as `slow`. When `fast` reaches the end, `slow` sits at the **midpoint** — no length calculation or division needed. (This same technique detects cycles in linked structures, though for a plain array the "find the middle" use case is the relevant one here.)

### Code

```java
public class SlowFastPointer {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5, 6, 7 };
        System.out.println("Middle index: " + findMiddle(arr));
    }

    static int findMiddle(int[] arr) {
        int slow = 0, fast = 0;
        while (fast < arr.length - 1 && fast + 1 < arr.length) {
            slow++;
            fast += 2;
        }
        return slow;
    }
}
```

### Example

```
Middle index: 3
```

For a 7-element array, index `3` is exactly the middle.

## 4. Partitioning

### Theory

Partitioning rearranges an array so everything **less than a pivot** ends up before everything **greater than or equal to** it — the core step of quicksort's partition scheme, and the basis of [Lesson 18: Dutch National Flag](<./18. In-place Manipulation.md#1-dutch-national-flag>) (which extends this to *three* categories instead of two). One pointer `i` marks the boundary of the "less than pivot" region; a scanning pointer `j` swaps qualifying elements into that region as it goes.

### Code

```java
import java.util.Arrays;

public class Partitioning {
    public static void main(String[] args) {
        int[] arr = { 8, 3, 5, 2, 9, 1 };
        int boundary = partition(arr, 5);
        System.out.println("Partitioned around pivot 5: " + Arrays.toString(arr));
        System.out.println("Boundary (count of elements < pivot): " + boundary);
    }

    static int partition(int[] arr, int pivot) {
        int i = 0;
        for (int j = 0; j < arr.length; j++) {
            if (arr[j] < pivot) {
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
                i++;
            }
        }
        return i;
    }
}
```

### Example

```
Partitioned around pivot 5: [3, 2, 1, 8, 9, 5]
Boundary (count of elements < pivot): 3
```

The first `3` elements (`3, 2, 1`) are all less than the pivot `5`; everything after is `≥ 5`.

## 5. Pair Sum

### Theory

Given a **sorted** array, find two elements summing to a target using [opposite-direction](#1-opposite-direction) pointers: if the current pair's sum is too small, move `left` right (to increase it); if too large, move `right` left (to decrease it) — **O(n)**, no nested loop needed.

### Code

```java
public class PairSum {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 6 };
        int[] indices = findPairSum(arr, 6);
        System.out.println("Indices summing to 6: [" + indices[0] + ", " + indices[1] + "]");
    }

    static int[] findPairSum(int[] arr, int target) {
        int left = 0, right = arr.length - 1;
        while (left < right) {
            int sum = arr[left] + arr[right];
            if (sum == target) return new int[] { left, right };
            if (sum < target) left++;
            else right--;
        }
        return new int[] { -1, -1 };
    }
}
```

### Example

```
Indices summing to 6: [1, 3]
```

`arr[1] + arr[3] = 2 + 4 = 6`.

## 6. Triplet Sum

### Theory

**3Sum:** find all triplets summing to a target (commonly `0`). Sort the array, then **fix one element** and run the [pair-sum](#5-pair-sum) two-pointer search on the rest — turning an O(n³) brute force into **O(n²)**. Skipping over duplicate values (for the fixed element *and* for the two pointers after finding a match) avoids duplicate triplets in the output.

### Code

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class TripletSum {
    public static void main(String[] args) {
        int[] arr = { -1, 0, 1, 2, -1, -4 };
        System.out.println("Triplets summing to 0: " + threeSum(arr));
    }

    static List<List<Integer>> threeSum(int[] arr) {
        Arrays.sort(arr);
        List<List<Integer>> result = new ArrayList<>();
        int n = arr.length;

        for (int i = 0; i < n; i++) {
            if (i > 0 && arr[i] == arr[i - 1]) continue; // skip duplicate fixed elements

            int left = i + 1, right = n - 1;
            while (left < right) {
                int sum = arr[i] + arr[left] + arr[right];
                if (sum == 0) {
                    result.add(Arrays.asList(arr[i], arr[left], arr[right]));
                    left++;
                    right--;
                    while (left < right && arr[left] == arr[left - 1]) left++;
                    while (left < right && arr[right] == arr[right + 1]) right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return result;
    }
}
```

### Example

```
Triplets summing to 0: [[-1, -1, 2], [-1, 0, 1]]
```

## 7. Four Sum

### Theory

**4Sum** generalizes further: **fix two elements** with a nested loop, then run the two-pointer pair-sum search on what remains — **O(n³)**. The duplicate-skipping logic applies at every fixed level (both outer loops, plus the two pointers) to avoid duplicate quadruplets.

### Code

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class FourSum {
    public static void main(String[] args) {
        int[] arr = { 1, 0, -1, 0, -2, 2 };
        System.out.println("Quadruplets summing to 0: " + fourSum(arr, 0));
    }

    static List<List<Integer>> fourSum(int[] arr, int target) {
        Arrays.sort(arr);
        List<List<Integer>> result = new ArrayList<>();
        int n = arr.length;

        for (int i = 0; i < n; i++) {
            if (i > 0 && arr[i] == arr[i - 1]) continue;
            for (int j = i + 1; j < n; j++) {
                if (j > i + 1 && arr[j] == arr[j - 1]) continue;

                int left = j + 1, right = n - 1;
                while (left < right) {
                    long sum = (long) arr[i] + arr[j] + arr[left] + arr[right];
                    if (sum == target) {
                        result.add(Arrays.asList(arr[i], arr[j], arr[left], arr[right]));
                        left++;
                        right--;
                        while (left < right && arr[left] == arr[left - 1]) left++;
                        while (left < right && arr[right] == arr[right + 1]) right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        return result;
    }
}
```

### Example

```
Quadruplets summing to 0: [[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]
```

## Quick Reference Cheat Sheet

| Variant | Pointer movement | Requires sorted input? |
|---|---|---|
| Opposite Direction | `left` and `right` converge | usually |
| Same Direction | both move forward, different roles | no |
| Slow/Fast Pointer | `fast` moves 2x speed of `slow` | no |
| Partitioning | scanning pointer swaps into a boundary pointer's position | no |
| Pair Sum | opposite-direction, O(n) | yes |
| Triplet Sum | fix 1, pair-sum the rest, O(n²) | yes |
| Four Sum | fix 2, pair-sum the rest, O(n³) | yes |
