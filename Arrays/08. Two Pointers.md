# 8. Two Pointers

The two-pointer pattern tracks two positions in an array simultaneously instead of one, cutting many O(n²) nested-loop problems down to **O(n)**. This lesson covers each directional flavor with a progression of classic problems — basic to advanced — rather than a single demo per technique.

## 1. Opposite Direction

### Theory

One pointer starts at index `0`, the other at `length-1`, moving **toward each other**. This works beautifully whenever comparing the two pointed-to values tells you unambiguously which pointer to move. Below is a basic-to-advanced progression of the classic problems this pattern solves.

### Problem 1 (Basic): Reverse an Array

Two pointers swap from both ends inward until they meet — full code already in [Lesson 2: Reversal](<./02. Basic Operations.md#9-reversal>).

### Problem 2 (Basic): Valid Palindrome

**Theory:** Skip non-alphanumeric characters and compare case-insensitively from both ends inward; any mismatch means it's not a palindrome.

```java
public class ValidPalindrome {
    public static void main(String[] args) {
        String s = "A man, a plan, a canal: Panama";
        System.out.println("Is palindrome: " + isPalindrome(s));
    }

    static boolean isPalindrome(String s) {
        int left = 0, right = s.length() - 1;
        while (left < right) {
            while (left < right && !Character.isLetterOrDigit(s.charAt(left))) left++;
            while (left < right && !Character.isLetterOrDigit(s.charAt(right))) right--;
            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }
            left++;
            right--;
        }
        return true;
    }
}
```

```
Is palindrome: true
```

### Problem 3 (Medium): Container With Most Water

**Theory:** Given heights at each index, find two lines that trap the most water between them (area = width × shorter height). Start at both ends (maximum possible width) and always move the pointer at the **shorter** line inward — moving the taller one could only shrink the width without ever increasing the limiting height, so it can never improve the answer.

```java
public class ContainerWithMostWater {
    public static void main(String[] args) {
        int[] height = { 1, 8, 6, 2, 5, 4, 8, 3, 7 };
        System.out.println("Max area: " + maxArea(height));
    }

    static int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int best = 0;
        while (left < right) {
            int area = Math.min(height[left], height[right]) * (right - left);
            best = Math.max(best, area);
            if (height[left] < height[right]) left++;
            else right--;
        }
        return best;
    }
}
```

```
Max area: 49
```

### Problem 4 (Advanced): Trapping Rain Water

**Theory:** Given elevation heights, find how much water is trapped after rain. Water trapped above index `i` is `min(maxToLeft, maxToRight) - height[i]`. Instead of precomputing both max arrays (O(n) space), track a running `leftMax`/`rightMax` with two pointers: whichever side currently has the **smaller** height/max is the one whose water level is already determined (the other side's wall, whatever it turns out to be, is guaranteed tall enough) — so process that side and move its pointer inward.

```java
public class TrappingRainWater {
    public static void main(String[] args) {
        int[] height = { 0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1 };
        System.out.println("Water trapped: " + trap(height));
    }

    static int trap(int[] height) {
        int left = 0, right = height.length - 1;
        int leftMax = 0, rightMax = 0;
        int water = 0;

        while (left < right) {
            if (height[left] < height[right]) {
                leftMax = Math.max(leftMax, height[left]);
                water += leftMax - height[left];
                left++;
            } else {
                rightMax = Math.max(rightMax, height[right]);
                water += rightMax - height[right];
                right--;
            }
        }
        return water;
    }
}
```

```
Water trapped: 6
```

## 2. Same Direction

### Theory

Both pointers start at the beginning and move **forward**, typically a `slow` pointer marking "where the next valid element should go" and a `fast` pointer scanning ahead.

### Problem 1 (Basic): Remove Duplicates from Sorted Array

**Theory:** `fast` scans every element; `slow` only advances (copying `arr[fast]` into place) when a genuinely new value is found.

```java
import java.util.Arrays;

public class RemoveDuplicates {
    public static void main(String[] args) {
        int[] arr = { 1, 1, 2, 2, 3 };
        int newLength = removeDuplicates(arr);
        System.out.println("New length: " + newLength);
        System.out.println("Array prefix: " + Arrays.toString(Arrays.copyOf(arr, newLength)));
    }

    static int removeDuplicates(int[] arr) {
        if (arr.length == 0) return 0;
        int slow = 0;
        for (int fast = 1; fast < arr.length; fast++) {
            if (arr[fast] != arr[slow]) {
                slow++;
                arr[slow] = arr[fast];
            }
        }
        return slow + 1;
    }
}
```

```
New length: 3
Array prefix: [1, 2, 3]
```

### Problem 2 (Medium): Remove Duplicates from Sorted Array II (allow at most 2)

**Theory:** A small but sharp variant: each value may appear **up to twice**. Compare the candidate against the element **two slots back** in the result (`nums[slow - 2]`) instead of one — if they differ, this is still allowed.

```java
import java.util.Arrays;

public class RemoveDuplicatesII {
    public static void main(String[] args) {
        int[] arr = { 1, 1, 1, 2, 2, 3 };
        int newLength = removeDuplicates(arr);
        System.out.println("New length: " + newLength);
        System.out.println("Array prefix: " + Arrays.toString(Arrays.copyOf(arr, newLength)));
    }

    static int removeDuplicates(int[] nums) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (slow < 2 || nums[fast] != nums[slow - 2]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        return slow;
    }
}
```

```
New length: 5
Array prefix: [1, 1, 2, 2, 3]
```

Three `1`s collapse to two, but `2` still gets to appear twice — the two-back comparison is what allows exactly two copies through.

## 3. Slow/Fast Pointer

### Theory

A special case of same-direction pointers where `fast` moves **twice as quickly** as `slow` — used to find midpoints or detect cycles (see [Lesson 17: Duplicate Number](<./17. Cyclic Sort.md#2-duplicate-number>) for the cycle-detection application on arrays-as-implicit-linked-lists).

### Problem 1 (Basic): Find the Middle of an Array

```java
public class FindMiddle {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5, 6, 7 };
        System.out.println("Middle index: " + findMiddle(arr));
    }

    static int findMiddle(int[] arr) {
        int slow = 0, fast = 0;
        while (fast < arr.length - 1 && fast + 1 < arr.length) {
            slow++;
            fast += 2;
        }
        return slow;
    }
}
```

```
Middle index: 3
```

### Problem 2 (Medium): Squares of a Sorted Array

**Theory:** Given a sorted array that may contain negatives, produce a sorted array of squares. Squaring can flip the order (a very negative number squares to something large), but the **largest square always comes from one of the two ends** — the most negative or most positive value. Fill the result **from the back**, comparing both ends and advancing whichever produced the larger square.

```java
public class SortedSquares {
    public static void main(String[] args) {
        int[] arr = { -4, -1, 0, 3, 10 };
        System.out.println("Sorted squares: " + java.util.Arrays.toString(sortedSquares(arr)));
    }

    static int[] sortedSquares(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];
        int left = 0, right = n - 1;

        for (int pos = n - 1; pos >= 0; pos--) {
            int leftSquare = nums[left] * nums[left];
            int rightSquare = nums[right] * nums[right];
            if (leftSquare > rightSquare) {
                result[pos] = leftSquare;
                left++;
            } else {
                result[pos] = rightSquare;
                right--;
            }
        }
        return result;
    }
}
```

```
Sorted squares: [0, 1, 9, 16, 100]
```

## 4. Partitioning

### Theory

Partitioning rearranges an array so everything satisfying some condition ends up before everything that doesn't — the core step of quicksort's partition scheme.

### Problem 1 (Basic): Partition Around a Pivot Value

```java
import java.util.Arrays;

public class Partitioning {
    public static void main(String[] args) {
        int[] arr = { 8, 3, 5, 2, 9, 1 };
        int boundary = partition(arr, 5);
        System.out.println("Partitioned around pivot 5: " + Arrays.toString(arr));
        System.out.println("Boundary (count of elements < pivot): " + boundary);
    }

    static int partition(int[] arr, int pivot) {
        int i = 0;
        for (int j = 0; j < arr.length; j++) {
            if (arr[j] < pivot) {
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
                i++;
            }
        }
        return i;
    }
}
```

```
Partitioned around pivot 5: [3, 2, 1, 8, 9, 5]
Boundary (count of elements < pivot): 3
```

### Problem 2 (Advanced): Dutch National Flag (Sort Colors)

**Theory:** A three-way generalization — partition into *three* regions (`0`s, `1`s, `2`s) in a single pass using **three** pointers (`low`, `mid`, `high`) instead of two. Full theory and code: [Lesson 18: Dutch National Flag](<./18. In-place Manipulation.md#1-dutch-national-flag>).

## 5. Pair Sum

### Theory

Given a **sorted** array, find elements matching a target condition using opposite-direction pointers.

### Problem 1 (Basic): Two Sum II (sorted array)

```java
public class PairSum {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 6 };
        int[] indices = findPairSum(arr, 6);
        System.out.println("Indices summing to 6: [" + indices[0] + ", " + indices[1] + "]");
    }

    static int[] findPairSum(int[] arr, int target) {
        int left = 0, right = arr.length - 1;
        while (left < right) {
            int sum = arr[left] + arr[right];
            if (sum == target) return new int[] { left, right };
            if (sum < target) left++;
            else right--;
        }
        return new int[] { -1, -1 };
    }
}
```

```
Indices summing to 6: [1, 3]
```

### Problem 2 (Advanced): Boat to Save People

**Theory:** Each boat carries at most 2 people and has a weight limit. Minimize the number of boats. Greedy + two pointers: sort by weight, always try to pair the **heaviest** remaining person with the **lightest** remaining person; if they fit together, both go; otherwise the heaviest goes alone. This is provably optimal — the heaviest person always needs a boat regardless, so pairing them with whoever fits best (the lightest) never wastes capacity.

```java
import java.util.Arrays;

public class BoatToSavePeople {
    public static void main(String[] args) {
        int[] people = { 3, 2, 2, 1 };
        System.out.println("Minimum boats: " + numRescueBoats(people, 3));
    }

    static int numRescueBoats(int[] people, int limit) {
        Arrays.sort(people);
        int left = 0, right = people.length - 1;
        int boats = 0;

        while (left <= right) {
            if (people[left] + people[right] <= limit) {
                left++; // lightest person fits alongside the heaviest
            }
            right--; // heaviest person always leaves in this boat, paired or not
            boats++;
        }
        return boats;
    }
}
```

```
Minimum boats: 3
```

## 6. Triplet Sum

### Theory

**3Sum:** find triplets matching a target. Sort the array, fix one element, then run [pair-sum](#5-pair-sum) on the rest — O(n³) brute force becomes **O(n²)**.

### Problem 1 (Medium): 3Sum (triplets summing to zero)

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class TripletSum {
    public static void main(String[] args) {
        int[] arr = { -1, 0, 1, 2, -1, -4 };
        System.out.println("Triplets summing to 0: " + threeSum(arr));
    }

    static List<List<Integer>> threeSum(int[] arr) {
        Arrays.sort(arr);
        List<List<Integer>> result = new ArrayList<>();
        int n = arr.length;

        for (int i = 0; i < n; i++) {
            if (i > 0 && arr[i] == arr[i - 1]) continue;

            int left = i + 1, right = n - 1;
            while (left < right) {
                int sum = arr[i] + arr[left] + arr[right];
                if (sum == 0) {
                    result.add(Arrays.asList(arr[i], arr[left], arr[right]));
                    left++;
                    right--;
                    while (left < right && arr[left] == arr[left - 1]) left++;
                    while (left < right && arr[right] == arr[right + 1]) right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return result;
    }
}
```

```
Triplets summing to 0: [[-1, -1, 2], [-1, 0, 1]]
```

### Problem 2 (Advanced): 3Sum Closest

**Theory:** Instead of finding an exact match, find the triplet whose sum is **closest** to the target. Same fix-one-element-then-two-pointer skeleton, but track the best-seen difference instead of collecting exact matches, and narrow the search based on whether the current sum is above or below target.

```java
public class ThreeSumClosest {
    public static void main(String[] args) {
        int[] arr = { -1, 2, 1, -4 };
        System.out.println("Closest sum to 1: " + threeSumClosest(arr, 1));
    }

    static int threeSumClosest(int[] arr, int target) {
        java.util.Arrays.sort(arr);
        int n = arr.length;
        int closest = arr[0] + arr[1] + arr[2];

        for (int i = 0; i < n; i++) {
            int left = i + 1, right = n - 1;
            while (left < right) {
                int sum = arr[i] + arr[left] + arr[right];
                if (Math.abs(sum - target) < Math.abs(closest - target)) {
                    closest = sum;
                }
                if (sum < target) left++;
                else if (sum > target) right--;
                else return sum; // exact match, can't get any closer
            }
        }
        return closest;
    }
}
```

```
Closest sum to 1: 2
```

## 7. Four Sum

### Theory

**4Sum** generalizes further: fix two elements with a nested loop, then two-pointer the rest — **O(n³)**.

### Problem 1 (Advanced): 4Sum

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class FourSum {
    public static void main(String[] args) {
        int[] arr = { 1, 0, -1, 0, -2, 2 };
        System.out.println("Quadruplets summing to 0: " + fourSum(arr, 0));
    }

    static List<List<Integer>> fourSum(int[] arr, int target) {
        Arrays.sort(arr);
        List<List<Integer>> result = new ArrayList<>();
        int n = arr.length;

        for (int i = 0; i < n; i++) {
            if (i > 0 && arr[i] == arr[i - 1]) continue;
            for (int j = i + 1; j < n; j++) {
                if (j > i + 1 && arr[j] == arr[j - 1]) continue;

                int left = j + 1, right = n - 1;
                while (left < right) {
                    long sum = (long) arr[i] + arr[j] + arr[left] + arr[right];
                    if (sum == target) {
                        result.add(Arrays.asList(arr[i], arr[j], arr[left], arr[right]));
                        left++;
                        right--;
                        while (left < right && arr[left] == arr[left - 1]) left++;
                        while (left < right && arr[right] == arr[right + 1]) right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }
        return result;
    }
}
```

```
Quadruplets summing to 0: [[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]
```

> **Beyond 4Sum:** the same fix-k-2-elements-then-two-pointer-the-rest idea generalizes to **kSum** for any `k`, recursively reducing each level to a smaller kSum until reaching the 2-pointer base case — at the cost of O(n^(k-1)) time.
