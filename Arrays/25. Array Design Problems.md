## 1. Randomized Arrays

### Theory

**Classic example — Insert Delete GetRandom O(1):** design a set supporting `insert`, `remove`, and `getRandom` (uniformly, from currently present elements) — all in **O(1) average**. An array alone gives O(1) `getRandom` (just index randomly) but O(n) `remove` (shifting elements); a `HashMap` alone gives O(1) `remove` but no way to `getRandom` uniformly. Combining both is the key design insight: an **array** stores the actual values (for `getRandom`), and a **HashMap** maps each value to its array index (for O(1) lookup). Removing an element **swaps it with the last element** first (updating the map), so the actual removal is always from the array's end — O(1).

### Code

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import java.util.Random;

public class RandomizedSet {
    private final List<Integer> values = new ArrayList<>();
    private final Map<Integer, Integer> valueToIndex = new HashMap<>();
    private final Random random = new Random();

    public static void main(String[] args) {
        RandomizedSet set = new RandomizedSet();
        System.out.println("insert(1): " + set.insert(1));
        System.out.println("insert(2): " + set.insert(2));
        System.out.println("remove(1): " + set.remove(1));
        System.out.println("insert(2) again: " + set.insert(2)); // already present
        System.out.println("Remaining values: " + set.values);
    }

    public boolean insert(int val) {
        if (valueToIndex.containsKey(val)) return false;
        valueToIndex.put(val, values.size());
        values.add(val);
        return true;
    }

    public boolean remove(int val) {
        if (!valueToIndex.containsKey(val)) return false;

        int indexToRemove = valueToIndex.get(val);
        int lastValue = values.get(values.size() - 1);

        values.set(indexToRemove, lastValue);       // move the last value into the removed slot
        valueToIndex.put(lastValue, indexToRemove);  // update its index in the map

        values.remove(values.size() - 1);
        valueToIndex.remove(val);
        return true;
    }

    public int getRandom() {
        return values.get(random.nextInt(values.size()));
    }
}
```

### Example

```
insert(1): true
insert(2): true
remove(1): true
insert(2) again: false
Remaining values: [2]
```

## 2. Shuffle

### Theory

**Classic example — Shuffle an Array:** design a class with `reset()` (restore the original array) and `shuffle()` (return a new random permutation). `reset()` just needs a saved copy of the original array; `shuffle()` is the [Fisher-Yates shuffle](<../Maths/19. Probability.md#3-random-selection>) from Maths Lesson 19, applied to a **fresh copy** each time so the stored original is never mutated.

### Code

```java
import java.util.Arrays;
import java.util.Random;

public class ShuffleArray {
    private final int[] original;
    private int[] current;
    private final Random random = new Random();

    public ShuffleArray(int[] nums) {
        this.original = nums.clone();
        this.current = nums.clone();
    }

    public static void main(String[] args) {
        ShuffleArray shuffler = new ShuffleArray(new int[] { 1, 2, 3 });
        System.out.println("Shuffled: " + Arrays.toString(shuffler.shuffle()));
        System.out.println("Reset: " + Arrays.toString(shuffler.reset()));
    }

    public int[] reset() {
        current = original.clone();
        return current;
    }

    public int[] shuffle() {
        current = original.clone();
        for (int i = current.length - 1; i > 0; i--) {
            int j = random.nextInt(i + 1);
            int temp = current[i];
            current[i] = current[j];
            current[j] = temp;
        }
        return current;
    }
}
```

### Example

```
Shuffled: [2, 3, 1]
Reset: [1, 2, 3]
```

> Exact shuffled output depends on `Random`'s seed — see [Maths Lesson 19](<../Maths/19. Probability.md#3-random-selection>) for the seeded, fully-reproducible version of this exact algorithm.

## 3. Reservoir Sampling

### Theory

**Classic example — Random Pick Index:** given an array (possibly with duplicate values), design `pick(target)` returning a uniformly random index among all indices holding that value — without knowing in advance how many there are (as if reading the array as a stream). Direct application of [reservoir sampling](<../Maths/19. Probability.md#3-random-selection>): walk the array, and each time the target value is seen (the `count`-th time), replace the currently-held answer with probability `1/count`.

### Code

```java
import java.util.Random;

public class RandomPickIndex {
    private final int[] nums;
    private final Random random;

    public RandomPickIndex(int[] nums, Random random) {
        this.nums = nums;
        this.random = random;
    }

    public static void main(String[] args) {
        RandomPickIndex picker = new RandomPickIndex(new int[] { 1, 2, 3, 3, 3 }, new Random(1));
        System.out.println("Random index of value 3: " + picker.pick(3));
    }

    public int pick(int target) {
        int result = -1;
        int count = 0;
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] == target) {
                count++;
                if (random.nextInt(count) == 0) {
                    result = i;
                }
            }
        }
        return result;
    }
}
```

### Example

```
Random index of value 3: 3
```

Index `3` is one of the three valid positions (`2`, `3`, `4`) holding the value `3` — with seed `1`, reservoir sampling deterministically lands on index `3`.

## 4. Circular Queue

### Theory

Design a **fixed-capacity FIFO queue** backed by an array, supporting `enQueue`, `deQueue`, `Front`, `Rear`, `isEmpty`, `isFull` — all **O(1)**. The trick: track a `head` index and a `count` of current elements (rather than a separate `tail`), computing positions with **modular arithmetic** so the array wraps around instead of needing to shift elements or grow. When full, `enQueue` is **rejected** (returns `false`) — this is the key behavioral difference from [Circular Buffer](#5-circular-buffer) below.

### Code

```java
public class MyCircularQueue {
    private final int[] data;
    private int head = 0;
    private int count = 0;
    private final int capacity;

    public MyCircularQueue(int k) {
        this.data = new int[k];
        this.capacity = k;
    }

    public static void main(String[] args) {
        MyCircularQueue queue = new MyCircularQueue(3);
        System.out.println(queue.enQueue(1) + " " + queue.enQueue(2) + " " + queue.enQueue(3) + " " + queue.enQueue(4));
        System.out.println("Rear: " + queue.Rear());
        System.out.println("isFull: " + queue.isFull());
        System.out.println("deQueue: " + queue.deQueue());
        System.out.println("enQueue(4): " + queue.enQueue(4));
        System.out.println("Rear: " + queue.Rear());
    }

    public boolean enQueue(int value) {
        if (isFull()) return false;
        data[(head + count) % capacity] = value;
        count++;
        return true;
    }

    public boolean deQueue() {
        if (isEmpty()) return false;
        head = (head + 1) % capacity;
        count--;
        return true;
    }

    public int Front() {
        return isEmpty() ? -1 : data[head];
    }

    public int Rear() {
        return isEmpty() ? -1 : data[(head + count - 1) % capacity];
    }

    public boolean isEmpty() {
        return count == 0;
    }

    public boolean isFull() {
        return count == capacity;
    }
}
```

### Example

```
true true true false
Rear: 3
isFull: true
deQueue: true
enQueue(4): true
Rear: 4
```

The 4th `enQueue` fails while full (capacity `3`); after one `deQueue` frees a slot, `enQueue(4)` succeeds and `4` wraps around to become the new rear.

## 5. Circular Buffer

### Theory

A **circular (ring) buffer** looks similar to a circular queue but has different full-capacity behavior: instead of **rejecting** new writes when full, it **overwrites the oldest** data — the standard behavior for streaming logs, audio buffers, or "keep the last N events" use cases. Track a single `writeIndex` that always wraps via modulo; once the buffer has filled up once, every new write silently overwrites whatever was oldest.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class CircularBuffer {
    private final int[] data;
    private final int capacity;
    private int writeIndex = 0;
    private int size = 0;

    public CircularBuffer(int capacity) {
        this.data = new int[capacity];
        this.capacity = capacity;
    }

    public static void main(String[] args) {
        CircularBuffer buffer = new CircularBuffer(3);
        for (int value : new int[] { 1, 2, 3, 4, 5 }) {
            buffer.write(value);
        }
        System.out.println("Contents after overwriting: " + buffer.readAllInOrder());
    }

    public void write(int value) {
        data[writeIndex] = value;
        writeIndex = (writeIndex + 1) % capacity;
        size = Math.min(size + 1, capacity);
    }

    public List<Integer> readAllInOrder() {
        List<Integer> result = new ArrayList<>();
        if (size < capacity) {
            for (int i = 0; i < size; i++) result.add(data[i]);
        } else {
            for (int i = 0; i < capacity; i++) {
                result.add(data[(writeIndex + i) % capacity]);
            }
        }
        return result;
    }
}
```

### Example

```
Contents after overwriting: [3, 4, 5]
```

Writing `1..5` into a capacity-`3` buffer keeps only the **last 3** values (`3, 4, 5`) — `1` and `2` were silently overwritten, unlike a circular queue, which would have rejected `4` and `5` outright once full.

## Quick Reference Cheat Sheet

| Problem | Key design insight | Full/overflow behavior |
|---|---|---|
| Insert Delete GetRandom O(1) | array (for random access) + HashMap (for O(1) lookup), swap-with-last on removal | — |
| Shuffle an Array | store original separately, Fisher-Yates on a fresh copy each time | — |
| Random Pick Index | reservoir sampling, replace answer with probability `1/count` | — |
| Circular Queue | `head` + `count`, positions via modulo, no shifting | rejects new writes when full |
| Circular Buffer | single `writeIndex` via modulo | **overwrites** oldest data when full |
