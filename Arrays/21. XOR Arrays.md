## 2. Single Number

### Theory

The basic case — one value appears once while everything else appears **exactly twice** — is solved by XORing the whole array (full coverage in [Maths Lesson 23](<../Maths/23. Bit Mathematics.md#2-xor-math>)). Two harder escalations change the pairing rule entirely:

### Problem 1 (Advanced): Single Number II (others appear 3 times)

**Theory:** When every value except one appears **three** times, plain XOR no longer works (three copies don't cancel to zero). Instead, count how many numbers have each **bit position** set; since triplicated values contribute a multiple of 3 to each bit's count, any bit whose count **isn't** a multiple of 3 must belong to the single unique number.

```java
public class SingleNumberII {
    public static void main(String[] args) {
        int[] nums = { 2, 2, 3, 2 };
        System.out.println("Single number (others appear 3x): " + singleNumber(nums));
    }

    static int singleNumber(int[] nums) {
        int result = 0;
        for (int bit = 0; bit < 32; bit++) {
            int count = 0;
            for (int n : nums) {
                count += (n >> bit) & 1;
            }
            if (count % 3 != 0) {
                result |= (1 << bit);
            }
        }
        return result;
    }
}
```

```
Single number (others appear 3x): 3
```

### Problem 2 (Advanced): Single Number III (two uniques, rest appear twice)

**Theory:** When **two** distinct values each appear once while everything else appears twice, XORing everything together leaves `a ^ b` (the XOR of the two unique values) — not directly useful by itself, but its lowest **set bit** identifies a bit where `a` and `b` differ. Partition the whole array by that bit into two groups; since `a` and `b` land in different groups, and every paired value lands in the *same* group as its pair (so it still cancels out), XORing each group separately isolates `a` and `b` individually.

```java
import java.util.Arrays;

public class SingleNumberIII {
    public static void main(String[] args) {
        int[] nums = { 1, 2, 1, 3, 2, 5 };
        System.out.println("Two unique numbers: " + Arrays.toString(singleNumbers(nums)));
    }

    static int[] singleNumbers(int[] nums) {
        int xorAll = 0;
        for (int n : nums) xorAll ^= n;

        int diffBit = xorAll & (-xorAll); // isolates the lowest set bit

        int a = 0, b = 0;
        for (int n : nums) {
            if ((n & diffBit) != 0) {
                a ^= n;
            } else {
                b ^= n;
            }
        }
        int[] result = { a, b };
        Arrays.sort(result);
        return result;
    }
}
```

```
Two unique numbers: [3, 5]
```

## 3. Pair XOR

### Theory

**Classic example — Count Pairs With XOR Equal to K:** rather than checking every pair (O(n²)), use the same [index-mapping](<./11. Hashing.md#5-index-mapping>) idea as Two Sum — since `a ^ b = k` implies `b = a ^ k`, track how many times each value has been **seen so far**, and for each new element, look up how many previously-seen values would pair with it to hit the target XOR.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class PairXor {
    public static void main(String[] args) {
        int[] arr = { 5, 4, 10, 15, 7, 6 };
        System.out.println("Pairs with XOR = 5: " + countPairsWithXor(arr, 5));
    }

    static int countPairsWithXor(int[] arr, int k) {
        Map<Integer, Integer> seenCount = new HashMap<>();
        int count = 0;

        for (int value : arr) {
            int complement = value ^ k;
            count += seenCount.getOrDefault(complement, 0);
            seenCount.merge(value, 1, Integer::sum);
        }
        return count;
    }
}
```

### Example

```
Pairs with XOR = 5: 1
```

Only `10 ^ 15 = 5` matches (10 in binary is `1010`, 15 is `1111`, XOR is `0101 = 5`) — the single qualifying pair in this array.

## 4. Prefix XOR

### Theory

Full theory for the prefix XOR array itself is in [Lesson 5: Prefix XOR](<./05. Prefix Techniques.md#3-prefix-xor>). Combining it with a hash map — the same pattern as [Lesson 11: Prefix Hashing](<./11. Hashing.md#6-prefix-hashing>), but for XOR instead of sum — solves **Count Subarrays With XOR Equal to K**: a subarray's XOR equals `k` exactly when its ending prefix XOR, XORed with `k`, matches a **previously seen** prefix XOR (since `x ^ x = 0` cancels the shared prefix cleanly, unlike subtraction, which is why this works even more directly than the sum version).

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class PrefixXorCount {
    public static void main(String[] args) {
        int[] arr = { 4, 2, 2, 6, 4 };
        System.out.println("Subarrays with XOR = 6: " + countSubarraysWithXorK(arr, 6));
    }

    static int countSubarraysWithXorK(int[] arr, int k) {
        Map<Integer, Integer> prefixXorCounts = new HashMap<>();
        prefixXorCounts.put(0, 1); // an empty prefix (XOR 0) has occurred once, before any elements

        int count = 0, prefixXor = 0;
        for (int value : arr) {
            prefixXor ^= value;
            count += prefixXorCounts.getOrDefault(prefixXor ^ k, 0);
            prefixXorCounts.merge(prefixXor, 1, Integer::sum);
        }
        return count;
    }
}
```

### Example

```
Subarrays with XOR = 6: 4
```

`[4,2,2,6,4]` has 4 subarrays whose elements XOR together to `6`: `[2,2,6]`... and three others found the same way, all via O(n) prefix-XOR bookkeeping rather than checking every subarray directly.

## Quick Reference Cheat Sheet

| Problem | Technique | Complexity |
|---|---|---|
| Missing Number | XOR every index and value together | O(n) time, O(1) space |
| Single Number (pairs) | XOR the whole array | O(n) time, O(1) space |
| Single Number II (triples) | per-bit count, keep bits not divisible by 3 | O(32n) time, O(1) space |
| Single Number III (two uniques) | XOR all, split by lowest differing bit, XOR each group | O(n) time, O(1) space |
| Pair XOR | hashmap of seen values, look up `value ^ k` | O(n) time, O(n) space |
| Prefix XOR (subarray count) | hashmap of prefix-XOR counts, look up `prefixXor ^ k` | O(n) time, O(n) space |
