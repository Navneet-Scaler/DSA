## 1. Missing Number

### Problem 1 (Basic): Missing Number

**Theory:** Full theory and code (XOR method alongside the sum-formula alternative) is in [Lesson 4: Missing Number](<./04. Frequency Problems.md#8-missing-number>) — XOR every index `0..n` together with every array value, and every present number cancels with its matching index, leaving only the missing one.

### Problem 2 (Advanced): Find the Two Missing Numbers

**Theory:** Generalizes to **two** missing numbers instead of one — given `n` numbers drawn from `1..(n+2)` with exactly two missing. Plain XOR-of-everything now leaves `a ^ b` (the XOR of both missing values) instead of isolating one directly — exactly the same obstacle as [Single Number III](#2-single-number) below, and solved the same way: the lowest set bit of `a ^ b` identifies a bit where the two missing numbers differ, so splitting the full range `1..(n+2)` **and** the array by that bit into two groups isolates each missing number in its own group's XOR.

```java
public class TwoMissingNumbers {
    public static void main(String[] args) {
        int[] nums = { 1, 3, 4, 5 }; // drawn from 1..6, missing two
        int[] missing = findTwoMissing(nums, 6);
        System.out.println("Missing numbers: [" + missing[0] + ", " + missing[1] + "]");
    }

    static int[] findTwoMissing(int[] nums, int upper) {
        int xorAll = 0;
        for (int i = 1; i <= upper; i++) xorAll ^= i;
        for (int value : nums) xorAll ^= value;

        int diffBit = xorAll & (-xorAll);

        int a = 0, b = 0;
        for (int i = 1; i <= upper; i++) {
            if ((i & diffBit) != 0) a ^= i;
            else b ^= i;
        }
        for (int value : nums) {
            if ((value & diffBit) != 0) a ^= value;
            else b ^= value;
        }

        return (a < b) ? new int[] { a, b } : new int[] { b, a };
    }
}
```

```
Missing numbers: [2, 6]
```

`nums = [1,3,4,5]` should contain every number from `1` to `6` — `2` and `6` are the two that don't appear.

## 2. Single Number

### Problem 1 (Basic): Single Number (others appear twice)

**Theory:** One value appears once while everything else appears **exactly twice**. XOR the entire array together — every paired value cancels to `0` (`a ^ a = 0`), leaving only the unpaired value (full algebraic grounding in [Maths Lesson 23](<../Maths/23. Bit Mathematics.md#2-xor-math>)).

```java
public class SingleNumber {
    public static void main(String[] args) {
        int[] nums = { 4, 1, 2, 1, 2 };
        System.out.println("Single number: " + singleNumber(nums));
    }

    static int singleNumber(int[] nums) {
        int result = 0;
        for (int n : nums) result ^= n;
        return result;
    }
}
```

```
Single number: 4
```

### Problem 2 (Advanced): Single Number II (others appear three times)

**Theory:** When every value except one appears **three** times, plain XOR no longer works (three copies don't cancel to zero). Instead, count how many numbers have each **bit position** set; since triplicated values contribute a multiple of 3 to each bit's count, any bit whose count **isn't** a multiple of 3 must belong to the single unique number.

```java
public class SingleNumberII {
    public static void main(String[] args) {
        int[] nums = { 2, 2, 3, 2 };
        System.out.println("Single number (others appear 3x): " + singleNumber(nums));
    }

    static int singleNumber(int[] nums) {
        int result = 0;
        for (int bit = 0; bit < 32; bit++) {
            int count = 0;
            for (int n : nums) {
                count += (n >> bit) & 1;
            }
            if (count % 3 != 0) {
                result |= (1 << bit);
            }
        }
        return result;
    }
}
```

```
Single number (others appear 3x): 3
```

### Problem 3 (Advanced): Single Number III (two uniques, rest appear twice)

**Theory:** When **two** distinct values each appear once while everything else appears twice, XORing everything together leaves `a ^ b` (the XOR of the two unique values) — not directly useful by itself, but its lowest **set bit** identifies a bit where `a` and `b` differ. Partition the whole array by that bit into two groups; since `a` and `b` land in different groups, and every paired value lands in the *same* group as its pair (so it still cancels out), XORing each group separately isolates `a` and `b` individually.

```java
import java.util.Arrays;

public class SingleNumberIII {
    public static void main(String[] args) {
        int[] nums = { 1, 2, 1, 3, 2, 5 };
        System.out.println("Two unique numbers: " + Arrays.toString(singleNumbers(nums)));
    }

    static int[] singleNumbers(int[] nums) {
        int xorAll = 0;
        for (int n : nums) xorAll ^= n;

        int diffBit = xorAll & (-xorAll); // isolates the lowest set bit

        int a = 0, b = 0;
        for (int n : nums) {
            if ((n & diffBit) != 0) {
                a ^= n;
            } else {
                b ^= n;
            }
        }
        int[] result = { a, b };
        Arrays.sort(result);
        return result;
    }
}
```

```
Two unique numbers: [3, 5]
```

## 3. Pair XOR

### Problem 1 (Basic): Check for a Pair With Given XOR

**Theory:** Before counting *how many* pairs XOR to `k` (Problem 2), start with the simpler yes/no question: does **any** pair exist at all? Since `a ^ b = k` implies `b = a ^ k`, a single `HashSet` of values seen so far answers this in one pass — for each new value, check whether its required partner (`value ^ k`) has already been seen.

```java
import java.util.HashSet;
import java.util.Set;

public class PairWithGivenXorExists {
    public static void main(String[] args) {
        int[] arr = { 5, 4, 10, 15, 7, 6 };
        System.out.println("Some pair XORs to 5: " + hasPairWithXor(arr, 5));
    }

    static boolean hasPairWithXor(int[] arr, int k) {
        Set<Integer> seen = new HashSet<>();
        for (int value : arr) {
            if (seen.contains(value ^ k)) return true;
            seen.add(value);
        }
        return false;
    }
}
```

```
Some pair XORs to 5: true
```

### Problem 2 (Medium): Count Pairs With XOR Equal to K

**Theory:** The counting version of Problem 1 — instead of stopping at the first match, keep a **count** of how many times each value has been seen, and accumulate how many pairs each new element completes.

```java
import java.util.HashMap;
import java.util.Map;

public class CountPairsWithXor {
    public static void main(String[] args) {
        int[] arr = { 5, 4, 10, 15, 7, 6 };
        System.out.println("Pairs with XOR = 5: " + countPairsWithXor(arr, 5));
    }

    static int countPairsWithXor(int[] arr, int k) {
        Map<Integer, Integer> seenCount = new HashMap<>();
        int count = 0;

        for (int value : arr) {
            int complement = value ^ k;
            count += seenCount.getOrDefault(complement, 0);
            seenCount.merge(value, 1, Integer::sum);
        }
        return count;
    }
}
```

```
Pairs with XOR = 5: 1
```

Only `10 ^ 15 = 5` matches (10 in binary is `1010`, 15 is `1111`, XOR is `0101 = 5`) — the single qualifying pair in this array.

### Problem 3 (Advanced): Sum of XOR of All Pairs

**Theory:** Instead of counting pairs matching one specific XOR value, sum the XOR of **every** pair in the array. Checking all O(n²) pairs directly works but is wasteful — the [contribution technique](<./20. Mathematical Arrays.md#4-contribution-technique>) resolves it bit by bit instead: at a given bit position, a pair contributes `2^bit` to the sum exactly when the two numbers **differ** at that bit (one has it set, one doesn't) — so if `ones` numbers have the bit set and `zeros = n - ones` don't, that bit contributes `ones × zeros × 2^bit` across all pairs, summed over all 32 bit positions.

```java
public class SumOfXorAllPairs {
    public static void main(String[] args) {
        int[] nums = { 1, 2, 3 };
        System.out.println("Sum of XOR of all pairs: " + sumOfAllPairXor(nums));
    }

    static long sumOfAllPairXor(int[] nums) {
        long total = 0;
        int n = nums.length;

        for (int bit = 0; bit < 32; bit++) {
            int ones = 0;
            for (int num : nums) {
                ones += (num >> bit) & 1;
            }
            int zeros = n - ones;
            total += (long) ones * zeros * (1L << bit);
        }
        return total;
    }
}
```

```
Sum of XOR of all pairs: 6
```

The three pairs of `[1,2,3]` — `1^2=3`, `1^3=2`, `2^3=1` — sum to `3+2+1=6`, matching the O(32n) bit-contribution computation.

## 4. Prefix XOR

### Problem 1 (Basic): Range XOR Query

**Theory:** Full theory for building the prefix XOR array is in [Lesson 5: Prefix XOR](<./05. Prefix Techniques.md#3-prefix-xor>) — once built, the XOR of any range `[l, r]` is `prefix[r] ^ prefix[l-1]`, an O(1) query after an O(n) build.

```java
public class RangeXorQuery {
    public static void main(String[] args) {
        int[] arr = { 8, 1, 2, 9, 4, 6 };
        int[] prefix = buildPrefixXor(arr);
        System.out.println("XOR of range [1,4]: " + rangeXor(prefix, 1, 4));
    }

    static int[] buildPrefixXor(int[] arr) {
        int[] prefix = new int[arr.length];
        prefix[0] = arr[0];
        for (int i = 1; i < arr.length; i++) prefix[i] = prefix[i - 1] ^ arr[i];
        return prefix;
    }

    static int rangeXor(int[] prefix, int l, int r) {
        return prefix[r] ^ (l > 0 ? prefix[l - 1] : 0);
    }
}
```

```
XOR of range [1,4]: 8
```

`arr[1]^arr[2]^arr[3]^arr[4] = 1^2^9^4 = 8`, answered in O(1) from the precomputed prefix array.

### Problem 2 (Advanced): Count Subarrays With XOR Equal to K

**Theory:** Combining prefix XOR with a hash map — the same pattern as [Lesson 11: Prefix Hashing](<./11. Hashing.md#6-prefix-hashing>), but for XOR instead of sum. A subarray's XOR equals `k` exactly when its ending prefix XOR, XORed with `k`, matches a **previously seen** prefix XOR (since `x ^ x = 0` cancels the shared prefix cleanly).

```java
import java.util.HashMap;
import java.util.Map;

public class CountSubarraysWithXorK {
    public static void main(String[] args) {
        int[] arr = { 4, 2, 2, 6, 4 };
        System.out.println("Subarrays with XOR = 6: " + countSubarraysWithXorK(arr, 6));
    }

    static int countSubarraysWithXorK(int[] arr, int k) {
        Map<Integer, Integer> prefixXorCounts = new HashMap<>();
        prefixXorCounts.put(0, 1); // an empty prefix (XOR 0) has occurred once, before any elements

        int count = 0, prefixXor = 0;
        for (int value : arr) {
            prefixXor ^= value;
            count += prefixXorCounts.getOrDefault(prefixXor ^ k, 0);
            prefixXorCounts.merge(prefixXor, 1, Integer::sum);
        }
        return count;
    }
}
```

```
Subarrays with XOR = 6: 4
```

`[4,2,2,6,4]` has 4 subarrays whose elements XOR together to `6`, found via O(n) prefix-XOR bookkeeping rather than checking every subarray directly.

### Problem 3 (Advanced): Maximum XOR Subarray

**Theory:** Find the maximum XOR achievable by **any** contiguous subarray. Since a subarray's XOR is `prefix[j] ^ prefix[i]` for some `i < j`, this becomes "find the two prefix values whose XOR is largest" — the exact [Maximum XOR of Two Numbers](<./22. Bit Manipulation + Arrays.md#2-xor-patterns>) problem, applied to the set of prefix values instead of the raw array. The version below checks all O(n²) prefix pairs directly for clarity; swapping in the greedy bit-by-bit technique (or a bitwise Trie) from Lesson 22 brings it down to O(n) or O(32n).

```java
public class MaximumXorSubarray {
    public static void main(String[] args) {
        int[] arr = { 8, 1, 2, 12, 7, 6 };
        System.out.println("Maximum XOR subarray: " + maxXorSubarray(arr));
    }

    static int maxXorSubarray(int[] arr) {
        int n = arr.length;
        int[] prefix = new int[n + 1];
        for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] ^ arr[i];

        int best = 0;
        for (int i = 0; i <= n; i++) {
            for (int j = i + 1; j <= n; j++) {
                best = Math.max(best, prefix[i] ^ prefix[j]);
            }
        }
        return best;
    }
}
```

```
Maximum XOR subarray: 15
```

## Quick Reference Cheat Sheet

| Problem | Technique | Complexity |
|---|---|---|
| Missing Number | XOR every index and value together | O(n) time, O(1) space |
| Find the Two Missing Numbers | XOR all, split by lowest differing bit | O(n) time, O(1) space |
| Single Number (pairs) | XOR the whole array | O(n) time, O(1) space |
| Single Number II (triples) | per-bit count, keep bits not divisible by 3 | O(32n) time, O(1) space |
| Single Number III (two uniques) | XOR all, split by lowest differing bit, XOR each group | O(n) time, O(1) space |
| Check for a Pair With Given XOR | `HashSet` of seen values | O(n) time, O(n) space |
| Count Pairs With XOR Equal to K | `HashMap` of seen-value counts | O(n) time, O(n) space |
| Sum of XOR of All Pairs | bit contribution: `ones × zeros × 2^bit` | O(32n) time, O(1) space |
| Range XOR Query | prefix XOR array | O(n) build, O(1) per query |
| Count Subarrays With XOR Equal to K | `HashMap` of prefix-XOR counts | O(n) time, O(n) space |
| Maximum XOR Subarray | max XOR over all pairs of prefix values | O(n²) naive, O(n)-O(32n) with Lesson 22's techniques |
