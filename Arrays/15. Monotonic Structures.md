# 15. Monotonic Structures

A monotonic stack or queue maintains its elements in strictly increasing or decreasing order at all times, popping whatever would break that order **before** pushing a new element. This simple discipline turns an apparent O(n²) "for each element, scan for its next greater value" into an **O(n)** single pass, because each element is pushed and popped at most once.

## 1. Monotonic Stack

### Theory

A **monotonic stack** keeps its elements ordered (increasing or decreasing) from bottom to top, by popping elements that would violate that order before pushing the current one. The elements popped along the way are exactly the ones for which the current element is their "answer" — this is what makes next/previous-greater/smaller problems solvable in O(n): every element is pushed once and popped **at most** once, so the total work across the whole array is linear.

---

## 2. Monotonic Queue

### Theory

A **monotonic queue** (implemented as a deque) is the same discipline applied to a **sliding window** instead of the whole array — elements can be removed from **both** ends: the back, to maintain monotonic order when pushing, and the front, when the oldest element falls outside the current window. Full worked example: [Lesson 7: Maximum Window / Minimum Window](<./07. Sliding Window.md#5-maximum-window>).

## 3. Next Greater Element

### Theory

For every element, find the **next** element to its right that's strictly greater. Scan left to right with a **decreasing** stack (of indices): whenever the current value is greater than the stack's top, that top element has found its answer — pop it and record the current value, repeating until the stack top is no longer smaller (or the stack empties). Then push the current index.

### Code

```java
import java.util.Arrays;

public class NextGreaterElement {
    public static void main(String[] args) {
        int[] arr = { 2, 1, 2, 4, 3 };
        System.out.println("Next greater: " + Arrays.toString(nextGreater(arr)));
    }

    static int[] nextGreater(int[] arr) {
        int n = arr.length;
        int[] result = new int[n];
        Arrays.fill(result, -1);
        java.util.Deque<Integer> stack = new java.util.ArrayDeque<>(); // holds indices, decreasing values

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && arr[stack.peek()] < arr[i]) {
                result[stack.pop()] = arr[i];
            }
            stack.push(i);
        }
        return result;
    }
}
```

### Example

```
Next greater: [4, 2, 4, -1, -1]
```

`arr[0]=2`'s next greater is `4`; `arr[3]=4` and `arr[4]=3` have no greater element to their right, so they stay `-1`.

## 4. Next Smaller Element

### Theory

The mirror of [Next Greater Element](#3-next-greater-element): use an **increasing** stack instead, popping whenever the current value is **smaller** than the stack's top.

### Code

```java
import java.util.Arrays;
import java.util.ArrayDeque;
import java.util.Deque;

public class NextSmallerElement {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 2, 10, 8 };
        System.out.println("Next smaller: " + Arrays.toString(nextSmaller(arr)));
    }

    static int[] nextSmaller(int[] arr) {
        int n = arr.length;
        int[] result = new int[n];
        Arrays.fill(result, -1);
        Deque<Integer> stack = new ArrayDeque<>(); // holds indices, increasing values

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && arr[stack.peek()] > arr[i]) {
                result[stack.pop()] = arr[i];
            }
            stack.push(i);
        }
        return result;
    }
}
```

### Example

```
Next smaller: [2, 2, -1, 8, -1]
```

## 5. Previous Greater Element

### Theory

For every element, find the **nearest** element to its **left** that's strictly greater. Still a single left-to-right pass, but the logic flips: pop elements from the stack that are `≤` the current value (they can never be *anyone's* previous-greater, since the current element is both closer and at least as large) — whatever remains on top of the stack (if anything) is the current element's previous greater.

### Code

```java
import java.util.Arrays;
import java.util.ArrayDeque;
import java.util.Deque;

public class PreviousGreaterElement {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 2, 10, 8 };
        System.out.println("Previous greater: " + Arrays.toString(previousGreater(arr)));
    }

    static int[] previousGreater(int[] arr) {
        int n = arr.length;
        int[] result = new int[n];
        Deque<Integer> stack = new ArrayDeque<>(); // holds indices, decreasing values

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && arr[stack.peek()] <= arr[i]) {
                stack.pop(); // can never serve as anyone's previous-greater from here on
            }
            result[i] = stack.isEmpty() ? -1 : arr[stack.peek()];
            stack.push(i);
        }
        return result;
    }
}
```

### Example

```
Previous greater: [-1, -1, 5, -1, 10]
```

`arr[2]=2`'s nearest greater element to the left is `5` (at index 1); `arr[4]=8`'s is `10` (at index 3).

## 6. Previous Smaller Element

### Theory

The mirror of [Previous Greater Element](#5-previous-greater-element): pop elements `≥` the current value instead.

### Code

```java
import java.util.Arrays;
import java.util.ArrayDeque;
import java.util.Deque;

public class PreviousSmallerElement {
    public static void main(String[] args) {
        int[] arr = { 4, 5, 2, 10, 8 };
        System.out.println("Previous smaller: " + Arrays.toString(previousSmaller(arr)));
    }

    static int[] previousSmaller(int[] arr) {
        int n = arr.length;
        int[] result = new int[n];
        Deque<Integer> stack = new ArrayDeque<>(); // holds indices, increasing values

        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && arr[stack.peek()] >= arr[i]) {
                stack.pop();
            }
            result[i] = stack.isEmpty() ? -1 : arr[stack.peek()];
            stack.push(i);
        }
        return result;
    }
}
```

### Example

```
Previous smaller: [-1, 4, -1, 2, 2]
```

## 7. Largest Rectangle in Histogram

### Theory

Given bar heights, find the largest rectangle that fits entirely under the histogram's outline — a genuinely advanced application combining [Previous Smaller Element](#6-previous-smaller-element)-style logic in both directions at once. Maintain an **increasing** stack of indices; whenever a shorter bar arrives, it means every taller bar still on the stack has just found its **right boundary** — pop each one and compute the rectangle it could form, using the (now-revealed) previous smaller element as its left boundary and the current index as its right boundary. A sentinel `0`-height bar appended at the end flushes any bars still on the stack once the real input is exhausted.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class LargestRectangleHistogram {
    public static void main(String[] args) {
        int[] heights = { 2, 1, 5, 6, 2, 3 };
        System.out.println("Largest rectangle area: " + largestRectangleArea(heights));
    }

    static int largestRectangleArea(int[] heights) {
        Deque<Integer> stack = new ArrayDeque<>(); // indices, increasing heights
        int maxArea = 0;
        int n = heights.length;

        for (int i = 0; i <= n; i++) {
            int currentHeight = (i == n) ? 0 : heights[i]; // sentinel flushes the stack at the end
            while (!stack.isEmpty() && heights[stack.peek()] > currentHeight) {
                int height = heights[stack.pop()];
                int width = stack.isEmpty() ? i : i - stack.peek() - 1;
                maxArea = Math.max(maxArea, height * width);
            }
            stack.push(i);
        }
        return maxArea;
    }
}
```

### Example

```
Largest rectangle area: 10
```

Bars at indices `2` and `3` (heights `5` and `6`) are both at least `5` tall — treating them as one rectangle of height `5` and width `2` gives area `10`, the largest that fits under this histogram.

## 8. Trapping Rain Water (Monotonic Stack)

### Theory

[Lesson 8](<./08. Two Pointers.md#1-opposite-direction>) solved this with two pointers; a monotonic stack offers a different lens on the same problem — processing water **layer by layer** instead of column by column. Maintain a **decreasing** stack of indices; when a taller bar arrives, it means water can now be trapped above the bar just popped, bounded on the left by the new stack top and on the right by the current bar — compute that layer's volume (width × bounded height) and add it in.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class TrappingRainWaterStack {
    public static void main(String[] args) {
        int[] height = { 0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1 };
        System.out.println("Water trapped: " + trap(height));
    }

    static int trap(int[] height) {
        Deque<Integer> stack = new ArrayDeque<>(); // indices, decreasing heights
        int water = 0;

        for (int i = 0; i < height.length; i++) {
            while (!stack.isEmpty() && height[stack.peek()] < height[i]) {
                int top = stack.pop();
                if (stack.isEmpty()) break; // no left boundary -- nothing trapped above 'top'

                int distance = i - stack.peek() - 1;
                int boundedHeight = Math.min(height[i], height[stack.peek()]) - height[top];
                water += distance * boundedHeight;
            }
            stack.push(i);
        }
        return water;
    }
}
```

### Example

```
Water trapped: 6
```

Same answer as the two-pointer version in [Lesson 8](<./08. Two Pointers.md#1-opposite-direction>) — confirming both techniques solve the identical problem.

## Quick Reference Cheat Sheet

| Pattern | Stack order maintained | Pop condition |
|---|---|---|
| Next Greater Element | decreasing | `arr[top] < current` |
| Next Smaller Element | increasing | `arr[top] > current` |
| Previous Greater Element | decreasing | `arr[top] <= current` |
| Previous Smaller Element | increasing | `arr[top] >= current` |
| Monotonic Queue (sliding window) | increasing or decreasing | same idea, but also evicts from the front when out of window range |
| Largest Rectangle in Histogram | increasing | pop reveals a bar's left+right boundary at once |
| Trapping Rain Water (stack) | decreasing | pop reveals a water layer bounded left+right |
