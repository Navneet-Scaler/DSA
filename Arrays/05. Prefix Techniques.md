# 5. Prefix Techniques

Prefix arrays precompute cumulative results so that a range query (sum, product, XOR, min, max over `[l, r]`) becomes **O(1)** instead of O(n) — one `O(n)` preprocessing pass buys unlimited fast queries. This lesson covers each prefix variant, and the "difference array" trick that does the same for **updates** instead of queries.

## 1. Prefix Sum

### Theory

`prefix[i]` stores the sum of `arr[0..i]`. Once built, the sum of **any** range `[l, r]` is a single subtraction:

```
prefix[i] = prefix[i-1] + arr[i]
rangeSum(l, r) = prefix[r] - prefix[l-1]     (treat prefix[-1] as 0)
```

### Code

```java
public class PrefixSum {
    public static void main(String[] args) {
        int[] arr = { 2, 4, 6, 8, 10 };
        int[] prefix = buildPrefixSum(arr);

        System.out.println("Range sum [1,3]: " + rangeSum(prefix, 1, 3));
    }

    static int[] buildPrefixSum(int[] arr) {
        int[] prefix = new int[arr.length];
        prefix[0] = arr[0];
        for (int i = 1; i < arr.length; i++) {
            prefix[i] = prefix[i - 1] + arr[i];
        }
        return prefix;
    }

    static int rangeSum(int[] prefix, int l, int r) {
        return prefix[r] - (l > 0 ? prefix[l - 1] : 0);
    }
}
```

### Example

```
Range sum [1,3]: 18
```

`arr[1] + arr[2] + arr[3] = 4 + 6 + 8 = 18` — answered without touching the array itself.

### Problem 2 (Medium): Find Pivot Index

**Theory:** Find the index where the sum of everything to its left equals the sum of everything to its right. Rather than recomputing both sides from scratch at every candidate index (O(n²)), track a running `leftSum` while walking forward, and derive the right sum as `total - leftSum - arr[i]` in O(1) — one prefix sum pass total.

```java
public class FindPivotIndex {
    public static void main(String[] args) {
        int[] arr = { 1, 7, 3, 6, 5, 6 };
        System.out.println("Pivot index: " + pivotIndex(arr));
    }

    static int pivotIndex(int[] arr) {
        int total = 0;
        for (int value : arr) total += value;

        int leftSum = 0;
        for (int i = 0; i < arr.length; i++) {
            int rightSum = total - leftSum - arr[i];
            if (leftSum == rightSum) return i;
            leftSum += arr[i];
        }
        return -1;
    }
}
```

```
Pivot index: 3
```

At index `3` (value `6`): left side `1+7+3=11`, right side `5+6=11` — balanced.

## 2. Prefix Product

### Theory

Same idea, multiplication instead of addition. The range query divides instead of subtracting:

```
rangeProduct(l, r) = prefix[r] / prefix[l-1]
```

> **Gotcha:** this division-based query **breaks if any element in `[0, l-1]` is zero** (division by zero, or a wrong answer if zeros cancel incorrectly). If zeros are possible, track their positions separately, or fall back to a segment tree.

### Code

```java
public class PrefixProduct {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4 };
        long[] prefix = buildPrefixProduct(arr);

        System.out.println("Range product [1,3]: " + rangeProduct(prefix, 1, 3));
    }

    static long[] buildPrefixProduct(int[] arr) {
        long[] prefix = new long[arr.length];
        prefix[0] = arr[0];
        for (int i = 1; i < arr.length; i++) {
            prefix[i] = prefix[i - 1] * arr[i];
        }
        return prefix;
    }

    static long rangeProduct(long[] prefix, int l, int r) {
        return prefix[r] / (l > 0 ? prefix[l - 1] : 1);
    }
}
```

### Example

```
Range product [1,3]: 24
```

`arr[1] × arr[2] × arr[3] = 2 × 3 × 4 = 24`.

### Problem (Advanced): Product of the Last K Numbers

**Theory:** Design a class supporting `add(num)` (append a number to a running stream) and `getProduct(k)` (return the product of the last `k` numbers added). A running prefix-product array answers `getProduct(k)` as `prefix[last] / prefix[last-k]` in O(1) — but the [division gotcha](#2-prefix-product) above becomes a real design problem here: any `add(0)` would permanently zero out every future prefix product. The fix: whenever `0` is added, **reset** the prefix array to start fresh from that point (`getProduct(k)` for a `k` reaching back past the reset can then immediately return `0` without even computing anything, since a `0` is known to be in that range).

```java
import java.util.ArrayList;
import java.util.List;

public class ProductOfLastKNumbers {
    private List<Long> prefix = new ArrayList<>(List.of(1L));

    public static void main(String[] args) {
        ProductOfLastKNumbers p = new ProductOfLastKNumbers();
        for (int x : new int[] { 3, 0, 2, 5, 4 }) p.add(x);

        System.out.println("Product of last 2: " + p.getProduct(2));
        System.out.println("Product of last 3: " + p.getProduct(3));
        System.out.println("Product of last 4: " + p.getProduct(4)); // spans the reset point (the 0)
    }

    void add(int num) {
        if (num == 0) {
            prefix = new ArrayList<>(List.of(1L)); // reset -- a zero breaks the running product
        } else {
            prefix.add(prefix.get(prefix.size() - 1) * num);
        }
    }

    long getProduct(int k) {
        int n = prefix.size() - 1;
        if (k > n) return 0; // the requested range spans a reset point (a zero)
        return prefix.get(n) / prefix.get(n - k);
    }
}
```

```
Product of last 2: 20
Product of last 3: 40
Product of last 4: 0
```

`getProduct(4)` reaches back far enough to include the `0` that was added — correctly returning `0` instead of dividing by a stale zero-containing prefix.

## 3. Prefix XOR

### Theory

Same pattern again, using XOR — and since `a ^ a = 0` (see [Maths Lesson 23: XOR Math](<../Maths/23. Bit Mathematics.md#2-xor-math>)), the range query is a clean cancellation, no division risk at all:

```
rangeXor(l, r) = prefix[r] ^ prefix[l-1]
```

### Code

```java
public class PrefixXor {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        int[] prefix = buildPrefixXor(arr);

        System.out.println("Range XOR [1,3]: " + rangeXor(prefix, 1, 3));
    }

    static int[] buildPrefixXor(int[] arr) {
        int[] prefix = new int[arr.length];
        prefix[0] = arr[0];
        for (int i = 1; i < arr.length; i++) {
            prefix[i] = prefix[i - 1] ^ arr[i];
        }
        return prefix;
    }

    static int rangeXor(int[] prefix, int l, int r) {
        return prefix[r] ^ (l > 0 ? prefix[l - 1] : 0);
    }
}
```

### Example

```
Range XOR [1,3]: 5
```

`arr[1] ^ arr[2] ^ arr[3] = 2 ^ 3 ^ 4 = 5`.

### Problem (Medium): XOR Queries of a Subarray

**Theory:** Given many `[l, r]` range-XOR queries up front (not just one), build the prefix XOR array **once**, then answer every query in O(1) — turning what looks like `q × n` work into `n + q`.

```java
import java.util.Arrays;

public class XorQueriesOfSubarray {
    public static void main(String[] args) {
        int[] arr = { 1, 3, 4, 8 };
        int[][] queries = { { 0, 1 }, { 1, 2 }, { 0, 3 }, { 3, 3 } };
        System.out.println("Query results: " + Arrays.toString(xorQueries(arr, queries)));
    }

    static int[] xorQueries(int[] arr, int[][] queries) {
        int n = arr.length;
        int[] prefix = new int[n + 1];
        for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] ^ arr[i];

        int[] result = new int[queries.length];
        for (int i = 0; i < queries.length; i++) {
            int l = queries[i][0], r = queries[i][1];
            result[i] = prefix[r + 1] ^ prefix[l];
        }
        return result;
    }
}
```

```
Query results: [2, 7, 14, 8]
```

## 4. Prefix Min & Prefix Max

### Theory

`prefixMin[i]` (or `prefixMax[i]`) tracks the smallest (or largest) value seen from the start up through index `i`. Unlike sum/product/XOR, min and max **can't be "undone"** — there's no inverse operation to remove an element from a running min/max, so this only answers queries of the form `[0, r]`, not arbitrary `[l, r]` ranges.

### Code

```java
import java.util.Arrays;

public class PrefixMinMax {
    public static void main(String[] args) {
        int[] arr = { 5, 3, 8, 1, 9 };

        System.out.println("Prefix min: " + Arrays.toString(buildPrefixMin(arr)));
        System.out.println("Prefix max: " + Arrays.toString(buildPrefixMax(arr)));
    }

    static int[] buildPrefixMin(int[] arr) {
        int[] prefixMin = new int[arr.length];
        prefixMin[0] = arr[0];
        for (int i = 1; i < arr.length; i++) {
            prefixMin[i] = Math.min(prefixMin[i - 1], arr[i]);
        }
        return prefixMin;
    }

    static int[] buildPrefixMax(int[] arr) {
        int[] prefixMax = new int[arr.length];
        prefixMax[0] = arr[0];
        for (int i = 1; i < arr.length; i++) {
            prefixMax[i] = Math.max(prefixMax[i - 1], arr[i]);
        }
        return prefixMax;
    }
}
```

### Example

```
Prefix min: [5, 3, 3, 1, 1]
Prefix max: [5, 5, 8, 8, 9]
```

### Problem 1 (Basic): Best Time to Buy and Sell Stock

**Theory:** Full theory and code in [Lesson 13: Best Time to Buy and Sell Stock](<./13. Kadane Family.md#6-best-time-to-buy-and-sell-stock>) — at heart, it's a **running prefix minimum** (the lowest price seen so far), compared against each day's price to find the best possible profit.

### Problem 2 (Medium): Check if Array Can Be Split at a Point (Max-Left ≤ Min-Right)

**Theory:** Find a split point where **every** element to the left is `≤` every element to the right. Rather than comparing every left element against every right element (O(n²)), it's enough to compare just the **prefix max** of the left part against the **suffix min** of the right part at each candidate split — if the largest element on the left doesn't exceed the smallest on the right, the whole split is automatically valid.

```java
public class SplitAtMaxLeftMinRight {
    public static void main(String[] args) {
        System.out.println("Can split [1,2,3,4]: " + canSplit(new int[] { 1, 2, 3, 4 }));
        System.out.println("Can split [3,2,1]: " + canSplit(new int[] { 3, 2, 1 }));
    }

    static boolean canSplit(int[] arr) {
        int n = arr.length;
        int[] prefixMax = new int[n];
        prefixMax[0] = arr[0];
        for (int i = 1; i < n; i++) prefixMax[i] = Math.max(prefixMax[i - 1], arr[i]);

        int[] suffixMin = new int[n];
        suffixMin[n - 1] = arr[n - 1];
        for (int i = n - 2; i >= 0; i--) suffixMin[i] = Math.min(suffixMin[i + 1], arr[i]);

        for (int i = 0; i < n - 1; i++) {
            if (prefixMax[i] <= suffixMin[i + 1]) return true;
        }
        return false;
    }
}
```

```
Can split [1,2,3,4]: true
Can split [3,2,1]: false
```

`[1,2,3,4]` splits cleanly after index `0` (`1 ≤ 2,3,4`); `[3,2,1]` is decreasing, so no split point ever has every left element `≤` every right element.

## 5. Difference Array

### Theory

A **difference array** is the mirror image of a prefix sum — instead of speeding up *queries*, it speeds up *range updates*. To add `val` to every element in `[l, r]`, touch only **two** positions in a helper array `diff`:

```
diff[l]   += val
diff[r+1] -= val
```

Taking the prefix sum of `diff` afterward reconstructs the fully-updated array. This turns **many O(n) range updates** into **many O(1) updates followed by a single O(n) reconstruction pass** — a huge win when there are lots of range updates to apply before reading the final array.

### Code

```java
import java.util.Arrays;

public class DifferenceArray {
    public static void main(String[] args) {
        int n = 5;
        int[] diff = new int[n + 1];

        applyRangeUpdate(diff, 1, 3, 3); // add 3 to indices 1..3
        applyRangeUpdate(diff, 0, 2, 2); // add 2 to indices 0..2

        int[] result = reconstruct(diff, n);
        System.out.println("Final array: " + Arrays.toString(result));
    }

    static void applyRangeUpdate(int[] diff, int l, int r, int val) {
        diff[l] += val;
        diff[r + 1] -= val;
    }

    static int[] reconstruct(int[] diff, int n) {
        int[] result = new int[n];
        result[0] = diff[0];
        for (int i = 1; i < n; i++) {
            result[i] = result[i - 1] + diff[i];
        }
        return result;
    }
}
```

### Example

```
Final array: [2, 5, 5, 3, 0]
```

Starting from all zeros: adding `3` to indices `1-3` gives `[0,3,3,3,0]`; adding `2` to indices `0-2` on top gives `[2,5,5,3,0]` — exactly what the difference array reconstructs, without ever looping over a range directly.

## 6. Range Updates

### Theory

"Range update" is the general name for the pattern in [Difference Array](#5-difference-array): applying the same change across every element in `[l, r]`. Without a difference array, `q` range updates over an array of size `n` cost **O(q × n)** in the worst case; with one, they cost **O(q + n)** — each update is O(1), and one final O(n) pass reconstructs the array.

### Problem (Medium): Corporate Flight Bookings

**Theory:** Given a list of `[firstFlight, lastFlight, seats]` bookings (1-indexed, inclusive ranges), find the total seats reserved on every flight. Each booking is exactly one range update — apply all of them to a difference array, then reconstruct once at the end.

```java
import java.util.Arrays;

public class CorporateFlightBookings {
    public static void main(String[] args) {
        int[][] bookings = { { 1, 2, 10 }, { 2, 3, 20 }, { 2, 5, 25 } };
        System.out.println("Seats per flight: " + Arrays.toString(corpFlightBookings(bookings, 5)));
    }

    static int[] corpFlightBookings(int[][] bookings, int n) {
        int[] diff = new int[n + 1];
        for (int[] booking : bookings) {
            int first = booking[0], last = booking[1], seats = booking[2];
            diff[first - 1] += seats; // convert to 0-indexed
            diff[last] -= seats;
        }

        int[] result = new int[n];
        result[0] = diff[0];
        for (int i = 1; i < n; i++) {
            result[i] = result[i - 1] + diff[i];
        }
        return result;
    }
}
```

```
Seats per flight: [10, 55, 45, 25, 25]
```

Flight 2 (index 1) appears in all three bookings — `10 + 20 + 25 = 55` seats reserved.

## 7. Range Queries

### Theory

"Range query" is the general name for the pattern in [Prefix Sum](#1-prefix-sum) (and its Product/XOR/Min/Max variants): answering "what's the aggregate over `[l, r]`?" repeatedly. Without a prefix array, `q` queries cost **O(q × n)**; with one, they cost **O(n + q)** — one O(n) preprocessing pass, then O(1) per query.

> **Rule of thumb:** many range **queries** → build a prefix array. Many range **updates** → build a difference array. Needing both frequently on the same array is what motivates more advanced structures (Segment Trees, Fenwick/BIT), which are outside the scope of this lesson.

### Problem (Basic): Range Sum Query - Immutable

**Theory:** The canonical "many range queries" design problem: given an array that never changes, answer many `sumRange(l, r)` calls efficiently. Precompute the prefix sum once in the constructor; every query is then a single subtraction.

```java
public class RangeSumQueryImmutable {
    private int[] prefix;

    public static void main(String[] args) {
        RangeSumQueryImmutable nsq = new RangeSumQueryImmutable(new int[] { -2, 0, 3, -5, 2, -1 });
        System.out.println("sumRange(0,2): " + nsq.sumRange(0, 2));
        System.out.println("sumRange(2,5): " + nsq.sumRange(2, 5));
        System.out.println("sumRange(0,5): " + nsq.sumRange(0, 5));
    }

    RangeSumQueryImmutable(int[] nums) {
        prefix = new int[nums.length + 1];
        for (int i = 0; i < nums.length; i++) {
            prefix[i + 1] = prefix[i] + nums[i];
        }
    }

    int sumRange(int l, int r) {
        return prefix[r + 1] - prefix[l];
    }
}
```

```
sumRange(0,2): 1
sumRange(2,5): -1
sumRange(0,5): -3
```

## 8. Continuous Subarray Sum

### Theory

Given an array, does it contain a subarray of length **≥ 2** whose sum is a multiple of `k`? Two prefix sums that leave the **same remainder mod k** mean everything between them sums to a multiple of `k` (their difference is divisible by `k` by definition). Track the **earliest index** each remainder was first seen; if the same remainder reappears at least `2` indices later, a qualifying subarray exists.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class ContinuousSubarraySum {
    public static void main(String[] args) {
        int[] nums = { 23, 2, 4, 6, 7 };
        System.out.println("Has subarray sum multiple of 6: " + checkSubarraySum(nums, 6));
    }

    static boolean checkSubarraySum(int[] nums, int k) {
        Map<Integer, Integer> remainderFirstSeenAt = new HashMap<>();
        remainderFirstSeenAt.put(0, -1); // an empty prefix has remainder 0, "seen" before index 0

        int prefixSum = 0;
        for (int i = 0; i < nums.length; i++) {
            prefixSum += nums[i];
            int remainder = (k == 0) ? prefixSum : prefixSum % k;

            if (remainderFirstSeenAt.containsKey(remainder)) {
                if (i - remainderFirstSeenAt.get(remainder) > 1) {
                    return true;
                }
            } else {
                remainderFirstSeenAt.put(remainder, i);
            }
        }
        return false;
    }
}
```

### Example

```
Has subarray sum multiple of 6: true
```

`[2, 4]` sums to `6` (length `2`, a multiple of `6`) — found via the remainder-repeat trick without ever summing that subarray directly.

## 9. Maximum Size Subarray Sum Equals K

### Theory

A close relative of [Lesson 11: Subarray Sum Equals K](<./11. Hashing.md#6-prefix-hashing>) — instead of *counting* how many subarrays sum to `k`, find the **longest** one. Track the **earliest** index each prefix sum was seen (not a count); whenever `prefixSum - k` has been seen before, the subarray between that earlier index and now is a candidate — and since it uses the earliest occurrence, it's the longest possible subarray ending here with that sum.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class MaxSizeSubarraySumEqualsK {
    public static void main(String[] args) {
        int[] nums = { 1, -1, 5, -2, 3 };
        System.out.println("Longest subarray summing to 3: " + maxSubArrayLen(nums, 3));
    }

    static int maxSubArrayLen(int[] nums, int k) {
        Map<Integer, Integer> firstSeenAt = new HashMap<>();
        firstSeenAt.put(0, -1);

        int prefixSum = 0, best = 0;
        for (int i = 0; i < nums.length; i++) {
            prefixSum += nums[i];
            if (firstSeenAt.containsKey(prefixSum - k)) {
                best = Math.max(best, i - firstSeenAt.get(prefixSum - k));
            }
            firstSeenAt.putIfAbsent(prefixSum, i); // keep the EARLIEST index only
        }
        return best;
    }
}
```

### Example

```
Longest subarray summing to 3: 4
```

`[1, -1, 5, -2]` sums to `3` and has length `4` — the longest qualifying subarray.

## Quick Reference Cheat Sheet

| Variant | Build formula | Problems covered |
|---|---|---|
| Prefix Sum | `prefix[i] = prefix[i-1] + arr[i]` | Range Sum → Find Pivot Index |
| Prefix Product | `prefix[i] = prefix[i-1] * arr[i]` | Range Product → Product of the Last K Numbers |
| Prefix XOR | `prefix[i] = prefix[i-1] ^ arr[i]` | Range XOR → XOR Queries of a Subarray |
| Prefix Min/Max | running `min`/`max` | Best Time to Buy/Sell Stock → Split at Max-Left ≤ Min-Right |
| Difference Array | `diff[l] += val; diff[r+1] -= val` | reconstruct via prefix sum of `diff` |
| Range Updates | see Difference Array | Corporate Flight Bookings |
| Range Queries | see Prefix Sum | Range Sum Query - Immutable |
| Continuous Subarray Sum | prefix sum `% k`, track first-seen index per remainder | subarray exists if a remainder repeats `≥2` indices later |
| Max Size Subarray Sum Equals K | prefix sum, track first-seen index per sum value | `best = max(best, i - firstSeenAt[prefixSum - k])` |
