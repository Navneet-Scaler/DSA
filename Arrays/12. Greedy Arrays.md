## 1. Jump Game

### Theory

Given an array where `arr[i]` is the maximum jump length from index `i`, can you reach the last index? Greedily track the **farthest reachable index** seen so far while scanning left to right; if you ever reach a position **beyond** the farthest-reachable mark before updating it, you're stuck.

### Code

```java
public class JumpGame {
    public static void main(String[] args) {
        System.out.println("Can reach end [2,3,1,1,4]: " + canJump(new int[] { 2, 3, 1, 1, 4 }));
        System.out.println("Can reach end [3,2,1,0,4]: " + canJump(new int[] { 3, 2, 1, 0, 4 }));
    }

    static boolean canJump(int[] nums) {
        int maxReach = 0;
        for (int i = 0; i < nums.length; i++) {
            if (i > maxReach) return false; // this index is unreachable
            maxReach = Math.max(maxReach, i + nums[i]);
        }
        return true;
    }
}
```

### Example

```
Can reach end [2,3,1,1,4]: true
Can reach end [3,2,1,0,4]: false
```

In the second case, index `3` (value `0`) traps anything that lands on it, and no earlier jump reaches far enough to skip past it.

## 2. Gas Station

### Theory

Given gas available and cost to travel between stations arranged in a circle, find the starting station that allows completing the full circuit (guaranteed unique if a solution exists and total gas ≥ total cost). Greedy insight: if the tank ever goes negative starting from some station `start`, **no station between `start` and the failure point** could have worked either (they'd fail even sooner) — so jump the candidate start past the failure point and keep going.

### Code

```java
public class GasStation {
    public static void main(String[] args) {
        int[] gas = { 1, 2, 3, 4, 5 };
        int[] cost = { 3, 4, 5, 1, 2 };
        System.out.println("Starting station: " + canCompleteCircuit(gas, cost));
    }

    static int canCompleteCircuit(int[] gas, int[] cost) {
        int totalTank = 0, currentTank = 0, start = 0;
        for (int i = 0; i < gas.length; i++) {
            int diff = gas[i] - cost[i];
            totalTank += diff;
            currentTank += diff;
            if (currentTank < 0) {
                start = i + 1; // no station up to and including i can be the start
                currentTank = 0;
            }
        }
        return (totalTank >= 0) ? start : -1;
    }
}
```

### Example

```
Starting station: 3
```

## 3. Interval-like Array Problems

### Theory

Arrays of `[start, end]` pairs come up constantly — meetings, jobs, ranges. **Classic example — maximum non-overlapping intervals** (activity selection): sort by **end time**, then greedily pick every interval whose start is `≥` the last picked interval's end. Picking by earliest end time (rather than earliest start, or shortest duration) is the greedy choice provably maximizing the count.

### Code

```java
import java.util.Arrays;
import java.util.Comparator;

public class IntervalLikeProblems {
    public static void main(String[] args) {
        int[][] intervals = { { 1, 2 }, { 2, 3 }, { 3, 4 }, { 1, 3 } };
        System.out.println("Max non-overlapping intervals: " + maxNonOverlapping(intervals));
    }

    static int maxNonOverlapping(int[][] intervals) {
        Arrays.sort(intervals, Comparator.comparingInt(interval -> interval[1]));

        int count = 0;
        int lastEnd = Integer.MIN_VALUE;
        for (int[] interval : intervals) {
            if (interval[0] >= lastEnd) {
                count++;
                lastEnd = interval[1];
            }
        }
        return count;
    }
}
```

### Example

```
Max non-overlapping intervals: 3
```

`[1,2]`, `[2,3]`, `[3,4]` are chosen (skipping `[1,3]`, which overlaps).

## 4. Rearrangement

### Theory

Some problems ask for the **best possible rearrangement** under a custom notion of "best." **Classic example — Largest Number:** given non-negative integers, arrange them to form the largest possible concatenated number. The greedy comparator: place `a` before `b` if `a+b` (as strings) is numerically larger than `b+a` — sorting by this custom rule (rather than plain numeric or lexicographic order) is what makes the arrangement optimal.

### Code

```java
import java.util.Arrays;

public class Rearrangement {
    public static void main(String[] args) {
        int[] nums = { 3, 30, 34, 5, 9 };
        System.out.println("Largest number: " + largestNumber(nums));
    }

    static String largestNumber(int[] nums) {
        String[] strs = new String[nums.length];
        for (int i = 0; i < nums.length; i++) strs[i] = String.valueOf(nums[i]);

        Arrays.sort(strs, (a, b) -> (b + a).compareTo(a + b)); // custom greedy comparator

        if (strs[0].equals("0")) return "0"; // handles the all-zero case, e.g. [0,0]

        StringBuilder result = new StringBuilder();
        for (String s : strs) result.append(s);
        return result.toString();
    }
}
```

### Example

```
Largest number: 9534330
```

## 5. Scheduling with Arrays

### Theory

**Classic example — Minimum Meeting Rooms:** given meeting `[start, end]` intervals, find the minimum number of rooms needed so no two overlapping meetings share a room. Greedy approach: sort start times and end times **separately**; sweep through start times, and every time a meeting starts before the earliest currently-ongoing meeting ends, that's another room needed — otherwise, a room has freed up and can be reused.

### Code

```java
import java.util.Arrays;

public class SchedulingWithArrays {
    public static void main(String[] args) {
        int[][] meetings = { { 0, 30 }, { 5, 10 }, { 15, 20 } };
        System.out.println("Minimum meeting rooms: " + minMeetingRooms(meetings));
    }

    static int minMeetingRooms(int[][] meetings) {
        int n = meetings.length;
        int[] starts = new int[n];
        int[] ends = new int[n];
        for (int i = 0; i < n; i++) {
            starts[i] = meetings[i][0];
            ends[i] = meetings[i][1];
        }
        Arrays.sort(starts);
        Arrays.sort(ends);

        int rooms = 0, maxRooms = 0;
        int s = 0, e = 0;
        while (s < n) {
            if (starts[s] < ends[e]) {
                rooms++;
                s++;
                maxRooms = Math.max(maxRooms, rooms);
            } else {
                rooms--;
                e++;
            }
        }
        return maxRooms;
    }
}
```

### Example

```
Minimum meeting rooms: 2
```

`[0,30]` overlaps with both `[5,10]` and `[15,20]` at different points, but `[5,10]` and `[15,20]` never overlap each other — so `2` rooms is enough, but `1` is not.

## Quick Reference Cheat Sheet

| Problem | Greedy rule |
|---|---|
| Jump Game | track farthest reachable index; fail if current index exceeds it |
| Gas Station | reset the candidate start right after any point the tank goes negative |
| Interval Scheduling | sort by end time, greedily pick non-overlapping intervals |
| Rearrangement (Largest Number) | sort by custom comparator: `a+b` vs `b+a` |
| Scheduling (Meeting Rooms) | sort starts/ends separately, sweep to track concurrent overlaps |
