## 1. Dutch National Flag

### Theory

Given an array with only three distinct values (classically `0`, `1`, `2` — like the stripes of the Dutch flag), sort it in **one pass** using **three pointers**: `low` (boundary of the `0`s region), `mid` (current element being examined), and `high` (boundary of the `2`s region). Swap `0`s to the front, leave `1`s in place, and swap `2`s to the back — `mid` only advances when it's safe to do so (after a `0` or `1` swap; after swapping in a `2`, `mid` must re-examine the newly-swapped-in value).

### Code

```java
import java.util.Arrays;

public class DutchNationalFlag {
    public static void main(String[] args) {
        int[] arr = { 2, 0, 2, 1, 1, 0 };
        sortColors(arr);
        System.out.println("Sorted colors: " + Arrays.toString(arr));
    }

    static void sortColors(int[] arr) {
        int low = 0, mid = 0, high = arr.length - 1;

        while (mid <= high) {
            if (arr[mid] == 0) {
                swap(arr, low, mid);
                low++;
                mid++;
            } else if (arr[mid] == 1) {
                mid++;
            } else {
                swap(arr, mid, high);
                high--; // don't advance mid -- the swapped-in value still needs examining
            }
        }
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### Example

```
Sorted colors: [0, 0, 1, 1, 2, 2]
```

## 2. Move Zeroes

### Theory

Move every `0` to the end of the array **while preserving the relative order** of the non-zero elements — the same [same-direction two-pointer](<./08. Two Pointers.md#2-same-direction>) skeleton as Remove Duplicates: `slow` marks where the next non-zero value should go, `fast` scans ahead.

### Code

```java
import java.util.Arrays;

public class MoveZeroes {
    public static void main(String[] args) {
        int[] arr = { 0, 1, 0, 3, 12 };
        moveZeroes(arr);
        System.out.println("After moving zeroes: " + Arrays.toString(arr));
    }

    static void moveZeroes(int[] arr) {
        int slow = 0;
        for (int fast = 0; fast < arr.length; fast++) {
            if (arr[fast] != 0) {
                int temp = arr[slow];
                arr[slow] = arr[fast];
                arr[fast] = temp;
                slow++;
            }
        }
    }
}
```

### Example

```
After moving zeroes: [1, 3, 12, 0, 0]
```

## 4. Remove Elements

### Theory

Given a target value, remove **every** occurrence of it in place, returning the new logical length. Same `slow`/`fast` skeleton again: `fast` scans every element, `slow` only keeps (and advances past) elements that **aren't** the target.

### Code

```java
import java.util.Arrays;

public class RemoveElements {
    public static void main(String[] args) {
        int[] arr = { 3, 2, 2, 3 };
        int newLength = removeElement(arr, 3);
        System.out.println("New length: " + newLength);
        System.out.println("Array prefix: " + Arrays.toString(Arrays.copyOf(arr, newLength)));
    }

    static int removeElement(int[] arr, int val) {
        int slow = 0;
        for (int fast = 0; fast < arr.length; fast++) {
            if (arr[fast] != val) {
                arr[slow] = arr[fast];
                slow++;
            }
        }
        return slow;
    }
}
```

### Example

```
New length: 2
Array prefix: [2, 2]
```

## 5. Wiggle Sort

### Theory

Rearrange an array so it alternates `arr[0] ≤ arr[1] ≥ arr[2] ≤ arr[3] ...`. Two variants, in increasing difficulty:

1. **Wiggle Sort I** (non-strict `≤`/`≥`) — a single **greedy pass**: whenever the current pair violates the expected direction for that position, swap them. No sorting needed at all.
2. **Wiggle Sort II** (strict `<`/`>`, the harder LeetCode variant) — the greedy pass can fail with strict inequalities and duplicate values, so instead: fully sort the array, then **interleave** the smaller half and larger half, each reversed, into the even and odd positions respectively. Reversing each half before interleaving keeps equal values from ending up adjacent.

### Code

```java
import java.util.Arrays;

public class WiggleSort {
    public static void main(String[] args) {
        int[] arr1 = { 3, 5, 2, 1, 6, 4 };
        wiggleSortI(arr1);
        System.out.println("Wiggle Sort I: " + Arrays.toString(arr1));

        int[] arr2 = { 1, 5, 1, 1, 6, 4 };
        System.out.println("Wiggle Sort II: " + Arrays.toString(wiggleSortII(arr2)));
    }

    // Greedy one-pass, non-strict inequalities, O(n)
    static void wiggleSortI(int[] arr) {
        for (int i = 0; i < arr.length - 1; i++) {
            boolean shouldBeLessEqual = (i % 2 == 0);
            boolean isViolated = shouldBeLessEqual ? arr[i] > arr[i + 1] : arr[i] < arr[i + 1];
            if (isViolated) {
                int temp = arr[i];
                arr[i] = arr[i + 1];
                arr[i + 1] = temp;
            }
        }
    }

    // Sort + interleave halves, strict inequalities, O(n log n)
    static int[] wiggleSortII(int[] arr) {
        int[] sorted = arr.clone();
        Arrays.sort(sorted);

        int n = arr.length;
        int[] result = new int[n];
        int mid = (n + 1) / 2;

        int smallIndex = mid - 1; // walk the smaller half backward (reversed)
        int largeIndex = n - 1;   // walk the larger half backward (reversed)

        for (int i = 0; i < n; i++) {
            if (i % 2 == 0) {
                result[i] = sorted[smallIndex--];
            } else {
                result[i] = sorted[largeIndex--];
            }
        }
        return result;
    }
}
```

### Example

```
Wiggle Sort I: [3, 5, 1, 6, 2, 4]
Wiggle Sort II: [1, 6, 1, 5, 1, 4]
```

Both satisfy their respective wiggle patterns — Wiggle Sort I in a single greedy pass, Wiggle Sort II by construction (interleaving pre-sorted, reversed halves guarantees the strict alternation even with repeated values like the three `1`s here).

## 6. Rearrangement

### Theory

**Advanced in-place trick — encode two values in one array slot.** Given an array where every value is itself a valid index (`0..n-1`), rearrange it so `newArr[i] = arr[arr[i]]` **without** allocating a second array. The trick: temporarily store **both** the old and new value in the same cell using `newValue = oldValue + (encodedOtherValue % n) * n` — since every value is `< n`, `value % n` always recovers the original, while `value / n` recovers the encoded piece. A final pass divides every cell by `n` to leave only the new values behind.

### Code

```java
import java.util.Arrays;

public class Rearrangement {
    public static void main(String[] args) {
        int[] arr = { 2, 0, 1, 3 };
        System.out.println("Rearranged so arr[i] = arr[arr[i]]: " + Arrays.toString(rearrange(arr)));
    }

    static int[] rearrange(int[] arr) {
        int n = arr.length;

        // Encode: each cell temporarily holds both its old value and part of its new value
        for (int i = 0; i < n; i++) {
            int newValue = arr[arr[i]] % n;
            arr[i] += newValue * n;
        }

        // Decode: strip away the old value, keeping only the new one
        for (int i = 0; i < n; i++) {
            arr[i] /= n;
        }
        return arr;
    }
}
```

### Example

```
Rearranged so arr[i] = arr[arr[i]]: [1, 2, 0, 3]
```

For the original `[2, 0, 1, 3]`: `newArr[0] = arr[arr[0]] = arr[2] = 1`, `newArr[1] = arr[arr[1]] = arr[0] = 2`, `newArr[2] = arr[arr[2]] = arr[1] = 0`, `newArr[3] = arr[arr[3]] = arr[3] = 3` — matching `[1, 2, 0, 3]`.

## Quick Reference Cheat Sheet

| Problem | Technique | Complexity |
|---|---|---|
| Dutch National Flag | three pointers (`low`, `mid`, `high`), single pass | O(n) time, O(1) space |
| Move Zeroes | same-direction two pointers, swap non-zero into place | O(n) time, O(1) space |
| Remove Duplicates | same-direction two pointers | see [Lesson 8](<./08. Two Pointers.md#2-same-direction>) |
| Remove Elements | same-direction two pointers, keep non-target values | O(n) time, O(1) space |
| Wiggle Sort I | greedy single pass, swap on violation | O(n) time, O(1) space |
| Wiggle Sort II | sort, then interleave reversed halves | O(n log n) time, O(n) space |
| Rearrangement (`arr[i]=arr[arr[i]]`) | encode two values per cell via `value + (new % n) * n` | O(n) time, O(1) space |
