# 7. Sliding Window

The sliding window pattern avoids recomputing a range's aggregate from scratch every time the range shifts by one — instead, it updates incrementally as a window of elements slides across the array, turning many O(n²) brute-force scans into a single **O(n)** pass.

## 1. Fixed Window

### Theory

A **fixed-size window** always spans exactly `k` elements. Instead of resumming all `k` elements at each new position, slide the window by **adding the incoming element and removing the outgoing one** — O(1) per shift, O(n) total instead of O(n×k).

### Code

```java
public class FixedWindow {
    public static void main(String[] args) {
        int[] arr = { 2, 1, 5, 1, 3, 2 };
        System.out.println("Max sum, window size 3: " + maxSumFixedWindow(arr, 3));
    }

    static int maxSumFixedWindow(int[] arr, int k) {
        int windowSum = 0;
        for (int i = 0; i < k; i++) windowSum += arr[i];

        int best = windowSum;
        for (int i = k; i < arr.length; i++) {
            windowSum += arr[i] - arr[i - k]; // add new element, drop the one leaving the window
            best = Math.max(best, windowSum);
        }
        return best;
    }
}
```

### Example

```
Max sum, window size 3: 9
```

The window `[5, 1, 3]` sums to `9` — the best of any 3 consecutive elements.

## 2. Variable Window

### Theory

A **variable-size window** grows and shrinks based on a condition, using two pointers (`left`, `right`) instead of a fixed offset. **Classic example — Longest Substring Without Repeating Characters:** expand `right` one step at a time; whenever the new character has been seen **inside the current window**, jump `left` past its previous occurrence.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class VariableWindow {
    public static void main(String[] args) {
        System.out.println("Longest substring without repeats: " + longestNoRepeat("abcabcbb"));
    }

    static int longestNoRepeat(String s) {
        Map<Character, Integer> lastSeen = new HashMap<>();
        int left = 0, best = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
                left = lastSeen.get(c) + 1; // jump left past the repeated character
            }
            lastSeen.put(c, right);
            best = Math.max(best, right - left + 1);
        }
        return best;
    }
}
```

### Example

```
Longest substring without repeats: 3
```

`"abc"` is the longest run with no repeated character (length 3).

## 3. Longest Window

### Theory

"Find the longest window satisfying a condition" is the general shape behind [Variable Window](#2-variable-window). **Another example — longest subarray with sum ≤ target:** expand `right` to grow the window; whenever the running sum exceeds the target, shrink from `left` until it's valid again, recording the best length along the way.

### Code

```java
public class LongestWindow {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("Longest subarray with sum <= 8: " + longestSubarraySumAtMost(arr, 8));
    }

    static int longestSubarraySumAtMost(int[] arr, int target) {
        int left = 0, sum = 0, best = 0;
        for (int right = 0; right < arr.length; right++) {
            sum += arr[right];
            while (sum > target) {
                sum -= arr[left];
                left++;
            }
            best = Math.max(best, right - left + 1);
        }
        return best;
    }
}
```

### Example

```
Longest subarray with sum <= 8: 3
```

`[1, 2, 3]` sums to `6 ≤ 8` and has length `3` — the longest such run (adding `4` would push the sum past `8`).

## 4. Shortest Window

### Theory

The mirror problem: "find the **shortest** window satisfying a condition." **Classic example — Minimum Size Subarray Sum:** expand `right` to accumulate sum; the moment the sum reaches the target, greedily shrink from `left` as much as possible while still satisfying the condition, recording the shortest valid length found.

### Code

```java
public class ShortestWindow {
    public static void main(String[] args) {
        int[] arr = { 2, 3, 1, 2, 4, 3 };
        System.out.println("Shortest subarray with sum >= 7: " + shortestSubarraySumAtLeast(arr, 7));
    }

    static int shortestSubarraySumAtLeast(int[] arr, int target) {
        int left = 0, sum = 0, best = Integer.MAX_VALUE;
        for (int right = 0; right < arr.length; right++) {
            sum += arr[right];
            while (sum >= target) {
                best = Math.min(best, right - left + 1);
                sum -= arr[left];
                left++;
            }
        }
        return (best == Integer.MAX_VALUE) ? 0 : best;
    }
}
```

### Example

```
Shortest subarray with sum >= 7: 2
```

`[4, 3]` sums to `7` in just `2` elements — shorter than any other qualifying run.

## 5. Maximum Window

### Theory

"Maximum window" here means finding the **maximum value inside every fixed-size window** as it slides — not the window's size. A **monotonic decreasing deque** (full treatment in [Lesson 15](<./15. Monotonic Structures.md>)) does this in O(n) total: keep the deque's front always holding the current window's maximum, popping smaller trailing values before adding a new one.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class MaximumWindow {
    public static void main(String[] args) {
        int[] arr = { 1, 3, -1, -3, 5, 3, 6, 7 };
        System.out.print("Sliding window maximums (k=3): ");
        for (int value : slidingWindowMax(arr, 3)) System.out.print(value + " ");
        System.out.println();
    }

    static int[] slidingWindowMax(int[] arr, int k) {
        Deque<Integer> indices = new ArrayDeque<>(); // stores indices, values decreasing front-to-back
        int[] result = new int[arr.length - k + 1];

        for (int i = 0; i < arr.length; i++) {
            while (!indices.isEmpty() && arr[indices.peekLast()] <= arr[i]) {
                indices.pollLast();
            }
            indices.addLast(i);

            if (indices.peekFirst() <= i - k) {
                indices.pollFirst(); // this index fell out of the window
            }
            if (i >= k - 1) {
                result[i - k + 1] = arr[indices.peekFirst()];
            }
        }
        return result;
    }
}
```

### Example

```
Sliding window maximums (k=3): 3 3 5 5 6 7 
```

## 6. Minimum Window

### Theory

The mirror of [Maximum Window](#5-maximum-window): the **minimum value inside every fixed-size window**, using a monotonic **increasing** deque instead.

> Don't confuse this with **"Minimum Window Substring"** (a separate, harder problem: find the shortest substring of `s` containing every character of `t`) — that one combines the [Variable Window](#2-variable-window) shrink/expand pattern with a character-frequency map, rather than the fixed-window deque technique shown here.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class MinimumWindow {
    public static void main(String[] args) {
        int[] arr = { 1, 3, -1, -3, 5, 3, 6, 7 };
        System.out.print("Sliding window minimums (k=3): ");
        for (int value : slidingWindowMin(arr, 3)) System.out.print(value + " ");
        System.out.println();
    }

    static int[] slidingWindowMin(int[] arr, int k) {
        Deque<Integer> indices = new ArrayDeque<>(); // stores indices, values increasing front-to-back
        int[] result = new int[arr.length - k + 1];

        for (int i = 0; i < arr.length; i++) {
            while (!indices.isEmpty() && arr[indices.peekLast()] >= arr[i]) {
                indices.pollLast();
            }
            indices.addLast(i);

            if (indices.peekFirst() <= i - k) {
                indices.pollFirst();
            }
            if (i >= k - 1) {
                result[i - k + 1] = arr[indices.peekFirst()];
            }
        }
        return result;
    }
}
```

### Example

```
Sliding window minimums (k=3): -1 -3 -3 -3 3 3 
```

## 7. Window Expansion

### Theory

**Expansion** is the "grow" half of the variable-window mechanic: advancing the `right` pointer to bring in a new element, updating the window's running state (sum, frequency map, distinct-count, etc.) to include it. Every variable-window algorithm in this lesson expands unconditionally at every step of its main loop — expansion never has a condition gating it, only shrinking does.

```
expand:
    include arr[right] into the window's running state
    right++
```

## 8. Window Shrinking

### Theory

**Shrinking** is the "shrink" half: advancing the `left` pointer to drop the leftmost element, when the window has grown to violate (or, for a "longest" search, is about to violate) the problem's constraint. This is where the two window search flavors diverge:

- **Longest-window search** ([Section 3](#3-longest-window)): shrink only while the constraint is **violated** (a `while (invalid)` loop), then measure.
- **Shortest-window search** ([Section 4](#4-shortest-window)): shrink as much as possible **while the constraint still holds** (a `while (valid)` loop), measuring at every successful shrink, since a smaller *valid* window is always at least as good.

```
shrink (longest-window style):
    while window state is invalid:
        remove arr[left] from the window's running state
        left++

shrink (shortest-window style):
    while window state is still valid:
        record this window as a candidate answer
        remove arr[left] from the window's running state
        left++
```

## Quick Reference Cheat Sheet

| Variant | Pointer behavior | Example problem |
|---|---|---|
| Fixed Window | slide by 1: add incoming, remove outgoing | max sum of any `k` consecutive elements |
| Variable Window | `left`/`right` move independently | longest substring without repeating characters |
| Longest Window | shrink only while invalid | longest subarray with sum ≤ target |
| Shortest Window | shrink while still valid, recording each step | shortest subarray with sum ≥ target |
| Maximum Window | monotonic decreasing deque | sliding window maximum |
| Minimum Window | monotonic increasing deque | sliding window minimum |
| Window Expansion | unconditional, every iteration | — |
| Window Shrinking | conditional, depends on longest vs shortest goal | — |
