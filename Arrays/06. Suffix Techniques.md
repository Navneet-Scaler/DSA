# 6. Suffix Techniques

Suffix arrays are [Prefix Techniques](<./05. Prefix Techniques.md>) computed from the **other direction** — cumulative results built from the end of the array backward. On their own they answer "aggregate from here to the end" queries; combined with a prefix array, they unlock problems that need information from **both sides** of every index at once.

## 1. Suffix Sum

### Theory

`suffix[i]` stores the sum of `arr[i..n-1]` — built from the **last** index backward to the first:

```
suffix[n-1] = arr[n-1]
suffix[i] = suffix[i+1] + arr[i]
```

### Code

```java
import java.util.Arrays;

public class SuffixSum {
    public static void main(String[] args) {
        int[] arr = { 2, 4, 6, 8, 10 };
        System.out.println("Suffix sum: " + Arrays.toString(buildSuffixSum(arr)));
    }

    static int[] buildSuffixSum(int[] arr) {
        int n = arr.length;
        int[] suffix = new int[n];
        suffix[n - 1] = arr[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffix[i] = suffix[i + 1] + arr[i];
        }
        return suffix;
    }
}
```

### Example

```
Suffix sum: [30, 28, 24, 18, 10]
```

`suffix[0] = 30` is the total sum — the whole array is a suffix of itself starting at index `0`.

### Problem 2 (Medium): Number of Ways to Split Array

**Theory:** Count split points where the left part's sum is `≥` the right part's sum. Track a running `leftSum` while scanning left to right; the right sum is always `total - leftSum` — no suffix array needs to be materialized at all, since only the total and a running left sum are needed (a nice reminder that "suffix sum" is sometimes better computed on the fly than precomputed in full).

```java
public class WaysToSplitArray {
    public static void main(String[] args) {
        int[] nums = { 10, 4, -8, 7 };
        System.out.println("Valid split points: " + waysToSplitArray(nums));
    }

    static int waysToSplitArray(int[] nums) {
        long total = 0;
        for (int n : nums) total += n;

        long leftSum = 0;
        int count = 0;
        for (int i = 0; i < nums.length - 1; i++) {
            leftSum += nums[i];
            long rightSum = total - leftSum;
            if (leftSum >= rightSum) count++;
        }
        return count;
    }
}
```

```
Valid split points: 2
```

## 2. Suffix Product

### Theory

Same idea, multiplication instead of addition:

```
suffix[n-1] = arr[n-1]
suffix[i] = suffix[i+1] * arr[i]
```

### Code

```java
import java.util.Arrays;

public class SuffixProduct {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4 };
        System.out.println("Suffix product: " + Arrays.toString(buildSuffixProduct(arr)));
    }

    static int[] buildSuffixProduct(int[] arr) {
        int n = arr.length;
        int[] suffix = new int[n];
        suffix[n - 1] = arr[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffix[i] = suffix[i + 1] * arr[i];
        }
        return suffix;
    }
}
```

### Example

```
Suffix product: [24, 24, 12, 4]
```

### Problem (Medium): Maximum Product Subarray

**Theory:** Full theory and code in [Lesson 13: Maximum Product Subarray](<./13. ## 3. Suffix Min & Suffix Max

### Theory

`suffixMin[i]` (or `suffixMax[i]`) is the smallest (largest) value in `arr[i..n-1]` — the mirror of [Prefix Min & Prefix Max](<./05. Prefix Techniques.md#4-prefix-min--prefix-max>), built from the end backward.

### Code

```java
import java.util.Arrays;

public class SuffixMinMax {
    public static void main(String[] args) {
        int[] arr = { 5, 3, 8, 1, 9 };

        System.out.println("Suffix min: " + Arrays.toString(buildSuffixMin(arr)));
        System.out.println("Suffix max: " + Arrays.toString(buildSuffixMax(arr)));
    }

    static int[] buildSuffixMin(int[] arr) {
        int n = arr.length;
        int[] suffixMin = new int[n];
        suffixMin[n - 1] = arr[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffixMin[i] = Math.min(suffixMin[i + 1], arr[i]);
        }
        return suffixMin;
    }

    static int[] buildSuffixMax(int[] arr) {
        int n = arr.length;
        int[] suffixMax = new int[n];
        suffixMax[n - 1] = arr[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffixMax[i] = Math.max(suffixMax[i + 1], arr[i]);
        }
        return suffixMax;
    }
}
```

### Example

```
Suffix min: [1, 1, 1, 1, 9]
Suffix max: [9, 9, 9, 9, 9]
```

### Problem (Advanced): Maximum Width Ramp

**Theory:** A "ramp" is a pair of indices `i < j` with `arr[i] ≤ arr[j]`; find the widest one. Precompute a **suffix max** array, then for each candidate left index `i`, use a second pointer `j` (only ever moving forward, never backward) to find how far right a valid ramp extends — advancing `i` forward whenever `arr[i]` exceeds the suffix max at the current `j`, since no ramp starting at `i` could possibly reach that far. The two pointers together make this O(n) despite superficially looking like it needs checking every pair.

```java
public class MaximumWidthRamp {
    public static void main(String[] args) {
        int[] nums = { 6, 0, 8, 2, 1, 5 };
        System.out.println("Maximum width ramp: " + maxWidthRamp(nums));
    }

    static int maxWidthRamp(int[] nums) {
        int n = nums.length;
        int[] suffixMax = new int[n];
        suffixMax[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffixMax[i] = Math.max(suffixMax[i + 1], nums[i]);
        }

        int best = 0, i = 0;
        for (int j = 0; j < n; j++) {
            while (i < j && nums[i] > suffixMax[j]) {
                i++;
            }
            if (i <= j && nums[i] <= suffixMax[j]) {
                best = Math.max(best, j - i);
            }
        }
        return best;
    }
}
```

```
Maximum width ramp: 4
```

`nums[1] = 0 ≤ nums[5] = 5`, a ramp of width `5 - 1 = 4` — the widest possible in this array.

## 4. Prefix + Suffix Combination

### Theory

Some problems need, for **every index**, an aggregate of everything **before** it *and* everything **after** it — neither a prefix array nor a suffix array alone can answer that, but together they can, in **O(n)** total. **Classic example — Product of Array Except Self:** for each index `i`, the answer is the product of every element except `arr[i]`, computed without division:

```
answer[i] = (product of everything before i) × (product of everything after i)
```

A clean way to compute this in a **single pass** avoids materializing separate prefix/suffix arrays: walk left-to-right accumulating a running prefix product into the result array, then walk right-to-left multiplying in a running suffix product.

### Code

```java
import java.util.Arrays;

public class PrefixSuffixCombination {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4 };
        System.out.println("Product except self: " + Arrays.toString(productExceptSelf(arr)));
    }

    static int[] productExceptSelf(int[] arr) {
        int n = arr.length;
        int[] result = new int[n];

        // Pass 1: result[i] = product of everything before i
        int prefix = 1;
        for (int i = 0; i < n; i++) {
            result[i] = prefix;
            prefix *= arr[i];
        }

        // Pass 2: multiply in the product of everything after i
        int suffix = 1;
        for (int i = n - 1; i >= 0; i--) {
            result[i] *= suffix;
            suffix *= arr[i];
        }

        return result;
    }
}
```

### Example

```
Product except self: [24, 12, 8, 6]
```

For index `2` (value `3`): product of everything else is `1 × 2 × 4 = 8` — matches `result[2] = 8`.

### Problem 2 (Advanced): Trapping Rain Water (Prefix/Suffix Max Version)

**Theory:** A third way to solve [Trapping Rain Water](<./08. Two Pointers.md#1-opposite-direction>) (alongside the two-pointer and monotonic-stack versions already shown): the water trapped above index `i` is `min(maxToLeft[i], maxToRight[i]) - height[i]`. Precompute a **prefix max** array and a **suffix max** array directly, then sum the trapped water at every index in one final pass. Simpler to reason about than the two-pointer version, at the cost of O(n) extra space for the two arrays.

```java
public class TrappingRainWaterPrefixSuffix {
    public static void main(String[] args) {
        int[] height = { 0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1 };
        System.out.println("Water trapped: " + trap(height));
    }

    static int trap(int[] height) {
        int n = height.length;
        if (n == 0) return 0;

        int[] leftMax = new int[n];
        int[] rightMax = new int[n];

        leftMax[0] = height[0];
        for (int i = 1; i < n; i++) leftMax[i] = Math.max(leftMax[i - 1], height[i]);

        rightMax[n - 1] = height[n - 1];
        for (int i = n - 2; i >= 0; i--) rightMax[i] = Math.max(rightMax[i + 1], height[i]);

        int water = 0;
        for (int i = 0; i < n; i++) {
            water += Math.min(leftMax[i], rightMax[i]) - height[i];
        }
        return water;
    }
}
```

```
Water trapped: 6
```

Same answer as the two-pointer ([Lesson 8](<./08. Two Pointers.md#1-opposite-direction>)) and monotonic-stack ([Lesson 15](<./15. Monotonic Structures.md#8-trapping-rain-water-monotonic-stack>)) versions — three different tools converging on the same result.

## Quick Reference Cheat Sheet

| Variant | Build formula | Problems covered |
|---|---|---|
| Suffix Sum | `suffix[i] = suffix[i+1] + arr[i]`, from the last index backward | recap → Number of Ways to Split Array |

| Suffix Min/Max | running `min`/`max`, from the last index backward | Maximum Width Ramp |
