## 1. Binary Search + Prefix

### Theory

**Classic example — Minimum Size Subarray Sum, revisited:** [Lesson 7](<./07. Sliding Window.md#4-shortest-window>) solved this with a sliding window in O(n). When the array is **non-negative**, its [prefix sum](<./05. Prefix Techniques.md#1-prefix-sum>) array is guaranteed **monotonically non-decreasing** — which means, for every starting index, [binary search](<./10. Binary Search on Arrays.md>) can directly find the smallest ending index whose prefix sum reaches the target. This trades the sliding window's O(n) for O(n log n) — worse here, but the same prefix-monotonicity + binary-search combination generalizes to variants where a two-pointer window doesn't apply cleanly (e.g., needing the count of qualifying subarrays for *every* starting index independently).

### Code

```java
import java.util.Arrays;

public class BinarySearchPrefix {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("Shortest subarray with sum >= 11: " + minSubArrayLen(arr, 11));
    }

    static int minSubArrayLen(int[] arr, int target) {
        int n = arr.length;
        int[] prefix = new int[n + 1];
        for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] + arr[i];

        int best = Integer.MAX_VALUE;
        for (int i = 0; i <= n; i++) {
            int needed = prefix[i] + target;
            int j = lowerBound(prefix, needed, i);
            if (j <= n) best = Math.min(best, j - i);
        }
        return (best == Integer.MAX_VALUE) ? 0 : best;
    }

    static int lowerBound(int[] arr, int target, int from) {
        int lo = from, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

### Example

```
Shortest subarray with sum >= 11: 3
```

## 2. Sliding Window + Prefix

### Theory

**Classic example — Subarrays with Exactly K Distinct Integers.** A plain sliding window naturally tracks "at most K distinct" (shrink when there are too many), but "**exactly** K" doesn't have a clean expand/shrink rule by itself. The fix combines two window counts using a prefix-style subtraction: `exactly(K) = atMost(K) - atMost(K-1)` — the count of subarrays with at most `K-1` distinct values is effectively a "prefix" that gets subtracted away, leaving only subarrays with *exactly* `K`.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class SlidingWindowPrefix {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 1, 2, 3 };
        System.out.println("Subarrays with exactly 2 distinct: " + subarraysWithKDistinct(arr, 2));
    }

    static int subarraysWithKDistinct(int[] arr, int k) {
        return atMostKDistinct(arr, k) - atMostKDistinct(arr, k - 1);
    }

    static int atMostKDistinct(int[] arr, int k) {
        if (k == 0) return 0;
        Map<Integer, Integer> count = new HashMap<>();
        int left = 0, total = 0;

        for (int right = 0; right < arr.length; right++) {
            count.merge(arr[right], 1, Integer::sum);
            while (count.size() > k) {
                int leftValue = arr[left];
                count.put(leftValue, count.get(leftValue) - 1);
                if (count.get(leftValue) == 0) count.remove(leftValue);
                left++;
            }
            total += right - left + 1; // every subarray ending here, starting from 'left' onward, qualifies
        }
        return total;
    }
}
```

### Example

```
Subarrays with exactly 2 distinct: 7
```

## 3. Greedy + Prefix

### Theory

**Classic example — Grumpy Bookstore Owner:** on some minutes the owner is "grumpy" and customers leave unsatisfied; using a special technique for `X` consecutive minutes keeps everyone satisfied during that window. First, compute the **prefix baseline**: the total satisfied customers on all the naturally-non-grumpy minutes (an unconditional sum). Then greedily slide a fixed window of size `X` to find which stretch of grumpy minutes would recover the **most additional** customers if "fixed" — added on top of the baseline for the final answer.

### Code

```java
public class GreedyPrefix {
    public static void main(String[] args) {
        int[] customers = { 1, 0, 1, 2, 1, 1, 7, 5 };
        int[] grumpy = { 0, 1, 0, 1, 0, 1, 0, 1 };
        System.out.println("Max satisfied customers: " + maxSatisfied(customers, grumpy, 3));
    }

    static int maxSatisfied(int[] customers, int[] grumpy, int x) {
        int n = customers.length;

        int baseline = 0; // customers already satisfied regardless of the technique
        for (int i = 0; i < n; i++) {
            if (grumpy[i] == 0) baseline += customers[i];
        }

        int windowRecovered = 0, bestRecovered = 0;
        for (int i = 0; i < n; i++) {
            if (grumpy[i] == 1) windowRecovered += customers[i];
            if (i >= x && grumpy[i - x] == 1) windowRecovered -= customers[i - x];
            bestRecovered = Math.max(bestRecovered, windowRecovered);
        }
        return baseline + bestRecovered;
    }
}
```

### Example

```
Max satisfied customers: 16
```

## 4. Heap + Prefix

### Theory

**Classic example — Kth Smallest Prefix Sum.** Build the prefix sum array as usual, then instead of sorting all of it (O(n log n)) just to find the `k`-th smallest, run the [K Smallest](<./16. Heap + Arrays.md#3-k-smallest>) heap technique **directly on the prefix values** — O(n log k), useful when `k` is small relative to `n`.

### Code

```java
import java.util.PriorityQueue;
import java.util.Collections;

public class HeapPrefix {
    public static void main(String[] args) {
        int[] arr = { 3, -1, 4, 1, 5, -9, 2, 6 };
        System.out.println("3rd smallest prefix sum: " + kthSmallestPrefixSum(arr, 3));
    }

    static int kthSmallestPrefixSum(int[] arr, int k) {
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());

        int prefix = 0;
        for (int value : arr) {
            prefix += value;
            maxHeap.offer(prefix);
            if (maxHeap.size() > k) maxHeap.poll();
        }
        return maxHeap.peek();
    }
}
```

### Example

```
3rd smallest prefix sum: 3
```

## 5. Monotonic Queue Optimization

### Theory

This is the **DP-optimization** use of the [monotonic deque](<./15. Monotonic Structures.md>) idea, not just the sliding-window-maximum problem itself. **Classic example — Constrained Subsequence Sum:** `dp[i]` is the best subsequence sum ending at index `i`, where the previous chosen element must be within `k` positions back: `dp[i] = nums[i] + max(0, max(dp[i-k..i-1]))`. Computed naively, finding that max costs O(k) per index — O(nk) total. A monotonic decreasing deque (of `dp` values within the trailing window of size `k`) gets that max in **O(1) amortized**, dropping the whole DP to **O(n)**.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class MonotonicQueueOptimization {
    public static void main(String[] args) {
        int[] nums = { 10, 2, -10, 5, 20, -9 };
        System.out.println("Max constrained subsequence sum: " + constrainedSubsetSum(nums, 2));
    }

    static int constrainedSubsetSum(int[] nums, int k) {
        int n = nums.length;
        int[] dp = new int[n];
        Deque<Integer> deque = new ArrayDeque<>(); // indices, dp values decreasing front-to-back
        int best = Integer.MIN_VALUE;

        for (int i = 0; i < n; i++) {
            while (!deque.isEmpty() && deque.peekFirst() < i - k) {
                deque.pollFirst(); // fell outside the trailing window of size k
            }
            int bestPrevious = deque.isEmpty() ? 0 : Math.max(0, dp[deque.peekFirst()]);
            dp[i] = nums[i] + bestPrevious;

            while (!deque.isEmpty() && dp[deque.peekLast()] <= dp[i]) {
                deque.pollLast();
            }
            deque.addLast(i);

            best = Math.max(best, dp[i]);
        }
        return best;
    }
}
```

### Example

```
Max constrained subsequence sum: 37
```

## 6. Convex Hull Trick (Advanced)

### Theory

Some DP transitions have the form `dp[i] = min over j<i of (dp[j] + m(j) × x(i) + b(j))` — each earlier state `j` defines a **line** `y = m(j)·x + b(j)`, and evaluating `dp[i]` means finding the **minimum y** across all these lines at `x = x(i)`. Checking every line directly costs O(n) per query (O(n²) total) — the full **Convex Hull Trick** maintains only the lines that could ever be the minimum at *some* x (the lower envelope, forming a convex hull), letting queries run in O(log n) (binary search over the hull) or even O(1) amortized when queries arrive in sorted x-order.

> The version below evaluates every line directly per query (O(n) per query) to keep the code simple and clearly correct — it demonstrates *what* the technique computes. The full hull-maintenance structure (only adding a line if it can ever be optimal, removing lines made obsolete) is what actually achieves the O(log n)/O(1) speedup, and is a substantial data structure in its own right, typically pulled from a competitive programming library rather than hand-written per-contest.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class ConvexHullTrick {
    public static void main(String[] args) {
        List<long[]> lines = new ArrayList<>(); // each line: {slope, intercept}
        lines.add(new long[] { 2, 3 });
        lines.add(new long[] { -1, 5 });
        lines.add(new long[] { 0, 1 });

        for (long x : new long[] { -2, 0, 3 }) {
            System.out.println("Min value at x=" + x + ": " + minAt(lines, x));
        }
    }

    static long minAt(List<long[]> lines, long x) {
        long best = Long.MAX_VALUE;
        for (long[] line : lines) {
            long y = line[0] * x + line[1];
            best = Math.min(best, y);
        }
        return best;
    }
}
```

### Example

```
Min value at x=-2: -1
Min value at x=0: 1
Min value at x=3: 1
```

At `x=-2`: line `y=2x+3` gives `-1`, the smallest among all three lines at that point — exactly what a real hull-based CHT structure would return in O(log n) instead of this O(n) scan.

## 7. Divide and Conquer Optimization (Advanced)

### Theory

Applies to DP transitions of the form `dp[k][j] = min over i<j of (dp[k-1][i] + cost(i, j))`, **when the optimal split point `opt(j)` is monotonically non-decreasing as `j` increases** (true whenever `cost` satisfies the quadrangle inequality — a technical condition met by many natural cost functions, like sum-of-squares of a contiguous group). Instead of trying every `i` for every `j` (O(n²) per DP layer), recursively solve for the **middle** `j` first, find its optimal `i`, then recurse on the left and right halves of `j` — each restricted to search only between the known boundaries — cutting the per-layer cost to **O(n log n)**.

**Classic example:** partition an array into exactly `m` contiguous groups to minimize the **sum of squares** of each group's sum.

### Code

```java
public class DivideAndConquerOptimization {
    static int[] prefix;
    static long[] dpPrev, dpCur;

    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("Minimum sum of squares, 3 groups: " + minCostPartition(arr, 3));
    }

    static long minCostPartition(int[] arr, int m) {
        int n = arr.length;
        prefix = new int[n + 1];
        for (int i = 0; i < n; i++) prefix[i + 1] = prefix[i] + arr[i];

        dpPrev = new long[n + 1];
        for (int j = 0; j <= n; j++) dpPrev[j] = cost(0, j); // k = 1 group

        for (int k = 2; k <= m; k++) {
            dpCur = new long[n + 1];
            java.util.Arrays.fill(dpCur, Long.MAX_VALUE);
            compute(k - 1, n, 0, n);
            dpPrev = dpCur;
        }
        return dpPrev[n];
    }

    static long cost(int i, int j) {
        long sum = prefix[j] - prefix[i];
        return sum * sum;
    }

    // Solves dpCur[j] for j in [lo, hi], knowing the optimal split point lies in [optLo, optHi]
    static void compute(int lo, int hi, int optLo, int optHi) {
        if (lo > hi) return;
        int mid = (lo + hi) / 2;

        long bestVal = Long.MAX_VALUE;
        int bestOpt = optLo;
        for (int i = optLo; i <= Math.min(mid, optHi); i++) {
            if (dpPrev[i] == Long.MAX_VALUE) continue;
            long val = dpPrev[i] + cost(i, mid);
            if (val < bestVal) {
                bestVal = val;
                bestOpt = i;
            }
        }
        dpCur[mid] = bestVal;

        compute(lo, mid - 1, optLo, bestOpt);
        compute(mid + 1, hi, bestOpt, optHi);
    }
}
```

### Example

```
Minimum sum of squares, 3 groups: 77
```

Splitting `[1,2,3,4,5]` into 3 groups as `{1,2}, {3,4}, {5}` gives sums `3, 7, 5`, and `3²+7²+5² = 9+49+25 = 83`... the D&C search finds the true optimum `{1,2,3},{4},{5}` → sums `6,4,5` → `36+16+25=77`, the minimum achievable.

## Quick Reference Cheat Sheet

| Technique | Combines | Complexity |
|---|---|---|
| Binary Search + Prefix | monotonic prefix sums + binary search per start index | O(n log n) |
| Sliding Window + Prefix | `atMost(K) - atMost(K-1)` subtraction trick | O(n) |
| Greedy + Prefix | unconditional baseline sum + greedy sliding-window improvement | O(n) |
| Heap + Prefix | top-k heap applied directly to prefix values | O(n log k) |
| Monotonic Queue Optimization | monotonic deque powering an O(1)-amortized DP transition | O(n) |
| Convex Hull Trick | maintain a lower envelope of lines for min/max-of-linear-functions DP | O(log n) or O(1) amortized (hull-maintained version) |
| Divide and Conquer Optimization | exploit monotonic optimal split points in a DP layer | O(n log n) per layer |
