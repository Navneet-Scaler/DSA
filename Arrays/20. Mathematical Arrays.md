## 4. Contribution Technique

### Theory

Many "sum over all subarrays" problems look like they need O(n²) or O(n³) — enumerate every subarray, compute something, add it up. The **Contribution Technique** flips the perspective: instead of asking "what's the answer for this subarray?", ask **"how many subarrays does this individual element affect, and by how much?"** — then sum each element's total contribution directly. This turns many such problems into **O(n)**.

### Problem 1 (Basic): Sum of All Subarray Sums

**Theory:** Given an array, sum the sums of **every** contiguous subarray. Instead of generating every subarray, notice that element `arr[i]` appears in exactly `(i+1) × (n-i)` subarrays — `(i+1)` choices for where the subarray starts (anywhere from `0` to `i`) times `(n-i)` choices for where it ends (anywhere from `i` to `n-1`). Sum each element's value times its contribution count.

```java
public class SumOfAllSubarraySums {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        System.out.println("Sum of all subarray sums: " + sumOfSubarrays(arr));
    }

    static long sumOfSubarrays(int[] arr) {
        int n = arr.length;
        long total = 0;
        for (int i = 0; i < n; i++) {
            long contributionCount = (long) (i + 1) * (n - i);
            total += arr[i] * contributionCount;
        }
        return total;
    }
}
```

```
Sum of all subarray sums: 20
```

The six subarrays of `[1,2,3]` — `[1],[2],[3],[1,2],[2,3],[1,2,3]` — sum to `1+2+3+3+5+6 = 20`, matching the O(n) contribution-based computation.

### Problem 2 (Advanced): Sum of Subarray Minimums

**Theory:** Sum the **minimum** of every contiguous subarray — a much harder version, since "minimum" isn't as simple to decompose as "sum" is. Element `arr[i]` is the minimum of a subarray exactly when that subarray's boundaries stay between `arr[i]`'s **nearest smaller element to the left** and its **nearest smaller-or-equal element to the right** (the asymmetric `<` vs `<=` choice avoids double-counting equal elements). Both distances come from a [monotonic stack](<./15. Monotonic Structures.md>) — combining Lesson 15's technique with this lesson's contribution counting.

```java
public class SumOfSubarrayMinimums {
    public static void main(String[] args) {
        int[] arr = { 3, 1, 2, 4 };
        System.out.println("Sum of subarray minimums: " + sumSubarrayMins(arr));
    }

    static long sumSubarrayMins(int[] arr) {
        int n = arr.length;
        int[] prevLess = new int[n];       // distance to nearest strictly-smaller element on the left
        int[] nextLessEq = new int[n];     // distance to nearest smaller-or-equal element on the right

        java.util.Deque<Integer> stack = new java.util.ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && arr[stack.peek()] >= arr[i]) stack.pop();
            prevLess[i] = stack.isEmpty() ? (i + 1) : (i - stack.peek());
            stack.push(i);
        }

        stack.clear();
        for (int i = n - 1; i >= 0; i--) {
            while (!stack.isEmpty() && arr[stack.peek()] > arr[i]) stack.pop();
            nextLessEq[i] = stack.isEmpty() ? (n - i) : (stack.peek() - i);
            stack.push(i);
        }

        long total = 0;
        long mod = 1_000_000_007L;
        for (int i = 0; i < n; i++) {
            total = (total + (long) arr[i] * prevLess[i] % mod * nextLessEq[i]) % mod;
        }
        return total;
    }
}
```

```
Sum of subarray minimums: 17
```

For `[3,1,2,4]`: the `10` subarrays and their minimums are `3,1,2,4,1,1,2,1,1,1` — summing to `17`, matching the O(n) contribution-based computation via the monotonic stack.

### Problem 3 (Advanced): Sum of Subarray Ranges

**Theory:** Sum the **range** (max − min) of every contiguous subarray — a direct combination of Problems 1 and 2's ideas: `sum of ranges = (sum of subarray maximums) − (sum of subarray minimums)`. Reuse the exact same monotonic-stack contribution technique from Problem 2 twice — once finding each element's contribution as a maximum, once as a minimum — and subtract.

```java
public class SumOfSubarrayRanges {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        System.out.println("Sum of subarray ranges: " + subArrayRanges(arr));
    }

    static long subArrayRanges(int[] arr) {
        return sumOfExtremes(arr, true) - sumOfExtremes(arr, false);
    }

    // isMax = true computes sum of subarray maximums; false computes sum of subarray minimums
    static long sumOfExtremes(int[] arr, boolean isMax) {
        int n = arr.length;
        int[] prevBound = new int[n];
        int[] nextBound = new int[n];

        java.util.Deque<Integer> stack = new java.util.ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            while (!stack.isEmpty() && compare(arr[stack.peek()], arr[i], isMax) < 0) stack.pop();
            prevBound[i] = stack.isEmpty() ? (i + 1) : (i - stack.peek());
            stack.push(i);
        }

        stack.clear();
        for (int i = n - 1; i >= 0; i--) {
            while (!stack.isEmpty() && compare(arr[stack.peek()], arr[i], isMax) <= 0) stack.pop();
            nextBound[i] = stack.isEmpty() ? (n - i) : (stack.peek() - i);
            stack.push(i);
        }

        long total = 0;
        for (int i = 0; i < n; i++) {
            total += (long) arr[i] * prevBound[i] * nextBound[i];
        }
        return total;
    }

    // For isMax: returns positive if a > b (a stays on the stack as the bigger max-candidate)
    // For isMin: returns positive if a < b (a stays on the stack as the smaller min-candidate)
    static int compare(int a, int b, boolean isMax) {
        return isMax ? Integer.compare(a, b) : Integer.compare(b, a);
    }
}
```

```
Sum of subarray ranges: 4
```

The six subarrays of `[1,2,3]` have ranges `0,0,0,1,1,2` — summing to `4`, matching `(sum of maximums) - (sum of minimums)` computed via two contribution passes.
