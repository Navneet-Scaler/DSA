# 13. Kadane Family

Kadane's Algorithm answers "what's the best contiguous subarray?" in a single **O(n)** pass, by tracking a running best-so-far and deciding, at every element, whether to extend the current run or start fresh. This lesson covers the maximum and minimum variants, the circular-array extension, and an alternate prefix-sum-based derivation.

## 1. Maximum Subarray

### Theory

At each element, decide: does extending the **previous** subarray (add this element to it) beat **starting fresh** at this element alone? Whichever is larger becomes the new running sum; track the best running sum seen at any point.

```
current = max(arr[i], current + arr[i])
best = max(best, current)
```

### Code

```java
public class MaximumSubarray {
    public static void main(String[] args) {
        int[] arr = { -2, 1, -3, 4, -1, 2, 1, -5, 4 };
        System.out.println("Maximum subarray sum: " + maxSubarray(arr));
    }

    static int maxSubarray(int[] arr) {
        int best = arr[0];
        int current = arr[0];
        for (int i = 1; i < arr.length; i++) {
            current = Math.max(arr[i], current + arr[i]);
            best = Math.max(best, current);
        }
        return best;
    }
}
```

### Example

```
Maximum subarray sum: 6
```

`[4, -1, 2, 1]` sums to `6` — the best contiguous run in the array.

## 2. Minimum Subarray

### Theory

The mirror image: at each step, take the **smaller** of extending or restarting, tracking the smallest running sum seen.

### Code

```java
public class MinimumSubarray {
    public static void main(String[] args) {
        int[] arr = { -2, 1, -3, 4, -1, 2, 1, -5, 4 };
        System.out.println("Minimum subarray sum: " + minSubarray(arr));
    }

    static int minSubarray(int[] arr) {
        int best = arr[0];
        int current = arr[0];
        for (int i = 1; i < arr.length; i++) {
            current = Math.min(arr[i], current + arr[i]);
            best = Math.min(best, current);
        }
        return best;
    }
}
```

### Example

```
Minimum subarray sum: -5
```

The single element `-5` is worse (more negative) than any combination involving it.

## 3. Circular Maximum

### Theory

In a **circular** array, a subarray can wrap from the end back to the start. There are exactly two cases to consider:

1. The best subarray **doesn't wrap** — that's just the ordinary [Maximum Subarray](#1-maximum-subarray).
2. The best subarray **does wrap** — equivalent to removing some *middle* chunk from the array and keeping both ends, which is the same as `(total sum) - (minimum subarray sum)`.

Take the larger of the two — **except** when every element is negative, in which case the "wrapping" case would incorrectly suggest an empty (all-elements-removed) subarray summing to `0`; guard against this by falling back to the plain maximum subarray whenever it's negative.

### Code

```java
public class CircularMaximum {
    public static void main(String[] args) {
        System.out.println("Circular max [5,-3,5]: " + maxSubarraySumCircular(new int[] { 5, -3, 5 }));
        System.out.println("Circular max all-negative [-3,-2,-3]: "
                + maxSubarraySumCircular(new int[] { -3, -2, -3 }));
    }

    static int maxSubarraySumCircular(int[] arr) {
        int total = 0;
        for (int value : arr) total += value;

        int maxNormal = maxSubarray(arr);
        int minNormal = minSubarray(arr);

        if (maxNormal < 0) return maxNormal; // all elements negative -- wrapping case is invalid here

        int maxWrap = total - minNormal;
        return Math.max(maxNormal, maxWrap);
    }

    static int maxSubarray(int[] arr) {
        int best = arr[0], current = arr[0];
        for (int i = 1; i < arr.length; i++) {
            current = Math.max(arr[i], current + arr[i]);
            best = Math.max(best, current);
        }
        return best;
    }

    static int minSubarray(int[] arr) {
        int best = arr[0], current = arr[0];
        for (int i = 1; i < arr.length; i++) {
            current = Math.min(arr[i], current + arr[i]);
            best = Math.min(best, current);
        }
        return best;
    }
}
```

### Example

```
Circular max [5,-3,5]: 10
Circular max all-negative [-3,-2,-3]: -2
```

`[5,-3,5]` wraps: `5 + 5 = 10` by taking both ends and excluding the middle `-3`. In the all-negative case, the fallback correctly returns `-2` (the least-bad single element) rather than an invalid `0`.

## 4. Prefix Variant

### Theory

Maximum subarray sum can also be derived directly from [prefix sums](<./05. Prefix Techniques.md#1-prefix-sum>) without Kadane's running-sum reframing: the best subarray ending at index `i` is `prefix[i] - (smallest prefix sum seen before i)` — subtracting off the lowest point still leaves the largest possible "rise" from that low point to `i`. This is mathematically the same algorithm as Kadane's, viewed through a different lens.

### Code

```java
public class PrefixVariant {
    public static void main(String[] args) {
        int[] arr = { -2, 1, -3, 4, -1, 2, 1, -5, 4 };
        System.out.println("Maximum subarray via prefix variant: " + maxSubarrayPrefix(arr));
    }

    static int maxSubarrayPrefix(int[] arr) {
        int prefix = 0;
        int minPrefix = 0;
        int best = arr[0];

        for (int value : arr) {
            prefix += value;
            best = Math.max(best, prefix - minPrefix);
            minPrefix = Math.min(minPrefix, prefix);
        }
        return best;
    }
}
```

### Example

```
Maximum subarray via prefix variant: 6
```

Same answer as [Maximum Subarray](#1-maximum-subarray) — confirming these are two derivations of the same result.

## 5. Maximum Product Subarray

### Theory

Extending Kadane's idea to **products** breaks the "just extend or restart" logic, because multiplying by a **negative** number flips a large positive running product into a large negative one — and vice versa. The fix: track **both** a running max **and** a running min product at every step; when the current element is negative, the previous min (likely very negative) becomes the new max after multiplying, so swap the roles before updating.

### Code

```java
public class MaximumProductSubarray {
    public static void main(String[] args) {
        System.out.println("Max product [2,3,-2,4]: " + maxProduct(new int[] { 2, 3, -2, 4 }));
        System.out.println("Max product [-2,0,-1]: " + maxProduct(new int[] { -2, 0, -1 }));
    }

    static int maxProduct(int[] nums) {
        int maxProd = nums[0], minProd = nums[0], result = nums[0];

        for (int i = 1; i < nums.length; i++) {
            int n = nums[i];
            if (n < 0) {
                int temp = maxProd;
                maxProd = minProd;
                minProd = temp;
            }
            maxProd = Math.max(n, maxProd * n);
            minProd = Math.min(n, minProd * n);
            result = Math.max(result, maxProd);
        }
        return result;
    }
}
```

### Example

```
Max product [2,3,-2,4]: 6
Max product [-2,0,-1]: 0
```

`[2,3]` gives the max product `6` in the first case (including `-2` or `4` would tank it). The `0` in the second array resets everything, so the best achievable is `0` itself.

## 6. Best Time to Buy and Sell Stock

### Theory

A close cousin of Kadane's Algorithm dressed up as a finance problem: given daily prices, find the maximum profit from **one** buy followed by **one** later sell. Track the **minimum price seen so far** while scanning forward; at each day, the best possible profit if selling *today* is `price[i] - minSoFar`. This is really [Maximum Subarray](#1-maximum-subarray) applied to the array of **day-to-day price differences** — but tracking a running minimum directly is simpler and avoids building that differences array at all.

### Code

```java
public class BestTimeToBuyAndSellStock {
    public static void main(String[] args) {
        int[] prices = { 7, 1, 5, 3, 6, 4 };
        System.out.println("Max profit: " + maxProfit(prices));
    }

    static int maxProfit(int[] prices) {
        int minPriceSoFar = Integer.MAX_VALUE;
        int best = 0;

        for (int price : prices) {
            minPriceSoFar = Math.min(minPriceSoFar, price);
            best = Math.max(best, price - minPriceSoFar);
        }
        return best;
    }
}
```

### Example

```
Max profit: 5
```

Buy at `1` (day 2), sell at `6` (day 5): profit `5` — the best achievable single buy-sell pair.

## Quick Reference Cheat Sheet

| Variant | Core idea | Complexity |
|---|---|---|
| Maximum Subarray | `current = max(arr[i], current + arr[i])` | O(n) |
| Minimum Subarray | `current = min(arr[i], current + arr[i])` | O(n) |
| Circular Maximum | `max(normal max, total - normal min)`, guard all-negative case | O(n) |
| Prefix Variant | `best subarray ending at i = prefix[i] - min(prefix before i)` | O(n) |
| Maximum Product Subarray | track running max **and** min, swap on negative multiplier | O(n) |
| Best Time to Buy and Sell Stock | track running min price, best = `price - minSoFar` | O(n) |
