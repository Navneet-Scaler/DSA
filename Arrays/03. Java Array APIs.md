# 3. Java Array APIs

Java's `java.util.Arrays` and `System` classes provide battle-tested implementations of the operations from [Lesson 2](<./02. Basic Operations.md>) — sorting, filling, copying, searching, comparing, and printing. Prefer these over hand-rolled loops whenever they fit; they're well-tested and usually faster than a naive implementation.

## 1. Arrays.sort()

### Theory

`Arrays.sort(arr)` sorts in place, ascending. For primitives it uses a **Dual-Pivot Quicksort** variant (O(n log n) average); for object arrays it uses a stable **Timsort** (merge-sort-based), since object arrays might need a custom order via `Comparator` — see [Lesson 9: Custom Comparator](<./09. Sorting Based Arrays.md#4-custom-comparator>).

### Code

```java
import java.util.Arrays;

public class ArraysSort {
    public static void main(String[] args) {
        int[] arr = { 5, 2, 8, 1, 9 };
        Arrays.sort(arr);
        System.out.println("Sorted: " + Arrays.toString(arr));
    }
}
```

### Example

```
Sorted: [1, 2, 5, 8, 9]
```

## 2. Arrays.fill()

### Theory

`Arrays.fill(arr, value)` sets **every** element to the same value in one call — useful for initializing a DP array to a sentinel like `-1` or `Integer.MAX_VALUE`.

### Code

```java
import java.util.Arrays;

public class ArraysFill {
    public static void main(String[] args) {
        int[] arr = new int[5];
        Arrays.fill(arr, -1);
        System.out.println("Filled with -1: " + Arrays.toString(arr));
    }
}
```

### Example

```
Filled with -1: [-1, -1, -1, -1, -1]
```

## 3. Arrays.copyOf()

### Theory

`Arrays.copyOf(arr, newLength)` returns a **new array** of the given length, copying elements from the start — truncating if `newLength` is smaller, or padding with default values if larger. This is the built-in version of the manual [Resize](<./02. Basic Operations.md#12-resize>) operation.

### Code

```java
import java.util.Arrays;

public class ArraysCopyOf {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        System.out.println("copyOf(arr, 5): " + Arrays.toString(Arrays.copyOf(arr, 5)));
        System.out.println("copyOf(arr, 2): " + Arrays.toString(Arrays.copyOf(arr, 2)));
    }
}
```

### Example

```
copyOf(arr, 5): [1, 2, 3, 0, 0]
copyOf(arr, 2): [1, 2]
```

## 4. Arrays.copyOfRange()

### Theory

`Arrays.copyOfRange(arr, from, to)` copies a **sub-range** `[from, to)` — `from` inclusive, `to` exclusive — into a new array. This is the built-in version of the manual [Split](<./02. Basic Operations.md#11-split>) operation.

### Code

```java
import java.util.Arrays;

public class ArraysCopyOfRange {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3, 4, 5 };
        System.out.println("copyOfRange(arr, 1, 4): " + Arrays.toString(Arrays.copyOfRange(arr, 1, 4)));
    }
}
```

### Example

```
copyOfRange(arr, 1, 4): [2, 3, 4]
```

## 5. Arrays.binarySearch()

### Theory

`Arrays.binarySearch(arr, key)` requires a **sorted** array. If found, it returns the index. If not found, it returns a **negative** value encoding where the key *would* go: `-(insertionPoint) - 1`. This lets you recover the insertion point with `-(result) - 1` when the search misses.

### Code

```java
import java.util.Arrays;

public class ArraysBinarySearch {
    public static void main(String[] args) {
        int[] arr = { 1, 3, 5, 7, 9 };

        System.out.println("binarySearch(5): " + Arrays.binarySearch(arr, 5));
        int missResult = Arrays.binarySearch(arr, 4);
        System.out.println("binarySearch(4) [not found]: " + missResult);
        System.out.println("Recovered insertion point: " + (-missResult - 1));
    }
}
```

### Example

```
binarySearch(5): 2
binarySearch(4) [not found]: -3
Recovered insertion point: 2
```

`4` isn't in the array, but it would be inserted at index `2` (between `3` and `5`) to keep it sorted.

## 6. Arrays.equals()

### Theory

`Arrays.equals(a, b)` checks whether two arrays have the **same length and same elements in the same order** — `==` on two arrays only checks if they're the *same object reference*, which is almost never what you want.

### Code

```java
import java.util.Arrays;

public class ArraysEquals {
    public static void main(String[] args) {
        int[] a = { 1, 2, 3 };
        int[] b = { 1, 2, 3 };

        System.out.println("a == b (reference equality): " + (a == b));
        System.out.println("Arrays.equals(a, b) (content equality): " + Arrays.equals(a, b));
    }
}
```

### Example

```
a == b (reference equality): false
Arrays.equals(a, b) (content equality): true
```

## 7. Arrays.deepEquals()

### Theory

`Arrays.equals()` only compares one level deep — for a 2D array (an array of array references), it would compare the inner arrays by reference, not content. `Arrays.deepEquals()` recurses into nested arrays, comparing their actual contents.

### Code

```java
import java.util.Arrays;

public class ArraysDeepEquals {
    public static void main(String[] args) {
        int[][] a = { { 1, 2 }, { 3, 4 } };
        int[][] b = { { 1, 2 }, { 3, 4 } };

        System.out.println("Arrays.equals(a, b) [shallow]: " + Arrays.equals(a, b));
        System.out.println("Arrays.deepEquals(a, b) [recursive]: " + Arrays.deepEquals(a, b));
    }
}
```

### Example

```
Arrays.equals(a, b) [shallow]: false
Arrays.deepEquals(a, b) [recursive]: true
```

`Arrays.equals` sees two different inner-array objects (even though their contents match) and reports `false`; `Arrays.deepEquals` looks inside and correctly reports `true`.

## 8. Arrays.toString()

### Theory

`Arrays.toString(arr)` produces a readable `[a, b, c]` representation for a **1D** array — printing an array directly with `System.out.println(arr)` instead gives a useless type-and-hashcode string, since arrays don't override `toString()`.

### Code

```java
import java.util.Arrays;

public class ArraysToString {
    public static void main(String[] args) {
        int[] arr = { 1, 2, 3 };
        System.out.println("Direct print (unhelpful): " + arr);
        System.out.println("Arrays.toString: " + Arrays.toString(arr));
    }
}
```

### Example

```
Direct print (unhelpful): [I@1b6d3586
Arrays.toString: [1, 2, 3]
```

> The exact hash portion (`1b6d3586`) will differ on every run — the point is that direct printing is never useful, only `Arrays.toString` is.

## 9. Arrays.deepToString()

### Theory

`Arrays.toString()` on a 2D array prints each row's *reference* string, not its contents — same shallow problem as `Arrays.equals()`. `Arrays.deepToString()` recurses, printing nested arrays' actual contents.

### Code

```java
import java.util.Arrays;

public class ArraysDeepToString {
    public static void main(String[] args) {
        int[][] grid = { { 1, 2 }, { 3, 4 } };
        System.out.println("Arrays.toString [shallow]: " + Arrays.toString(grid));
        System.out.println("Arrays.deepToString [recursive]: " + Arrays.deepToString(grid));
    }
}
```

### Example

```
Arrays.toString [shallow]: [[I@1b6d3586, [I@4554617c]
Arrays.deepToString [recursive]: [[1, 2], [3, 4]]
```

## 10. System.arraycopy()

### Theory

`System.arraycopy(src, srcPos, dest, destPos, length)` copies elements between arrays (or within the same array) very efficiently — it's the native method that `Arrays.copyOf` and `Arrays.copyOfRange` are built on top of internally. Useful when you need to copy **into an existing array** at a specific offset, rather than getting a brand-new array back.

### Code

```java
import java.util.Arrays;

public class SystemArraycopy {
    public static void main(String[] args) {
        int[] src = { 1, 2, 3, 4, 5 };
        int[] dest = new int[5];

        System.arraycopy(src, 1, dest, 0, 3); // copy 3 elements starting at src[1] into dest[0..]
        System.out.println("dest after arraycopy: " + Arrays.toString(dest));
    }
}
```

### Example

```
dest after arraycopy: [2, 3, 4, 0, 0]
```

## Quick Reference Cheat Sheet

| Method | Purpose |
|---|---|
| `Arrays.sort(arr)` | in-place ascending sort |
| `Arrays.fill(arr, v)` | set every element to `v` |
| `Arrays.copyOf(arr, n)` | new array of length `n`, truncated or padded |
| `Arrays.copyOfRange(arr, from, to)` | new array from a sub-range |
| `Arrays.binarySearch(arr, key)` | index if found, else `-(insertionPoint)-1` |
| `Arrays.equals(a, b)` | shallow content equality (1D) |
| `Arrays.deepEquals(a, b)` | recursive content equality (nested arrays) |
| `Arrays.toString(arr)` | readable string, 1D only |
| `Arrays.deepToString(arr)` | readable string, recursive into nested arrays |
| `System.arraycopy(...)` | low-level, fast copy into an existing array |
