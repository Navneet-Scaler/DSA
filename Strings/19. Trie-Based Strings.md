# 19. Trie-Based Strings

A **trie** (prefix tree) stores a set of strings as a tree where each path from the root spells out a prefix — shared prefixes share nodes, so lookups, prefix checks, and dictionary-style problems all become **O(word length)** instead of scanning every stored word. This lesson builds the data structure itself, then works through the classic problems it unlocks, closing with a trie-powered upgrade to [Lesson 18's Word Search](<./18. Backtracking.md#5-word-search>).

## 1. Trie Implementation

### Theory

**Classic example — Implement Trie (Prefix Tree):** support `insert`, `search` (exact word match), and `startsWith` (prefix match). Each node holds a map of `character → child node` and a boolean flag marking whether a **complete word** ends there (distinguishing `"app"` as a prefix of `"apple"` from `"app"` being a word in its own right). `insert` walks/creates nodes character by character, then marks the final node as a word-end; `search` and `startsWith` both walk the same path, differing only in whether they also check the final node's word-end flag.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class Trie {
    static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        boolean isEnd = false;
    }

    private final TrieNode root = new TrieNode();

    public static void main(String[] args) {
        Trie trie = new Trie();
        trie.insert("apple");
        System.out.println("search('apple'): " + trie.search("apple"));
        System.out.println("search('app'): " + trie.search("app"));
        System.out.println("startsWith('app'): " + trie.startsWith("app"));
    }

    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node = node.children.computeIfAbsent(c, k -> new TrieNode());
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        TrieNode node = find(word);
        return node != null && node.isEnd;
    }

    public boolean startsWith(String prefix) {
        return find(prefix) != null;
    }

    private TrieNode find(String s) {
        TrieNode node = root;
        for (char c : s.toCharArray()) {
            node = node.children.get(c);
            if (node == null) return null;
        }
        return node;
    }
}
```

### Example

```
search('apple'): true
search('app'): false
startsWith('app'): true
```

`"app"` was never inserted as a complete word — only as a prefix of `"apple"` — so `search` correctly returns `false` while `startsWith` returns `true`.

> **Java idiom:** `node.children.computeIfAbsent(c, k -> new TrieNode())` replaces the "check if child exists, create if not, descend" three-line pattern with one line.

## 2. Wildcard Search

### Theory

**Classic example — Design Add and Search Words Data Structure:** support adding words and searching, where the search pattern may include `.` to match **any single character**. A plain trie lookup breaks down at a `.`, since there's no single child to follow — instead, **branch into every child** at that position and recursively try each one, succeeding if **any** branch completes the match. This turns the search from a simple walk into a DFS over the trie.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class WordDictionary {
    static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        boolean isEnd = false;
    }

    private final TrieNode root = new TrieNode();

    public static void main(String[] args) {
        WordDictionary wd = new WordDictionary();
        wd.addWord("bad");
        wd.addWord("dad");
        wd.addWord("mad");
        System.out.println("search('pad'): " + wd.search("pad"));
        System.out.println("search('bad'): " + wd.search("bad"));
        System.out.println("search('.ad'): " + wd.search(".ad"));
        System.out.println("search('b..'): " + wd.search("b.."));
    }

    public void addWord(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node = node.children.computeIfAbsent(c, k -> new TrieNode());
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        return dfs(root, word, 0);
    }

    private boolean dfs(TrieNode node, String word, int index) {
        if (index == word.length()) return node.isEnd;

        char c = word.charAt(index);
        if (c == '.') {
            for (TrieNode child : node.children.values()) {
                if (dfs(child, word, index + 1)) return true;
            }
            return false;
        } else {
            TrieNode child = node.children.get(c);
            return child != null && dfs(child, word, index + 1);
        }
    }
}
```

### Example

```
search('pad'): false
search('bad'): true
search('.ad'): true
search('b..'): true
```

`".ad"` matches `"bad"`, `"dad"`, or `"mad"` — the wildcard branches into all three first-letter children and finds a match. `"b.."` matches `"bad"` by wildcarding both remaining positions.

## 3. Replace Words

### Theory

**Classic example — Replace Words:** given a dictionary of word "roots" and a sentence, replace every word in the sentence with the **shortest root** that is a prefix of it (if any root matches). Insert every root into a trie; for each sentence word, walk the trie character by character, and stop **as soon as** a node marked `isEnd` is reached — that's the shortest matching root by construction, since the walk proceeds in prefix order.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class ReplaceWords {
    static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        boolean isEnd = false;
    }

    private final TrieNode root = new TrieNode();

    public static void main(String[] args) {
        ReplaceWords rw = new ReplaceWords();
        System.out.println("Result: " + rw.replaceWords(
                java.util.List.of("cat", "bat", "rat"),
                "the cattle was rattled by the battery"));
    }

    public String replaceWords(java.util.List<String> dictionary, String sentence) {
        for (String rootWord : dictionary) insert(rootWord);

        StringBuilder result = new StringBuilder();
        for (String word : sentence.split(" ")) {
            if (result.length() > 0) result.append(' ');
            result.append(findRoot(word));
        }
        return result.toString();
    }

    private void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node = node.children.computeIfAbsent(c, k -> new TrieNode());
        }
        node.isEnd = true;
    }

    private String findRoot(String word) {
        TrieNode node = root;
        StringBuilder prefix = new StringBuilder();
        for (char c : word.toCharArray()) {
            node = node.children.get(c);
            if (node == null) return word; // no root matches -- keep the original word
            prefix.append(c);
            if (node.isEnd) return prefix.toString();
        }
        return word;
    }
}
```

### Example

```
Result: the cat was rat by the bat
```

`"cattle"` → shortest matching root `"cat"`; `"rattled"` → `"rat"`; `"battery"` → `"bat"` — each replaced word stops at the first complete root found while walking, guaranteeing the shortest one.

## 4. Longest Word in Dictionary

### Theory

**Classic example — Longest Word in Dictionary:** find the longest word that can be **built one character at a time** by other words already in the list (every prefix of the word, down to length 1, must also be a complete word in the dictionary). Insert all words into a trie; for each word, walk its path and confirm **every node along the way** (not just the final one) is itself marked as a complete word — that's exactly the "buildable one letter at a time" condition. Among all qualifying words, keep the longest, breaking ties by lexicographically smallest.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class LongestWordInDictionary {
    static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        boolean isEnd = false;
    }

    private final TrieNode root = new TrieNode();

    public static void main(String[] args) {
        LongestWordInDictionary solver = new LongestWordInDictionary();
        System.out.println("longestWord: " + solver.longestWord(
                new String[] { "w", "wo", "wor", "worl", "world" }));

        LongestWordInDictionary solver2 = new LongestWordInDictionary();
        System.out.println("longestWord: " + solver2.longestWord(
                new String[] { "a", "banana", "app", "appl", "ap", "apply", "apple" }));
    }

    public String longestWord(String[] words) {
        for (String word : words) insert(word);

        String best = "";
        for (String word : words) {
            if (allPrefixesExist(word)) {
                if (word.length() > best.length()
                        || (word.length() == best.length() && word.compareTo(best) < 0)) {
                    best = word;
                }
            }
        }
        return best;
    }

    private void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node = node.children.computeIfAbsent(c, k -> new TrieNode());
        }
        node.isEnd = true;
    }

    private boolean allPrefixesExist(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            node = node.children.get(c);
            if (node == null || !node.isEnd) return false;
        }
        return true;
    }
}
```

### Example

```
longestWord: world
longestWord: apple
```

`"world"` is buildable: `"w"`, `"wo"`, `"wor"`, `"worl"`, `"world"` are all present. `"apple"` beats `"apply"` (same length) lexicographically, and beats `"banana"` despite `"banana"` being longer, since `"banana"`'s prefixes (`"b"`, `"ba"`, ...) aren't all present in the dictionary.

## 5. Word Search II

### Theory

**Classic example — Word Search II (advanced):** given a grid and a **list** of words (not just one, as in [Lesson 18's Word Search](<./18. Backtracking.md#5-word-search>)), find all words that can be traced through adjacent cells. Running Lesson 18's single-word backtracking search once per word would repeatedly re-walk the same grid paths. Instead, insert **all** words into one trie, then do a **single** DFS pass over the grid: at each cell, only descend into the trie if a matching child exists — this prunes entire branches the moment no remaining word could possibly match, and finds all words in one combined traversal. Marking a found word's trie node (e.g., removing it) avoids reporting the same word twice via a different path.

### Code

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class WordSearchII {
    static class TrieNode {
        Map<Character, TrieNode> children = new HashMap<>();
        String word = null; // non-null exactly at nodes completing one of the target words
    }

    public static void main(String[] args) {
        char[][] board = {
                { 'o', 'a', 'a', 'n' },
                { 'e', 't', 'a', 'e' },
                { 'i', 'h', 'k', 'r' },
                { 'i', 'f', 'l', 'v' }
        };
        String[] words = { "oath", "pea", "eat", "rain" };
        System.out.println("findWords: " + findWords(board, words));
    }

    static List<String> findWords(char[][] board, String[] words) {
        TrieNode root = new TrieNode();
        for (String word : words) {
            TrieNode node = root;
            for (char c : word.toCharArray()) {
                node = node.children.computeIfAbsent(c, k -> new TrieNode());
            }
            node.word = word;
        }

        List<String> result = new ArrayList<>();
        int rows = board.length, cols = board[0].length;
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                dfs(board, r, c, root, result);
            }
        }
        return result;
    }

    static void dfs(char[][] board, int r, int c, TrieNode node, List<String> result) {
        if (r < 0 || r >= board.length || c < 0 || c >= board[0].length || board[r][c] == '#') return;

        char ch = board[r][c];
        TrieNode next = node.children.get(ch);
        if (next == null) return;

        if (next.word != null) {
            result.add(next.word);
            next.word = null; // avoid duplicate reporting
        }

        board[r][c] = '#';
        dfs(board, r + 1, c, next, result);
        dfs(board, r - 1, c, next, result);
        dfs(board, r, c + 1, next, result);
        dfs(board, r, c - 1, next, result);
        board[r][c] = ch;
    }
}
```

### Example

```
findWords: [eat, oath]
```

`"pea"` and `"rain"` aren't traceable through this grid, so only `"eat"` and `"oath"` are found — both discovered in a **single** combined DFS pass, rather than one separate search per word.
