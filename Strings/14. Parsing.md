## 1. Reverse Words in a String

### Theory

**Classic example — Reverse Words in a String:** given a sentence with possibly irregular spacing (leading, trailing, or multiple spaces between words), reverse the **order of the words** (not the characters within each word), collapsing all whitespace down to single spaces. Java's `String.split` with a regex for one-or-more whitespace characters handles the messy-whitespace tokenizing directly.

### Code

```java
import java.util.Arrays;
import java.util.Collections;
import java.util.List;

public class ReverseWords {
    public static void main(String[] args) {
        System.out.println("'" + reverseWords("  hello world  ") + "'");
        System.out.println("'" + reverseWords("the sky is blue") + "'");
    }

    static String reverseWords(String s) {
        String[] words = s.trim().split("\\s+");
        Collections.reverse(Arrays.asList(words));
        return String.join(" ", words);
    }
}
```

### Example

```
'world hello'
'blue is sky the'
```

`s.trim().split("\\s+")` strips leading/trailing whitespace and splits on any run of whitespace, so `"  hello world  "` cleanly tokenizes to `["hello", "world"]` before reversing.

> **Java idiom:** `Collections.reverse(Arrays.asList(words))` reverses an array in place via the `List` view — no manual two-pointer swap loop needed. `String.join(" ", words)` is the collapsing-inverse of `split`.

## 2. String to Integer (atoi)

### Theory

**Classic example — String to Integer (atoi):** implement the C-style `atoi` conversion: skip leading whitespace, read an optional sign, read as many digits as available, stop at the first non-digit character, and **clamp** the result to the 32-bit signed integer range (`Integer.MIN_VALUE` to `Integer.MAX_VALUE`) if it overflows. The core discipline here is following the exact rule order precisely — whitespace, then sign, then digits, then clamp — since the "stop at first invalid character" behavior (rather than rejecting the whole string) is what real `atoi` implementations do.

### Code

```java
public class StringToInteger {
    public static void main(String[] args) {
        System.out.println("myAtoi('42'): " + myAtoi("42"));
        System.out.println("myAtoi('   -42'): " + myAtoi("   -42"));
        System.out.println("myAtoi('4193 with words'): " + myAtoi("4193 with words"));
        System.out.println("myAtoi('words and 987'): " + myAtoi("words and 987"));
    }

    static int myAtoi(String s) {
        int i = 0, n = s.length();
        while (i < n && s.charAt(i) == ' ') i++;

        int sign = 1;
        if (i < n && (s.charAt(i) == '+' || s.charAt(i) == '-')) {
            if (s.charAt(i) == '-') sign = -1;
            i++;
        }

        long num = 0;
        while (i < n && Character.isDigit(s.charAt(i))) {
            num = num * 10 + (s.charAt(i) - '0');
            if (sign * num > Integer.MAX_VALUE) return Integer.MAX_VALUE;
            if (sign * num < Integer.MIN_VALUE) return Integer.MIN_VALUE;
            i++;
        }
        return (int) (sign * num);
    }
}
```

### Example

```
myAtoi('42'): 42
myAtoi('   -42'): -42
myAtoi('4193 with words'): 4193
myAtoi('words and 987'): 0
```

`"4193 with words"` stops reading digits at the space, returning `4193` rather than failing. `"words and 987"` has no leading digits at all, so the result is `0`.

## 3. Roman Numeral Conversion

### Theory

**Classic example — Roman to Integer / Integer to Roman**, two directions of the same mapping:

- **Roman to Integer:** normally, symbol values add up left to right — but a smaller-value symbol placed **before** a larger one (like `IV` = 4) means **subtract** it instead. Scan left to right, comparing each symbol's value to the next one's: if it's smaller, subtract it; otherwise add it.
- **Integer to Roman:** the reverse — greedily subtract the **largest** possible Roman value (including the subtractive pairs like `"CM"` = 900 and `"IV"` = 4, treated as single units) from the number, appending the matching symbol each time, until the number reaches zero.

### Code

```java
public class RomanNumeralConversion {
    public static void main(String[] args) {
        System.out.println("romanToInt 'III': " + romanToInt("III"));
        System.out.println("romanToInt 'LVIII': " + romanToInt("LVIII"));
        System.out.println("romanToInt 'MCMXCIV': " + romanToInt("MCMXCIV"));
        System.out.println("intToRoman 1994: " + intToRoman(1994));
        System.out.println("intToRoman 58: " + intToRoman(58));
    }

    static int romanToInt(String s) {
        java.util.Map<Character, Integer> values = java.util.Map.of(
                'I', 1, 'V', 5, 'X', 10, 'L', 50, 'C', 100, 'D', 500, 'M', 1000);

        int total = 0;
        for (int i = 0; i < s.length(); i++) {
            int value = values.get(s.charAt(i));
            if (i + 1 < s.length() && value < values.get(s.charAt(i + 1))) {
                total -= value;
            } else {
                total += value;
            }
        }
        return total;
    }

    static String intToRoman(int num) {
        int[] values = { 1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1 };
        String[] symbols = { "M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I" };

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < values.length; i++) {
            while (num >= values[i]) {
                sb.append(symbols[i]);
                num -= values[i];
            }
        }
        return sb.toString();
    }
}
```

### Example

```
romanToInt 'III': 3
romanToInt 'LVIII': 58
romanToInt 'MCMXCIV': 1994
intToRoman 1994: MCMXCIV
intToRoman 58: LVIII
```

`"MCMXCIV"`: `M`(1000) + `CM`(900, since `C` < `M`) + `XC`(90, since `X` < `C`) + `IV`(4, since `I` < `V`) = `1994` — and converting `1994` back through the greedy symbol list reproduces exactly `"MCMXCIV"`.

## 4. Expression Evaluation

### Theory

**Classic example — Basic Calculator:** evaluate a string expression containing `+`, `-`, digits, spaces, and parentheses (no multiplication/division). Track a running `result`, the current `number` being built, and a `sign` to apply once the number is complete. Parentheses are handled with a stack: on `(`, push the result-so-far and the sign that applied to the parenthesized group, then reset to evaluate the group fresh; on `)`, fold the completed group's value back into what was popped.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class BasicCalculator {
    public static void main(String[] args) {
        System.out.println("calculate('1 + 1'): " + calculate("1 + 1"));
        System.out.println("calculate(' 2-1 + 2 '): " + calculate(" 2-1 + 2 "));
        System.out.println("calculate('(1+(4+5+2)-3)+(6+8)'): " + calculate("(1+(4+5+2)-3)+(6+8)"));
    }

    static int calculate(String s) {
        Deque<Integer> stack = new ArrayDeque<>();
        int result = 0, number = 0, sign = 1;

        for (char c : s.toCharArray()) {
            if (Character.isDigit(c)) {
                number = number * 10 + (c - '0');
            } else if (c == '+' || c == '-') {
                result += sign * number;
                number = 0;
                sign = (c == '+') ? 1 : -1;
            } else if (c == '(') {
                stack.push(result);
                stack.push(sign);
                result = 0;
                sign = 1;
            } else if (c == ')') {
                result += sign * number;
                number = 0;
                result *= stack.pop(); // sign that applied before this '('
                result += stack.pop(); // result accumulated before this '('
            }
        }
        return result + sign * number;
    }
}
```

### Example

```
calculate('1 + 1'): 2
calculate(' 2-1 + 2 '): 3
calculate('(1+(4+5+2)-3)+(6+8)'): 23
```

### Problem 2 (Advanced): Basic Calculator II

**Theory:** A different variant — `+`, `-`, `*`, `/` but **no parentheses**. Since `*` and `/` bind tighter than `+` and `-`, use a stack of already-resolved terms: push a number directly for `+`, push its negation for `-`, but for `*`/`/`, **pop the previous number and combine it immediately** (since multiplication/division must happen before addition finalizes) rather than pushing a fresh value. The final answer is the sum of everything left on the stack.

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class BasicCalculatorII {
    public static void main(String[] args) {
        System.out.println("calculate('3+2*2'): " + calculate("3+2*2"));
        System.out.println("calculate(' 3/2 '): " + calculate(" 3/2 "));
        System.out.println("calculate(' 3+5 / 2 '): " + calculate(" 3+5 / 2 "));
    }

    static int calculate(String s) {
        Deque<Integer> stack = new ArrayDeque<>();
        int num = 0;
        char sign = '+';
        s = s.trim();

        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            if (Character.isDigit(c)) {
                num = num * 10 + (c - '0');
            }
            if ((!Character.isDigit(c) && c != ' ') || i == s.length() - 1) {
                switch (sign) {
                    case '+' -> stack.push(num);
                    case '-' -> stack.push(-num);
                    case '*' -> stack.push(stack.pop() * num);
                    case '/' -> stack.push(stack.pop() / num);
                }
                sign = c;
                num = 0;
            }
        }

        int result = 0;
        for (int value : stack) result += value;
        return result;
    }
}
```

```
calculate('3+2*2'): 7
calculate(' 3/2 '): 1
calculate(' 3+5 / 2 '): 5
```

`3+2*2`: the `*2` combines with the previously-pushed `2` immediately (giving `4` on the stack), so the sum is `3 + 4 = 7`, correctly respecting operator precedence without a full parser.

## 5. Valid Number

### Theory

**Classic example — Valid Number:** determine whether a string represents a valid decimal or scientific-notation number (like `"2e10"`, `"-90E3"`, `"0089"`, `".1"`), rejecting malformed ones (like `"e"`, `"1a"`, `"99e2.5"`, `"--6"`). This is naturally a **state-machine** problem: track whether a digit, a decimal point, or an exponent marker (`e`/`E`) has been seen so far, and enforce the ordering rules (only one `.`, only one `e`, no `.` after `e`, a sign only allowed at the very start or immediately after `e`, and a digit must appear somewhere before and after any `e`).

### Code

```java
public class ValidNumber {
    public static void main(String[] args) {
        String[] tests = { "0", "e", ".", "2e10", "-90E3", "1e", "99e2.5", "--6", "-+3", "95a54e53" };
        for (String t : tests) {
            System.out.println("isNumber('" + t + "'): " + isNumber(t));
        }
    }

    static boolean isNumber(String s) {
        boolean seenDigit = false, seenDot = false, seenExponent = false;

        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            if (Character.isDigit(c)) {
                seenDigit = true;
            } else if (c == '+' || c == '-') {
                if (i > 0 && s.charAt(i - 1) != 'e' && s.charAt(i - 1) != 'E') return false;
            } else if (c == '.') {
                if (seenDot || seenExponent) return false;
                seenDot = true;
            } else if (c == 'e' || c == 'E') {
                if (seenExponent || !seenDigit) return false;
                seenExponent = true;
                seenDigit = false; // require at least one digit after 'e' too
            } else {
                return false;
            }
        }
        return seenDigit;
    }
}
```

### Example

```
isNumber('0'): true
isNumber('e'): false
isNumber('.'): false
isNumber('2e10'): true
isNumber('-90E3'): true
isNumber('1e'): false
isNumber('99e2.5'): false
isNumber('--6'): false
isNumber('-+3'): false
isNumber('95a54e53'): false
```

`"1e"` fails because no digit follows the `e`; `"99e2.5"` fails because a `.` appears after `e`; `"--6"` and `"-+3"` fail because a sign only belongs at the start or right after `e`/`E`; `"95a54e53"` fails on the invalid character `a`.
