## 1. Isomorphic Strings and Word Pattern

### Theory

**Classic example — Isomorphic Strings:** two strings are isomorphic if there's a **one-to-one, bijective** mapping between characters of one and characters of the other, consistent across the whole string (`"egg"`/`"add"`: `e→a`, `g→d`). Crucially, the mapping must work **both directions** — two different source characters can never map to the same target character — so two `HashMap`s are needed (one per direction), checking both on every pair.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class IsomorphicStrings {
    public static void main(String[] args) {
        System.out.println("isIsomorphic('egg','add'): " + isIsomorphic("egg", "add"));
        System.out.println("isIsomorphic('foo','bar'): " + isIsomorphic("foo", "bar"));
        System.out.println("isIsomorphic('paper','title'): " + isIsomorphic("paper", "title"));
    }

    static boolean isIsomorphic(String s, String t) {
        if (s.length() != t.length()) return false;

        Map<Character, Character> mapST = new HashMap<>();
        Map<Character, Character> mapTS = new HashMap<>();

        for (int i = 0; i < s.length(); i++) {
            char a = s.charAt(i), b = t.charAt(i);
            if (mapST.containsKey(a) && mapST.get(a) != b) return false;
            if (mapTS.containsKey(b) && mapTS.get(b) != a) return false;
            mapST.put(a, b);
            mapTS.put(b, a);
        }
        return true;
    }
}
```

### Example

```
isIsomorphic('egg','add'): true
isIsomorphic('foo','bar'): false
isIsomorphic('paper','title'): true
```

`"foo"`/`"bar"`: `f→b`, but then `o→a` for the first `o`... the second `o` also needs `o→a` (consistent), yet `r` at the end would need `o→r` too — actually the failure is that both `o`s map fine to `a`, but there's no consistent single-character partner for `f` and the trailing `r` breaks the length-matched positional check. The two-map check catches any such inconsistency immediately.

### Problem 2 (Related): Word Pattern

**Theory:** The exact same bijective-mapping idea, but between a pattern's **characters** and a sentence's **words** instead of character-to-character. Split the sentence into words, then run the identical two-map consistency check between `pattern.charAt(i)` and `words[i]`.

```java
import java.util.HashMap;
import java.util.Map;

public class WordPattern {
    public static void main(String[] args) {
        System.out.println("wordPattern('abba','dog cat cat dog'): "
                + wordPattern("abba", "dog cat cat dog"));
        System.out.println("wordPattern('abba','dog cat cat fish'): "
                + wordPattern("abba", "dog cat cat fish"));
        System.out.println("wordPattern('aaaa','dog cat cat dog'): "
                + wordPattern("aaaa", "dog cat cat dog"));
    }

    static boolean wordPattern(String pattern, String s) {
        String[] words = s.split(" ");
        if (pattern.length() != words.length) return false;

        Map<Character, String> mapCharToWord = new HashMap<>();
        Map<String, Character> mapWordToChar = new HashMap<>();

        for (int i = 0; i < pattern.length(); i++) {
            char c = pattern.charAt(i);
            String word = words[i];
            if (mapCharToWord.containsKey(c) && !mapCharToWord.get(c).equals(word)) return false;
            if (mapWordToChar.containsKey(word) && mapWordToChar.get(word) != c) return false;
            mapCharToWord.put(c, word);
            mapWordToChar.put(word, c);
        }
        return true;
    }
}
```

```
wordPattern('abba','dog cat cat dog'): true
wordPattern('abba','dog cat cat fish'): false
wordPattern('aaaa','dog cat cat dog'): false
```

`"aaaa"` against `"dog cat cat dog"` fails: `a` would need to map to both `"dog"` **and** `"cat"`, violating the one-to-one requirement.

## 2. One Edit Distance

### Theory

**Classic example — One Edit Distance:** determine whether two strings are **exactly one** edit apart (insert, delete, or replace one character) — a decision problem, distinct from [Lesson 17's full Edit Distance](<./17. Dynamic Programming on Strings.md#1-edit-distance>), which computes the minimum number of edits regardless of what it is. If lengths differ by more than `1`, it's immediately impossible. Otherwise, walk both strings together until the first mismatch: if the lengths are **equal**, the rest of both strings (after skipping the one mismatched character on each side) must be identical (a substitution); if lengths differ by exactly `1`, the rest of the longer string (after skipping just its one extra character) must match the rest of the shorter one (an insertion/deletion). If no mismatch is ever found, they're one edit apart only if the lengths actually differ (an extra trailing character was inserted).

### Code

```java
public class OneEditDistance {
    public static void main(String[] args) {
        System.out.println("isOneEditDistance('ab','acb'): " + isOneEditDistance("ab", "acb"));
        System.out.println("isOneEditDistance('cab','ad'): " + isOneEditDistance("cab", "ad"));
        System.out.println("isOneEditDistance('1203','1213'): " + isOneEditDistance("1203", "1213"));
        System.out.println("isOneEditDistance('ab','ab'): " + isOneEditDistance("ab", "ab"));
    }

    static boolean isOneEditDistance(String s, String t) {
        int m = s.length(), n = t.length();
        if (Math.abs(m - n) > 1) return false;

        if (m > n) return isOneEditDistance(t, s); // ensure s is the shorter (or equal-length) string

        for (int i = 0; i < m; i++) {
            if (s.charAt(i) != t.charAt(i)) {
                if (m == n) {
                    return s.substring(i + 1).equals(t.substring(i + 1)); // substitution
                } else {
                    return s.substring(i).equals(t.substring(i + 1)); // insertion into s
                }
            }
        }
        return m != n; // identical prefix throughout -- valid only if t has exactly one extra trailing char
    }
}
```

### Example

```
isOneEditDistance('ab','acb'): true
isOneEditDistance('cab','ad'): false
isOneEditDistance('1203','1213'): true
isOneEditDistance('ab','ab'): false
```

`"ab"` → `"acb"`: inserting `'c'` between `a` and `b` makes them one edit apart. `"ab"` vs `"ab"` is **zero** edits, not one — correctly rejected. `"cab"` vs `"ad"`: length difference is `1`, but the characters don't align into a single valid insertion/deletion, so it needs more than one edit.

## 3. Custom Sort String

### Theory

**Classic example — Custom Sort String:** given a string `order` defining a custom character priority, sort `s` so its characters appear in that relative order (characters not in `order` can go anywhere, conventionally at the end). Rather than writing a custom `Comparator`, count the frequency of every character in `s`, then rebuild the output by walking through `order` and appending each character its full count of times, finally appending any leftover characters (not mentioned in `order`) in whatever order remains.

### Code

```java
import java.util.LinkedHashMap;
import java.util.Map;

public class CustomSortString {
    public static void main(String[] args) {
        System.out.println("customSortString('cba','abcd'): " + customSortString("cba", "abcd"));
    }

    static String customSortString(String order, String s) {
        Map<Character, Integer> count = new LinkedHashMap<>();
        for (char c : s.toCharArray()) count.merge(c, 1, Integer::sum);

        StringBuilder result = new StringBuilder();
        for (char c : order.toCharArray()) {
            if (count.containsKey(c)) {
                result.append(String.valueOf(c).repeat(count.get(c)));
                count.remove(c);
            }
        }
        for (Map.Entry<Character, Integer> entry : count.entrySet()) {
            result.append(String.valueOf(entry.getKey()).repeat(entry.getValue()));
        }
        return result.toString();
    }
}
```

### Example

```
customSortString('cba','abcd'): cbad
```

`order = "cba"` places `c` first, then `b`, then `a`; `d` isn't in `order`, so it's appended at the end — `"cbad"`.

## 4. Group Shifted Strings

### Theory

**Classic example — Group Shifted Strings:** group strings that can be transformed into each other by **shifting every character by the same amount** (cyclically, so `z` shifts to `a`) — e.g. `"abc"`, `"bcd"`, and `"xyz"` are all in the same group, since each is a uniform shift of the others. The key insight: what defines the group isn't the actual characters, but the **sequence of differences** between consecutive characters (mod 26) — shifting every character by the same amount preserves all those differences unchanged. So the grouping key is the tuple of `(char[i] - char[0]) mod 26` for every position `i`, which is identical for all strings in the same shift-group.

### Code

```java
import java.util.*;

public class GroupShiftedStrings {
    public static void main(String[] args) {
        String[] strings = { "abc", "bcd", "acef", "xyz", "az", "ba", "a", "z" };
        System.out.println("Groups: " + groupStrings(strings));
    }

    static List<List<String>> groupStrings(String[] strings) {
        Map<String, List<String>> groups = new LinkedHashMap<>();

        for (String s : strings) {
            StringBuilder keyBuilder = new StringBuilder();
            for (int i = 0; i < s.length(); i++) {
                int diff = ((s.charAt(i) - s.charAt(0)) % 26 + 26) % 26;
                keyBuilder.append(diff).append(',');
            }
            groups.computeIfAbsent(keyBuilder.toString(), k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(groups.values());
    }
}
```

### Example

```
Groups: [[abc, bcd, xyz], [acef], [az, ba], [a, z]]
```

`"abc"`, `"bcd"`, `"xyz"` share the difference pattern `(0, 1, 2)` — each character is one more than the previous, cyclically. `"az"` and `"ba"` share `(0, 25)` — `az`: `a→z` is a difference of `25` mod 26; `ba`: `b→a` is also `25` mod 26 (`-1 mod 26 = 25`).

## 5. Determine if Two Strings Are Close

### Theory

**Classic example — Determine if Two Strings Are Close (advanced):** two operations are allowed — swap any two **characters** within a string (any number of times), or pick two characters and swap **every occurrence** of one with the other throughout the string. Two strings are "close" if one can be turned into the other using any number of these operations. Since character-position swaps are unrestricted, character **order never matters**. What must match: the **set of distinct characters** used (an operation can rename a character but never introduce a brand-new one), and the **multiset of frequency counts** (swapping two characters' identities swaps their counts, but the counts themselves, sorted, stay the same set of numbers).

### Code

```java
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

public class CloseStrings {
    public static void main(String[] args) {
        System.out.println("closeStrings('abc','bca'): " + closeStrings("abc", "bca"));
        System.out.println("closeStrings('a','aa'): " + closeStrings("a", "aa"));
        System.out.println("closeStrings('cabbba','abbccc'): " + closeStrings("cabbba", "abbccc"));
    }

    static boolean closeStrings(String word1, String word2) {
        if (word1.length() != word2.length()) return false;

        Map<Character, Integer> count1 = new HashMap<>();
        Map<Character, Integer> count2 = new HashMap<>();
        for (char c : word1.toCharArray()) count1.merge(c, 1, Integer::sum);
        for (char c : word2.toCharArray()) count2.merge(c, 1, Integer::sum);

        if (!count1.keySet().equals(count2.keySet())) return false;

        Integer[] freq1 = count1.values().toArray(new Integer[0]);
        Integer[] freq2 = count2.values().toArray(new Integer[0]);
        Arrays.sort(freq1);
        Arrays.sort(freq2);
        return Arrays.equals(freq1, freq2);
    }
}
```

### Example

```
closeStrings('abc','bca'): true
closeStrings('a','aa'): false
closeStrings('cabbba','abbccc'): true
```

`"abc"`/`"bca"` use the same characters with identical frequencies (`1` each), just reordered. `"a"`/`"aa"` differ in length, immediately disqualified. `"cabbba"` (`a:2, b:3, c:1`) and `"abbccc"` (`a:1, b:2, c:3`) use the same character set, and their frequency **multisets** both equal `{1, 2, 3}` — swapping the roles of `a` and `c` throughout turns one into the other.

## Quick Reference Cheat Sheet

| Technique | Complexity | Problems covered |
|---|---|---|
| Isomorphic Strings and Word Pattern | O(n) | bijective character/word mapping, two-map consistency check |
| One Edit Distance | O(n) | decision version of [Lesson 17's Edit Distance](<./17. Dynamic Programming on Strings.md#1-edit-distance>) |

| Group Shifted Strings | O(total characters) | grouping key: sequence of `(char[i]-char[0]) mod 26` |
