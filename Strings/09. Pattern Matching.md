# 9. Pattern Matching

Finding a pattern inside a text is one of the oldest problems in computer science, and it has an entire family of algorithms — each exploiting a different structural insight to beat the naive O(n×m) approach. This lesson builds each one from scratch, in order of increasing sophistication.

## 1. Naive Matching

### Problem 1 (Basic): Find First Occurrence (strStr)

**Theory:** Try every starting position in the text; at each, compare character by character against the pattern. **O(n×m)** worst case (e.g. text `"aaaa...a"`, pattern `"aaa...b"` — every position almost matches before failing at the last character).

```java
public class NaiveMatching {
    public static void main(String[] args) {
        System.out.println("strStr('hello', 'll'): " + strStr("hello", "ll"));
        System.out.println("strStr('aaaaa', 'bba'): " + strStr("aaaaa", "bba"));
    }

    static int strStr(String haystack, String needle) {
        int n = haystack.length(), m = needle.length();
        for (int i = 0; i + m <= n; i++) {
            if (haystack.substring(i, i + m).equals(needle)) return i;
        }
        return -1;
    }
}
```

```
strStr('hello', 'll'): 2
strStr('aaaaa', 'bba'): -1
```

### Problem 2 (Basic): Count All Occurrences (Overlapping)

**Theory:** Same idea, but don't stop at the first match — and don't skip past a match either, since occurrences are allowed to **overlap**.

```java
public class CountOccurrences {
    public static void main(String[] args) {
        System.out.println("Occurrences of 'aa' in 'aaaa': " + countOccurrences("aaaa", "aa"));
    }

    static int countOccurrences(String text, String pattern) {
        int n = text.length(), m = pattern.length();
        int count = 0;
        for (int i = 0; i + m <= n; i++) {
            if (text.substring(i, i + m).equals(pattern)) count++;
        }
        return count;
    }
}
```

```
Occurrences of 'aa' in 'aaaa': 3
```

`"aaaa"` contains `"aa"` at positions `0`, `1`, and `2` — overlapping matches all count.

## 2. Prefix Function (LPS)

### Theory

The **prefix function** (also called the **failure function**, or **LPS array** — "Longest Proper Prefix which is also a Suffix") is the piece of precomputation that makes [KMP](#3-kmp-algorithm) fast. `lps[i]` stores the length of the longest prefix of the pattern that is also a suffix of `pattern[0..i]` — this tells KMP exactly how far it can safely "slide" the pattern after a mismatch, without re-checking characters it already knows must match.

### Problem (Medium): Repeated Substring Pattern

**Theory:** Check whether a string is built entirely from repetitions of some shorter substring (e.g. `"abab"` = `"ab"` repeated twice). The LPS array of the whole string reveals this directly: if `n - lps[n-1]` (the length of the "non-repeating unit") evenly divides `n`, the string is exactly that unit repeated.

```java
public class RepeatedSubstringPattern {
    public static void main(String[] args) {
        System.out.println("'abab' is repeated substring: " + repeatedSubstringPattern("abab"));
        System.out.println("'aba' is repeated substring: " + repeatedSubstringPattern("aba"));
        System.out.println("'abcabcabcabc' is repeated substring: " + repeatedSubstringPattern("abcabcabcabc"));
    }

    static boolean repeatedSubstringPattern(String s) {
        int[] lps = computeLPS(s);
        int n = s.length();
        int unitLength = n - lps[n - 1];
        return lps[n - 1] > 0 && n % unitLength == 0;
    }

    static int[] computeLPS(String pattern) {
        int m = pattern.length();
        int[] lps = new int[m];
        int length = 0, i = 1;
        while (i < m) {
            if (pattern.charAt(i) == pattern.charAt(length)) {
                length++;
                lps[i] = length;
                i++;
            } else if (length != 0) {
                length = lps[length - 1];
            } else {
                lps[i] = 0;
                i++;
            }
        }
        return lps;
    }
}
```

```
'abab' is repeated substring: true
'aba' is repeated substring: false
'abcabcabcabc' is repeated substring: true
```

## 3. KMP Algorithm

### Problem 1 (Basic): Pattern Search

**Theory:** Knuth-Morris-Pratt uses the [LPS array](#2-prefix-function-lps) to avoid re-examining characters after a mismatch — on a mismatch at pattern position `j`, jump `j` back to `lps[j-1]` instead of restarting from `0`, since that many characters are already known to match. **O(n + m)** — the naive algorithm's worst case, eliminated entirely.

```java
import java.util.ArrayList;
import java.util.List;

public class KmpSearch {
    public static void main(String[] args) {
        System.out.println("KMP matches of 'ab' in 'ababab': " + search("ababab", "ab"));
    }

    static List<Integer> search(String text, String pattern) {
        int[] lps = computeLPS(pattern);
        List<Integer> result = new ArrayList<>();
        int i = 0, j = 0;

        while (i < text.length()) {
            if (text.charAt(i) == pattern.charAt(j)) {
                i++;
                j++;
                if (j == pattern.length()) {
                    result.add(i - j);
                    j = lps[j - 1]; // continue searching for overlapping matches
                }
            } else if (j != 0) {
                j = lps[j - 1]; // don't restart from scratch -- skip known-matching characters
            } else {
                i++;
            }
        }
        return result;
    }

    static int[] computeLPS(String pattern) {
        int m = pattern.length();
        int[] lps = new int[m];
        int length = 0, i = 1;
        while (i < m) {
            if (pattern.charAt(i) == pattern.charAt(length)) {
                length++;
                lps[i] = length;
                i++;
            } else if (length != 0) {
                length = lps[length - 1];
            } else {
                lps[i] = 0;
                i++;
            }
        }
        return lps;
    }
}
```

```
KMP matches of 'ab' in 'ababab': [0, 2, 4]
```

### Problem 2 (Advanced): Shortest Palindrome

**Theory:** Given a string, find the shortest palindrome formed by adding characters **only in front of it**. Clever trick: build `combined = s + '#' + reverse(s)` (the `#` separator prevents false overlaps), and compute its LPS array. The last LPS value tells you the **longest prefix of `s` that is already a palindrome** — everything after that prefix needs to be mirrored and prepended.

```java
public class ShortestPalindrome {
    public static void main(String[] args) {
        System.out.println("Shortest palindrome from 'aacecaaa': " + shortestPalindrome("aacecaaa"));
        System.out.println("Shortest palindrome from 'abcd': " + shortestPalindrome("abcd"));
    }

    static String shortestPalindrome(String s) {
        String combined = s + "#" + new StringBuilder(s).reverse();
        int[] lps = computeLPS(combined);

        int longestPalindromicPrefixLength = lps[combined.length() - 1];
        String suffixToMirror = s.substring(longestPalindromicPrefixLength);
        String toPrepend = new StringBuilder(suffixToMirror).reverse().toString();

        return toPrepend + s;
    }

    static int[] computeLPS(String pattern) {
        int m = pattern.length();
        int[] lps = new int[m];
        int length = 0, i = 1;
        while (i < m) {
            if (pattern.charAt(i) == pattern.charAt(length)) {
                length++;
                lps[i] = length;
                i++;
            } else if (length != 0) {
                length = lps[length - 1];
            } else {
                lps[i] = 0;
                i++;
            }
        }
        return lps;
    }
}
```

```
Shortest palindrome from 'aacecaaa': aaacecaaa
Shortest palindrome from 'abcd': dcbabcd
```

## 4. Rabin-Karp

### Theory

Compare a **rolling hash** of the pattern against a rolling hash of every text window of the same length (see [Lesson 8: Rolling Hash](<./08. Hashing.md#5-rolling-hash>)) — updating the window's hash in O(1) as it slides. When hashes match, do a direct character comparison to rule out a hash collision (a **false positive**, unlikely but possible). Average **O(n + m)**, though worst case (many collisions) degrades toward O(n×m).

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class RabinKarp {
    static final long BASE = 256, MOD = 1_000_000_007L;

    public static void main(String[] args) {
        System.out.println("Rabin-Karp matches of 'ab' in 'ababab': " + search("ababab", "ab"));
    }

    static List<Integer> search(String text, String pattern) {
        int n = text.length(), m = pattern.length();
        List<Integer> result = new ArrayList<>();
        if (m > n) return result;

        long patternHash = 0, textHash = 0, highOrder = 1;
        for (int i = 0; i < m - 1; i++) highOrder = (highOrder * BASE) % MOD;

        for (int i = 0; i < m; i++) {
            patternHash = (patternHash * BASE + pattern.charAt(i)) % MOD;
            textHash = (textHash * BASE + text.charAt(i)) % MOD;
        }

        for (int i = 0; i <= n - m; i++) {
            if (patternHash == textHash && text.substring(i, i + m).equals(pattern)) {
                result.add(i); // hash matched AND direct comparison confirmed -- no false positive
            }
            if (i < n - m) {
                textHash = (BASE * (textHash - text.charAt(i) * highOrder % MOD) + text.charAt(i + m)) % MOD;
                textHash = ((textHash % MOD) + MOD) % MOD; // keep it non-negative
            }
        }
        return result;
    }
}
```

### Example

```
Rabin-Karp matches of 'ab' in 'ababab': [0, 2, 4]
```

## 5. Z Algorithm

### Theory

The **Z-array** for a string `s`: `z[i]` is the length of the longest substring starting at `i` that matches a **prefix** of `s`. Computing it in O(n) (using a maintained `[l, r]` window of the rightmost known prefix-match found so far) lets you find every occurrence of a pattern in a text: concatenate `pattern + separator + text` (a separator character not in either), compute the Z-array of that combined string, and every position where `z[i] == pattern.length()` marks a match.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class ZAlgorithm {
    public static void main(String[] args) {
        System.out.println("Z-algorithm matches of 'ab' in 'ababab': " + search("ababab", "ab"));
    }

    static List<Integer> search(String text, String pattern) {
        String combined = pattern + "$" + text;
        int[] z = buildZArray(combined);

        List<Integer> result = new ArrayList<>();
        int m = pattern.length();
        for (int i = m + 1; i < combined.length(); i++) {
            if (z[i] == m) {
                result.add(i - m - 1); // translate back to an index in the original text
            }
        }
        return result;
    }

    static int[] buildZArray(String s) {
        int n = s.length();
        int[] z = new int[n];
        int l = 0, r = 0;

        for (int i = 1; i < n; i++) {
            if (i < r) {
                z[i] = Math.min(r - i, z[i - l]);
            }
            while (i + z[i] < n && s.charAt(z[i]) == s.charAt(i + z[i])) {
                z[i]++;
            }
            if (i + z[i] > r) {
                l = i;
                r = i + z[i];
            }
        }
        return z;
    }
}
```

### Example

```
Z-algorithm matches of 'ab' in 'ababab': [0, 2, 4]
```

## 6. Boyer-Moore

### Theory

Boyer-Moore compares the pattern against the text **right to left** instead of left to right, and on a mismatch, uses the **bad character rule**: jump the pattern forward so its last occurrence of the mismatched text character lines up with where the mismatch happened (or past it entirely, if that character never appears in the pattern). This lets Boyer-Moore skip large chunks of text without examining every character — often **sub-linear** in practice, though O(n×m) worst case without the additional "good suffix rule" (a further refinement not shown here for simplicity).

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class BoyerMoore {
    public static void main(String[] args) {
        System.out.println("Boyer-Moore first match, 'ABC' in 'ABAAABCD': " + search("ABAAABCD", "ABC"));
    }

    static int search(String text, String pattern) {
        int n = text.length(), m = pattern.length();
        Map<Character, Integer> lastOccurrence = new HashMap<>();
        for (int i = 0; i < m; i++) lastOccurrence.put(pattern.charAt(i), i);

        int shift = 0;
        while (shift <= n - m) {
            int j = m - 1;
            while (j >= 0 && pattern.charAt(j) == text.charAt(shift + j)) {
                j--;
            }
            if (j < 0) {
                return shift; // full match found
            }
            char mismatchChar = text.charAt(shift + j);
            int lastIndex = lastOccurrence.getOrDefault(mismatchChar, -1);
            shift += Math.max(1, j - lastIndex);
        }
        return -1;
    }
}
```

### Example

```
Boyer-Moore first match, 'ABC' in 'ABAAABCD': 4
```

## 7. Finite Automata Matching

### Theory

Build a **deterministic finite automaton (DFA)** ahead of time: a state machine with one state per pattern position, where the transition function `dfa[state][character]` tells you exactly which state to move to for any possible next character — including on a mismatch, since it's precomputed for **every** character in the alphabet, not just the pattern's own characters. Once built (O(m × alphabet size)), matching the text takes exactly **O(n)** with **no backtracking at all** — every character is read exactly once, unlike KMP which conceptually "reconsiders" via the LPS jump (though also O(n) amortized).

### Code

```java
import java.util.HashMap;
import java.util.Map;
import java.util.TreeSet;

public class FiniteAutomataMatching {
    public static void main(String[] args) {
        String text = "ABABDABACDABABCABAB";
        String pattern = "ABABCABAB";
        System.out.println("DFA match index: " + search(text, pattern));
    }

    static int search(String text, String pattern) {
        TreeSet<Character> alphabetSet = new TreeSet<>();
        for (char c : text.toCharArray()) alphabetSet.add(c);
        for (char c : pattern.toCharArray()) alphabetSet.add(c);

        Map<Character, Integer> charIndex = new HashMap<>();
        int idx = 0;
        for (char c : alphabetSet) charIndex.put(c, idx++);

        int[][] dfa = buildDFA(pattern, charIndex);

        int state = 0;
        for (int i = 0; i < text.length(); i++) {
            state = dfa[state][charIndex.get(text.charAt(i))];
            if (state == pattern.length()) {
                return i - pattern.length() + 1;
            }
        }
        return -1;
    }

    static int[][] buildDFA(String pattern, Map<Character, Integer> charIndex) {
        int m = pattern.length(), alphabetSize = charIndex.size();
        int[][] dfa = new int[m][alphabetSize];

        dfa[0][charIndex.get(pattern.charAt(0))] = 1;
        int x = 0; // "restart" state, tracks where we'd be after a mismatch

        for (int j = 1; j < m; j++) {
            for (char c : charIndex.keySet()) {
                dfa[j][charIndex.get(c)] = dfa[x][charIndex.get(c)]; // copy mismatch behavior from state x
            }
            dfa[j][charIndex.get(pattern.charAt(j))] = j + 1; // the one "correct" transition
            x = dfa[x][charIndex.get(pattern.charAt(j))];
        }
        return dfa;
    }
}
```

### Example

```
DFA match index: 10
```

## Quick Reference Cheat Sheet

| Algorithm | Complexity | Core idea |
|---|---|---|
| Naive Matching | O(n×m) | brute-force character comparison at every position |
| Prefix Function (LPS) | O(m) to build | longest prefix-that's-also-a-suffix, per position |
| KMP | O(n+m) | use LPS to skip re-checking known-matching characters |
| Rabin-Karp | O(n+m) average | rolling hash comparison, verify on hash match |
| Z Algorithm | O(n+m) | Z-array of `pattern + separator + text` |
| Boyer-Moore | sub-linear in practice, O(n×m) worst case | right-to-left scan, bad-character skip |
| Finite Automata Matching | O(n) after O(m×alphabet) preprocessing | precomputed transition table, no backtracking |
