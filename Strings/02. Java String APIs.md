## 1. length()

### Theory

Returns the number of `char` units in the string — O(1), since it's a cached field, not computed by scanning.

### Code

```java
public class LengthMethod {
    public static void main(String[] args) {
        System.out.println("hello".length());
    }
}
```

```
5
```

## 2. charAt()

### Theory

Returns the character at a 0-indexed position — O(1). Throws `StringIndexOutOfBoundsException` outside `[0, length())`.

### Code

```java
public class CharAtMethod {
    public static void main(String[] args) {
        System.out.println("hello".charAt(1));
    }
}
```

```
e
```

## 3. substring()

### Theory

`substring(begin)` returns everything from `begin` to the end; `substring(begin, end)` returns `[begin, end)` — `end` **exclusive**, matching the convention used throughout [Arrays copyOfRange](<../Arrays/03. Java Array APIs.md#4-arrayscopyofrange>). O(n) since Java 7 (it copies the range; older Java versions shared the backing array, which caused memory-leak issues fixed by making it copy).

### Code

```java
public class SubstringMethod {
    public static void main(String[] args) {
        String s = "hello world";
        System.out.println(s.substring(6));
        System.out.println(s.substring(0, 5));
    }
}
```

```
world
hello
```

## 4. indexOf() & lastIndexOf()

### Theory

`indexOf(str)` finds the **first** occurrence's starting index (`-1` if absent); `lastIndexOf(str)` finds the **last**. Both have overloads accepting a starting search position. Naive implementation is O(n×m); [Lesson 9](<./09. Pattern Matching.md>) covers faster algorithms for repeated/large-scale searching.

### Code

```java
public class IndexOfMethods {
    public static void main(String[] args) {
        String s = "hello world";
        System.out.println(s.indexOf("o"));
        System.out.println(s.indexOf("o", 5));
        System.out.println(s.lastIndexOf("o"));
    }
}
```

```
4
7
7
```

## 5. contains(), startsWith() & endsWith()

### Theory

`contains(seq)` checks for a substring anywhere; `startsWith(prefix)` / `endsWith(suffix)` anchor the check to a specific end. All three are thin wrappers around `indexOf`-style scanning.

### Code

```java
public class ContainsStartsEndsWith {
    public static void main(String[] args) {
        String s = "hello world";
        System.out.println(s.contains("lo w"));
        System.out.println(s.startsWith("hel"));
        System.out.println(s.endsWith("rld"));
    }
}
```

```
true
true
true
```

## 6. equals() & equalsIgnoreCase()

### Theory

`equals()` checks character-by-character content equality (case-sensitive); `equalsIgnoreCase()` does the same while normalizing case first. Always prefer these over `==` for content comparison — full reasoning in [Lesson 1](<./01. String Fundamentals.md#4-java-string-pool-memory-model--interning>).

### Code

```java
public class EqualsMethods {
    public static void main(String[] args) {
        System.out.println("apple".equals("Apple"));
        System.out.println("apple".equalsIgnoreCase("Apple"));
    }
}
```

```
false
true
```

## 8. split()

### Theory

Splits a string by a **regex** delimiter into a `String[]`. By default, **trailing empty strings are removed** from the result (a common surprise) — empty strings in the *middle* are kept. A second `limit` argument controls this (negative limit keeps trailing empties, positive limit caps the number of splits).

### Code

```java
import java.util.Arrays;

public class SplitMethod {
    public static void main(String[] args) {
        System.out.println(Arrays.toString("one,two,,three".split(",")));
        System.out.println(Arrays.toString("a,b,c,,".split(",")));       // trailing empties dropped
        System.out.println(Arrays.toString("a,b,c,,".split(",", -1)));   // -1 keeps them
    }
}
```

```
[one, two, , three]
[a, b, c]
[a, b, c, , ]
```

## 9. join()

### Theory

The inverse of `split()`: `String.join(delimiter, elements...)` concatenates elements with a delimiter between each — no delimiter added before the first or after the last.

### Code

```java
public class JoinMethod {
    public static void main(String[] args) {
        System.out.println(String.join("-", "a", "b", "c"));
        System.out.println(String.join(", ", java.util.List.of("x", "y", "z")));
    }
}
```

```
a-b-c
x, y, z
```

## 10. trim() & strip()

### Theory

Both remove leading/trailing whitespace, but define "whitespace" differently: `trim()` (legacy, pre-Unicode-aware) removes any character `≤ U+0020`; `strip()` (Java 11+) uses `Character.isWhitespace()`, which correctly recognizes a broader set of Unicode whitespace characters. They agree for plain ASCII spaces, but differ for characters like `U+2000` (EN QUAD, a Unicode space-separator character above `U+0020`).

### Code

```java
public class TrimVsStrip {
    public static void main(String[] args) {
        String s = " hello "; // Unicode EN QUAD space on both sides
        System.out.println("trim():   [" + s.trim() + "]");
        System.out.println("strip():  [" + s.strip() + "]");
    }
}
```

```
trim():   [ hello ]
strip():  [hello]
```

`trim()` doesn't recognize `U+2000` as whitespace (it's above `U+0020`), so it's left in place; `strip()` correctly removes it.

## 11. replace() & replaceAll()

### Theory

`replace(CharSequence, CharSequence)` does a **literal** (non-regex) find-and-replace-all. `replaceAll(regex, replacement)` treats its first argument as a **regular expression** — a frequent source of bugs when the "literal" string you meant to replace happens to contain regex metacharacters (`.`, `*`, `+`, ...).

### Code

```java
public class ReplaceMethods {
    public static void main(String[] args) {
        System.out.println("hello".replace('l', 'L'));
        System.out.println("a.b.c".replace(".", "-"));       // literal dot
        System.out.println("hello123world456".replaceAll("[0-9]+", "-")); // regex
    }
}
```

```
heLLo
a-b-c
hello-world-
```

## 12. matches()

### Theory

Checks whether the **entire** string matches a given regex (implicitly anchored at both ends — unlike `Pattern.find()`, which can match a substring).

### Code

```java
public class MatchesMethod {
    public static void main(String[] args) {
        System.out.println("12345".matches("[0-9]+"));
        System.out.println("12345".matches("[0-9]{3}")); // whole-string match fails, only 3 digits expected
    }
}
```

```
true
false
```

## 13. toCharArray()

### Theory

Copies the string's characters into a new `char[]` — O(n). Necessary whenever an algorithm needs **in-place character mutation** (strings themselves are immutable, so you can't mutate a `String` directly — see [Lesson 1](<./01. String Fundamentals.md#3-mutable-vs-immutable-strings>)).

### Code

```java
import java.util.Arrays;

public class ToCharArrayMethod {
    public static void main(String[] args) {
        char[] chars = "abc".toCharArray();
        System.out.println(Arrays.toString(chars));
    }
}
```

```
[a, b, c]
```

## 14. valueOf()

### Theory

Converts virtually any type to its string representation — overloaded for primitives, `char[]`, and `Object` (delegating to `toString()`). The static-method form (`String.valueOf(x)`) is null-safe for objects, unlike calling `.toString()` directly on a possibly-`null` reference.

### Code

```java
public class ValueOfMethod {
    public static void main(String[] args) {
        System.out.println(String.valueOf(123));
        System.out.println(String.valueOf(true));
        System.out.println(String.valueOf((Object) null)); // null-safe
    }
}
```

```
123
true
null
```

## 15. format()

### Theory

Builds a string using `printf`-style format specifiers (`%d` integer, `%s` string, `%.2f` fixed-decimal float, ...) — the readable alternative to manual concatenation when interpolating multiple values.

### Code

```java
public class FormatMethod {
    public static void main(String[] args) {
        System.out.println(String.format("%d apples cost $%.2f", 5, 2.5));
    }
}
```

```
5 apples cost $2.50
```

## 16. repeat() & isBlank()

### Theory

`repeat(n)` (Java 11+) concatenates `n` copies of the string — replacing the older manual-loop-with-StringBuilder idiom for this specific case. `isBlank()` (Java 11+) checks whether a string is **empty or entirely whitespace** — stricter than `isEmpty()`, which only checks for zero length.

### Code

```java
public class RepeatAndIsBlank {
    public static void main(String[] args) {
        System.out.println("ab".repeat(3));
        System.out.println("   ".isBlank());
        System.out.println("   ".isEmpty());
    }
}
```

```
ababab
true
false
```

## Quick Reference Cheat Sheet

| Method | Purpose |
|---|---|
| `length()` / `charAt(i)` | O(1) size / indexed access |
| `substring(a, b)` | copy of range `[a, b)` |
| `indexOf` / `lastIndexOf` | first / last occurrence index |
| `contains` / `startsWith` / `endsWith` | substring presence checks |
| `equals` / `equalsIgnoreCase` | content comparison |
| `compareTo` | lexicographic ordering |
| `split(regex)` | tokenize by regex, drops trailing empties by default |
| `join(delim, ...)` | inverse of split |
| `trim()` / `strip()` | remove surrounding whitespace (ASCII-only vs Unicode-aware) |
| `replace` / `replaceAll` | literal vs regex find-and-replace |
| `matches(regex)` | whole-string regex match |
| `toCharArray()` | copy into a mutable `char[]` |
| `valueOf(x)` | null-safe conversion to `String` |
| `format(...)` | printf-style templated string building |
| `repeat(n)` / `isBlank()` | repeat n times / whitespace-only check |
