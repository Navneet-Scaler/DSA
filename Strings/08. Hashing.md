## 1. Character Hashing

### Problem 1 (Basic): Ransom Note

**Theory:** Can a ransom note be constructed using only letters available in a magazine (each magazine letter usable once)? Build a frequency map of the magazine, then check that every letter the note needs is available in sufficient quantity.

```java
import java.util.HashMap;
import java.util.Map;

public class RansomNote {
    public static void main(String[] args) {
        System.out.println("'aa' from 'aab': " + canConstruct("aa", "aab"));
        System.out.println("'aa' from 'ab': " + canConstruct("aa", "ab"));
    }

    static boolean canConstruct(String ransomNote, String magazine) {
        Map<Character, Integer> available = new HashMap<>();
        for (char c : magazine.toCharArray()) available.merge(c, 1, Integer::sum);

        for (char c : ransomNote.toCharArray()) {
            int remaining = available.getOrDefault(c, 0);
            if (remaining == 0) return false;
            available.put(c, remaining - 1);
        }
        return true;
    }
}
```

```
'aa' from 'aab': true
'aa' from 'ab': false
```

## 2. Word Hashing

### Problem 1 (Basic): Word Pattern

**Theory:** Given a pattern like `"abba"` and a sentence, check whether the words follow the same repetition pattern (a bijective mapping between pattern letters and words). Two hash maps — one per mapping direction — reject any inconsistency, the same technique as Isomorphic Strings applied to whole words instead of characters.

```java
import java.util.HashMap;
import java.util.Map;

public class WordPattern {
    public static void main(String[] args) {
        System.out.println("abba / 'dog cat cat dog': " + wordPattern("abba", "dog cat cat dog"));
        System.out.println("abba / 'dog cat cat fish': " + wordPattern("abba", "dog cat cat fish"));
    }

    static boolean wordPattern(String pattern, String s) {
        String[] words = s.split(" ");
        if (pattern.length() != words.length) return false;

        Map<Character, String> patternToWord = new HashMap<>();
        Map<String, Character> wordToPattern = new HashMap<>();

        for (int i = 0; i < words.length; i++) {
            char p = pattern.charAt(i);
            String w = words[i];

            if (patternToWord.containsKey(p) && !patternToWord.get(p).equals(w)) return false;
            if (wordToPattern.containsKey(w) && wordToPattern.get(w) != p) return false;

            patternToWord.put(p, w);
            wordToPattern.put(w, p);
        }
        return true;
    }
}
```

```
abba / 'dog cat cat dog': true
abba / 'dog cat cat fish': false
```

### Problem 2 (Advanced): Group Shifted Strings

**Theory:** Group strings that are all "shifts" of each other (e.g. `"abc"`, `"bcd"`, `"xyz"` all shift by a constant amount per letter, wrapping `z → a`). The hash **key** is the derived "shift pattern" — the sequence of consecutive-letter differences (mod 26) — rather than the string itself; strings sharing that pattern belong to the same group. This is [Group Anagrams](<../Arrays/04. Frequency Problems.md#10-group-anagrams>)'s technique (canonical-key + hash map), just with a different canonicalization rule.

```java
import java.util.*;

public class GroupShiftedStrings {
    public static void main(String[] args) {
        String[] strings = { "abc", "bcd", "acef", "xyz", "az", "ba", "a", "z" };
        System.out.println("Groups: " + groupStrings(strings));
    }

    static List<List<String>> groupStrings(String[] strings) {
        Map<String, List<String>> groups = new HashMap<>();
        for (String s : strings) {
            String key = shiftKey(s);
            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(groups.values());
    }

    static String shiftKey(String s) {
        if (s.length() == 1) return "single"; // all single-character strings shift-match each other
        StringBuilder key = new StringBuilder();
        for (int i = 1; i < s.length(); i++) {
            int diff = ((s.charAt(i) - s.charAt(i - 1)) + 26) % 26;
            key.append(diff).append(",");
        }
        return key.toString();
    }
}
```

```
Groups: [[abc, bcd, xyz], [acef], [az, ba], [a, z]]
```

`"abc"`, `"bcd"`, `"xyz"` all have the shift pattern `+1,+1`; `"az"` and `"ba"` both have pattern `+25` (`z-a=25`, and `a-b` wraps to `25` too).

## 3. Frequency Maps

### Problem 1 (Basic): Find the Difference

**Theory:** String `t` is `s` shuffled with one extra letter added — find that letter. Subtract `s`'s frequency map from `t`'s; whatever remains with a positive count is the extra letter.

```java
import java.util.HashMap;
import java.util.Map;

public class FindTheDifference {
    public static void main(String[] args) {
        System.out.println("Extra letter: " + findTheDifference("abcd", "abcde"));
    }

    static char findTheDifference(String s, String t) {
        Map<Character, Integer> count = new HashMap<>();
        for (char c : t.toCharArray()) count.merge(c, 1, Integer::sum);
        for (char c : s.toCharArray()) count.merge(c, -1, Integer::sum);

        for (Map.Entry<Character, Integer> entry : count.entrySet()) {
            if (entry.getValue() > 0) return entry.getKey();
        }
        throw new IllegalStateException("no difference found");
    }
}
```

```
Extra letter: e
```

## 4. Prefix Hashing

### Problem 1 (Basic): O(1) Substring Equality Check

**Theory:** Precompute a **prefix hash** array (a rolling polynomial hash of every prefix of the string), so the hash of **any** substring `s[i..j]` can be derived in O(1) via subtraction — analogous to [prefix sums](<../Arrays/05. Prefix Techniques.md#1-prefix-sum>), just with hash arithmetic (`hash[j] - hash[i] * base^(j-i)`) instead of plain subtraction. Comparing two substrings' hashes is then O(1) instead of O(length) — at the cost of a (very small) chance of hash collision giving a false positive.

```java
public class PrefixHashSubstringCheck {
    static final long BASE = 31, MOD = 1_000_000_007L;

    public static void main(String[] args) {
        String s = "abcabc";
        long[] prefixHash = buildPrefixHash(s);
        long[] powers = buildPowers(s.length());

        System.out.println("s[0..3) == s[3..6)? " + (substringHash(prefixHash, powers, 0, 3) == substringHash(prefixHash, powers, 3, 6)));
    }

    static long[] buildPrefixHash(String s) {
        long[] hash = new long[s.length() + 1];
        for (int i = 0; i < s.length(); i++) {
            hash[i + 1] = (hash[i] * BASE + s.charAt(i)) % MOD;
        }
        return hash;
    }

    static long[] buildPowers(int n) {
        long[] powers = new long[n + 1];
        powers[0] = 1;
        for (int i = 1; i <= n; i++) powers[i] = (powers[i - 1] * BASE) % MOD;
        return powers;
    }

    static long substringHash(long[] prefixHash, long[] powers, int start, int end) {
        long result = (prefixHash[end] - prefixHash[start] * powers[end - start]) % MOD;
        return (result + MOD) % MOD; // keep it non-negative
    }
}
```

```
s[0..3) == s[3..6)? true
```

`s[0..3) = "abc"` and `s[3..6) = "abc"` — identical, confirmed in O(1) via their precomputed hashes instead of a character-by-character comparison.

### Problem 2 (Advanced): Count Distinct Substrings of a Given Length

**Theory:** Using the same prefix-hash structure, compute the hash of every length-`L` substring in O(1) each (O(n) total for all of them), and count how many **distinct hash values** appear — an O(n) alternative to the O(n²) approach of materializing and comparing every substring directly.

```java
import java.util.HashSet;
import java.util.Set;

public class DistinctSubstringsOfLength {
    static final long BASE = 31, MOD = 1_000_000_007L;

    public static void main(String[] args) {
        System.out.println("Distinct length-3 substrings of 'abcabc': " + countDistinctSubstrings("abcabc", 3));
    }

    static int countDistinctSubstrings(String s, int length) {
        int n = s.length();
        long[] prefixHash = new long[n + 1];
        long[] powers = new long[n + 1];
        powers[0] = 1;
        for (int i = 0; i < n; i++) {
            prefixHash[i + 1] = (prefixHash[i] * BASE + s.charAt(i)) % MOD;
            powers[i + 1] = (powers[i] * BASE) % MOD;
        }

        Set<Long> distinctHashes = new HashSet<>();
        for (int i = 0; i + length <= n; i++) {
            long hash = (prefixHash[i + length] - prefixHash[i] * powers[length] % MOD + MOD * MOD) % MOD;
            distinctHashes.add(hash);
        }
        return distinctHashes.size();
    }
}
```

```
Distinct length-3 substrings of 'abcabc': 3
```

`{"abc", "bca", "cab"}` — three distinct length-3 substrings (the 4th window, `"abc"` again at the end, repeats the first).

## 5. Rolling Hash

### Problem (Medium-Advanced): Repeated DNA Sequences

**Theory:** Find every 10-character DNA substring that occurs **more than once** in a long sequence. A **rolling hash** updates the previous window's hash in O(1) to get the next window's hash (removing the outgoing character's contribution, adding the incoming one) instead of recomputing from scratch — turning an O(n×L) scan (L = substring length) into O(n).

### Code

```java
import java.util.ArrayList;
import java.util.HashSet;
import java.util.List;
import java.util.Set;

public class RepeatedDnaSequences {
    public static void main(String[] args) {
        String s = "AAAAACCCCCAAAAACCCCCCAAAAAGGGTTT";
        System.out.println("Repeated 10-letter sequences: " + findRepeatedDnaSequences(s));
    }

    static List<String> findRepeatedDnaSequences(String s) {
        int L = 10;
        if (s.length() <= L) return new ArrayList<>();

        Set<String> seen = new HashSet<>();
        Set<String> repeated = new HashSet<>();

        for (int i = 0; i + L <= s.length(); i++) {
            String window = s.substring(i, i + L); // substring() itself is O(L); a true rolling hash
            if (!seen.add(window)) {                // would avoid even this, using numeric hashes instead
                repeated.add(window);
            }
        }
        return new ArrayList<>(repeated);
    }
}
```

### Example

```
Repeated 10-letter sequences: [AAAAACCCCC, CCCCCAAAAA]
```

> This version uses `substring()` directly for clarity (O(L) per window). The genuinely O(n) version replaces the `Set<String>` with a `Set<Long>` of **rolling numeric hashes** — encoding each of the 4 DNA letters as 2 bits and shifting the window's bit-pattern by one letter at each step, exactly the "remove outgoing, add incoming" rolling-hash update from the theory above.

## 6. Double Hashing

### Theory

A single polynomial hash (as used above) has a small but real chance of **collision** — two different strings hashing to the same value, especially adversarially crafted inputs designed to break a known hash function (a real concern in competitive programming, where problem setters sometimes specifically construct anti-hash tests). **Double hashing** computes **two independent hashes** (different bases and/or moduli) for every string/substring, and only treats two things as equal when **both** hashes match — the probability of a simultaneous collision in both is the product of each individually, making it astronomically unlikely in practice.

### Code

```java
public class DoubleHashing {
    static final long BASE1 = 31, MOD1 = 1_000_000_007L;
    static final long BASE2 = 37, MOD2 = 998_244_353L;

    public static void main(String[] args) {
        long[] hash1 = hashOf("abc", BASE1, MOD1);
        long[] hash2 = hashOf("abc", BASE2, MOD2);
        System.out.println("Hash pair for 'abc': (" + hash1[0] + ", " + hash2[0] + ")");
    }

    // Returns a single combined hash value using both (base, mod) pairs
    static long[] hashOf(String s, long base, long mod) {
        long hash = 0;
        for (char c : s.toCharArray()) {
            hash = (hash * base + c) % mod;
        }
        return new long[] { hash };
    }
}
```

```
Hash pair for 'abc': (96354, 136518)
```

Two strings are only considered equal when **both** components of this pair match — the technique underlying robust solutions to hard problems like **Longest Duplicate Substring** (binary search on length + double-hashed rolling comparison to check whether a duplicate of that length exists).
