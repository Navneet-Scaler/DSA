# 4. String Traversal

Every string algorithm starts with a way to walk through the characters — forward, backward, from both ends at once, or word by word. This lesson covers each traversal shape with a representative problem, plus the different Java idioms for iterating characters.

## 1. Forward Traversal

### Theory

The default: walk index `0` to `length-1`. **Example — count vowels:** a single forward pass checking each character against a vowel set.

### Intuition

Whenever a problem asks for a total, count, or "does any character satisfy X" that doesn't depend on character *order* or *position relative to each other*, a single forward pass is the default first instinct — there's no reason to look backward or track state across positions, so reach for the simplest tool before considering anything fancier.

### Code

```java
public class ForwardTraversal {
    public static void main(String[] args) {
        System.out.println("Vowel count in 'hello world': " + countVowels("hello world"));
    }

    static int countVowels(String s) {
        String vowels = "aeiouAEIOU";
        int count = 0;
        for (int i = 0; i < s.length(); i++) {
            if (vowels.indexOf(s.charAt(i)) != -1) count++;
        }
        return count;
    }
}
```

### Example

```
Vowel count in 'hello world': 3
```

## 2. Reverse Traversal

### Theory

Walk `length-1` down to `0` — useful whenever logic depends on distance from the **end** rather than the start. **Example — reverse a string manually** (though `new StringBuilder(s).reverse()` is the idiomatic one-liner in real code, shown alongside for contrast).

### Intuition

The giveaway is the word "reverse" (or anything phrased in terms of "from the end", "last occurrence first", "suffix-relative position") — that's the signal to flip the loop direction rather than reach for extra data structures. Building the result by appending characters in reverse-visited order is the most direct translation of "reversed" into code.

### Code

```java
public class ReverseTraversal {
    public static void main(String[] args) {
        String s = "hello";
        System.out.println("Manual reverse: " + reverseManual(s));
        System.out.println("StringBuilder.reverse(): " + new StringBuilder(s).reverse());
    }

    static String reverseManual(String s) {
        StringBuilder result = new StringBuilder();
        for (int i = s.length() - 1; i >= 0; i--) {
            result.append(s.charAt(i));
        }
        return result.toString();
    }
}
```

### Example

```
Manual reverse: olleh
StringBuilder.reverse(): olleh
```

## 3. Two-Pass Traversal

### Theory

Some problems need information gathered from a **first** pass before a **second** pass can compute the answer. **Classic example — First Unique Character:** pass 1 builds a character frequency map (see [Lesson 5](<./05. Frequency Counting.md>)); pass 2 walks the string again looking for the first character whose frequency is exactly `1`.

### Intuition

The trap here is trying to answer "is this the first unique character?" while only looking **forward** from the current position — you can't know a character's *total* frequency without seeing the whole string first. Whenever a question needs a global property (total count, global max, "does this appear anywhere else") to decide something about *each* position, that's the signal for two passes: one to gather the global information, one to use it. Trying to force this into a single pass usually means silently computing the wrong thing.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class TwoPassTraversal {
    public static void main(String[] args) {
        System.out.println("First unique char index, 'leetcode': " + firstUniqChar("leetcode"));
        System.out.println("First unique char index, 'loveleetcode': " + firstUniqChar("loveleetcode"));
    }

    static int firstUniqChar(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum); // pass 1

        for (int i = 0; i < s.length(); i++) {                          // pass 2
            if (freq.get(s.charAt(i)) == 1) return i;
        }
        return -1;
    }
}
```

### Example

```
First unique char index, 'leetcode': 0
First unique char index, 'loveleetcode': 2
```

## 4. Bidirectional Traversal

### Theory

Two pointers moving **inward from both ends simultaneously** — the string equivalent of [Arrays: Opposite Direction](<../Arrays/08. Two Pointers.md#1-opposite-direction>). **Classic example — Reverse Vowels of a String:** only swap when **both** pointers currently sit on a vowel; otherwise advance whichever pointer isn't.

### Intuition

"Reverse **only** the vowels, but leave everything else exactly where it is" is the tell — a plain forward or reverse traversal can't do this alone, because consonants must stay fixed while vowels swap across arbitrary distances. Anchoring one pointer at each end and only advancing a pointer when it's *not* looking at something relevant (a vowel) is the natural way to "skip past" the characters that don't participate, while still processing the whole string in one pass.

### Code

```java
public class BidirectionalTraversal {
    public static void main(String[] args) {
        System.out.println("Reverse vowels 'hello': " + reverseVowels("hello"));
        System.out.println("Reverse vowels 'leetcode': " + reverseVowels("leetcode"));
    }

    static String reverseVowels(String s) {
        char[] arr = s.toCharArray();
        String vowels = "aeiouAEIOU";
        int left = 0, right = arr.length - 1;

        while (left < right) {
            if (vowels.indexOf(arr[left]) == -1) {
                left++;
            } else if (vowels.indexOf(arr[right]) == -1) {
                right--;
            } else {
                char temp = arr[left];
                arr[left] = arr[right];
                arr[right] = temp;
                left++;
                right--;
            }
        }
        return new String(arr);
    }
}
```

### Example

```
Reverse vowels 'hello': holle
Reverse vowels 'leetcode': leotcede
```

## 5. Character Iteration

### Theory

Java offers a few equivalent ways to iterate a string's characters, each with different trade-offs:

- **`charAt(i)` in a for-loop** — most explicit, O(1) per access, no extra allocation.
- **`toCharArray()` then enhanced for-loop** — copies into a `char[]` first (O(n) upfront), then iterates that array directly; slightly more idiomatic-looking, marginally faster for very hot loops since it avoids repeated method-call overhead.
- **`chars()` (returns an `IntStream`)** — the modern Stream-based approach, useful for chaining functional operations (`filter`, `map`, `count`) directly, at the cost of boxing/stream overhead for simple loops.

### Code

```java
public class CharacterIteration {
    public static void main(String[] args) {
        String s = "hello";

        System.out.print("charAt loop: ");
        for (int i = 0; i < s.length(); i++) System.out.print(s.charAt(i));
        System.out.println();

        System.out.print("toCharArray loop: ");
        for (char c : s.toCharArray()) System.out.print(c);
        System.out.println();

        long vowelCount = s.chars().filter(c -> "aeiou".indexOf(c) != -1).count();
        System.out.println("chars() stream, vowel count: " + vowelCount);
    }
}
```

### Example

```
charAt loop: hello
toCharArray loop: hello
chars() stream, vowel count: 2
```

## 6. Word Traversal

### Theory

Some problems operate on **words** rather than characters — split first ([Lesson 2: split()](<./02. Java String APIs.md#8-split>)), then traverse the resulting array. **Classic example — Reverse Words in a String:** split on whitespace (which also collapses multiple spaces and trims — `split("\\s+")` after a `trim()`, or simply `split(" ")` combined with filtering empties), reverse the word order, and rejoin.

### Intuition

The moment a problem talks about "words" instead of "characters" — reordering them, counting them, comparing them — treating individual characters as the unit of work makes the logic needlessly fiddly (tracking word boundaries by hand). Split into words first, so the *word* becomes the array element, and every array technique already covered ([Arrays series](<../Arrays/README.md>)) applies directly — here, reversing an array of words is just [Reversal](<../Arrays/02. Basic Operations.md#9-reversal>) one level up.

### Code

```java
import java.util.Arrays;
import java.util.Collections;
import java.util.List;

public class WordTraversal {
    public static void main(String[] args) {
        System.out.println("'" + reverseWords("the sky is blue") + "'");
        System.out.println("'" + reverseWords("  hello world  ") + "'");
    }

    static String reverseWords(String s) {
        String[] words = s.trim().split("\\s+");
        List<String> wordList = Arrays.asList(words);
        Collections.reverse(wordList);
        return String.join(" ", wordList);
    }
}
```

### Example

```
'blue is sky the'
'world hello'
```

Extra/leading/trailing spaces in the second example are cleaned up naturally by `trim()` + `split("\\s+")` before the reversal.
