# Strings for DSA

Strings are arrays of characters with their own quirks — immutability, encoding, and a huge family of pattern-matching and DP techniques that don't apply to plain arrays. This section goes from "what a string actually is in memory in Java" through the full pattern library: two pointers, sliding window, hashing, pattern matching (KMP, Z-algorithm, Rabin-Karp), palindromes, tries, and string-specific DP.

Each lesson follows the same structure as [Arrays](../Arrays/README.md) and [Maths](../Maths/README.md):

- **Theory** — the concept explained simply, with rules and edge cases.
- **Code** — a working Java implementation.
- **Example** — sample input, a dry run, and the output.

Most lessons cover a **progression of named problems per subtopic** (basic → advanced), and call out Java-specific idioms (`String` API methods, `StringBuilder`, `Collectors.joining`, etc.) alongside the raw algorithm.

> **Language used:** Java
