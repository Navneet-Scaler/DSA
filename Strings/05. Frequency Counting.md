## 1. Frequency Array

### Theory

When the character set is small and known (lowercase letters, digits), a fixed `int[26]` (or `int[128]` for ASCII) bucket array counts occurrences faster than a hash map — no hashing overhead, direct indexing via `c - 'a'`.

### Intuition

The question to ask is: "how many *distinct* characters could possibly appear?" If the answer is small and fixed (26 lowercase letters, 10 digits), an array indexed by `c - 'a'` is strictly better than a hash map — same idea as choosing [Array Frequency over HashMap Frequency](<../Arrays/04. Frequency Problems.md#3-array-frequency>) for bounded-range integers.

### Code

```java
public class FrequencyArray {
    public static void main(String[] args) {
        int[] freq = countFrequency("hello");
        System.out.println("'l' count: " + freq['l' - 'a']);
        System.out.println("'o' count: " + freq['o' - 'a']);
    }

    static int[] countFrequency(String s) {
        int[] freq = new int[26];
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }
        return freq;
    }
}
```

### Example

```
'l' count: 2
'o' count: 1
```

## 2. HashMap Frequency

### Theory

For an unbounded or unpredictable character set (Unicode text, mixed case, punctuation), a `HashMap<Character, Integer>` is the general-purpose fallback — full theory in [Arrays Lesson 4: HashMap Frequency](<../Arrays/04. Frequency Problems.md#2-hashmap-frequency>).

### Intuition

If the input could contain *anything* — emoji, accented letters, symbols — a fixed-size bucket array either wastes huge amounts of memory (indexing by full Unicode code point) or simply doesn't cover the input. A `HashMap` only pays for the distinct characters actually present, which is why it's the safe default whenever the character set isn't tightly bounded ahead of time.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapFrequencyString {
    public static void main(String[] args) {
        Map<Character, Integer> freq = countFrequency("mississippi");
        System.out.println("Frequency map: " + freq);
    }

    static Map<Character, Integer> countFrequency(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) {
            freq.merge(c, 1, Integer::sum);
        }
        return freq;
    }
}
```

### Example

```
Frequency map: {p=2, s=4, i=4, m=1}
```

## 3. HashSet

### Theory

When the question is purely "which distinct characters appear" (not how many times), a `HashSet<Character>` tracks presence without counting.

### Intuition

If a problem only ever asks "has this been seen?" or "how many *distinct* values are there?" — never "how many times" — carrying a full frequency count is wasted bookkeeping. Recognizing that the count itself is irrelevant to the answer is what tells you to reach for a `Set` instead of a `Map`.

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class HashSetString {
    public static void main(String[] args) {
        Set<Character> distinctChars = distinctCharacters("mississippi");
        System.out.println("Distinct characters: " + distinctChars);
        System.out.println("Distinct count: " + distinctChars.size());
    }

    static Set<Character> distinctCharacters(String s) {
        Set<Character> seen = new HashSet<>();
        for (char c : s.toCharArray()) seen.add(c);
        return seen;
    }
}
```

### Example

```
Distinct characters: [p, s, i, m]
Distinct count: 4
```

## 4. Counting Characters (Sort by Frequency)

### Theory

**Problem 1 (Basic): Most Frequent Character** — build a frequency map, then scan for the maximum. **Problem 2 (Advanced): Sort Characters By Frequency** — sort every *distinct* character by how often it occurs (descending), then rebuild the string repeating each character that many times.

### Intuition

For the basic version: once you have counts per character, "which is most frequent" is just a max-search over the map's values — no new idea beyond frequency counting itself. The advanced version adds a twist: the *output* needs characters grouped and ordered by count, which means the frequency map isn't the final answer — it's an intermediate step that then needs **sorting** (by count, descending) before rebuilding the string. Recognizing "count first, then sort by that count" is the bridge between simple frequency problems and this one.

### Code

```java
import java.util.*;

public class CountingCharacters {
    public static void main(String[] args) {
        System.out.println("Most frequent char in 'aabbbcc': " + mostFrequentChar("aabbbcc"));
        System.out.println("Sorted by frequency, 'tree': " + frequencySort("tree"));
    }

    static char mostFrequentChar(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);

        char best = s.charAt(0);
        int bestCount = 0;
        for (Map.Entry<Character, Integer> entry : freq.entrySet()) {
            if (entry.getValue() > bestCount) {
                bestCount = entry.getValue();
                best = entry.getKey();
            }
        }
        return best;
    }

    static String frequencySort(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);

        List<Character> chars = new ArrayList<>(freq.keySet());
        chars.sort((a, b) -> freq.get(b) - freq.get(a)); // descending by count

        StringBuilder result = new StringBuilder();
        for (char c : chars) {
            result.append(String.valueOf(c).repeat(freq.get(c)));
        }
        return result.toString();
    }
}
```

### Example

```
Most frequent char in 'aabbbcc': b
Sorted by frequency, 'tree': eetr
```

`'e'` appears twice (most frequent, so it comes first), followed by `'t'` and `'r'` (one each).

## 5. Counting Words

### Theory

**Classic example — Most Common Word:** given a paragraph and a list of banned words, find the most frequent word that isn't banned. Tokenize into lowercase words (stripping punctuation via regex), then apply the same frequency-counting pattern to whole words instead of characters.

### Intuition

This is character-frequency counting's twin, just at the word granularity — the only new work is getting clean word tokens out of messy punctuation-laden text before counting can even start. Once tokenized, "most common, excluding a banned set" is exactly [Most Frequent Character](#4-counting-characters-sort-by-frequency) with an extra filter condition applied during counting.

### Code

```java
import java.util.HashMap;
import java.util.HashSet;
import java.util.Map;
import java.util.Set;

public class CountingWords {
    public static void main(String[] args) {
        String paragraph = "Bob hit a ball, the hit BALL flew far after it was hit.";
        String[] banned = { "hit" };
        System.out.println("Most common non-banned word: " + mostCommonWord(paragraph, banned));
    }

    static String mostCommonWord(String paragraph, String[] banned) {
        Set<String> bannedSet = new HashSet<>();
        for (String b : banned) bannedSet.add(b.toLowerCase());

        String[] words = paragraph.toLowerCase().split("[^a-z]+");
        Map<String, Integer> freq = new HashMap<>();
        for (String word : words) {
            if (!word.isEmpty() && !bannedSet.contains(word)) {
                freq.merge(word, 1, Integer::sum);
            }
        }

        String best = null;
        int bestCount = 0;
        for (Map.Entry<String, Integer> entry : freq.entrySet()) {
            if (entry.getValue() > bestCount) {
                bestCount = entry.getValue();
                best = entry.getKey();
            }
        }
        return best;
    }
}
```

### Example

```
Most common non-banned word: ball
```

`"hit"` appears 3 times but is banned; `"ball"` appears twice (`"ball"` and `"BALL"`, matched case-insensitively) and wins.

## 6. Duplicate Characters

### Theory

**Classic example — Find All Duplicate Characters:** report every character that appears more than once. Build the frequency map, then filter for entries with count `> 1`.

### Intuition

This is frequency counting used as a **filter** rather than a search for an extremum (unlike "most frequent") — once you have the counts, the question "which ones repeat" is answered by scanning the map for a threshold (`count > 1`), not by finding a single maximum. Recognizing which of these two shapes ("find the best" vs. "find everything matching a condition") a problem needs tells you whether to track just one running answer or collect a full list.

### Code

```java
import java.util.HashMap;
import java.util.Map;
import java.util.TreeSet;

public class DuplicateCharacters {
    public static void main(String[] args) {
        System.out.println("Duplicates in 'programming': " + findDuplicates("programming"));
    }

    static TreeSet<Character> findDuplicates(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);

        TreeSet<Character> duplicates = new TreeSet<>(); // TreeSet keeps output sorted for a stable example
        for (Map.Entry<Character, Integer> entry : freq.entrySet()) {
            if (entry.getValue() > 1) duplicates.add(entry.getKey());
        }
        return duplicates;
    }
}
```

### Example

```
Duplicates in 'programming': [g, m, r]
```

## 7. Unique Characters

### Theory

**Classic example — Determine if a String Has All Unique Characters:** the mirror of duplicate detection — confirm **no** character repeats. The [HashSet](#3-hashset) approach works directly: if the set's size after adding every character equals the string's length, nothing collided.

### Intuition

"All unique" and "has a duplicate" are logical opposites, so it's tempting to reuse the exact same frequency-map code and just flip a condition — but since the *count* doesn't matter here (only "did this collide at all"), a `HashSet` is a cleaner match for the question than a full frequency map: add each character and bail out the instant `add()` reports it was already present, without ever needing to know exact counts.

### Code

```java
import java.util.HashSet;
import java.util.Set;

public class UniqueCharacters {
    public static void main(String[] args) {
        System.out.println("'abcdef' all unique: " + hasAllUniqueCharacters("abcdef"));
        System.out.println("'hello' all unique: " + hasAllUniqueCharacters("hello"));
    }

    static boolean hasAllUniqueCharacters(String s) {
        Set<Character> seen = new HashSet<>();
        for (char c : s.toCharArray()) {
            if (!seen.add(c)) return false; // add() returns false if c was already present
        }
        return true;
    }
}
```

### Example

```
'abcdef' all unique: true
'hello' all unique: false
```
