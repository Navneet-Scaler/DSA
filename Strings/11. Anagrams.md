# 11. Anagrams

Two strings are anagrams if one can be rearranged into the other — same characters, same counts, different order. This lesson works through the whole family: checking a single pair, grouping many strings by anagram class, finding anagram substrings inside a larger text, and measuring "how far" two strings are from being anagrams.

## 1. Anagram Check

### Theory

**Classic example — Valid Anagram.** Two approaches, both worth knowing:

- **Sorting:** sort both strings' characters — anagrams become **identical** strings. Simple, but **O(n log n)**.
- **Frequency counting:** tally each character's count in one string, then subtract the counts from the other — if every count lands back at zero, they're anagrams. **O(n)**, and the better default for interviews.

### Code

```java
import java.util.Arrays;

public class ValidAnagram {
    public static void main(String[] args) {
        System.out.println("Sort-based, 'anagram'/'nagaram': " + isAnagramSort("anagram", "nagaram"));
        System.out.println("Sort-based, 'rat'/'car': " + isAnagramSort("rat", "car"));
        System.out.println("Count-based, 'anagram'/'nagaram': " + isAnagramCount("anagram", "nagaram"));
    }

    // O(n log n) -- sorting approach
    static boolean isAnagramSort(String s, String t) {
        if (s.length() != t.length()) return false;
        char[] sChars = s.toCharArray();
        char[] tChars = t.toCharArray();
        Arrays.sort(sChars);
        Arrays.sort(tChars);
        return Arrays.equals(sChars, tChars);
    }

    // O(n) -- frequency-counting approach (assumes lowercase a-z)
    static boolean isAnagramCount(String s, String t) {
        if (s.length() != t.length()) return false;
        int[] count = new int[26];
        for (char c : s.toCharArray()) count[c - 'a']++;
        for (char c : t.toCharArray()) count[c - 'a']--;
        for (int c : count) if (c != 0) return false;
        return true;
    }
}
```

### Example

```
Sort-based, 'anagram'/'nagaram': true
Sort-based, 'rat'/'car': false
Count-based, 'anagram'/'nagaram': true
```

## 2. Group Anagrams

### Theory

**Classic example — Group Anagrams:** given a list of strings, group every anagram of each other together. Every anagram shares the same **sorted-character signature** (`"eat"` and `"tea"` both sort to `"aet"`), so that signature makes a natural `HashMap` key — bucket each string under its signature, then return the buckets.

### Code

```java
import java.util.*;

public class GroupAnagrams {
    public static void main(String[] args) {
        String[] strs = { "eat", "tea", "tan", "ate", "nat", "bat" };
        System.out.println("Groups: " + groupAnagrams(strs));
    }

    static List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> groups = new HashMap<>();
        for (String s : strs) {
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);
            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(groups.values());
    }
}
```

### Example

```
Groups: [[eat, tea, ate], [tan, nat], [bat]]
```

> **Java idiom:** `groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s)` replaces the classic "check `containsKey`, then either create or fetch, then add" four-line pattern with one line — a common cleanup once you're comfortable with lambdas. The equivalent using streams: `Arrays.stream(strs).collect(Collectors.groupingBy(s -> { char[] c = s.toCharArray(); Arrays.sort(c); return new String(c); }))` returns a `Map` directly via `Collectors.groupingBy`, Java's built-in "bucket by key" collector.

## 3. Sliding Window Anagram Search

### Theory

**Classic example — Find All Anagrams in a String:** find every starting index in `s` where a substring is an anagram of `p`. Since anagrams share the same character-count signature, slide a **fixed-size window** (size `= p.length()`) across `s`, maintaining a running frequency count — no need to re-sort or re-count from scratch at every position, just add the entering character and remove the leaving one.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class FindAllAnagrams {
    public static void main(String[] args) {
        System.out.println("Anagram start indices: " + findAnagrams("cbaebabacd", "abc"));
    }

    static List<Integer> findAnagrams(String s, String p) {
        List<Integer> result = new ArrayList<>();
        int n = s.length(), k = p.length();
        if (n < k) return result;

        int[] need = new int[26];
        int[] window = new int[26];
        for (char c : p.toCharArray()) need[c - 'a']++;

        for (int i = 0; i < n; i++) {
            window[s.charAt(i) - 'a']++;
            if (i >= k) {
                window[s.charAt(i - k) - 'a']--;
            }
            if (i >= k - 1 && java.util.Arrays.equals(need, window)) {
                result.add(i - k + 1);
            }
        }
        return result;
    }
}
```

### Example

```
Anagram start indices: [0, 6]
```

`"cba"` (index 0) and `"bac"` (index 6) are both anagrams of `"abc"`.

### Problem 2 (Medium): Permutation in String

**Theory:** A close variant — instead of finding **all** anagram positions, just check whether `s2` **contains** any permutation of `s1` at all (return as soon as one is found). Identical fixed-size sliding window; the only change is stopping early with `true` on the first match instead of collecting every index.

```java
public class PermutationInString {
    public static void main(String[] args) {
        System.out.println("'ab' permutation in 'eidbaooo': " + checkInclusion("ab", "eidbaooo"));
        System.out.println("'ab' permutation in 'eidboaoo': " + checkInclusion("ab", "eidboaoo"));
    }

    static boolean checkInclusion(String s1, String s2) {
        int n1 = s1.length(), n2 = s2.length();
        if (n1 > n2) return false;

        int[] need = new int[26];
        int[] window = new int[26];
        for (char c : s1.toCharArray()) need[c - 'a']++;

        for (int i = 0; i < n2; i++) {
            window[s2.charAt(i) - 'a']++;
            if (i >= n1) {
                window[s2.charAt(i - n1) - 'a']--;
            }
            if (i >= n1 - 1 && java.util.Arrays.equals(need, window)) {
                return true;
            }
        }
        return false;
    }
}
```

```
'ab' permutation in 'eidbaooo': true
'ab' permutation in 'eidboaoo': false
```

`"eidbaooo"` contains `"ba"` (an anagram of `"ab"`) at index 3; `"eidboaoo"` has no contiguous 2-character window that's a permutation of `"ab"`.

## 4. Minimum Steps to Make an Anagram

### Theory

**Classic example — Minimum Number of Steps to Make Two Strings Anagram:** given two same-length strings, find the minimum number of characters to change in one so it becomes an anagram of the other. Count characters in both strings and subtract; every character where the first string has a **surplus** (positive difference) represents one character that must be changed — sum only the positive differences (summing all differences, positive and negative, would double-count or cancel out incorrectly).

### Code

```java
public class MinStepsToMakeAnagram {
    public static void main(String[] args) {
        System.out.println("Min steps, 'bab'->'aba': " + minSteps("bab", "aba"));
        System.out.println("Min steps, 'leetcode'->'practice': " + minSteps("leetcode", "practice"));
    }

    static int minSteps(String s, String t) {
        int[] count = new int[26];
        for (char c : s.toCharArray()) count[c - 'a']++;
        for (char c : t.toCharArray()) count[c - 'a']--;

        int steps = 0;
        for (int c : count) {
            if (c > 0) steps += c;
        }
        return steps;
    }
}
```

### Example

```
Min steps, 'bab'->'aba': 1
Min steps, 'leetcode'->'practice': 5
```

`"bab"` (`b:2, a:1`) vs `"aba"` (`a:2, b:1`): `s` has one surplus `b` — changing it to `a` makes them anagrams, so `1` step.

## 5. Generalized Anagram Distance

### Theory

**Classic example — K-Anagrams:** given two same-length strings and an integer `k`, determine whether one can be turned into an anagram of the other by changing **at most `k`** characters. This directly reuses [Minimum Steps to Make an Anagram](#4-minimum-steps-to-make-an-anagram)'s exact computation — the "minimum steps" value **is** the minimum number of changes required, so the check is simply whether that value is `≤ k`.

### Code

```java
public class KAnagrams {
    public static void main(String[] args) {
        System.out.println("'anagram'->'grammar' within 2 changes: " + areKAnagrams("anagram", "grammar", 2));
        System.out.println("'anagram'->'grammar' within 1 change: " + areKAnagrams("anagram", "grammar", 1));
    }

    static boolean areKAnagrams(String s1, String s2, int k) {
        if (s1.length() != s2.length()) return false;

        int[] count = new int[26];
        for (char c : s1.toCharArray()) count[c - 'a']++;
        for (char c : s2.toCharArray()) count[c - 'a']--;

        int surplus = 0;
        for (int c : count) {
            if (c > 0) surplus += c;
        }
        return surplus <= k;
    }
}
```

### Example

```
'anagram'->'grammar' within 2 changes: true
'anagram'->'grammar' within 1 change: false
```

`"anagram"` (`a:3,n:1,g:1,r:1,m:1`) vs `"grammar"` (`g:1,r:2,a:2,m:2`): surplus characters in `"anagram"` are one extra `a` and one extra `n` — `2` changes needed, so it fits within `k=2` but not `k=1`.
