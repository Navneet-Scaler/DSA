## 1. Opposite Ends

### Theory

**Classic example — Reverse String (in-place):** two pointers start at both ends of a character array and swap inward until they meet — the string-specific instance of [Arrays: Reversal](<../Arrays/02. Basic Operations.md#9-reversal>).

### Intuition

Reversal is the textbook opposite-ends signal: the character that belongs at position `0` in the output is whatever currently sits at the **last** position, and vice versa — so working from both ends inward finishes the whole swap in a single pass, touching each element exactly once.

### Code

```java
import java.util.Arrays;

public class OppositeEnds {
    public static void main(String[] args) {
        char[] chars = { 'h', 'e', 'l', 'l', 'o' };
        reverseString(chars);
        System.out.println(Arrays.toString(chars));
    }

    static void reverseString(char[] chars) {
        int left = 0, right = chars.length - 1;
        while (left < right) {
            char temp = chars[left];
            chars[left] = chars[right];
            chars[right] = temp;
            left++;
            right--;
        }
    }
}
```

### Example

```
[o, l, l, e, h]
```

## 2. Same Direction

### Theory

**Classic example — Is Subsequence:** given strings `s` and `t`, check whether `s`'s characters appear in `t`, in order (not necessarily contiguously). Two pointers, `i` for `s` and `j` for `t`, both **only ever move forward**: `j` scans every character of `t`, and `i` advances only when the characters match.

### Intuition

The key realization: you never need to backtrack in `t` to check subsequence membership — once a character in `t` is consumed as a match (or skipped as a non-match), it's irrelevant to everything after. That "never go backward" property is exactly what licenses a single forward pass with two independently-advancing pointers, instead of anything resembling backtracking or recursion.

### Code

```java
public class SameDirectionTwoPointers {
    public static void main(String[] args) {
        System.out.println("'abc' subsequence of 'ahbgdc': " + isSubsequence("abc", "ahbgdc"));
        System.out.println("'axc' subsequence of 'ahbgdc': " + isSubsequence("axc", "ahbgdc"));
    }

    static boolean isSubsequence(String s, String t) {
        int i = 0;
        for (int j = 0; j < t.length() && i < s.length(); j++) {
            if (s.charAt(i) == t.charAt(j)) {
                i++;
            }
        }
        return i == s.length();
    }
}
```

### Example

```
'abc' subsequence of 'ahbgdc': true
'axc' subsequence of 'ahbgdc': false
```

## 3. Skip Characters

### Theory

**Classic example — Backspace String Compare:** given two strings containing `#` (representing a backspace, deleting the previous character), determine if they produce the same final text. Processing from the **end backward** with a pointer per string, skipping characters that get "eaten" by a pending backspace count, avoids building the fully-processed strings at all.

### Intuition

Processing forward is awkward because a `#` deletes something you've *already* emitted — you'd need to undo output you already produced. Flipping to a backward pass sidesteps this entirely: reading from the end, a `#` just means "skip the next real character you encounter" (track a pending-skip counter), so you only ever emit characters that are guaranteed to survive, never needing to retract anything.

### Code

```java
public class SkipCharacters {
    public static void main(String[] args) {
        System.out.println("'ab#c' == 'ad#c': " + backspaceCompare("ab#c", "ad#c"));
    }

    static boolean backspaceCompare(String s, String t) {
        int i = s.length() - 1, j = t.length() - 1;

        while (i >= 0 || j >= 0) {
            i = nextValidIndex(s, i);
            j = nextValidIndex(t, j);

            if (i >= 0 && j >= 0) {
                if (s.charAt(i) != t.charAt(j)) return false;
            } else if (i >= 0 || j >= 0) {
                return false; // one string ran out before the other
            }
            i--;
            j--;
        }
        return true;
    }

    static int nextValidIndex(String s, int index) {
        int skip = 0;
        while (index >= 0) {
            if (s.charAt(index) == '#') {
                skip++;
                index--;
            } else if (skip > 0) {
                skip--;
                index--;
            } else {
                break; // found a character that survives
            }
        }
        return index;
    }
}
```

### Example

```
'ab#c' == 'ad#c': true
```

Both reduce to `"ac"` — `"ab#c"` deletes the `b`, `"ad#c"` deletes the `d`.

## 4. Palindrome Checking

### Theory

**Problem 1 (Basic): Valid Palindrome** — check whether a string reads the same forward and backward, ignoring case and non-alphanumeric characters. Full code already in [Arrays Lesson 8: Opposite Direction](<../Arrays/08. Two Pointers.md#1-opposite-direction>). **Problem 2 (Advanced): Valid Palindrome II** — same check, but allow **deleting at most one character** to make it a palindrome.

### Intuition

The basic check is a direct opposite-ends scan. The advanced twist — "at most one deletion allowed" — means the very first mismatch isn't necessarily a failure: it could be fixed by dropping *either* the left or the right character at that mismatch point. So the natural extension is: on the first mismatch, try both "skip the left character" and "skip the right character" as separate candidate palindromes, and succeed if **either** one checks out from there. This only needs to branch once (any further mismatch means no single deletion could have fixed it).

### Code

```java
public class PalindromeChecking {
    public static void main(String[] args) {
        System.out.println("'abca' valid with one deletion: " + validPalindrome("abca"));
    }

    static boolean validPalindrome(String s) {
        int left = 0, right = s.length() - 1;
        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) {
                return isPalindromeRange(s, left + 1, right) || isPalindromeRange(s, left, right - 1);
            }
            left++;
            right--;
        }
        return true;
    }

    static boolean isPalindromeRange(String s, int left, int right) {
        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) return false;
            left++;
            right--;
        }
        return true;
    }
}
```

### Example

```
'abca' valid with one deletion: true
```

`"abca"` isn't a palindrome as-is, but deleting either the `b` (giving `"aca"`) or the `c` (giving `"aba"`) makes it one.

## 5. String Comparison

### Theory

**Classic example — Compare Version Numbers:** given two version strings like `"1.01"` and `"1.001"`, compare them numerically segment-by-segment (dot-separated), where each segment is compared as an **integer** (so leading zeros don't affect the comparison) and a missing trailing segment counts as `0`.

### Intuition

Treating version strings as plain strings and comparing lexicographically fails immediately (`"1.10"` vs `"1.9"` would compare wrong character-by-character). The fix is recognizing the string is really a **sequence of numbers** in disguise — split on `.`, parse each segment to an integer to strip leading zeros and get correct numeric comparison, and walk both segment sequences with two pointers, treating a shorter version's missing segments as implicit zeros.

### Code

```java
public class StringComparison {
    public static void main(String[] args) {
        System.out.println("compareVersion('1.01', '1.001'): " + compareVersion("1.01", "1.001"));
        System.out.println("compareVersion('1.0', '1.0.0'): " + compareVersion("1.0", "1.0.0"));
    }

    static int compareVersion(String version1, String version2) {
        String[] parts1 = version1.split("\\.");
        String[] parts2 = version2.split("\\.");
        int n = Math.max(parts1.length, parts2.length);

        for (int i = 0; i < n; i++) {
            int v1 = (i < parts1.length) ? Integer.parseInt(parts1[i]) : 0;
            int v2 = (i < parts2.length) ? Integer.parseInt(parts2[i]) : 0;
            if (v1 != v2) return Integer.compare(v1, v2);
        }
        return 0;
    }
}
```

### Example

```
compareVersion('1.01', '1.001'): 0
compareVersion('1.0', '1.0.0'): 0
```

`"1.01"` parses to segment `1` (leading zero stripped), same as `"1.001"`'s `1`; `"1.0.0"`'s extra `.0` segment is treated as an implicit `0` on the shorter version, so both compare equal.

## 6. Merge Strings

### Theory

**Classic example — Merge Strings Alternately:** given two strings, build a new string by alternating characters from each, appending any leftover tail once the shorter string is exhausted.

### Intuition

This is the string-specific case of [Arrays: Merge](<../Arrays/02. Basic Operations.md#10-merge>), but simpler — merging *sorted* arrays needs a comparison to decide which side to take next, while alternating merge doesn't compare anything; it just strictly alternates turns. Two pointers, one per string, each advancing on its own turn, naturally stop needing special-casing once you let the loop condition be "either pointer still has characters left" rather than "both."

### Code

```java
public class MergeStrings {
    public static void main(String[] args) {
        System.out.println("Merge 'abc' & 'pqr': " + mergeAlternately("abc", "pqr"));
        System.out.println("Merge 'ab' & 'pqrs': " + mergeAlternately("ab", "pqrs"));
    }

    static String mergeAlternately(String word1, String word2) {
        StringBuilder result = new StringBuilder();
        int i = 0, j = 0;

        while (i < word1.length() || j < word2.length()) {
            if (i < word1.length()) result.append(word1.charAt(i++));
            if (j < word2.length()) result.append(word2.charAt(j++));
        }
        return result.toString();
    }
}
```

### Example

```
Merge 'abc' & 'pqr': apbqcr
Merge 'ab' & 'pqrs': apbqrs
```
