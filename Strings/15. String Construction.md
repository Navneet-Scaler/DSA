# 15. String Construction

Some string problems aren't about *analyzing* a given string but *building* a new one according to a rule — digit-by-digit addition, describing a string by its own run-lengths, laying characters into a zigzag pattern, or padding words into evenly justified lines. This lesson covers that construction family, always with `StringBuilder` as the tool doing the actual building.

## 1. Efficient String Building

### Theory

Java `String`s are **immutable** — every `+` concatenation in a loop creates a **brand-new** string object, copying all previous characters again, turning what looks like an O(n) loop into O(n²) total work. `StringBuilder` avoids this: it holds a **mutable, resizable character buffer**, so `.append()` is amortized O(1), and `.toString()` builds the final immutable `String` only once, at the end. Every problem in this lesson uses `StringBuilder` for exactly this reason.

```java
public class EfficientStringBuilding {
    public static void main(String[] args) {
        // Inefficient: O(n^2) -- each += creates an entirely new String
        String slow = "";
        for (int i = 0; i < 5; i++) {
            slow += i;
        }

        // Efficient: O(n) amortized -- one mutable buffer, one final toString()
        StringBuilder fast = new StringBuilder();
        for (int i = 0; i < 5; i++) {
            fast.append(i);
        }

        System.out.println("Built (slow way): " + slow);
        System.out.println("Built (StringBuilder): " + fast);
    }
}
```

```
Built (slow way): 01234
Built (StringBuilder): 01234
```

Same output either way — the difference is entirely about performance at scale, which is why every construction problem below reaches for `StringBuilder` by default.

## 2. Digit-by-Digit Construction

### Theory

**Classic example — Add Strings / Add Binary:** add two numbers given as strings (too large to fit in an `int`/`long`, or in a non-decimal base), without converting to a numeric type. Simulate elementary-school addition: walk both strings **from the end**, adding corresponding digits plus a carry, appending each result digit to a `StringBuilder`, then **reverse** the builder at the end (since digits were naturally produced least-significant-first).

### Code

```java
public class AddStringsAndBinary {
    public static void main(String[] args) {
        System.out.println("addStrings('11','123'): " + addStrings("11", "123"));
        System.out.println("addBinary('11','1'): " + addBinary("11", "1"));
    }

    static String addStrings(String num1, String num2) {
        StringBuilder result = new StringBuilder();
        int i = num1.length() - 1, j = num2.length() - 1, carry = 0;

        while (i >= 0 || j >= 0 || carry != 0) {
            int d1 = i >= 0 ? num1.charAt(i--) - '0' : 0;
            int d2 = j >= 0 ? num2.charAt(j--) - '0' : 0;
            int total = d1 + d2 + carry;
            result.append(total % 10);
            carry = total / 10;
        }
        return result.reverse().toString();
    }

    static String addBinary(String a, String b) {
        StringBuilder result = new StringBuilder();
        int i = a.length() - 1, j = b.length() - 1, carry = 0;

        while (i >= 0 || j >= 0 || carry != 0) {
            int d1 = i >= 0 ? a.charAt(i--) - '0' : 0;
            int d2 = j >= 0 ? b.charAt(j--) - '0' : 0;
            int total = d1 + d2 + carry;
            result.append(total % 2);
            carry = total / 2;
        }
        return result.reverse().toString();
    }
}
```

### Example

```
addStrings('11','123'): 134
addBinary('11','1'): 100
```

`"11" + "123"`: aligning from the right, `1+3=4`, `1+2=3`, `0+1=1` → `"134"`. `"11" + "1"` in binary: `1+1=10` (write `0`, carry `1`), `1+0+1=10` (write `0`, carry `1`), then the leftover carry `1` → `"100"`.

## 3. Count and Say

### Theory

**Classic example — Count and Say:** generate the `n`th term of a sequence where each term is the **"read-aloud" description** of the previous one — counting consecutive runs of the same digit and saying "count, digit". `"1"` → "one 1" → `"11"` → "two 1s" → `"21"` → "one 2, one 1" → `"1211"`, and so on. Each step is a run-length scan of the previous string, appending `(count, digit)` pairs to a `StringBuilder`.

### Code

```java
public class CountAndSay {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            System.out.println("countAndSay(" + i + "): " + countAndSay(i));
        }
    }

    static String countAndSay(int n) {
        String s = "1";
        for (int iteration = 1; iteration < n; iteration++) {
            StringBuilder next = new StringBuilder();
            int i = 0;
            while (i < s.length()) {
                char digit = s.charAt(i);
                int count = 0;
                while (i < s.length() && s.charAt(i) == digit) {
                    count++;
                    i++;
                }
                next.append(count).append(digit);
            }
            s = next.toString();
        }
        return s;
    }
}
```

### Example

```
countAndSay(1): 1
countAndSay(2): 11
countAndSay(3): 21
countAndSay(4): 1211
countAndSay(5): 111221
```

`"1211"` reads as "one 1, one 2, two 1s" → `"111221"`, matching term 5.

## 4. Zigzag Conversion

### Theory

**Classic example — Zigzag Conversion:** write a string in a zigzag pattern across `numRows` rows (down one column, then diagonally up, repeating), then read it back **row by row**. Simulate directly: maintain a `StringBuilder` **per row**, and a "current row" pointer that increments while going down and decrements while going up, flipping direction whenever it hits the top or bottom row. Concatenating the row builders at the end produces the final zigzag-read string.

### Code

```java
public class ZigzagConversion {
    public static void main(String[] args) {
        System.out.println("convert('PAYPALISHIRING', 3): " + convert("PAYPALISHIRING", 3));
        System.out.println("convert('PAYPALISHIRING', 4): " + convert("PAYPALISHIRING", 4));
    }

    static String convert(String s, int numRows) {
        if (numRows == 1) return s;

        StringBuilder[] rows = new StringBuilder[numRows];
        for (int i = 0; i < numRows; i++) rows[i] = new StringBuilder();

        int currentRow = 0;
        boolean goingDown = false;

        for (char c : s.toCharArray()) {
            rows[currentRow].append(c);
            if (currentRow == 0 || currentRow == numRows - 1) {
                goingDown = !goingDown;
            }
            currentRow += goingDown ? 1 : -1;
        }

        StringBuilder result = new StringBuilder();
        for (StringBuilder row : rows) result.append(row);
        return result.toString();
    }
}
```

### Example

```
convert('PAYPALISHIRING', 3): PAHNAPLSIIGYIR
convert('PAYPALISHIRING', 4): PINALSIGYAHRPI
```

With `numRows=3`, `"PAYPALISHIRING"` zigzags as:
```
P   A   H   N
A P L S I I G
Y   I   R
```
Reading row by row: `"PAHN" + "APLSIIG" + "YIR"` = `"PAHNAPLSIIGYIR"`.

## 5. Text Justification

### Theory

**Classic example — Text Justification (advanced):** given a list of words and a maximum line width, arrange them into fully justified lines — each line except the last padded with spaces so it's **exactly** `maxWidth` characters, with extra spaces distributed as evenly as possible **left to right** when they don't divide evenly. Greedily pack as many words as fit on each line, then handle three cases: the **last line** (or any line with only one word) is left-justified with trailing spaces; every other line divides `totalSpaces` among the `gaps` between words, giving any remainder (`totalSpaces % gaps`) to the leftmost gaps first.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class TextJustification {
    public static void main(String[] args) {
        String[] words = { "This", "is", "an", "example", "of", "text", "justification." };
        for (String line : fullJustify(words, 16)) {
            System.out.println("'" + line + "'");
        }
    }

    static List<String> fullJustify(String[] words, int maxWidth) {
        List<String> result = new ArrayList<>();
        int i = 0, n = words.length;

        while (i < n) {
            int j = i, lineLength = 0;
            while (j < n && lineLength + words[j].length() + (j - i) <= maxWidth) {
                lineLength += words[j].length();
                j++;
            }

            int numWords = j - i;
            StringBuilder line = new StringBuilder();

            if (numWords == 1 || j == n) {
                // Left-justify: single word, or the last line
                for (int k = i; k < j; k++) {
                    if (k > i) line.append(' ');
                    line.append(words[k]);
                }
                while (line.length() < maxWidth) line.append(' ');
            } else {
                int totalChars = lineLength;
                int totalSpaces = maxWidth - totalChars;
                int gaps = numWords - 1;
                int spaceEach = totalSpaces / gaps;
                int extra = totalSpaces % gaps;

                for (int k = i; k < j - 1; k++) {
                    line.append(words[k]);
                    int spaces = spaceEach + (k - i < extra ? 1 : 0);
                    for (int s = 0; s < spaces; s++) line.append(' ');
                }
                line.append(words[j - 1]);
            }

            result.add(line.toString());
            i = j;
        }
        return result;
    }
}
```

### Example

```
'This    is    an'
'example  of text'
'justification.  '
```

Line 1 (`"This"`, `"is"`, `"an"`): `8` characters of words, `8` spaces to distribute across `2` gaps → `4` spaces each, evenly — matches `"This" + "    " + "is" + "    " + "an"`. The last line is left-justified with trailing spaces padding it out to exactly `16` characters instead of being evenly distributed.
