## 1. Run-Length Encoding

### Theory

**Run-length encoding (RLE)** replaces consecutive repeated characters with the character followed by its count — `"aaabbbccd"` becomes `"a3b3c2d"` (a lone character, with no repeats, is written without a count). Scan through, counting how long each run of identical characters lasts, and append `(character, count)` to a `StringBuilder`.

### Code

```java
public class RunLengthEncode {
    public static void main(String[] args) {
        System.out.println("Encoded 'aaabbbccd': " + encode("aaabbbccd"));
        System.out.println("Encoded 'abc': " + encode("abc"));
    }

    static String encode(String s) {
        StringBuilder result = new StringBuilder();
        int i = 0;
        while (i < s.length()) {
            char c = s.charAt(i);
            int count = 0;
            while (i < s.length() && s.charAt(i) == c) {
                count++;
                i++;
            }
            result.append(c);
            if (count > 1) result.append(count);
        }
        return result.toString();
    }
}
```

### Example

```
Encoded 'aaabbbccd': a3b3c2d
Encoded 'abc': abc
```

`"abc"` has no repeated runs, so its "encoded" form is unchanged — RLE only helps when repetition is actually present.

## 2. Run-Length Decoding

### Theory

The inverse of [Run-Length Encoding](#1-run-length-encoding): reconstruct the original string from a `character, count` sequence. Read a character, then greedily read all following digits to form its count (a run with no digits means count `1`), and append the character repeated that many times.

### Code

```java
public class RunLengthDecode {
    public static void main(String[] args) {
        System.out.println("Decoded 'a3b3c2d1': " + decode("a3b3c2d1"));
        System.out.println("Decoded 'abc': " + decode("abc"));
    }

    static String decode(String s) {
        StringBuilder result = new StringBuilder();
        int i = 0;
        while (i < s.length()) {
            char c = s.charAt(i);
            i++;
            int numStart = i;
            while (i < s.length() && Character.isDigit(s.charAt(i))) i++;
            int count = (i > numStart) ? Integer.parseInt(s.substring(numStart, i)) : 1;
            result.append(String.valueOf(c).repeat(count));
        }
        return result.toString();
    }
}
```

### Example

```
Decoded 'a3b3c2d1': aaabbbccd
Decoded 'abc': abc
```

> **Java idiom:** `String.valueOf(c).repeat(count)` (Java 11+) replaces a manual character-repeating loop.

## 3. In-Place String Compression

### Theory

**Classic example — String Compression:** given a `char[]` array, compress it **in place** (no auxiliary array) using the same run-length idea, and return the new length. Use two pointers: `read` scans forward counting each run, `write` places the compressed output (character, then digits of the count if `> 1`) directly back into the same array — since the compressed form is never longer than the original, `write` never overtakes `read`.

### Code

```java
public class InPlaceStringCompression {
    public static void main(String[] args) {
        char[] chars1 = { 'a', 'a', 'b', 'b', 'c', 'c', 'c' };
        int len1 = compress(chars1);
        System.out.println("Compressed length: " + len1 + ", result: "
                + new String(chars1, 0, len1));

        char[] chars2 = { 'a' };
        int len2 = compress(chars2);
        System.out.println("Compressed length: " + len2 + ", result: "
                + new String(chars2, 0, len2));
    }

    static int compress(char[] chars) {
        int write = 0, read = 0, n = chars.length;

        while (read < n) {
            char c = chars[read];
            int count = 0;
            while (read < n && chars[read] == c) {
                read++;
                count++;
            }
            chars[write++] = c;
            if (count > 1) {
                for (char digit : String.valueOf(count).toCharArray()) {
                    chars[write++] = digit;
                }
            }
        }
        return write;
    }
}
```

### Example

```
Compressed length: 6, result: a2b2c3
Compressed length: 1, result: a
```

`['a','a','b','b','c','c','c']` compresses to `"a2b2c3"` (length `6`, down from `7`). A single `'a'` with no repeats stays `"a"` — count `1` is never written.

## 4. Encode and Decode Strings

### Theory

**Classic example — Encode and Decode Strings:** pack a `List<String>` into a **single** string for transmission/storage, and reverse the process, correctly handling strings that may contain **any** character (including delimiters like `#` or commas that a naive join would misinterpret as a separator). The robust technique: **length-prefix** each string (`"5#hello"` means "the next 5 characters are the string"), so the decoder always knows exactly where one string ends and the next begins — no character in the payload can be mistaken for a delimiter, since the delimiter `#` is only ever read immediately after a length number.

### Code

```java
import java.util.ArrayList;
import java.util.List;

public class EncodeDecodeStrings {
    public static void main(String[] args) {
        List<String> strs = List.of("hello", "world", "a#b");
        String encoded = encode(strs);
        System.out.println("Encoded: " + encoded);
        System.out.println("Decoded: " + decode(encoded));
    }

    static String encode(List<String> strs) {
        StringBuilder result = new StringBuilder();
        for (String s : strs) {
            result.append(s.length()).append('#').append(s);
        }
        return result.toString();
    }

    static List<String> decode(String s) {
        List<String> result = new ArrayList<>();
        int i = 0;
        while (i < s.length()) {
            int j = i;
            while (s.charAt(j) != '#') j++;
            int length = Integer.parseInt(s.substring(i, j));
            result.add(s.substring(j + 1, j + 1 + length));
            i = j + 1 + length;
        }
        return result;
    }
}
```

### Example

```
Encoded: 5#hello5#world3#a#b
Decoded: [hello, world, a#b]
```

`"a#b"` contains a `#` character, yet decodes correctly — because the decoder always reads exactly `3` characters right after seeing `"3#"`, it never re-interprets the embedded `#` as a new delimiter.

## 5. UTF-8 Validation

### Theory

**Classic example — UTF-8 Validation:** given a sequence of integers representing bytes, determine if it forms a valid UTF-8 encoding. UTF-8 encodes each character in **1 to 4 bytes**, where the **leading byte's high bits** announce the total byte count (`0xxxxxxx` = 1 byte, `110xxxxx` = 2 bytes, `1110xxxx` = 3 bytes, `11110xxx` = 4 bytes), and every **continuation byte** must match the pattern `10xxxxxx`. Track how many continuation bytes are still expected; check each byte against the pattern its role requires, and confirm the count reaches exactly zero by the end (a sequence cut off mid-character is invalid).

### Code

```java
public class UTF8Validation {
    public static void main(String[] args) {
        System.out.println("validUtf8([197,130,1]): " + validUtf8(new int[] { 197, 130, 1 }));
        System.out.println("validUtf8([235,140,4]): " + validUtf8(new int[] { 235, 140, 4 }));
    }

    static boolean validUtf8(int[] data) {
        int remaining = 0;

        for (int b : data) {
            if (remaining == 0) {
                if ((b >> 7) == 0b0) {
                    remaining = 0;
                } else if ((b >> 5) == 0b110) {
                    remaining = 1;
                } else if ((b >> 4) == 0b1110) {
                    remaining = 2;
                } else if ((b >> 3) == 0b11110) {
                    remaining = 3;
                } else {
                    return false;
                }
            } else {
                if ((b >> 6) != 0b10) return false;
                remaining--;
            }
        }
        return remaining == 0;
    }
}
```

### Example

```
validUtf8([197,130,1]): true
validUtf8([235,140,4]): false
```

`197` = `11000101` announces a 2-byte character; `130` = `10000010` is a valid continuation byte; `1` = `00000001` is a valid standalone 1-byte character — all valid. `235` = `11101011` announces a 3-byte character, but only `140` (a valid continuation) follows before `4` = `00000100`, which is a **new** 1-byte character starting before the 3-byte sequence finished — invalid.

## Quick Reference Cheat Sheet

| Technique | Complexity | Problems covered |
|---|---|---|
| Run-Length Encoding | O(n) | basic character/count encoding |
| Run-Length Decoding | O(n) | inverse of encoding (`String.repeat`) |
| In-Place String Compression | O(n), O(1) extra space | String Compression (two-pointer read/write) |
| Encode and Decode Strings | O(n) | length-prefix framing, delimiter-safe |
