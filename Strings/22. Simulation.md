# 22. Simulation

Simulation problems don't need a clever algorithmic trick — they need **careful, literal step-by-step execution** of exactly what the problem describes: typing keystrokes (including backspace), walking a robot's move sequence, resolving a Unix file path one segment at a time. The skill here is translating the described process directly into code without overcomplicating it.

## 1. Backspace String Compare

### Theory

**Classic example — Backspace String Compare:** given two strings where `#` represents a backspace keystroke, determine if they produce the same final text once all backspaces are applied. Simulate typing with a stack: push a regular character, and on `#`, pop the last character (if any exist — backspacing on empty input does nothing). Build the final result for both strings this way, then compare.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class BackspaceStringCompare {
    public static void main(String[] args) {
        System.out.println("backspaceCompare('ab#c','ad#c'): " + backspaceCompare("ab#c", "ad#c"));
        System.out.println("backspaceCompare('ab##','c#d#'): " + backspaceCompare("ab##", "c#d#"));
        System.out.println("backspaceCompare('a#c','b'): " + backspaceCompare("a#c", "b"));
    }

    static boolean backspaceCompare(String s, String t) {
        return buildFinal(s).equals(buildFinal(t));
    }

    static String buildFinal(String s) {
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c != '#') {
                stack.push(c);
            } else if (!stack.isEmpty()) {
                stack.pop();
            }
        }
        StringBuilder result = new StringBuilder();
        while (!stack.isEmpty()) result.append(stack.pop());
        return result.reverse().toString();
    }
}
```

### Example

```
backspaceCompare('ab#c','ad#c'): true
backspaceCompare('ab##','c#d#'): true
backspaceCompare('a#c','b'): false
```

`"ab#c"` types `a`, `b`, backspaces `b`, types `c` → `"ac"`. `"ad#c"` types `a`, `d`, backspaces `d`, types `c` → also `"ac"` — they match despite looking completely different before simulation.

## 2. Robot Return to Origin

### Theory

**Classic example — Robot Return to Origin:** given a sequence of moves (`U`, `D`, `L`, `R`), determine whether a robot starting at `(0,0)` ends back at the origin after executing all of them. Direct simulation: track `x` and `y`, adjusting by `±1` per matching move character, and check both are `0` at the end.

### Code

```java
public class RobotReturnToOrigin {
    public static void main(String[] args) {
        System.out.println("judgeCircle('UD'): " + judgeCircle("UD"));
        System.out.println("judgeCircle('LL'): " + judgeCircle("LL"));
    }

    static boolean judgeCircle(String moves) {
        int x = 0, y = 0;
        for (char move : moves.toCharArray()) {
            switch (move) {
                case 'U' -> y++;
                case 'D' -> y--;
                case 'L' -> x--;
                case 'R' -> x++;
            }
        }
        return x == 0 && y == 0;
    }
}
```

### Example

```
judgeCircle('UD'): true
judgeCircle('LL'): false
```

`"UD"` moves up then down, canceling out. `"LL"` moves left twice with no return move, ending at `x = -2`.

## 3. Rotate String

### Theory

**Classic example — Rotate String:** determine if `goal` can be obtained by rotating `s` some number of positions. Rather than trying every rotation offset manually, use a neat trick: **every rotation of `s` is a substring of `s + s`** (concatenating `s` with itself lines up every possible rotation point contiguously) — so the check reduces to a single substring search, provided the lengths match first (different lengths can never be rotations of each other).

### Code

```java
public class RotateString {
    public static void main(String[] args) {
        System.out.println("rotateString('abcde','cdeab'): " + rotateString("abcde", "cdeab"));
        System.out.println("rotateString('abcde','abced'): " + rotateString("abcde", "abced"));
    }

    static boolean rotateString(String s, String goal) {
        return s.length() == goal.length() && (s + s).contains(goal);
    }
}
```

### Example

```
rotateString('abcde','cdeab'): true
rotateString('abcde','abced'): false
```

`"abcde" + "abcde"` = `"abcdeabcde"` contains `"cdeab"` starting at index 2, confirming it's a valid rotation. `"abced"` has the same characters as `"abcde"` but in a different (non-rotational) order, so it never appears as a contiguous substring.

## 4. Repeated String Match

### Theory

**Classic example — Repeated String Match:** find the minimum number of times `a` must be repeated so that `b` becomes a substring of the result (or `-1` if impossible). Repeating `a` fewer than `⌈len(b) / len(a)⌉` times can never make it long enough to contain `b`, so start there; but occasionally **one more repeat** is needed to fully cover `b` if `b` starts mid-way through a copy of `a` — so check that count, then check one extra repeat, and give up if neither contains `b`.

### Code

```java
public class RepeatedStringMatch {
    public static void main(String[] args) {
        System.out.println("repeatedStringMatch('abcd','cdabcdab'): "
                + repeatedStringMatch("abcd", "cdabcdab"));
        System.out.println("repeatedStringMatch('a','aa'): " + repeatedStringMatch("a", "aa"));
        System.out.println("repeatedStringMatch('abc','wxyz'): " + repeatedStringMatch("abc", "wxyz"));
    }

    static int repeatedStringMatch(String a, String b) {
        int count = (int) Math.ceil((double) b.length() / a.length());
        StringBuilder repeated = new StringBuilder();
        for (int i = 0; i < count; i++) repeated.append(a);

        if (repeated.toString().contains(b)) return count;

        repeated.append(a); // try one more repeat, in case b straddles a boundary
        if (repeated.toString().contains(b)) return count + 1;

        return -1;
    }
}
```

### Example

```
repeatedStringMatch('abcd','cdabcdab'): 3
repeatedStringMatch('a','aa'): 2
repeatedStringMatch('abc','wxyz'): -1
```

`"abcd"` repeated `2` times gives `"abcdabcd"` (length 8, same as `b`), but `"cdabcdab"` doesn't appear there — repeating a `3`rd time (`"abcdabcdabcd"`) finally contains it.

## 5. Simplify Path

### Theory

**Classic example — Simplify Path (Unix-style):** given an absolute file path (possibly with `.` for current directory, `..` for parent directory, and redundant slashes), reduce it to its simplified canonical form. Split on `/`, and process each segment with a **stack**: ignore empty segments (from repeated slashes) and `.` segments (no-ops); on `..`, pop the stack (going up a directory, if there's anywhere to go up to); otherwise, push the segment name. Join whatever remains on the stack with `/`, prefixed by a single leading `/`.

### Code

```java
import java.util.ArrayDeque;
import java.util.Deque;

public class SimplifyPath {
    public static void main(String[] args) {
        System.out.println("simplifyPath('/home/'): " + simplifyPath("/home/"));
        System.out.println("simplifyPath('/../'): " + simplifyPath("/../"));
        System.out.println("simplifyPath('/home//foo/'): " + simplifyPath("/home//foo/"));
        System.out.println("simplifyPath('/a/./b/../../c/'): " + simplifyPath("/a/./b/../../c/"));
    }

    static String simplifyPath(String path) {
        Deque<String> stack = new ArrayDeque<>();
        for (String part : path.split("/")) {
            if (part.isEmpty() || part.equals(".")) {
                continue;
            } else if (part.equals("..")) {
                if (!stack.isEmpty()) stack.pollLast();
            } else {
                stack.addLast(part);
            }
        }
        return "/" + String.join("/", stack);
    }
}
```

### Example

```
simplifyPath('/home/'): /home
simplifyPath('/../'): /
simplifyPath('/home//foo/'): /home/foo
simplifyPath('/a/./b/../../c/'): /c
```

`"/../"` tries to go above the root, which is a no-op (an empty stack stays empty) — resulting in just `"/"`. `"/a/./b/../../c/"`: `.` is ignored, then both `..`s pop back past `b` and `a`, leaving only `"c"`.
