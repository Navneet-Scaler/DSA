## 1. Fixed Window

### Theory

**Classic example — Maximum Number of Vowels in a Substring of Given Length:** slide a fixed-size window of length `k` across the string, tracking a running vowel count — add the incoming character's contribution, remove the outgoing one.

### Intuition

"Substring of length k" is the string-world signal for a fixed window — the window size never changes, so there's no need for a shrink/expand decision at all. The only work per step is updating the running count by one add and one remove, exactly like [Arrays: Fixed Window](<../Arrays/07. Sliding Window.md#1-fixed-window>).

### Code

```java
public class FixedWindowVowels {
    public static void main(String[] args) {
        System.out.println("Max vowels, window 3: " + maxVowels("abciiidef", 3));
    }

    static int maxVowels(String s, int k) {
        String vowels = "aeiou";
        int count = 0;
        for (int i = 0; i < k; i++) {
            if (vowels.indexOf(s.charAt(i)) != -1) count++;
        }

        int best = count;
        for (int i = k; i < s.length(); i++) {
            if (vowels.indexOf(s.charAt(i)) != -1) count++;
            if (vowels.indexOf(s.charAt(i - k)) != -1) count--;
            best = Math.max(best, count);
        }
        return best;
    }
}
```

### Example

```
Max vowels, window 3: 3
```

`"iii"` is a window of 3 vowels — the maximum possible.

## 2. Variable Window

### Theory

**Classic example — Longest Substring Without Repeating Characters:** the canonical string sliding-window problem. Full algorithm and code already in [Arrays Lesson 7: Variable Window](<../Arrays/07. Sliding Window.md#2-variable-window>) — worth re-stating here since it's fundamentally a **string** problem that happens to generalize to arrays, not the other way around.

### Intuition

The window's validity condition — "no repeated character inside it" — can only be *violated* by expanding (adding a new character that happens to already be in the window); shrinking never introduces a repeat. That asymmetry (expansion can break things, shrinking can only fix things) is what makes the two-pointer expand/shrink shape work at all: grow greedily, and only pull the left edge forward when the most recently added character forces a violation.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class LongestSubstringNoRepeat {
    public static void main(String[] args) {
        System.out.println("Longest substring without repeats: " + lengthOfLongestSubstring("abcabcbb"));
    }

    static int lengthOfLongestSubstring(String s) {
        Map<Character, Integer> lastSeen = new HashMap<>();
        int left = 0, best = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
                left = lastSeen.get(c) + 1;
            }
            lastSeen.put(c, right);
            best = Math.max(best, right - left + 1);
        }
        return best;
    }
}
```

### Example

```
Longest substring without repeats: 3
```

## 3. Longest Substring

### Theory

**Classic example — Longest Repeating Character Replacement:** given a budget of `k` character replacements, find the longest substring achievable by replacing up to `k` characters so it becomes all one repeated letter. A window is valid whenever `(window length) - (count of its most frequent character) ≤ k` — i.e., the number of characters that would need replacing doesn't exceed the budget.

### Intuition

The trick that unlocks this problem: you don't need to know *which* letter the window is being converted to — you just need to know how many characters *don't* match the **most common** letter already in the window, since replacing those is always the cheapest way to make the window uniform. Once framed that way, it becomes an ordinary sliding window with a validity check based on `window length - max frequency`, and (subtly) the running `maxFrequency` never needs to *decrease* even as the window shrinks — the window only grows when a genuinely better maximum is found, so stale values don't cause incorrect shrinking.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class LongestRepeatingCharacterReplacement {
    public static void main(String[] args) {
        System.out.println("ABAB, k=2: " + characterReplacement("ABAB", 2));
        System.out.println("AABABBA, k=1: " + characterReplacement("AABABBA", 1));
    }

    static int characterReplacement(String s, int k) {
        Map<Character, Integer> count = new HashMap<>();
        int left = 0, maxFrequency = 0, best = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            count.merge(c, 1, Integer::sum);
            maxFrequency = Math.max(maxFrequency, count.get(c));

            if ((right - left + 1) - maxFrequency > k) {
                char leftChar = s.charAt(left);
                count.put(leftChar, count.get(leftChar) - 1);
                left++;
            }
            best = Math.max(best, right - left + 1);
        }
        return best;
    }
}
```

### Example

```
ABAB, k=2: 4
AABABBA, k=1: 4
```

## 5. Distinct Characters

### Theory

**Problem 1 (Basic): Longest Substring with At Most Two Distinct Characters** — the `k=2` special case of [Arrays Lesson 7: Longest Substring with At Most K Distinct Characters](<../Arrays/07. Sliding Window.md#2-variable-window>). **Problem 2:** the general `K` version is covered fully at that link — same algorithm, just parameterized.

### Intuition

Recognizing "at most 2 distinct characters" as a **special case** of "at most K distinct characters" (rather than a brand new problem) is the actual skill here — interview problems frequently present a simplified version of a pattern you already know, and the win is noticing you can reuse the general solution with `k` hardcoded to `2`, rather than re-deriving something from scratch.

### Code

```java
import java.util.HashMap;
import java.util.Map;

public class LongestTwoDistinct {
    public static void main(String[] args) {
        System.out.println("Longest substring, at most 2 distinct: " + lengthOfLongestSubstringTwoDistinct("eceba"));
    }

    static int lengthOfLongestSubstringTwoDistinct(String s) {
        Map<Character, Integer> count = new HashMap<>();
        int left = 0, best = 0;

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            count.merge(c, 1, Integer::sum);

            while (count.size() > 2) {
                char leftChar = s.charAt(left);
                count.put(leftChar, count.get(leftChar) - 1);
                if (count.get(leftChar) == 0) count.remove(leftChar);
                left++;
            }
            best = Math.max(best, right - left + 1);
        }
        return best;
    }
}
```

### Example

```
Longest substring, at most 2 distinct: 3
```

`"ece"` uses only `{e, c}` (2 distinct characters) and has length `3` — the longest such run.

## 6. Frequency Window

### Theory

**Problem 1 (Basic): Permutation in String** — check whether `s2` contains **any** permutation (rearrangement) of `s1` as a substring. Slide a fixed window of size `|s1|` across `s2`, comparing the window's character-frequency map against `s1`'s. **Problem 2 (Advanced): Find All Anagrams in a String** — the same check, but collect **every** starting index where a match occurs, instead of stopping at the first.

### Intuition

"Contains a permutation" doesn't care about character *order* at all — only *which characters, how many of each*. That's the signal to compare **frequency maps**, not substrings directly: two windows are "the same" for this problem exactly when their frequency maps are equal, regardless of arrangement. Once that reframing clicks, sliding a fixed-size window and comparing frequency maps (Problem 1) versus collecting every match instead of the first (Problem 2) is a one-line difference — the hard part is recognizing the frequency-map equivalence in the first place.

### Code

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class FrequencyWindow {
    public static void main(String[] args) {
        System.out.println("Permutation of 'ab' in 'eidbaooo': " + checkInclusion("ab", "eidbaooo"));
        System.out.println("All anagram start indices: " + findAnagrams("cbaebabacd", "abc"));
    }

    static boolean checkInclusion(String s1, String s2) {
        int k = s1.length();
        if (k > s2.length()) return false;

        Map<Character, Integer> need = buildFrequency(s1);
        Map<Character, Integer> window = new HashMap<>();

        for (int i = 0; i < s2.length(); i++) {
            window.merge(s2.charAt(i), 1, Integer::sum);
            if (i >= k) {
                removeFromWindow(window, s2.charAt(i - k));
            }
            if (i >= k - 1 && window.equals(need)) return true;
        }
        return false;
    }

    static List<Integer> findAnagrams(String s, String p) {
        int k = p.length();
        Map<Character, Integer> need = buildFrequency(p);
        Map<Character, Integer> window = new HashMap<>();
        List<Integer> result = new ArrayList<>();

        for (int i = 0; i < s.length(); i++) {
            window.merge(s.charAt(i), 1, Integer::sum);
            if (i >= k) {
                removeFromWindow(window, s.charAt(i - k));
            }
            if (i >= k - 1 && window.equals(need)) result.add(i - k + 1);
        }
        return result;
    }

    static Map<Character, Integer> buildFrequency(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) freq.merge(c, 1, Integer::sum);
        return freq;
    }

    static void removeFromWindow(Map<Character, Integer> window, char c) {
        int updated = window.get(c) - 1;
        if (updated == 0) window.remove(c);
        else window.put(c, updated);
    }
}
```

### Example

```
Permutation of 'ab' in 'eidbaooo': true
All anagram start indices: [0, 6]
```

`"eidbaooo"` contains `"ba"` (a permutation of `"ab"`) starting at index 3. `"cbaebabacd"` has anagrams of `"abc"` starting at indices `0` (`"cba"`) and `6` (`"bac"`).
